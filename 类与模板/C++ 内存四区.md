# C++ 内存四区详解

---

## 📚 总体概览

C++程序在运行时，内存被划分为**四个区域**：

| 区域   | 存放内容                 | 生命周期     | 管理者         |
| ------ | ------------------------ | ------------ | -------------- |
| 代码区 | 程序的机器指令           | 程序运行期间 | 操作系统       |
| 全局区 | 全局变量、静态变量、常量 | 程序运行期间 | 操作系统       |
| 栈区   | 局部变量、函数参数       | 函数调用期间 | 编译器自动管理 |
| 堆区   | 动态分配的内存           | 手动控制     | 程序员         |

---

## 1️⃣ 代码区（Code Segment / Text Segment）

### 什么是代码区？

代码区存放的是**CPU执行的机器指令**（也就是你写的代码编译后的二进制指令）。

### 特点：

```
✅ 共享性：多个进程可以共享同一份代码（比如同时运行多个相同程序）
✅ 只读性：防止程序意外修改自己的指令
✅ 程序运行前就已确定：编译时就固定了
```

### 形象理解：

> 🎬 代码区就像**电影胶片**——内容是固定的，只能"播放"（执行），不能修改。

---

## 2️⃣ 全局区（Global / Static Segment）

### 什么是全局区？

全局区存放**全局变量**、**静态变量**和**常量**。

### 存放内容详解：

```cpp
#include <iostream>
using namespace std;

// ========== 全局区存放内容 ==========

// 1. 全局变量
int g_a = 10;           // ✅ 在全局区
int g_b = 20;           // ✅ 在全局区

// 2. 静态变量（无论在哪定义）
static int s_a = 30;    // ✅ 在全局区

// 3. 常量
//    3.1 字符串常量
//    3.2 const修饰的全局常量
const int c_g_a = 40;   // ✅ 在全局区

void test() {
    // 函数内的静态变量也在全局区！
    static int s_b = 50; // ✅ 在全局区
    
    // 局部常量不在全局区！
    const int c_l_a = 60; // ❌ 在栈区
}

int main() {
    // 局部变量不在全局区
    int a = 100;         // ❌ 在栈区
    
    // 字符串常量在全局区
    cout << "hello" << endl;  // "hello" ✅ 在全局区
    
    return 0;
}
```

### 特点：

```
✅ 程序结束后由操作系统释放
✅ 全局变量和静态变量存放在一起
✅ 常量区（只读）也在全局区内
```

### 快速判断口诀：

> 🎯 **"全局的、静态的、字符串常量和全局const"** —— 都在全局区

## 3️⃣ 栈区（Stack）

### 什么是栈区？

栈区存放**函数的局部变量**、**函数参数**等，由编译器**自动分配和释放**。

### 核心代码示例：

```cpp
#include <iostream>
using namespace std;

// ⚠️ 危险示例：不要返回局部变量的地址！
int* func() {
    int a = 10;      // 局部变量，存放在栈区
    return &a;       // ❌ 危险！返回栈区地址
}

int main() {
    int* p = func();
    
    cout << *p << endl;  // 第一次可能输出10（编译器做了保留）
    cout << *p << endl;  // 第二次就是乱码了！内存已被释放
    
    return 0;
}
```

### 为什么会乱码？

```
调用 func() 时：
┌─────────────┐
│   a = 10    │  ← 栈区局部变量
│   (地址)    │
└─────────────┘

func() 结束后：
┌─────────────┐
│   ???       │  ← 内存被释放，数据不再有效
│   (地址)    │     （可能被其他数据覆盖）
└─────────────┘
```

### 栈区的特点：

```
✅ 自动管理：编译器自动分配和释放，无需程序员操心
✅ 速度快：入栈出栈操作非常高效
⚠️ 空间有限：通常只有几MB（Windows默认1MB，Linux默认8MB）
⚠️ 不能返回局部变量地址：函数结束后内存就释放了
```

### 栈的工作原理（LIFO - 后进先出）：

```
void funcA() {
    int x = 1;    // x 入栈
    funcB();      // 调用 funcB
}                 // x 出栈

void funcB() {
    int y = 2;    // y 入栈
}                 // y 出栈

调用过程中栈的变化：
───────────────────────────
        │      │
        │  y   │  ← funcB 的局部变量
        ├──────┤
        │  x   │  ← funcA 的局部变量
        └──────┘
───────────────────────────
```

## 4️⃣ 堆区（Heap）— 重点！

### 什么是堆区？

堆区是由**程序员手动分配和释放**的内存区域。如果程序员不释放，程序结束时由操作系统回收。

### 核心代码示例：

```cpp
#include <iostream>
using namespace std;

int* func() {
    // 使用 new 在堆区开辟内存
    int* p = new int(10);  // 在堆区创建一个int，值为10
    return p;               // ✅ 安全！堆区内存不会自动释放
}

int main() {
    int* p = func();
    
    cout << *p << endl;  // 输出 10
    cout << *p << endl;  // 仍然输出 10！
    cout << *p << endl;  // 仍然输出 10！
    
    // 用完后必须手动释放！
    delete p;
    p = nullptr;  // 良好习惯：释放后置空
    
    return 0;
}
```

### 内存分布图解：

```
┌─────────────────────────────────────┐
│           代码区                    │  ← 程序指令
├─────────────────────────────────────┤
│           全局区                    │  ← 全局/静态变量
├─────────────────────────────────────┤
│                                     │
│            堆区 ↓                   │  ← 向下增长（new分配）
│                                     │
│    ┌─────────────────────┐          │
│    │   10 (new int(10))  │          │  ← 堆区中的数据
│    └─────────────────────┘          │
│                                     │
├─────────────────────────────────────┤
│                                     │
│            栈区 ↑                   │  ← 向上增长（局部变量）
│                                     │
│    ┌─────────────────────┐          │
│    │   p (指针变量)      │──────────┼──→ 指向堆区的10
│    └─────────────────────┘          │
│                                     │
└─────────────────────────────────────┘
```

### new 和 delete 的用法：

```cpp
// ========== 基本语法 ==========

// 1. 单个变量
int* p1 = new int;        // 分配未初始化的int
int* p2 = new int(100);   // 分配并初始化为100
delete p1;
delete p2;

// 2. 数组
int* arr = new int[10];   // 分配10个int的数组
arr[0] = 1;
arr[1] = 2;
// ...
delete[] arr;             // ⚠️ 注意：数组要用 delete[]

// 3. 对象
class Person {
public:
    Person() { cout << "构造函数" << endl; }
    ~Person() { cout << "析构函数" << endl; }
};

Person* person = new Person;  // 调用构造函数
delete person;                // 调用析构函数
```

### 堆区的特点：

```
✅ 灵活：程序员完全控制分配和释放时机
✅ 空间大：比栈区大得多（理论上只受系统内存限制）
⚠️ 需要手动管理：忘记释放会导致内存泄漏
⚠️ 速度较慢：比栈区分配慢
⚠️ 容易出错：野指针、重复释放等问题
```

```
┌────────┬──────────────────┬──────────────┬─────────────┐
│  区域  │     存放内容     │   生命周期   │   管理者    │
├────────┼──────────────────┼──────────────┼─────────────┤
│ 代码区 │ 二进制机器指令   │ 程序运行期间 │  操作系统   │

│ 全局区 │ 全局变量         │ 程序运行期间 │  操作系统   │

│        │ 常量(字符串/const│              │             │
├────────┼──────────────────┼──────────────┼─────────────┤
│ 栈区   │ 局部变量         │ 函数调用期间 │ 编译器自动  │

└────────┴──────────────────┴──────────────┴─────────────┘
```

## 💡 实战记忆技巧

```
📦 代码区 → "剧本"    → 只能读，不能改
📦 全局区 → "仓库"    → 全程都在，谁都能用
📦 栈区   → "临时工"  → 用完就走，自动清理
📦 堆区   → "私人物品" → 自己管理，用完要收
```

## ❓ 常见面试题

**Q1: 局部变量和全局变量的区别？**

```
局部变量：栈区，函数结束即销毁
全局变量：全局区，程序结束才销毁
```

**Q2: 为什么不能返回局部变量的地址？**

```
因为局部变量在栈区，函数结束后栈内存被释放，
返回的地址指向的是无效内存（野指针）。
```

**Q3: new 和 malloc 的区别？**

```
new：C++关键字，会调用构造函数，返回具体类型指针
malloc：C函数，不调用构造函数，返回void*需要强转
```

void stackAreaDemo2() {

    static int s_a = 400;        // ✅ 虽然在函数内，但存放在全局区

int* dangerousFunction() {
    int a = 500;                 // ✅ 栈区局部变量

    // 函数结束后，a 的内存被释放
    // 返回的地址指向的是无效内存（野指针）

// 3.4 ✅ 正确示例：返回堆区地址是安全的
int* safeFunction() {
    int* p = new int(600);       // ✅ 在堆区分配内存
    return p;                    // ✅ 安全！堆区内存不会自动释放
    
    // 注意：调用者需要负责释放这块内存
}

void heapAreaDemo2() {

    // 3. 堆区地址
    cout << "\n【堆区】" << endl;
    int* heap_p1 = new int(900);
    int* heap_p2 = new int(1000);
    cout << "堆区变量1的地址:            " << (long)heap_p1 << endl;

    // 4. 代码区地址（函数地址）
    cout << "\n【代码区】" << endl;
    cout << "函数 main 的地址:           " << (long)&main << endl;
    cout << "函数 comprehensiveDemo 的地址: " << (long)&comprehensiveDemo << endl;
    
    // 清理堆区内存
    delete heap_p1;
    delete heap_p2;
    
    cout << "\n========== 地址分析 ==========" << endl;
    cout << "观察规律：" << endl;
    cout << "1. 全局区地址通常较小" << endl;
    cout << "2. 栈区地址通常较大，且相邻变量地址接近" << endl;
    cout << "3. 堆区地址在中间，且可能不连续" << endl;
    cout << "4. 代码区地址最小" << endl;
}

void commonMistakes() {
    cout << "\n========== 常见错误示例 ==========" << endl;
    
    // 错误1：返回局部变量地址
    cout << "\n【错误1】返回局部变量地址" << endl;
    int* p1 = dangerousFunction();
    cout << "第一次访问: " << *p1 << " (可能正确)" << endl;
    cout << "第二次访问: " << *p1 << " (可能乱码)" << endl;
    // ⚠️ 这是未定义行为，结果不可预测
    
    // 错误2：忘记释放堆区内存（内存泄漏）
    cout << "\n【错误2】内存泄漏示例" << endl;
    for (int i = 0; i < 3; i++) {
        int* leak = new int(i);  // ✅ 分配内存
        cout << "分配了内存，但忘记释放..." << endl;
        // ❌ 忘记 delete leak;  导致内存泄漏
    }
    
    // 错误3：重复释放
    cout << "\n【错误3】重复释放（已注释，运行会崩溃）" << endl;
    int* p2 = new int(100);
    delete p2;
    // delete p2;  // ❌ 重复释放，会导致程序崩溃
    // 正确做法：释放后置空
    p2 = nullptr;
    
    // 错误4：数组用错误的 delete
    cout << "\n【错误4】数组释放错误（已注释）" << endl;
    int* arr = new int[5];
    // delete arr;    // ❌ 错误！应该用 delete[]
    delete[] arr;     // ✅ 正确
}

2. 堆区：手动管理，空间大，速度慢，必须配对使用 new/delete
3. 全局区：程序运行期间一直存在，包括全局变量、静态变量、常量
4. 代码区：存放程序指令，只读且共享

✅ 释放指针后立即置空（p = nullptr）
✅ 数组用 delete[]，单个对象用 delete
✅ 避免返回局部变量的地址或引用
✅ 尽量使用栈区（RAII原则）

【常见错误】
❌ 返回局部变量地址 → 野指针
❌ 忘记释放堆内存 → 内存泄漏
❌ 重复释放 → 程序崩溃
