### 🟡 第二部分：必看但需倍速 (C++ 核心特性)

**理由：** 这是 C++ 区别于 C 的关键，也是写出“现代 C++”代码的基础。如果不学，你写的只是“带类的 C”。

- **程序的内存模型 (01-05):**
  - **重点听：** `new` 和 `delete` 运算符。这是 C++ 的动态内存管理，替代 C 的 `malloc/free`。
- **C++中的引用 (06-11):**
  - **⭐⭐⭐ 五星必看：** C 语言没有“引用” (`&`)。在算法题中，传递大对象（如 vector, string）必须用引用，否则会因为拷贝导致超时。
- **函数高级 (12-15):**
  - **重点听：** 函数重载 (Overloading)。
- **类和对象 (16-60):**
  - 这部分是 OOP 的核心。如果你只为了刷算法题，不需要钻得太深，但以下几个点必须懂：
  - **封装 (16-22):** `class` 和 `struct` 的区别，`public`/`private`。
  - **构造与析构 (23-28):** **⭐⭐⭐ 五星必看**。特别是**深拷贝与浅拷贝** (27节)。这是 C++ 最容易崩溃的地方。
  - **this 指针 (33):** 了解一下即可。
  - **运算符重载 (39-44):** **重点听**。在算法中，如果你想把自定义结构体放进优先队列 (Priority Queue) 或者排序，你需要重载 `<` 运算符。
- **模板 (01-18):**
  - **⭐⭐⭐ 五星必看**。不理解模板，你就看不懂 STL（标准模板库）。不用自己会写复杂的模板，但要理解它是怎么工作的。

### 🔴 第三部分：核心中的核心 (STL 标准模板库)

**理由：** **这是你学这门课最大的目的。** 刷算法题就是用这些工具。相当于 Python 的 list, dict, set。

- **STL初识 (19-22):** 了解容器、算法、迭代器。
- **String 容器 (23-30):** C++ 的字符串处理比 C 的 `char*` 方便一万倍。
- **Vector 容器 (31-37):** **⭐⭐⭐ 动态数组**。替代 C 数组的神器，算法题 80% 的时间都在用它。
- **Deque, Stack, Queue (38-48):** **⭐⭐⭐ 栈和队列**。广度优先搜索 (BFS)、深度优先搜索 (DFS) 必用。
- **List (49-56):** 双向链表。了解即可，刷题用得少（通常用数组模拟链表）。
- **Set/Map (57-69):** **⭐⭐⭐⭐⭐ 红黑树**。
  - `set`: 自动去重排序。
  - `map`: 键值对映射（字典）。算法题中处理映射关系的终极武器。
- **常用算法 (71-97):**
  - **重点听：** `sort` (排序), `find` (查找), `reverse` (反转)。剩下的用到再查。
