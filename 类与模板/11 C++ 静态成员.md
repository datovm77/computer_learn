# C++ 静态成员详解

> 说明：示例默认已包含必要头文件，例如 `<iostream>`、`<string>`，并默认使用 `std::cout`、`std::string` 等标准库名称。

## 一、为什么需要静态成员？

想象一个场景：你要写一个"银行账户"类，需要统计**总共创建了多少个账户**。如果用普通成员变量，每个对象都有自己的副本，无法统计总数。这时就需要**静态成员**。

## 二、静态成员变量（由浅入深）

### 1. 基础概念

```cpp
class BankAccount {
public:
    static int accountCount;  // 静态成员变量
    int balance;              // 普通成员变量
    
    BankAccount() {
        accountCount++;  // 每创建一个对象就加1
    }
};

// 非 inline 静态数据成员：需要在类外提供一个定义，可同时初始化
int BankAccount::accountCount = 0;
```

**关键特点：**

- 静态成员变量属于**整个类**，不属于某个具体对象
- 所有对象**共享同一份**静态成员变量
- 对本例这种非 `inline` 静态数据成员，通常需要在类外定义；C++17 起可用 `inline static` 在类内直接定义

### 2. 使用示例

```cpp
int main() {
    BankAccount acc1;
    BankAccount acc2;
    BankAccount acc3;
    
    // 三种访问方式都可以
    cout << BankAccount::accountCount << endl;  // 推荐：通过类名访问，输出3
    cout << acc1.accountCount << endl;          // 通过对象访问，输出3
    cout << acc2.accountCount << endl;          // 任何对象看到的都是同一个值，输出3
    
    return 0;
}
```

### 3. 内存理解

```
普通成员变量：
acc1对象: [balance: 1000]
acc2对象: [balance: 2000]
acc3对象: [balance: 3000]

静态成员变量：
静态存储期对象: [accountCount: 3]  ← 所有对象共享这一份
```

`accountCount` 具有**静态存储期**，生命周期通常贯穿整个程序。常见实现会把它放在静态/全局存储区域，但 C++ 标准关注的是存储期和生命周期，不规定具体内存分区名称。

## 三、静态成员函数

### 1. 基础用法

```cpp
class Student {
private:
    static int totalStudents;
    string name;
    
public:
    Student(string n) : name(n) {
        totalStudents++;
    }
    
    // 静态成员函数
    static int getTotalStudents() {
        return totalStudents;
    }
    
    // 普通成员函数
    string getName() {
        return name;
    }
};

int Student::totalStudents = 0;

int main() {
    Student s1("张三");
    Student s2("李四");
    
    // 不需要创建对象就能调用
    cout << Student::getTotalStudents() << endl;  // 输出2
    
    return 0;
}
```

### 2. 重要限制

```cpp
class Example {
private:
    static int staticVar;
    int normalVar;
    
public:
    static void staticFunc() {
        staticVar = 10;      // ✓ 可以访问静态成员
        // normalVar = 20;   // ✗ 错误！没有对象，不能直接访问普通成员
        // getName();        // ✗ 错误！没有 this，不能直接调用普通成员函数
    }

    static int readNormalVar(const Example& obj) {
        return obj.normalVar; // ✓ 可以通过明确的对象访问非静态成员
    }
    
    void normalFunc() {
        staticVar = 10;      // ✓ 普通函数可以访问静态成员
        normalVar = 20;      // ✓ 也可以访问普通成员
    }
    
    string getName() { return "test"; }
};
```

**为什么有这个限制？**

- 静态函数不属于任何对象，没有`this`指针
- 普通成员变量需要通过对象才能访问
- 静态函数不能**直接/隐式**访问普通成员；如果传入具体对象、引用或指针，仍然可以访问该对象的非静态成员

### 3. 类内初始化：C++17 的 `inline static`

```cpp
class Counter {
public:
    inline static int count = 0;              // C++17 起：类内就是定义
    inline static constexpr int maxCount = 100; // constexpr 静态数据成员隐含 inline
};
```

C++17 前，非 `inline` 静态数据成员在类内通常只是**声明**，如果被 ODR-use（例如取地址、作为运行期对象使用），需要在某个源文件里提供且只提供一个类外定义。C++17 起 `inline static` 能避免“头文件里不能放普通定义”的问题。

```cpp
class Config {
public:
    static const int version = 1; // C++17 前可作为整型编译期常量使用
};

// C++17 前如果需要 ODR-use，可能仍要类外定义：
// const int Config::version;
```

## 四、实际应用场景

### 场景1：计数器

```cpp
class Product {
private:
    static int productCount;
    int id;
    
public:
    Product() {
        id = ++productCount;  // 自动分配ID
    }
    
    int getID() const { return id; }
    static int getTotal() { return productCount; }
};

int Product::productCount = 0;

int main() {
    Product p1, p2, p3;
    cout << "产品1的ID: " << p1.getID() << endl;  // 1
    cout << "产品2的ID: " << p2.getID() << endl;  // 2
    cout << "总产品数: " << Product::getTotal() << endl;  // 3
}
```

### 场景2：工具函数

```cpp
class MathUtil {
public:
    inline static constexpr double PI = 3.14159;
    
    static double square(double x) {
        return x * x;
    }
    
    static double circleArea(double radius) {
        return PI * square(radius);
    }
};

int main() {
    // 不需要创建对象，直接使用
    cout << MathUtil::square(5) << endl;           // 25
    cout << MathUtil::circleArea(3) << endl;       // 28.27
}
```

## 五、常见易错点

### 1. 忘记类外定义

```cpp
class Test {
    static int x;  // 声明
};
// int Test::x = 0;  // 非 inline 静态数据成员被使用时，需要一个类外定义
```

注意这里更准确地说是“定义”，不是单纯“初始化”。定义负责为静态数据成员分配存储空间，初始化只是定义时可以顺便做的事。

### 2. 混淆访问方式

```cpp
// 推荐用类名访问静态成员
ClassName::staticMember    // ✓ 清晰
object.staticMember        // ✓ 可以，但容易误解
```

### 3. 在静态函数中使用this

```cpp
static void func() {
    // this->member;  // ✗ 错误！静态函数没有this指针
}
```

### 4. 多线程计数不是自动安全的

```cpp
#include <atomic>

class BankAccount {
public:
    inline static std::atomic<int> accountCount{0};

    BankAccount() {
        ++accountCount;
    }
};
```

普通 `static int` 的自增在多线程下会产生数据竞争；需要跨线程统计时，应使用 `std::atomic` 或锁。

## 六、总结对比表

| 特性     | 普通成员     | 静态成员           |
| -------- | ------------ | ------------------ |
| 属于     | 每个对象     | 整个类             |
| 内存     | 每个对象一份 | 所有对象共享一份   |
| 访问方式 | 必须通过对象 | 可通过类名直接访问 |
| this指针 | 有           | 无                 |

**记忆口诀：**

- 静态成员"高高在上"，属于类不属于对象
- 静态函数没有 `this`，不能直接访问非静态成员
- 非 `inline` 静态数据成员通常需要一个类外定义；C++17 起可优先考虑 `inline static`
- 静态共享数据跨线程修改时，要考虑数据竞争
