## 核心概念:成员的存储方式

C++类的成员变量和成员函数在内存中是**分开存储**的,这是理解类内存布局的关键。

------

## 1. 非静态成员变量 (`int m_A`)

```cpp
int m_A;
```

- **属于每个对象**:每个`Person`对象都有自己独立的`m_A`副本
- **占用对象内存**:这个变量存储在对象的内存空间中
- **示例**:

```cpp
Person p1, p2;
p1.m_A = 10;
p2.m_A = 20;  // p1和p2各有自己的m_A
```

------

## 2. 静态成员变量 (`static int m_B`)

```cpp
static int m_B;
int Person::m_B = 0;  // 必须在类外初始化
```

- **所有对象共享**:不管创建多少个`Person`对象,`m_B`只有一份
- **不占用对象内存**:存储在全局数据区,不计入对象大小
- **类外初始化**:必须在类外定义并初始化
- **示例**:

```cpp
Person p1, p2;
Person::m_B = 100;
// p1和p2访问的是同一个m_B
```

------

## 3. 非静态成员函数 (`void func()`)

```cpp
void func() {}
```

- **代码段存储**:函数代码存储在代码段,不占用对象内存
- **所有对象共享**:所有对象调用同一份函数代码
- **隐含this指针**:通过`this`指针区分是哪个对象调用的

```cpp
Person p1, p2;
p1.func();  // 实际是 func(&p1)
p2.func();  // 实际是 func(&p2)
```

------

## 4. 静态成员函数 (`static void func2()`)

```cpp
static void func2() {}
```

- **没有this指针**:无法访问非静态成员变量
- **只能访问静态成员**:因为它不属于任何具体对象
- **调用方式**:可以通过类名或对象调用

```cpp
Person::func2();  // 推荐
p1.func2();       // 也可以
```

------

## 实际验证

你可以用`sizeof`验证:

```cpp
cout << sizeof(Person);  // 输出4 (只计算m_A)
```

**为什么是4?**

- `m_A`(非静态变量):4字节 ✓
- `m_B`(静态变量):不计入 ✗
- `func()`和`func2()`(函数):不计入 ✗

------

## 总结表格

| 成员类型       | 是否属于对象 | 占用对象内存 | 共享性               |
| -------------- | ------------ | ------------ | -------------------- |
| 非静态成员变量 | ✓            | ✓            | 每个对象独立         |
| 静态成员变量   | ✗            | ✗            | 所有对象共享         |
| 非静态成员函数 | ✗            | ✗            | 所有对象共享(有this) |
| 静态成员函数   | ✗            | ✗            | 所有对象共享(无this) |

这种设计既节省了内存,又提高了效率。有什么不清楚的地方吗?
