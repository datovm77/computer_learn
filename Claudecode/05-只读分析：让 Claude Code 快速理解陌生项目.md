# 05-只读分析：让 Claude Code 快速理解陌生项目

本章目标：在不修改任何文件的前提下，让 Claude Code 快速建立项目地图，找出入口、核心模块、测试命令和后续学习路线。

## 使用场景

适合刚接手陌生仓库、准备做代码评审、排查线上问题前需要理解结构，或想让 Claude Code 帮你梳理项目约定但暂时不希望它改代码。

建议从项目根目录启动，并优先使用 Plan Mode 或明确要求“只读分析”。只读阶段应该产出这些信息：目录地图、运行入口、主要模块、配置文件、数据流、测试框架、常用命令、风险区域和待确认问题。

高风险提醒：这一章不要让 Claude Code 删除文件、批量重构、改权限、安装依赖、提交代码、联网搜索或处理密钥。看到 `.env`、私钥、token、证书时，只让它识别“存在敏感配置”，不要复制内容到对话里。

普通终端命令：

```bash
cd /path/to/project
claude --permission-mode plan
```

Claude Code 对话输入：

```markdown
请只读分析这个项目，不要修改任何文件。
请输出：
1. 项目目录地图
2. 主要入口文件和启动流程
3. 核心模块及职责
4. 测试框架和可能的测试命令
5. 配置文件、环境变量位置和敏感信息风险
6. 我继续阅读这个项目的建议顺序
```

## 推荐提示词模板

Claude Code 对话输入：

```markdown
你现在是只读代码向导。请只使用读取、搜索、列目录等操作，不要编辑、删除、安装依赖、提交代码或联网。

任务：快速理解当前仓库。

请按以下结构输出：
- 一句话说明项目用途
- 目录地图：只列关键目录和文件
- 运行入口：从命令到入口文件再到核心模块
- 核心模块：每个模块说明职责、关键文件、依赖关系
- 测试与验证：识别测试框架、测试目录、推荐运行命令
- 风险与待确认：列出你不能确定的地方，不要猜
- 下一步学习路线：给我 5 个最值得阅读的文件

约束：
- 不修改任何文件
- 不读取或复述密钥值
- 如果需要运行命令，先说明命令目的和风险，等我确认
```

如果你只想分析某个目录，可以缩小范围。

Claude Code 对话输入：

```markdown
请只读分析 @src 目录，重点找出用户登录相关代码。不要修改文件。请给出文件列表、调用链和我应该先读哪些文件。
```

## 错误示范

Claude Code 对话输入：

```markdown
帮我看看这个项目，顺便把你觉得不好的地方都改掉。
```

问题：范围太大，而且“顺便改掉”会把只读分析变成不受控修改。

Claude Code 对话输入：

```markdown
把整个仓库格式化一下，再告诉我架构。
```

问题：整库格式化会制造巨大 diff，干扰理解，也可能和团队格式化规则冲突。

普通终端命令：

```bash
claude -p "分析项目并自动修复所有问题"
```

问题：无边界的“自动修复”容易触发大范围编辑、安装依赖或运行高风险命令。

## 正确示范

Claude Code 对话输入：

```markdown
请先只读。目标是让我能在 30 分钟内理解这个仓库。
请先列出你准备查看的文件类型和搜索关键词，等我确认后再继续深入。
```

Claude Code 对话输入：

```markdown
请追踪一次请求从路由入口到数据库访问的路径。只输出路径和关键文件，不要修改代码。
```

Claude Code 对话输入：

```markdown
请识别测试框架和测试命令。不要运行会联网、安装依赖、改数据库、删除文件或提交代码的命令。如果命令可能有副作用，先问我。
```

普通终端命令：

```bash
claude --permission-mode plan -p "只读分析当前项目，输出目录地图、入口、核心模块和测试命令。不要修改文件。"
```

## 练习任务

1. 在一个陌生项目根目录启动 Claude Code。
2. 使用本章模板，让 Claude Code 只读输出项目地图。
3. 让它继续追踪一个核心功能的调用链。
4. 让它找出测试框架、测试目录和推荐测试命令，但不要实际运行。
5. 根据输出，手动打开 3 个关键文件核对它的结论。

进阶练习：让 Claude Code 生成一份“新成员阅读顺序”，要求每个文件后面写明阅读目的。

## 验收标准

- Claude Code 没有修改、删除、格式化任何文件。
- 输出能清楚说明项目用途、入口、关键模块和调用关系。
- 测试命令被识别出来，并标明是否可能有副作用。
- 敏感配置只被定位为风险点，没有泄露密钥内容。
- 结论中区分了“已确认”和“待确认”，没有把猜测当事实。
- 你能根据输出独立找到 5 个最值得优先阅读的文件。
