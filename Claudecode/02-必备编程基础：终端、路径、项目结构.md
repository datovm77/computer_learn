# 必备编程基础：终端、路径、项目结构

Claude Code 的效果很大程度取决于你是否在正确目录启动，以及你是否能描述项目结构。终端、路径和项目根目录是进阶使用的地基。

高风险提醒：不清楚当前目录时，不要让 Claude Code 删除文件、批量重构、修改权限、安装依赖或提交代码。路径写错可能影响完全不同的项目。

## 使用场景

你需要掌握这些基础，才能完成以下任务：

- 在正确项目根目录启动 Claude Code。
- 让 Claude Code 找入口文件、依赖文件和测试目录。
- 判断一个命令应该在项目根目录、子目录还是系统任意目录运行。
- 明确告诉 Claude Code 哪些目录可以改，哪些目录只读。

常见项目结构概念：

- 项目根目录：通常包含 `.git`、`README`、依赖文件和配置文件。
- 入口文件：程序启动时最先执行或框架约定加载的文件。
- 依赖文件：记录第三方库和版本，例如 `package.json`、`requirements.txt`、`pyproject.toml`、`Cargo.toml`。
- 测试目录：常见名称包括 `test`、`tests`、`__tests__`、`spec`。
- 配置文件：例如 `.env.example`、`tsconfig.json`、`vite.config.js`、`pytest.ini`。

普通终端命令：

```bash
pwd
ls
find . -maxdepth 2 -type f
```

Windows PowerShell 普通终端命令：

```powershell
Get-Location
Get-ChildItem
Get-ChildItem -Recurse -File -Depth 2
```

## 推荐提示词模板

Claude Code 对话输入：

```markdown
请只读分析当前项目结构，不要修改文件。

请找出：
- 项目根目录判断依据
- 入口文件候选
- 依赖文件
- 测试目录
- 配置文件
- 你还不能确定的信息

输出格式：
- 用表格列出路径、类型、判断依据。
- 最后给出我下一步应该阅读的 5 个文件。
```

Claude Code 对话输入：

```markdown
请帮我确认当前目录是否适合执行任务。

任务：修改登录页文案。
范围：只允许修改前端页面和相关测试。
禁止：删除文件、批量重构、安装依赖、联网、提交代码、处理真实密钥。

请先说明你认为相关文件在哪里，再等我确认。
```

## 错误示范

普通终端命令：

```bash
claude
```

问题：

- 命令本身没有错，但如果你不知道当前目录，就可能在错误项目中启动。
- 启动前应先运行 `pwd` 和 `git status`。

Claude Code 对话输入：

```markdown
把 src 下面都整理一下。
```

问题：

- `src` 可能很大。
- “整理”没有明确目标。
- 没有说明是否允许重命名、移动文件、批量重构或删除文件。

## 正确示范

普通终端命令：

```bash
pwd
git status
ls
```

Claude Code 对话输入：

```markdown
请先确认当前目录是否为项目根目录。不要修改文件。

请检查：
1. 是否存在 Git 仓库。
2. 是否存在 README 或依赖文件。
3. 是否存在源代码目录和测试目录。
4. 如果当前不是根目录，请说明我应该切换到哪个目录。
```

Claude Code 对话输入：

```markdown
请只修改 `docs/usage.md` 中的运行说明，不要改代码。

限制：
- 不删除文件。
- 不移动文件。
- 不改权限。
- 不安装依赖。
- 不提交代码。

完成后请告诉我修改了哪些段落，以及我应该如何检查。
```

## 路径基础

绝对路径从系统根位置开始，例如 `/home/user/project` 或 `C:\Users\user\project`。

相对路径从当前目录开始，例如 `src/main.js`、`../README.md`。

当前目录很重要，因为同一个相对路径在不同目录下指向不同文件。

普通终端命令：

```bash
pwd
cd ..
pwd
cd -
```

Windows PowerShell 普通终端命令：

```powershell
Get-Location
Set-Location ..
Get-Location
```

Claude Code 对话输入：

```markdown
请解释 `src/app`、`./src/app`、`../src/app` 在当前目录下分别指向哪里。不要修改文件。
```

## 练习任务

1. 找一个练习项目。
2. 在项目外层、项目根目录、项目子目录分别运行 `pwd` 或 `Get-Location`。
3. 判断哪里才是应该启动 Claude Code 的目录。
4. 让 Claude Code 只读列出项目结构。
5. 手动记录入口文件、依赖文件、测试目录和配置文件。

普通终端命令：

```bash
pwd
git status
find . -maxdepth 2 -type f
```

Claude Code 对话输入：

```markdown
请根据当前目录生成项目结构速查表。只读分析，不要修改文件。表格列为：路径、作用、我学习时应该关注什么。
```

## 验收标准

完成本章后，你应该能做到：

- 能解释项目根目录、入口文件、依赖文件、测试目录的含义。
- 能用普通终端命令确认当前路径。
- 能判断 Claude Code 是否在正确目录运行。
- 能把修改范围限定到具体目录或文件。
- 能识别路径不清导致的删除文件、批量重构、改权限、安装依赖、提交代码风险。
