# 14-安全边界：权限、密钥、危险命令与联网审核

Claude Code 能读写文件、运行命令、接入外部工具，并在合适配置下联网。这些能力让它接近真实开发者，也意味着必须建立安全边界。进阶用户的关键能力不是“让 Claude 做更多”，而是知道什么时候必须停下来审查。

高风险提醒：删除文件、批量重构、改权限、安装依赖、提交代码、联网、处理密钥都必须显式提醒、显式授权、显式验收。任何自动化流程都不应默认允许这些动作。

## 使用场景

需要重点设置安全边界的场景：

- 让 Claude Code 修改代码或文档。
- 让 Claude Code 运行测试、构建、格式化或脚本。
- 让 Claude Code 安装依赖、升级包、生成 lockfile。
- 让 Claude Code 操作 Git，例如提交、回退、变基、清理文件。
- 让 Claude Code 接触 `.env`、证书、token、SSH key、云凭据。
- 让 Claude Code 联网查资料、下载文件、调用 API。
- 让 Claude Code 使用 MCP 访问外部系统。

安全原则：

- 最小权限：只开放完成任务必要的文件、命令和工具。
- 先读后写：先做只读分析和计划，再执行修改。
- 小步变更：减少一次性批量修改。
- 人工确认：高风险动作前必须确认。
- 可审计：保留 diff、命令输出和决策原因。
- 可回滚但不乱回滚：不要随意 `git reset --hard` 或覆盖他人改动。

危险命令示例：

- 删除：`rm -rf`、`del /s`、`Remove-Item -Recurse`
- 改权限：`chmod -R`、`chown -R`
- Git 回退：`git reset --hard`、`git clean -fd`
- 依赖变更：`npm install`、`pip install`、`cargo update`
- 执行远程脚本：`curl ... | bash`
- 密钥输出：`cat .env`、`printenv`、读取证书和私钥

## 推荐提示词模板

Claude Code 对话输入：

```markdown
在执行任务前，请先做安全边界确认。

请回答：
- 需要读取哪些文件？
- 需要修改哪些文件？
- 是否需要运行命令？
- 是否涉及删除文件、批量重构、改权限、安装依赖、提交代码、联网、处理密钥？
- 哪些步骤需要人工确认？

在我确认前，不要修改文件，不要运行高风险命令。
```

Claude Code 对话输入：

```markdown
请只读检查当前 diff 是否包含安全风险。

重点：
- 密钥、token、证书、私钥。
- 删除文件。
- 权限变化。
- 依赖新增或升级。
- 联网调用。
- Git 提交、回退或清理命令。
- 大规模重构。

不要修改文件。按严重程度输出。
```

普通终端命令：

```bash
git status --short
git diff --stat
git diff --check
```

普通终端命令：

```bash
git diff -- . ':!package-lock.json' ':!pnpm-lock.yaml' ':!yarn.lock'
```

用途：先看代码和配置的实质改动，再单独审查 lockfile。

普通终端命令：

```bash
git diff --name-only | claude -p "请根据文件名判断是否有高风险改动。重点：密钥、权限、依赖、删除、Git 配置、CI、部署。不要修改文件。"
```

## 错误示范

Claude Code 对话输入：

```markdown
项目乱了，直接帮我 git reset --hard，然后重新做。
```

问题：

- `git reset --hard` 会丢弃未提交改动。
- 多 Agent 或多人协作时可能抹掉他人工作。
- 应先查看 `git status` 和 diff，并明确哪些改动可丢弃。

普通终端命令：

```bash
cat .env | claude -p "帮我检查环境变量配置"
```

问题：

- `.env` 可能包含密钥。
- 应脱敏后再分析，或只提供变量名和错误信息。

普通终端命令：

```bash
curl https://example.com/install.sh | bash
```

问题：

- 执行远程脚本风险极高。
- 应先下载、审查、校验来源，再决定是否运行。

Claude Code 对话输入：

```markdown
自动安装所有缺失依赖，能跑起来就行。
```

问题：

- 安装依赖会改变供应链风险和 lockfile。
- “能跑起来”不是安全验收标准。
- 应先列出缺失依赖、来源、版本、必要性。

Claude Code 对话输入：

```markdown
你可以联网查任何资料，顺便把找到的代码复制进项目。
```

问题：

- 联网需要来源审核。
- 复制代码可能有许可证和安全风险。
- 应要求引用来源、限制范围、人工确认。

## 正确示范

密钥处理：

普通终端命令：

```bash
sed 's/=.*/=<redacted>/' .env.example
```

Claude Code 对话输入：

```markdown
请根据下面脱敏后的环境变量名检查配置完整性。
不要要求我提供真实密钥。
不要输出任何 token、密码、证书或私钥。
```

危险命令审批：

Claude Code 对话输入：

```markdown
你建议运行的命令包含删除或回退风险时，请先停止并说明：
- 命令原文。
- 会影响哪些文件。
- 是否可恢复。
- 是否有更安全替代方案。
在我明确批准前，不要执行。
```

依赖安装审核：

Claude Code 对话输入：

```markdown
如果需要安装依赖，请先输出依赖评审：
- 包名和版本。
- 为什么必须安装。
- 是否有现有替代。
- 会修改哪些 lockfile。
- 是否需要联网。
不要直接安装。
```

联网审核：

Claude Code 对话输入：

```markdown
如果需要联网，请先说明：
- 要访问的域名。
- 目的。
- 是否会发送本地代码、日志或密钥。
- 如何引用来源。
得到确认前不要联网。
```

提交代码审核：

普通终端命令：

```bash
git status --short
git diff --stat
git diff --cached --stat
```

Claude Code 对话输入：

```markdown
请生成提交前检查报告。
要求：
- 不执行 git commit。
- 检查未暂存和已暂存改动。
- 高亮密钥、删除、权限、依赖、联网、批量重构风险。
- 给出建议提交信息草稿。
```

## 练习任务

练习 1：危险命令分类

Claude Code 对话输入：

```markdown
请把以下命令按风险分类，并说明是否需要人工确认：
- rm -rf dist
- git reset --hard
- npm test
- npm install lodash
- chmod -R 777 .
- git diff
- cat .env
```

练习 2：密钥脱敏流程

Claude Code 对话输入：

```markdown
请设计一个向 Claude Code 提供 .env 信息的安全流程。
要求：
- 不暴露真实值。
- 只提供变量名和错误信息。
- 说明哪些内容不能粘贴。
```

练习 3：联网审核清单

Claude Code 对话输入：

```markdown
请生成一个联网前审核清单。
场景：需要查看第三方库官方文档。
要求：
- 限制官方域名。
- 记录访问日期。
- 不上传本地密钥或私有代码。
```

练习 4：提交前安全检查

普通终端命令：

```bash
git diff --name-status
git diff --check
```

Claude Code 对话输入：

```markdown
请根据我提供的 git diff 输出做提交前安全检查。
不要提交代码。
重点检查删除、权限、依赖、密钥、联网、批量重构。
```

## 验收标准

完成本章后，你应该能够：

- 识别 Claude Code 工作流中的高风险动作。
- 在提示词中设置“先确认后执行”的安全边界。
- 安全处理 `.env`、token、证书、私钥等敏感内容。
- 对依赖安装、联网、提交代码和回退命令建立审批流程。
- 使用 `git status`、`git diff`、`git diff --check` 做基础审查。
- 明确拒绝或暂停会覆盖他人改动的操作。
- 为自动化和子 Agent 设置最小权限。
