# 算法学习流程

> 目标：在学完数据结构主线后，把常见算法思想系统串起来，做到“知道为什么这样做、能分析复杂度、能手写基础版本、能看懂经典题型”。  
> 依据：`课程.md` 中的完整播放列表。  
> 主线：`Abdul Bari 课程一` 为主，`王道考研` 用来补排序、字符串匹配、图算法细节。

## 推荐总路线

### 第一轮主线：先把算法框架搭起来

1. 算法分析基础
2. 递归关系与主定理
3. 分治算法
4. 排序算法
5. 贪心算法
6. 动态规划
7. 图算法
8. 回溯法
9. 分支限界法
10. 字符串匹配
11. NP 问题

### 第二轮强化：补证明、补题感、补模板

1. 手写常见模板
2. 刷经典题
3. 比较相似算法的差异
4. 复盘复杂度与适用边界

---

## 阶段五：贪心算法 ｜ ~1.8h

**目标**：理解“局部最优”为什么有时能导出全局最优。  
**来源**：`Abdul Bari 课程一`

### 主线视频

- 3 Greedy Method - Introduction
- 3.1 Knapsack Problem
- 3.2 Job Sequencing with Deadlines
- 3.3 Optimal Merge Pattern
- 3.4 Huffman Coding
- 3.5 Prim's and Kruskal's Algorithms
- 3.6 Dijkstra Algorithm

### 这一阶段必须会

- 贪心不是“随便选当前最好的”，而是要满足特定结构条件。
- 知道哪些问题适合贪心，哪些不适合。
- 会用排序、堆、并查集、图等结构配合贪心。

### 输出要求

- 自己讲清楚 3 个经典例子为什么能贪心。
- 比较分数背包和 0/1 背包的本质差异。

### 特别提醒（与图算法的巧妙重叠）

这一阶段里的 `Prim / Kruskal / Dijkstra` 已经和图算法有交叉。  
这里经典的思路是：**Abdul Bari 侧重于证明“为什么这三个算法具有贪心选择性质”，而王道侧重于“在图的邻接矩阵/邻接表上怎么把代码写出来”**。  
建议第一次学时重点先跟着 Abdul Bari 抓“贪心思想”，图上具体的结构实现细节留给数据结构/图算法阶段用王道来巩固。

1. 状态是什么？
2. 状态转移从哪里来？
3. 初始条件是什么？
4. 填表顺序为什么这样安排？

## 阶段七：图算法 ｜ ~2.5h

**目标**：把图上的遍历、最短路、最小生成树、拓扑排序串成一个体系。  
**来源**：`Abdul Bari 课程一 + 王道考研`

### 前置依赖

开始这部分前，建议你已经掌握：

- 图的邻接矩阵 / 邻接表表示
- 队列和 BFS
- 递归和 DFS
- 堆 / 优先队列的基本思想

### 主线视频

- 5.1 BFS & DFS（Abdul Bari）
- 5.2 Articulation Point（Abdul Bari）
- 6.4.1 最小生成树（王道）
- 6.4.2_1 最短路径 - BFS（王道）
- 6.4.2_2 最短路径 - Dijkstra（王道）
- 6.4.2_3 最短路径 - Floyd（王道）
- 6.4.4 拓扑排序（王道）
- 6.4.5 关键路径（王道）

### 说明与协同（重要）

这里不再重复“图的定义和存储结构”，默认那部分已经在数据结构流程中完成。  
你会发现这里罗列的 王道 6.4.x 系列视频与你在 `数据结构学习流程.md` 中 **“第二轮强化的图高级应用”** 是完全重合的！这不是冗余，而是说明图的经典算法（最短路、最小生成树、拓扑排序）既是数据结构的巅峰，也是算法设计（贪心/DP应用）的直观体现。  
**注**：你在上一个文件（数据结构）或者当前文件（算法）的任意一侧看懂这些视频即可，不需要看两遍。

### 这一阶段必须会

- BFS 和 DFS 的用途差异。
- 最小生成树和最短路径不是一回事。
- Dijkstra、Floyd、Bellman-Ford 各自适用场景。
- DAG 上的拓扑排序和关键路径思路。

### 输出要求

- 手写 BFS / DFS。
- 说清楚 Prim 和 Kruskal 的区别。
- 说清楚 Dijkstra 为什么不适合带负权边。
