## 第一章 队列的基本概念

### 1.1 什么是队列？

**队列（Queue）** 是一种 **操作受限的线性表**——它只允许在 **一端** 进行插入，在 **另一端** 进行删除。

生活类比：

| 场景 | 类比 |
|------|------|
| 排队买票 | 先来的人先买到票，后来的人排到队尾 |
| 打印机任务队列 | 先提交的任务先打印 |
| 餐厅叫号 | 先取号的人先被叫到 |
| 消息队列 | 先发送的消息先被接收方读到 |

共同规律：**先进来的，先出去。**

### 1.2 FIFO 原则

队列遵循 **FIFO（First In, First Out）——先进先出** 原则。

```
        ┌───────────────────────────────────────┐
   出队 ←│  1  │  2  │  3  │  4  │    │    │    │← 入队
  (front)└───────────────────────────────────────┘ (rear)
             ↑                           ↑
          队头（Front）              队尾（Rear）
```

**核心术语**：

- **队头（Front）**：允许删除（出队）的一端
- **队尾（Rear）**：允许插入（入队）的一端
- **入队 / 进队（Enqueue / Push）**：将元素放到队尾
- **出队 / 离队（Dequeue / Pop）**：将队头元素取走

> **关键理解**：队列本质上还是线性表（数组或链表），只是人为限制了操作的位置——只能在一端插入，另一端删除。这种"限制"恰好反映了现实生活中"先来先服务"的公平原则。

### 1.3 队列的抽象数据类型（ADT）

**ADT（Abstract Data Type，抽象数据类型）** 是一种只规定"能做什么"（提供哪些操作）、隐藏"如何实现"（用数组还是链表）的设计方法。它为使用者定义了一组统一的接口，无论底层存储结构如何变化，对外表现的行为不变。

不论用什么方式实现，队列都必须支持以下操作：

| 操作 | 描述 | 时间复杂度 |
|------|------|-----------|
| `InitQueue()` | 初始化空队列 | O(1) |
| `EnQueue(x)` | 元素 x 入队（到队尾） | O(1) |
| `DeQueue()` | 队头元素出队 | O(1) |
| `GetHead()` | 返回队头元素（不删除） | O(1) |
| `IsEmpty()` | 判断队列是否为空 | O(1) |
| `IsFull()` | 判断是否已满（仅顺序队列） | O(1) |

### 1.4 队列与栈的对比

| 维度 | 栈 | 队列 |
|------|-----|------|
| 原则 | LIFO（后进先出） | FIFO（先进先出） |
| 操作端 | 同一端（栈顶） | 两端（队尾入，队头出） |
| 类比 | 一摞盘子 | 排队买票 |
| 应用 | 括号匹配、表达式求值 | BFS、层序遍历、消息队列 |

## 第二章 队列的顺序存储实现

### 2.1 基本思路

用一个 **数组** 存放队列中的元素，再用两个变量 `front` 和 `rear` 来指示队头和队尾的位置。

```
  索引:    [0]  [1]  [2]  [3]  [4]  ...  [MaxSize-1]
  内容:     10   20   30    _    _   ...     _
              ↑              ↑
           front=0         rear=2（队尾元素在索引2）
```

**两种常见的指针约定**：

#### 约定一：front 指向队头元素的前一位置，rear 指向队尾元素

```
空队:   front = -1, rear = -1
入队:   先判断是否满，rear++，再赋值
出队:   先判断是否空，front++，取元素
队满:   rear == MaxSize - 1
队空:   front == rear
元素数: rear - front
```

这种写法中，真正的队头元素在 `front + 1` 位置；`front` 记录的是"已经出队的最后一个位置"。

#### 约定二：front 指向队头元素，rear 指向队尾元素的下一个位置（本教程采用）

```
空队:   front = 0, rear = 0
入队:   data[rear] = x; rear++;       先放元素，rear 再后移
出队:   x = data[front]; front++;     先取元素，front 再后移
队满:   rear == MaxSize
队空:   front == rear（初始时都为 0）
元素数: rear - front
```

### 2.2 最简单的顺序队列实现

```cpp
#include <iostream>
using namespace std;

const int MaxSize = 100;

struct SqQueue {
    int data[MaxSize];
    int front;  // 队头指针（指向队头元素）
    int rear;   // 队尾指针（指向队尾元素的下一位置）
};

void InitQueue(SqQueue &Q) {
    Q.front = 0;
    Q.rear = 0;
}

bool IsEmpty(SqQueue Q) {
    return Q.front == Q.rear;
}

bool IsFull(SqQueue Q) {
    return Q.rear == MaxSize;
}

// 入队
bool EnQueue(SqQueue &Q, int x) {
    if (IsFull(Q)) return false;
    Q.data[Q.rear] = x;
    Q.rear++;
    return true;
}

// 出队
bool DeQueue(SqQueue &Q, int &x) {
    if (IsEmpty(Q)) return false;
    x = Q.data[Q.front];
    Q.front++;
    return true;
}

// 获取队头元素
bool GetHead(SqQueue Q, int &x) {
    if (IsEmpty(Q)) return false;
    x = Q.data[Q.front];
    return true;
}
```

### 2.3 假溢出问题：为什么需要循环队列？

上面的"简单顺序队列"有一个严重问题——**假溢出**。

```
初始空队:   front=0, rear=0     [ _ , _ , _ , _ , _ ]

入队 A,B,C: front=0, rear=3     [ A , B , C , _ , _ ]

出队 A,B:   front=2, rear=3     [ _ , _ , C , _ , _ ]
                                   ↑
                                front=2

此时希望再入队 D,E,F：
  但 rear=3，rear++ 后变成 4,5
  当 rear=5 时，IsFull() 返回 true！
  可是数组中前两个位置（索引 0,1）明明是空的！
```

**这就是假溢出（假性溢出）**：数组前端明明还有空闲空间（并未真正存满），但 `rear` 指针已单向走到数组末尾，导致程序误判为"已满"。之所以称其为"假"，是因为剩余容量真实存在，只是 `rear` 无法自动回到头部。

### 2.4 循环队列：解决假溢出

**核心思想**：将数组视为一个**环形**空间。当 `rear` 走到数组末尾时，让它绕回到开头。

```
逻辑视图（环形）：
              [MaxSize-1]
              ┌─────────┐
         rear→│         │←front
              │         │
              │         │
              │         │
              └─────────┘
               [0]
```

**实现要点**：通过 **取模运算**（`%`，求余数）实现指针的循环移动。原理：当 `指针值 + 1` 等于 `MaxSize` 时，`MaxSize % MaxSize = 0`，指针自动从末尾回绕到头部。例如 `(5 + 1) % 6 = 0`，即 rear 从索引 5 回到索引 0。

```cpp
// 入队时 rear 的移动
Q.rear = (Q.rear + 1) % MaxSize;

// 出队时 front 的移动
Q.front = (Q.front + 1) % MaxSize;
```

### 2.5 循环队列的队空 vs 队满判定

循环队列中，如果不做额外约定，`front == rear` 无法区分队空和队满——因为队满时 `front` 也可能等于 `rear`！

有三种解决方案：

#### 方案一：牺牲一个存储单元（最常用）

**队满条件**：`(rear + 1) % MaxSize == front`

即 `rear` 指针再往前走一步就碰到 `front` 了，表示队列已满。这意味着数组中始终有一个位置是空的。

```
队满示意图（MaxSize=6）：
    索引:    0    1    2    3    4    5
    内容:  [a]  [b]  [c]  [d]  [e]  [ ]
             ↑                    ↑
           front=0              rear=5

    (rear+1)%6 = 0 == front → 队满 ✅
    索引5是"牺牲"的空位
```

**队空条件**：`front == rear`

```
队空示意图（MaxSize=6）：
    索引:    0    1    2    3    4    5
    内容:  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]
             ↑
         front=rear=0
```

**最大元素个数**：`MaxSize - 1`

之所以能区分：队空时 `front == rear` 指向同一个位置；队满时 `rear` 的下一个位置就是 `front`，两者不再相等，从而消除了二义性。

#### 方案二：增设 size 变量

额外维护一个 `size` 变量记录当前元素个数：

```
队空条件: size == 0
队满条件: size == MaxSize
```

#### 方案三：增设 tag 标志

增设一个 `tag` 变量，每次入队操作置 tag=1，出队操作置 tag=0。

```
队空条件: front == rear && tag == 0
队满条件: front == rear && tag == 1
```

**本教程以方案一（牺牲一个单元）为主**，这是考试和工程中最常用的方法。

### 2.6 循环队列完整实现

```cpp
#include <iostream>
using namespace std;

const int MaxSize = 6;  // 实际最多存 MaxSize-1=5 个元素

struct CircularQueue {
    int data[MaxSize];
    int front;  // 指向队头元素
    int rear;   // 指向队尾元素的下一位置
};

// ========== 初始化 ==========
void InitQueue(CircularQueue &Q) {
    Q.front = 0;
    Q.rear = 0;
}

// ========== 判空 ==========
bool IsEmpty(CircularQueue Q) {
    return Q.front == Q.rear;
}

// ========== 判满 ==========
bool IsFull(CircularQueue Q) {
    return (Q.rear + 1) % MaxSize == Q.front;
}

// ========== 获取队列长度 ==========
int QueueLength(CircularQueue Q) {
    return (Q.rear - Q.front + MaxSize) % MaxSize;
}

// ========== 入队 ==========
bool EnQueue(CircularQueue &Q, int x) {
    if (IsFull(Q)) {
        cout << "队满，无法入队！" << endl;
        return false;
    }
    Q.data[Q.rear] = x;
    Q.rear = (Q.rear + 1) % MaxSize;  // rear 循环后移
    return true;
}

// ========== 出队 ==========
bool DeQueue(CircularQueue &Q, int &x) {
    if (IsEmpty(Q)) {
        cout << "队空，无法出队！" << endl;
        return false;
    }
    x = Q.data[Q.front];
    Q.front = (Q.front + 1) % MaxSize;  // front 循环后移
    return true;
}

// ========== 获取队头 ==========
bool GetHead(CircularQueue Q, int &x) {
    if (IsEmpty(Q)) return false;
    x = Q.data[Q.front];
    return true;
}

// ========== 打印队列 ==========
void PrintQueue(CircularQueue Q) {
    if (IsEmpty(Q)) {
        cout << "队列为空" << endl;
        return;
    }
    cout << "队列: ";
    int i = Q.front;
    while (i != Q.rear) {
        cout << Q.data[i] << " ";
        i = (i + 1) % MaxSize;
    }
    cout << "(front=" << Q.front << ", rear=" << Q.rear << ")" << endl;
}

// ========== 测试 ==========
int main() {
    CircularQueue Q;
    InitQueue(Q);

    EnQueue(Q, 10);
    EnQueue(Q, 20);
    EnQueue(Q, 30);
    EnQueue(Q, 40);
    EnQueue(Q, 50);
    PrintQueue(Q);  // 队列: 10 20 30 40 50 (front=0, rear=5)

    // 此时应该队满（再入队会失败）
    EnQueue(Q, 60);  // 队满，无法入队！

    int x;
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 10
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 20
    PrintQueue(Q);  // 队列: 30 40 50 (front=2, rear=5)

    // 假溢出问题已解决：虽然 rear=5，但 front 已经前移，可以继续入队
    EnQueue(Q, 60);
    EnQueue(Q, 70);
    PrintQueue(Q);  // 队列: 30 40 50 60 70 (front=2, rear=1)

    // 继续出队
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 30
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 40
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 50
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 60
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 70
    DeQueue(Q, x);  // 队空，无法出队！

    return 0;
}
```

### 2.7 循环队列入队出队过程图解

追踪 MaxSize=6，`EnQueue(10,20,30,40,50), DeQueue(10,20), EnQueue(60,70)` 的完整过程：

```
初始:               front=0, rear=0     队列: []                空队

EnQueue(10):        front=0, rear=1     队列: [10, _, _, _, _, _]
EnQueue(20):        front=0, rear=2     队列: [10, 20, _, _, _, _]
EnQueue(30):        front=0, rear=3     队列: [10, 20, 30, _, _, _]
EnQueue(40):        front=0, rear=4     队列: [10, 20, 30, 40, _, _]
EnQueue(50):        front=0, rear=5     队列: [10, 20, 30, 40, 50, _]

队满了吗？ (rear+1)%6 = 0 == front → ✅ 队满

DeQueue()→10:       front=1, rear=5     队列: [_, 20, 30, 40, 50, _]
DeQueue()→20:       front=2, rear=5     队列: [_, _, 30, 40, 50, _]

EnQueue(60):        front=2, rear=0     队列: [_, _, 30, 40, 50, 60]
                (rear=5 → (5+1)%6=0)
EnQueue(70):        front=2, rear=1     队列: [70, _, 30, 40, 50, 60]

    索引:           0     1     2     3     4     5
                 [ 70 ] [  ] [ 30 ] [ 40 ] [ 50 ] [ 60 ]
                         ↑     ↑
                       rear=1 front=2
```

> **关键**：虽然物理上 70 在 30 前面，但在逻辑上 30 先入队且没出队，30 仍然是队头。队列的逻辑顺序由 front→rear 决定，而不是物理存储顺序。

### 2.8 循环队列的另一种设计：front 和 rear 都指向元素

有些教材采用 `front` 指向队头元素，`rear` 指向队尾元素的约定：

```
初始化:     front = 0, rear = -1     （或者 front=0, rear=MaxSize-1）
入队:       rear = (rear+1) % MaxSize;  data[rear] = x;
出队:       x = data[front]; front = (front+1) % MaxSize;
队空:       front == (rear+1) % MaxSize   （即 front 在 rear 的下一个）
队满:       (rear+2) % MaxSize == front   （牺牲一个单元时）
元素个数:   空队为 0；非空时为 (rear - front + MaxSize) % MaxSize + 1
```

这种写法中，`rear` 指向最后一个元素而不是下一位置，判空条件变成了 `front == (rear+1)%MaxSize`。如果继续使用"牺牲一个单元"来区分队空和队满，判满条件要写成 `(rear+2)%MaxSize == front`。不同教材写法不同，**考试时务必看清题目约定**。

### 2.9 两种指针约定对比

| | front 指向队头，rear 指向队尾的下一位置 | front 指向队头，rear 指向队尾 |
|---|---|---|
| 初始化 | `front=0, rear=0` | `front=0, rear=MaxSize-1` |
| 入队 | `data[rear]=x; rear=(rear+1)%M` | `rear=(rear+1)%M; data[rear]=x` |
| 出队 | `x=data[front]; front=(front+1)%M` | `x=data[front]; front=(front+1)%M` |
| 队空 | `front==rear` | `front==(rear+1)%M` |
| 队满 | `(rear+1)%M==front` | `(rear+2)%M==front` |
| 队列长度 | `(rear-front+M)%M` | 空队为 0；非空时为 `(rear-front+M)%M+1` |

**本教程采用第一种（rear指向下一位置）**，绝大多数教材和考研真题使用此约定。

## 第三章 队列的链式存储实现（链队列）

### 3.1 为什么需要链队列？

顺序队列（即使是循环队列）仍有缺点：
- **固定容量**：MaxSize 是常数，太小会溢出，太大会浪费
- **牺牲一个单元**：循环队列最多存 `MaxSize-1` 个元素

**链队列**用链表实现，每次入队动态分配节点，通常不需要预设容量。

### 3.2 设计思路

链队列需要一个头指针 `front` 和一个尾指针 `rear`：

```
链队列结构：
    front                    rear
      |                        |
      v                        v
    [data|next] -> [data|next] -> [data|next] -> nullptr
      ↑                          ↑
     队头                        队尾
```

- **入队**：在 `rear` 后面插入新节点，`rear` 后移（尾插法）
- **出队**：删除 `front` 指向的节点，`front` 后移（头删法）
- 两种实现风格：**带头节点** 和 **不带头节点**

### 3.3 带头节点的链队列

```cpp
#include <iostream>
using namespace std;

struct LinkNode {
    int data;
    LinkNode *next;
    LinkNode(int val = 0) : data(val), next(nullptr) {}
};

struct LinkQueue {
    LinkNode *front;  // 头节点（头节点 next 指向第一个数据节点）
    LinkNode *rear;   // 指向最后一个数据节点（队尾）
};

// ========== 初始化 ==========
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = new LinkNode;  // 创建头节点
    Q.front->next = nullptr;
}

// ========== 判空 ==========
bool IsEmpty(LinkQueue Q) {
    return Q.front == Q.rear;
    // 或者: return Q.front->next == nullptr;
}

// ========== 入队 ==========
void EnQueue(LinkQueue &Q, int x) {
    LinkNode *newNode = new LinkNode(x);
    Q.rear->next = newNode;  // 原队尾指向新节点
    Q.rear = newNode;        // rear 指向新队尾
}

// ========== 出队 ==========
bool DeQueue(LinkQueue &Q, int &x) {
    if (IsEmpty(Q)) return false;
    LinkNode *temp = Q.front->next;  // 第一个数据节点
    x = temp->data;
    Q.front->next = temp->next;      // 跳过该节点
    if (Q.rear == temp) {            // 如果队列只有一个元素
        Q.rear = Q.front;            // 出队后队列为空，rear 也指向头节点
    }
    delete temp;
    return true;
}

// ========== 获取队头 ==========
bool GetHead(LinkQueue Q, int &x) {
    if (IsEmpty(Q)) return false;
    x = Q.front->next->data;
    return true;
}

// ========== 销毁队列 ==========
void DestroyQueue(LinkQueue &Q) {
    while (Q.front != nullptr) {
        Q.rear = Q.front->next;  // 用 rear 暂存下一个节点
        delete Q.front;
        Q.front = Q.rear;
    }
}

// ========== 打印 ==========
void PrintQueue(LinkQueue Q) {
    if (IsEmpty(Q)) {
        cout << "队列为空" << endl;
        return;
    }
    cout << "队列: ";
    LinkNode *curr = Q.front->next;
    while (curr != nullptr) {
        cout << curr->data << " ";
        curr = curr->next;
    }
    cout << endl;
}

// ========== 测试 ==========
int main() {
    LinkQueue Q;
    InitQueue(Q);

    EnQueue(Q, 10);
    EnQueue(Q, 20);
    EnQueue(Q, 30);
    PrintQueue(Q);  // 队列: 10 20 30

    int x;
    GetHead(Q, x);
    cout << "队头: " << x << endl;  // 10

    DeQueue(Q, x); cout << "出队: " << x << endl;  // 10
    DeQueue(Q, x); cout << "出队: " << x << endl;  // 20
    PrintQueue(Q);  // 队列: 30

    DeQueue(Q, x); cout << "出队: " << x << endl;  // 30
    DeQueue(Q, x);  // 队空，无法出队！

    DestroyQueue(Q);
    return 0;
}
```

### 3.4 不带头节点的链队列

```cpp
struct LinkQueueNoHead {
    LinkNode *front;  // 指向第一个数据节点
    LinkNode *rear;   // 指向最后一个数据节点
};

void InitQueue(LinkQueueNoHead &Q) {
    Q.front = Q.rear = nullptr;  // 空队列
}

bool IsEmpty(LinkQueueNoHead Q) {
    return Q.front == nullptr;
}

void EnQueue(LinkQueueNoHead &Q, int x) {
    LinkNode *newNode = new LinkNode(x);
    if (IsEmpty(Q)) {
        Q.front = Q.rear = newNode;  // 第一个节点，front 和 rear 都指向它
    } else {
        Q.rear->next = newNode;
        Q.rear = newNode;
    }
}

bool DeQueue(LinkQueueNoHead &Q, int &x) {
    if (IsEmpty(Q)) return false;
    LinkNode *temp = Q.front;
    x = temp->data;
    Q.front = Q.front->next;
    if (Q.front == nullptr) {       // 队列变空，rear 也要置空
        Q.rear = nullptr;
    }
    delete temp;
    return true;
}
```

### 3.5 链队列过程图解

以带头节点的链队列为例，追踪 `EnQueue(10), EnQueue(20), DeQueue()` 的过程：

```
初始化后:
    front → [头节点|null]
    rear  → [头节点]     （front == rear）

EnQueue(10):
    front → [头节点|·] → [10|null]
                         ↑
                        rear

EnQueue(20):
    front → [头节点|·] → [10|·] → [20|null]
                                   ↑
                                  rear

DeQueue()→10:
    front → [头节点|·] → [20|null]
                         ↑
                        rear
    释放 [10] 节点
    
再次 DeQueue()→20:
    front → [头节点|null]
    rear  → [头节点]     （又回到 front==rear，队空）
    释放 [20] 节点
```

> **关键**：带头节点链队列中，`front` 始终指向头节点不变，只是 `front->next` 变化。不带头节点版本中，出队时 `front` 本身会后移。

### 3.6 顺序队列 vs 链队列 对比

| 维度 | 顺序队列（循环队列） | 链队列 |
|------|---------------------|--------|
| 存储方式 | 连续数组 | 离散节点+指针 |
| 队满 | 有固定容量限制（最多 MaxSize-1） | 通常无固定容量，但可能内存耗尽 |
| 空间 | 可能浪费或溢出 | 不预留连续空位，但每节点多一个指针 |
| 时间 | O(1)，无额外分配开销 | O(1)，但有 new/delete 开销 |
| 缓存 | ✅ 连续内存，缓存友好 | ❌ 节点分散，缓存不友好 |
| 实现复杂度 | 需要处理取模运算 | 指针操作，注意内存释放 |
| 适用 | 元素数量可预估，追求性能 | 元素数量变化大，不确定峰值 |

> **工程实践**：大多数场景用循环队列（如果容量可预估）或 `std::queue`。

## 第四章 双端队列（Deque）

### 4.1 什么是双端队列？

**双端队列（Double-Ended Queue，简称 Deque，读作 "deck"）** 是队列的推广——它允许在 **两端** 进行插入和删除操作。

```
         ┌─────────────────────────────────────┐
  前端出队←│                                     │←后端出队
  前端入队→│  双端队列（Deque）：两端都可操作    │→后端入队
         └─────────────────────────────────────┘
```

对比：

| 数据结构 | 插入端 | 删除端 |
|---------|--------|--------|
| 普通队列（Queue） | 仅后端（队尾） | 仅前端（队头） |
| 栈（Stack） | 仅同一端（栈顶） | 仅同一端（栈顶） |
| 双端队列（Deque） | 两端均可 | 两端均可 |

**Deque 的操作集**：

| 操作 | 描述 | 时间复杂度 |
|------|------|-----------|
| `push_front(x)` | 从队头入队 | O(1) |
| `push_back(x)` | 从队尾入队 | O(1) |
| `pop_front()` | 从队头出队 | O(1) |
| `pop_back()` | 从队尾出队 | O(1) |
| `front()` | 返回队头元素 | O(1) |
| `back()` | 返回队尾元素 | O(1) |

> **注意**：`deque` 是 C++ STL 中的双端队列容器，但标准只规定接口和复杂度，不规定具体底层实现。常见实现不是简单的"数组+两个指针"，而是分段连续空间。本章只讲逻辑概念，STL 实现细节无需掌握。

### 4.2 输出受限 vs 输入受限的双端队列

双端队列可以有进一步的限制：

**输出受限的双端队列**：两端都可以入队，但只有一端可以出队。

```
入队 → [   |   |   |   |   |   |   ] ← 入队
出队 ←
```

**输入受限的双端队列**：只有一端可以入队，但两端都可以出队。

```
          [   |   |   |   |   |   |   ] ← 入队（仅此端）
出队 ←                     出队 →
```

### 4.3 双端队列的典型应用

1. **滑动窗口最大值**：LeetCode 239，用 deque 在 O(n) 时间内解决
2. **撤销/重做功能**：编辑器中的撤销（undo）回退到旧状态，重做（redo）进到新状态
3. **回文检查**：从两端同时取字符比较
4. **任务调度**：某些任务需要优先处理（从队头插队），普通任务从队尾等待
5. **缓存 LRU（Least Recently Used，最近最少使用）的简化版**：最近使用的放前端，最久不用的从后端淘汰

### 4.4 Deque 基础实现（循环数组版）

```cpp
#include <iostream>
using namespace std;

const int MaxSize = 10;

struct Deque {
    int data[MaxSize];
    int front;  // 指向队头元素
    int rear;   // 指向队尾元素的下一位置
};

void InitDeque(Deque &D) {
    D.front = 0;
    D.rear = 0;
}

bool IsEmpty(Deque D) {
    return D.front == D.rear;
}

bool IsFull(Deque D) {
    return (D.rear + 1) % MaxSize == D.front;
}

// 队头入队
bool push_front(Deque &D, int x) {
    if (IsFull(D)) return false;
    D.front = (D.front - 1 + MaxSize) % MaxSize;  // front 前移
    D.data[D.front] = x;
    return true;
}

// 队尾入队
bool push_back(Deque &D, int x) {
    if (IsFull(D)) return false;
    D.data[D.rear] = x;
    D.rear = (D.rear + 1) % MaxSize;
    return true;
}

// 队头出队
bool pop_front(Deque &D, int &x) {
    if (IsEmpty(D)) return false;
    x = D.data[D.front];
    D.front = (D.front + 1) % MaxSize;
    return true;
}

// 队尾出队
bool pop_back(Deque &D, int &x) {
    if (IsEmpty(D)) return false;
    D.rear = (D.rear - 1 + MaxSize) % MaxSize;  // rear 前移
    x = D.data[D.rear];
    return true;
}
```

### 4.5 栈和队列能用 Deque 模拟吗？

**可以。** 因为 Deque 具备栈和队列的全部能力：

```
// 用 Deque 模拟栈（LIFO）
push_back(x), pop_back()    // 用同一端操作就是栈

// 用 Deque 模拟普通队列（FIFO）
push_back(x), pop_front()   // 尾进头出就是队列
```

这也是为什么 C++ 的 `std::stack` 和 `std::queue` 默认底层容器都是 `std::deque`——它足够灵活。

## 第五章 队列的应用

### 5.1 二叉树的层序遍历

**层序遍历（Level-Order Traversal）**：从上到下、从左到右依次访问二叉树的每个节点。

```
         ┌───┐
         │ 1 │ ← 第一层
         └───┘
        /     \
     ┌───┐   ┌───┐
     │ 2 │   │ 3 │ ← 第二层
     └───┘   └───┘
     /   \   /   \
   ┌─┐ ┌─┐ ┌─┐ ┌─┐
   │4│ │5│ │6│ │7│ ← 第三层
   └─┘ └─┘ └─┘ └─┘

层序遍历结果: 1, 2, 3, 4, 5, 6, 7
```

**算法思路**：

```
1. 根节点入队
2. 循环直到队列为空：
   ① 出队一个节点，访问它
   ② 如果它有左孩子，左孩子入队
   ③ 如果它有右孩子，右孩子入队
```

**为什么用队列？**

层序遍历要求"先访问的节点，它的孩子也先被访问"——这就是 FIFO。如果用栈（DFS，Depth-First Search，深度优先搜索），会优先处理后加入的节点，容易沿一条分支深入，无法保证按层访问。

**完整实现**：

```cpp
#include <iostream>
#include <queue>
using namespace std;

struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

// ========== 层序遍历（基础版） ==========
void levelOrder(TreeNode *root) {
    if (root == nullptr) return;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        TreeNode *curr = q.front(); q.pop();
        cout << curr->val << " ";   // 访问当前节点

        if (curr->left)  q.push(curr->left);
        if (curr->right) q.push(curr->right);
    }
}

// ========== 层序遍历（分层版） ==========
// 每层打印一行，方便看出层次结构
void levelOrderByLevel(TreeNode *root) {
    if (root == nullptr) return;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int levelSize = q.size();  // 当前层的节点数
        for (int i = 0; i < levelSize; i++) {
            TreeNode *curr = q.front(); q.pop();
            cout << curr->val << " ";
            if (curr->left)  q.push(curr->left);
            if (curr->right) q.push(curr->right);
        }
        cout << endl;  // 一层结束，换行
    }
}

int main() {
    /* 构建二叉树:
             1
           /   \
          2     3
         / \   / \
        4   5 6   7
    */
    TreeNode *root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);
    root->right->left = new TreeNode(6);
    root->right->right = new TreeNode(7);

    cout << "层序遍历: ";
    levelOrder(root);        // 输出: 1 2 3 4 5 6 7
    cout << endl;

    cout << "按层输出:" << endl;
    levelOrderByLevel(root);
    // 输出:
    // 1
    // 2 3
    // 4 5 6 7

    return 0;
}
```

第1轮: 出队 1

### 5.2 广度优先搜索（BFS）

**广度优先搜索（Breadth-First Search）** 是图和树中最重要的遍历算法之一，核心数据结构就是**队列**。

#### 图的 BFS

```cpp
#include <iostream>
#include <queue>
#include <vector>
using namespace std;

// 用邻接表表示图
void BFS(const vector<vector<int>> &graph, int start) {
    int n = graph.size();
    vector<bool> visited(n, false);

    queue<int> q;
    q.push(start);
    visited[start] = true;

    while (!q.empty()) {
        int curr = q.front(); q.pop();
        cout << curr << " ";  // 访问当前节点

        // 将所有未访问的邻居入队
        for (int neighbor : graph[curr]) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                q.push(neighbor);
            }
        }
    }
}

int main() {
    // 图: 0-1-2-3
    //     |   |
    //     4   5
    vector<vector<int>> graph = {
        {1, 4},       // 0 的邻居
        {0, 2},       // 1 的邻居
        {1, 3, 5},    // 2 的邻居
        {2},          // 3 的邻居
        {0},          // 4 的邻居
        {2}           // 5 的邻居
    };

    cout << "BFS 从 0 开始: ";
    BFS(graph, 0);  // 输出: 0 1 4 2 3 5
    cout << endl;

    return 0;
}
```

**BFS 为什么用队列？**

因为 BFS 要求"按层"访问——先访问离起点最近的节点，再访问稍远的节点。这正是 **FIFO** 的顺序：先入队的节点（离起点近的）先被处理，它的邻居随后入队，排在当前队列已有元素之后，所以稍后才会被处理。

**BFS 与队列的关系**：

```
BFS 的"层"和队列状态：
         初始层（距离0）:    起点
                            ↓ 入队
         队列: [起点]
                            ↓ 出队起点，入队其邻居
         第一层（距离1）:    起点的所有邻居
         队列: [邻居1, 邻居2, ...]
                            ↓ 依次出队，入队它们的邻居
         第二层（距离2）:    邻居的邻居
         ... 
```

**BFS 的应用**：

| 应用 | 描述 |
|------|------|
| 最短路径（无权图） | BFS 第一次到达目标节点时的路径即最短路径 |
| 连通分量 | 从某点 BFS，遍历到的所有节点属于同一连通分量 |
| 二分图（Bipartite Graph，顶点可被分为两组的图）检测 | BFS 中相邻节点不能同色，能成功染色的图即为二分图 |
| 单词阶梯 | LeetCode 127，每次改变一个字母，BFS 找最短路径 |
| 迷宫最短路径 | 从起点 BFS 到终点的步数即最短路径 |

### 5.3 其他应用场景

1. **CPU 任务调度**：先到达的任务先获得 CPU 时间片
2. **消息队列**：生产者-消费者模型，生产者将消息入队，消费者从队头取出处理
3. **打印任务队列**：先提交的文档先打印
4. **键盘缓冲区**：输入的字符先被缓存，然后按输入顺序处理
5. **广度优先的网络爬虫**：先爬取当前页面的链接，再爬取更深的链接

### 5.4 用两个栈模拟队列

这是队列与栈的经典转换题：**栈是 LIFO，队列是 FIFO**。一个栈无法直接实现 FIFO，但两个栈可以。

核心思路：

- `inStack` 专门负责入队，新元素直接压入 `inStack`
- `outStack` 专门负责出队，队头元素在 `outStack` 的栈顶
- 当 `outStack` 为空且需要出队时，把 `inStack` 的元素全部倒入 `outStack`

```
入队 1,2,3 后:

inStack:   栈顶 → 3
                  2
                  1

outStack:  空

第一次出队前，把 inStack 全部倒入 outStack:

inStack:   空

outStack:  栈顶 → 1   ← 最早入队的元素到了栈顶
                  2
                  3
```

完整实现：

```cpp
#include <iostream>
#include <stack>
using namespace std;

class MyQueue {
private:
    stack<int> inStack;
    stack<int> outStack;

    void moveInToOut() {
        while (!inStack.empty()) {
            outStack.push(inStack.top());
            inStack.pop();
        }
    }

public:
    void push(int x) {
        inStack.push(x);
    }

    bool pop(int &x) {
        if (outStack.empty()) moveInToOut();
        if (outStack.empty()) return false;
        x = outStack.top();
        outStack.pop();
        return true;
    }

    bool front(int &x) {
        if (outStack.empty()) moveInToOut();
        if (outStack.empty()) return false;
        x = outStack.top();
        return true;
    }

    bool empty() const {
        return inStack.empty() && outStack.empty();
    }
};

int main() {
    MyQueue q;
    q.push(1);
    q.push(2);
    q.push(3);

    int x;
    q.front(x); cout << "队头: " << x << endl;  // 1

    while (q.pop(x)) {
        cout << x << " ";
    }
    // 输出: 1 2 3
    return 0;
}
```

**复杂度分析**：

| 操作 | 最坏时间 | 均摊时间 | 说明 |
|------|----------|----------|------|
| `push` | O(1) | O(1) | 直接压入 `inStack` |
| `pop` | O(n) | O(1) | 偶尔需要整体倒栈 |
| `front` | O(n) | O(1) | 偶尔需要整体倒栈 |

为什么 `pop` 和 `front` 是均摊 O(1)？因为每个元素最多经历三步：进入 `inStack` 一次、转移到 `outStack` 一次、从 `outStack` 弹出一次。对 n 个元素来说总操作量是 O(n)，平均到每次队列操作就是 O(1)。

### 5.5 用两个队列模拟栈

栈要求 **后进先出**，而队列只能 **先进先出**。用两个队列模拟栈时，常见做法是：入栈直接进入主队列，出栈时把前面的元素临时搬走，只留下最后入队的元素。

```
入栈 1,2,3 后:

q1: [1, 2, 3]   ← 3 是最后入栈的元素，也是栈顶
q2: []

pop() 时:
1. 把 1,2 从 q1 搬到 q2
   q1: [3]
   q2: [1, 2]

2. 弹出 q1 中剩下的 3

3. 交换 q1 和 q2
   q1: [1, 2]
   q2: []
```

完整实现：

```cpp
#include <iostream>
#include <queue>
using namespace std;

class MyStack {
private:
    queue<int> q1;  // 主队列
    queue<int> q2;  // 临时队列

public:
    void push(int x) {
        q1.push(x);
    }

    bool pop(int &x) {
        if (q1.empty()) return false;

        while (q1.size() > 1) {
            q2.push(q1.front());
            q1.pop();
        }

        x = q1.front();
        q1.pop();
        q1.swap(q2);
        return true;
    }

    bool top(int &x) {
        if (q1.empty()) return false;

        while (q1.size() > 1) {
            q2.push(q1.front());
            q1.pop();
        }

        x = q1.front();
        q2.push(q1.front());  // top 只查看，不删除
        q1.pop();
        q1.swap(q2);
        return true;
    }

    bool empty() const {
        return q1.empty();
    }
};

int main() {
    MyStack st;
    st.push(1);
    st.push(2);
    st.push(3);

    int x;
    st.top(x); cout << "栈顶: " << x << endl;  // 3

    while (st.pop(x)) {
        cout << x << " ";
    }
    // 输出: 3 2 1
    return 0;
}
```

**复杂度**：

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| `push` | O(1) | 直接入队 |
| `pop` | O(n) | 需要搬运前 n-1 个元素 |
| `top` | O(n) | 需要搬运前 n-1 个元素 |
| `empty` | O(1) | 直接判断主队列 |

也可以采用另一种写法：每次 `push` 后主动调整队列顺序，让新元素排到队头。这样 `push` 是 O(n)，但 `pop` 和 `top` 可以做到 O(1)。

### 5.6 约瑟夫环问题（队列模拟）

**约瑟夫环**：n 个人围成一圈，从第 1 个人开始报数，每数到 k 的人出列，问出列顺序或最后剩下的人。

队列模拟的关键是：

- 没数到 k 的人：出队后再入队，相当于移动到队尾
- 数到 k 的人：出队后不再入队，表示出列

以 `n=5, k=2` 为例：

```
初始队列: [1, 2, 3, 4, 5]

第1轮: 1 报数1，移到队尾；2 报数2，出列
       队列: [3, 4, 5, 1]     出列: 2

第2轮: 3 报数1，移到队尾；4 报数2，出列
       队列: [5, 1, 3]        出列: 4

第3轮: 5 报数1，移到队尾；1 报数2，出列
       队列: [3, 5]           出列: 1

第4轮: 3 报数1，移到队尾；5 报数2，出列
       队列: [3]              出列: 5

第5轮: 3 出列
       出列顺序: 2, 4, 1, 5, 3
       最后剩下: 3
```

完整实现：

```cpp
#include <iostream>
#include <queue>
#include <vector>
using namespace std;

vector<int> josephus(int n, int k) {
    vector<int> order;
    if (n <= 0 || k <= 0) return order;

    queue<int> q;
    for (int i = 1; i <= n; i++) {
        q.push(i);
    }

    while (!q.empty()) {
        for (int count = 1; count < k; count++) {
            q.push(q.front());  // 没数到 k，移到队尾
            q.pop();
        }

        order.push_back(q.front());  // 数到 k，出列
        q.pop();
    }

    return order;
}

int main() {
    vector<int> order = josephus(5, 2);

    cout << "出列顺序: ";
    for (int x : order) {
        cout << x << " ";
    }
    cout << endl;

    if (!order.empty()) {
        cout << "最后剩下: " << order.back() << endl;
    }

    return 0;
}
```

**复杂度**：每出列一个人，需要旋转 `k-1` 次，所以时间复杂度是 O(nk)，空间复杂度是 O(n)。如果只求最后剩下的编号，可以用数学递推做到 O(n)，但队列模拟更适合理解过程。

### 5.7 单调队列与滑动窗口最大值

**单调队列**通常用 `deque` 实现。它不是普通 FIFO 队列，而是维护一个"从队头到队尾单调递减"的候选队列，让队头永远是当前窗口最大值。

解决 LeetCode 239「滑动窗口最大值」时，队列中一般存 **下标**，而不是直接存值：

- 队头下标如果已经离开窗口，就从队头删除
- 新元素入队前，把队尾所有小于等于它的元素删除
- 当前窗口形成后，队头下标对应的值就是最大值

```cpp
#include <iostream>
#include <deque>
#include <vector>
using namespace std;

vector<int> maxSlidingWindow(vector<int> &nums, int k) {
    vector<int> ans;
    deque<int> dq;  // 存下标，保证 nums[dq[0]] >= nums[dq[1]] >= ...

    for (int i = 0; i < (int)nums.size(); i++) {
        // 1. 删除已经离开窗口的下标
        while (!dq.empty() && dq.front() <= i - k) {
            dq.pop_front();
        }

        // 2. 删除队尾中不可能再成为最大值的元素
        while (!dq.empty() && nums[dq.back()] <= nums[i]) {
            dq.pop_back();
        }

        // 3. 当前元素入队
        dq.push_back(i);

        // 4. 窗口长度达到 k 后，记录最大值
        if (i >= k - 1) {
            ans.push_back(nums[dq.front()]);
        }
    }

    return ans;
}

int main() {
    vector<int> nums = {1, 3, -1, -3, 5, 3, 6, 7};
    int k = 3;

    vector<int> ans = maxSlidingWindow(nums, k);
    for (int x : ans) {
        cout << x << " ";
    }
    // 输出: 3 3 5 5 6 7

    return 0;
}
```

手动模拟 `nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3`：

| i | nums[i] | 单调队列中的值 | 当前窗口 | 输出 |
|---|---------|----------------|----------|------|
| 0 | 1 | [1] | 未形成 | - |
| 1 | 3 | [3] | 未形成 | - |
| 2 | -1 | [3, -1] | [1, 3, -1] | 3 |
| 3 | -3 | [3, -1, -3] | [3, -1, -3] | 3 |
| 4 | 5 | [5] | [-1, -3, 5] | 5 |
| 5 | 3 | [5, 3] | [-3, 5, 3] | 5 |
| 6 | 6 | [6] | [5, 3, 6] | 6 |
| 7 | 7 | [7] | [3, 6, 7] | 7 |

**为什么要删掉队尾较小或相等的元素？** 因为如果新元素不小于旧元素且下标更靠后，那么队尾那些旧元素不可能在未来任何窗口中成为最大值。

**复杂度**：每个下标最多入队一次、出队一次，所以总时间复杂度是 O(n)，空间复杂度是 O(k)。

### 5.8 队列在操作系统中的应用

队列在操作系统中非常常见，因为操作系统经常要处理"先到先服务"或"按到达顺序缓冲"的问题。

| 场景 | 队列作用 | 说明 |
|------|----------|------|
| 键盘缓冲区 | 保存输入字符 | 用户输入的字符先进入缓冲区，程序按输入顺序读取 |
| 生产者-消费者缓冲区 | 协调速度差 | 生产者把数据放入缓冲区，消费者按顺序取走 |
| SPOOLing（Simultaneous Peripheral Operations On-Line，假脱机）技术 | 模拟独占设备 | 多个作业的输出先进入输出井，再由打印进程按队列顺序输出 |
| FCFS 调度 | 先来先服务 | 作业或进程按到达顺序排队，队头先被调度 |
| 时间片轮转 | 就绪队列 | 进程用完时间片后回到就绪队列队尾 |

#### 缓冲区：解决速度不匹配

例如键盘输入和程序读取的速度不同。键盘输入字符时，操作系统先把字符放入缓冲区：

```
键盘输入:   a  b  c  d
             ↓  ↓  ↓  ↓
缓冲队列:  [a, b, c, d]
             ↑
          程序从队头读取
```

只要缓冲区没满，输入设备和程序就不必严格同步。这正是队列适合做缓冲区的原因。

#### SPOOLing（假脱机技术）：把独占设备改造成共享效果

打印机是典型的独占设备，同一时刻只能打印一个任务。SPOOLing 会把多个打印任务先放到磁盘上的输出井，再由系统打印进程按顺序取出：

```
作业A输出 ┐
作业B输出 ├──> 输出井 / 打印队列: [A, B, C] ──> 打印机
作业C输出 ┘
```

从用户角度看，多个作业似乎都能"同时提交打印"，但真正执行打印时仍然是队列顺序。

#### FCFS 与时间片轮转

FCFS（First Come, First Served）调度直接体现 FIFO：

```
就绪队列: [P1, P2, P3, P4]
            ↑
          先调度 P1
```

时间片轮转也会使用队列：队头进程获得一个时间片，如果时间片用完还没结束，就回到队尾等待下一轮。

### 5.9 优先队列底层：堆

优先队列不是 FIFO，它总是让**优先级最高**的元素先出队。C++ 的 `std::priority_queue` 默认使用**大顶堆**实现。

大顶堆满足：

- 堆是一棵完全二叉树
- 每个节点的值都大于或等于它的孩子
- 堆顶元素就是当前最大值

```
逻辑结构（大顶堆）:

             50
           /    \
         30      40
        /  \
      10    20

数组存储:
索引:   0   1   2   3   4
值:    [50, 30, 40, 10, 20]
```

堆通常用数组存储。对于下标 `i`：

| 关系 | 公式 |
|------|------|
| 父节点 | `(i - 1) / 2` |
| 左孩子 | `2 * i + 1` |
| 右孩子 | `2 * i + 2` |

#### 入队：上浮

插入新元素时，先放到数组末尾，再不断和父节点比较。如果比父节点更大，就交换位置。

```
插入 60:

原堆数组: [50, 30, 40, 10, 20]
先放末尾: [50, 30, 40, 10, 20, 60]

60 比父节点 40 大，交换:
[50, 30, 60, 10, 20, 40]

60 比父节点 50 大，继续交换:
[60, 30, 50, 10, 20, 40]
```

#### 出队：下沉

删除堆顶时，先用最后一个元素覆盖堆顶，再不断和较大的孩子交换，直到重新满足堆性质。

```
弹出 60:

原堆数组: [60, 30, 50, 10, 20, 40]
末尾 40 放到堆顶:
[40, 30, 50, 10, 20]

40 比较两个孩子 30 和 50，和较大的 50 交换:
[50, 30, 40, 10, 20]
```

复杂度：

| 操作 | 优先队列语义 | 堆中做了什么 | 时间复杂度 |
|------|--------------|--------------|------------|
| `push` | 插入元素 | 末尾插入 + 上浮 | O(log n) |
| `pop` | 删除最高优先级元素 | 删除堆顶 + 下沉 | O(log n) |
| `top` | 查看最高优先级元素 | 读取堆顶 | O(1) |
| `empty` / `size` | 状态查询 | 读取成员变量 | O(1) |

> **注意**：优先队列只保证每次取出的元素优先级最高，不保证相同优先级元素之间保持原入队顺序。也就是说，`priority_queue` **不是稳定（stable）的**——同优先级元素的出队顺序可能与入队顺序不同。

## 第六章 C++ STL 中的队列

### 6.1 std::queue

C++ 标准库提供了 `std::queue`，这是一个**容器适配器**（底层默认用 `std::deque`）。

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;

    q.push(10);
    q.push(20);
    q.push(30);

    cout << "队头: " << q.front() << endl;  // 10
    cout << "队尾: " << q.back() << endl;   // 30
    cout << "大小: " << q.size() << endl;   // 3

    while (!q.empty()) {
        cout << q.front() << " ";
        q.pop();   // 注意：STL 的 pop() 不返回值！
    }
    // 输出: 10 20 30
    return 0;
}
```

| 方法 | 功能 | 注意 |
|------|------|------|
| `push(x)` | 入队（队尾插入） | |
| `pop()` | 出队（队头删除） | **不返回值；空队列调用是未定义行为** |
| `front()` | 返回队头元素的引用 | 空队列调用是未定义行为 |
| `back()` | 返回队尾元素的引用 | 空队列调用是未定义行为 |
| `empty()` | 是否为空 | |
| `size()` | 元素个数 | |

### 6.2 std::deque

C++ `std::deque` 是**双端队列**，功能比 `queue` 更强：

```cpp
#include <iostream>
#include <deque>
using namespace std;

int main() {
    deque<int> dq;

    dq.push_back(20);   // 队尾入队: [20]
    dq.push_back(30);   // 队尾入队: [20, 30]
    dq.push_front(10);  // 队头入队: [10, 20, 30]
    dq.push_back(40);   // 队尾入队: [10, 20, 30, 40]

    cout << "队头: " << dq.front() << endl;  // 10
    cout << "队尾: " << dq.back() << endl;   // 40

    dq.pop_front();  // 出队头: [20, 30, 40]
    dq.pop_back();   // 出队尾: [20, 30]

    cout << "deque: ";
    for (int x : dq) cout << x << " ";  // 20 30
    cout << endl;

    // deque 支持随机访问！
    cout << "dq[0] = " << dq[0] << endl;   // 20
    cout << "dq[1] = " << dq[1] << endl;   // 30

    return 0;
}
```

| 方法 | 功能 | 注意 |
|------|------|------|
| `push_front(x)` | 队头插入 | |
| `push_back(x)` | 队尾插入 | |
| `pop_front()` | 队头删除 | |
| `pop_back()` | 队尾删除 | |
| `front()` | 返回队头引用 | |
| `back()` | 返回队尾引用 | |
| `operator[]` | 随机访问 | O(1) |
| `size()` | 元素个数 | |
| `empty()` | 是否为空 | |

> **std::deque 的底层**：标准不规定具体实现。常见实现不是简单的"循环数组"，而是一系列固定大小的连续内存块（block）加上一张索引表。这使得它在两端插入删除都是 O(1)，同时支持 O(1) 的随机访问（通常比 `std::list` 快，随机访问的常数因子通常比 `std::vector` 大）。

### 6.3 std::priority_queue（了解）

**优先队列**不是 FIFO——优先级最高的元素先出队。

```cpp
#include <iostream>
#include <queue>
#include <vector>
#include <functional>
using namespace std;

int main() {
    // 默认是大顶堆（最大值优先）
    priority_queue<int> pq;
    pq.push(30);
    pq.push(10);
    pq.push(50);
    pq.push(20);

    while (!pq.empty()) {
        cout << pq.top() << " ";  // 50 30 20 10
        pq.pop();
    }
    cout << endl;

    // 小顶堆（最小值优先）
    priority_queue<int, vector<int>, greater<int>> minPq;
    minPq.push(30);
    minPq.push(10);
    minPq.push(50);
    minPq.push(20);

    while (!minPq.empty()) {
        cout << minPq.top() << " ";  // 10 20 30 50
        minPq.pop();
    }
    cout << endl;

    return 0;
}
```

### 7.2 必须牢记的核心结论

#### 循环队列核心公式（rear 指向下一位置，牺牲一个单元）

| 条件 / 计算 | 公式 |
|-------------|------|
| 队空 | `front == rear` |
| 队满 | `(rear + 1) % MaxSize == front` |
| 入队 | `data[rear] = x; rear = (rear+1)%M` |
| 出队 | `x = data[front]; front = (front+1)%M` |
| 队列长度 | `(rear - front + MaxSize) % MaxSize` |
| 最多元素 | `MaxSize - 1` |

#### 链队列要点

| 要点 | 结论 |
|------|------|
| 入队操作 | 尾插法，修改 rear |
| 出队操作 | 头删法，修改 front->next |
| 队空条件 | `front == rear`（带头节点）或 `front == nullptr`（不带头节点） |
| 特殊处理 | 出队最后一个元素时，rear 也要回退到 front/头节点 |
| 何时用链队列 | 元素数量无法预估、变化大时 |

#### 栈与队列对比

| 对比项 | 栈 | 队列 |
|--------|-----|------|
| 原则 | LIFO | FIFO |
| 插入端 | 栈顶 | 队尾 |
| 删除端 | 栈顶 | 队头 |
| 底层容器 | 通常 deque/vector | 通常 deque |
| 典型应用 | 括号匹配、表达式、DFS | 层序遍历、BFS |

### 7.3 经典易错点

**❶ 循环队列队满判定别搞混**

```cpp
// ✅ 正确（牺牲一个单元法）
if ((Q.rear + 1) % MaxSize == Q.front)  // 队满

// ❌ 错误
if (Q.rear + 1 == Q.front)  // 没有取模！当 rear=MaxSize-1, front=0 时会出错
```

**❷ 循环队列长度公式必须加 MaxSize 再取模**

```cpp
// ✅ 正确
int len = (Q.rear - Q.front + MaxSize) % MaxSize;

// ❌ 错误：当 rear < front 时为负数
int len = Q.rear - Q.front;
```

**❸ 链队列删除最后一个节点后 rear 必须更新**

```cpp
// 带头节点链队列出队时：
if (Q.rear == temp) {       // temp 是出队节点
    Q.rear = Q.front;       // 必须更新 rear 指向头节点！
}
```

不带头节点版本：
```cpp
if (Q.front == nullptr) {   // 出队后队空
    Q.rear = nullptr;       // rear 也要置空
}
```

**❹ STL queue 的 pop() 不返回值**

```cpp
// ❌ 错误
int x = q.pop();  // 编译错误！pop() 返回 void

// ✅ 正确
int x = q.front();
q.pop();
```

**❺ 区分 front 和 rear 的含义**

有多种约定，考试务必看清题目用的是哪种：

```
本教程使用：front 指向队头，rear 指向队尾的下一位置
另一种约定：front 指向队头，rear 指向队尾
```

不同约定导致入队/出队的指针操作顺序不同、队空条件不同。

**❻ BFS 中需要在入队时标记 visited，而不是出队时**

```cpp
// ✅ 正确：入队时标记
q.push(neighbor);
visited[neighbor] = true;  // 立即标记

// ❌ 错误：出队时标记（会导致重复入队）
q.push(neighbor);
// visited[neighbor] = true;  // 忘了标记 → 弹出时才标记
// → 同一个节点可能被多个父节点重复加入队列！
```

```
这是一个经典问题——其实就是队列的简单模拟：

期望输出: 1, 2, 3, 4

到达 1：期望就是 1，直接通过 ✓          队列: []
到达 2：期望下一个是 2，直接通过 ✓      队列: []
到达 3：期望下一个是 3，直接通过 ✓      队列: []
到达 4：期望下一个是 4，直接通过 ✓      队列: []
```

但如果是：

```
到达顺序: 3, 1, 2, 4
期望输出: 1, 2, 3, 4

到达 3：期望是 1，非 3，入队              队列: [3]
到达 1：期望是 1 ✓，通过 ✓              队列: [3]
        → 输出后期望变为 2，检查队头 3 ≠ 2，不匹配
到达 2：期望是 2 ✓，直接通过 ✓          队列: [3]
        → 输出后期望变为 3
        检查队头 3 == 3，出队 ✓          队列: []
        → 输出后期望变为 4
到达 4：期望是 4 ✓，直接通过 ✓          队列: []
```

#### 题目4：判断输出序列能否通过队列实现

> 输入序列为 1,2,3,4,5（依次入队），判断以下输出序列是否可能是队列的输出序列：
> A. 1,2,3,4,5
> B. 5,4,3,2,1

**队列是 FIFO**，出队顺序必须和入队顺序一致。所以：

```
A. 1,2,3,4,5  ✅ 队列 FIFO，完全一致
B. 5,4,3,2,1  ❌ 这是栈（LIFO）的输出，队列无法实现
```

**答案：A 可以，B 不可以**

> **思考**：如果使用双端队列（deque），能否实现 B 的输出序列？
> 
> 可以。但需要合适的操作序列：push_back(1), push_back(2), push_back(3), push_back(4), push_back(5), pop_back(), pop_back(), pop_back(), pop_back(), pop_back()。注意这实际上是用 deque 模拟栈了。

#### 题目5：BFS 求最短路径

> 在无权图中，从节点 0 到节点 5，求最短路径长度（最少边数）。
> 邻接表：
> ```
> 0: 1, 2
> 1: 0, 3
> 2: 0, 3, 4
> 3: 1, 2, 5
> 4: 2, 5
> 5: 3, 4
> ```

```
BFS 过程（记录每个节点的距离）：

第0步: 从0出发
  队列: [0],   dist[0]=0

第1步: 出队0, 入队邻居 1, 2
  队列: [1,2], dist[1]=1, dist[2]=1

第2步: 出队1, 入队邻居 3（未访问过）
       出队2, 入队邻居 4（3已入队不再重复）
  队列: [3,4], dist[3]=2, dist[4]=2

第3步: 出队3, 入队邻居 5（未访问）
  到达目标5！dist[5]=3

最短路径长度: 3 (0→2→4→5 或 0→1→3→5 等)
```

**答案：最少需要 3 条边**

#### 题目6：循环队列指针变化

> 设循环队列采用牺牲一个单元法，数组长度 `MaxSize=8`，初始 `front=rear=0`。依次执行：
> `EnQueue(a), EnQueue(b), EnQueue(c), EnQueue(d), DeQueue(), DeQueue(), EnQueue(e), EnQueue(f), EnQueue(g)`。
> 求最终 `front`、`rear` 的值和队列中的逻辑序列。

```
初始: front=0, rear=0, 队列=[]

EnQueue(a): rear=1, 队列=[a]
EnQueue(b): rear=2, 队列=[a,b]
EnQueue(c): rear=3, 队列=[a,b,c]
EnQueue(d): rear=4, 队列=[a,b,c,d]

DeQueue():  front=1, 出队 a, 队列=[b,c,d]
DeQueue():  front=2, 出队 b, 队列=[c,d]

EnQueue(e): rear=5, 队列=[c,d,e]
EnQueue(f): rear=6, 队列=[c,d,e,f]
EnQueue(g): rear=7, 队列=[c,d,e,f,g]

#### 题目7：循环队列最多还能入队几个元素

> 循环队列数组长度 `MaxSize=10`，采用 `front` 指向队头、`rear` 指向队尾下一位置、牺牲一个单元法。当前 `front=7, rear=3`。问：
> 1. 当前队列长度是多少？
> 2. 队列是否已满？
> 3. 最多还能入队几个元素？

```
队列长度 = (rear - front + MaxSize) % MaxSize
         = (3 - 7 + 10) % 10
         = 6

最大容量 = MaxSize - 1 = 9
还能入队 = 9 - 6 = 3
```

判满也可以直接看：

```
(rear + 1) % MaxSize = (3 + 1) % 10 = 4
front = 7
4 != 7，所以未满
```

**答案：当前 6 个元素，未满，最多还能入队 3 个。**

#### 题目8：循环队列连续操作后的判空判满

> 循环队列数组长度 `MaxSize=5`，采用牺牲一个单元法。初始为空，依次执行：
> `EnQueue(1), EnQueue(2), EnQueue(3), EnQueue(4), DeQueue(), EnQueue(5), EnQueue(6)`。
> 哪些操作成功？最终队列状态如何？

```
MaxSize=5，最多只能存 4 个元素。

初始: front=0, rear=0

EnQueue(1): 成功，rear=1，队列=[1]
EnQueue(2): 成功，rear=2，队列=[1,2]
EnQueue(3): 成功，rear=3，队列=[1,2,3]
EnQueue(4): 成功，rear=4，队列=[1,2,3,4]

此时 (rear+1)%5 = 0 == front，队满。

DeQueue():  成功，front=1，出队 1，队列=[2,3,4]
EnQueue(5): 成功，rear=(4+1)%5=0，队列=[2,3,4,5]

此时 (rear+1)%5 = 1 == front，再次队满。

EnQueue(6): 失败，队列已满
```

**答案：最后一次 `EnQueue(6)` 失败；最终 `front=1, rear=0`，逻辑序列为 `2, 3, 4, 5`。**

#### 题目9：根据长度反推 rear

> 某循环队列数组长度 `MaxSize=12`，采用 `rear` 指向队尾下一位置的约定。已知 `front=9`，当前队列长度为 5，求 `rear` 的值。

由长度公式：

```
length = (rear - front + MaxSize) % MaxSize
5 = (rear - 9 + 12) % 12
5 = (rear + 3) % 12
```

所以：

```
rear = (front + length) % MaxSize
     = (9 + 5) % 12
     = 2
```

**答案：`rear=2`。**

#### 题目10：判断循环队列操作是否合法

> 循环队列数组长度 `MaxSize=6`，采用牺牲一个单元法。当前 `front=3, rear=1`。若连续执行 3 次入队操作，是否合法？

先计算当前长度：

```
length = (rear - front + MaxSize) % MaxSize
       = (1 - 3 + 6) % 6
       = 4
```

最大容量：

```
MaxSize - 1 = 5
```

当前已经有 4 个元素，只剩 1 个空位可用。因此：

```
第1次入队：成功
第2次入队：失败（队满）
第3次入队：失败（仍然队满）
```

**答案：不合法。最多只能再成功入队 1 次，不能连续成功执行 3 次入队。**

### 7.5 学习建议

1. **手画循环队列**：在纸上画一个环，手动模拟 front/rear 的移动，这是理解循环队列的最好方法
2. **区分假溢出和真溢出**：真溢出是数组中所有位置真的满了，无法再插入；假溢出是数组前半部分还有空位，但 rear 已单向走到数组末尾，无法继续入队。循环队列通过取模运算把 rear "绕回"数组开头，从而解决假溢出
3. **链队列注意边界**：空队列入队、最后一个元素出队，这两个边界情况代码处理不同
4. **层序遍历先理解再写代码**：先手动模拟几次层序遍历的过程，再理解队列在其中扮演的角色
5. **BFS 的 visited 数组**：一定在入队时设置 visited，防止重复入队
6. **STL 优先用起来**：实际做题用 `std::queue` 和 `std::deque`，手写版本只在考试中需要
