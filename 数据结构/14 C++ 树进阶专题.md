## 1.1 树的基本术语

```
          A          ← 根节点 (Root)
        /   \
       B     C       ← A 的孩子节点
      / \     \
     D   E     F     ← D、F 是叶子节点；E 还有孩子
        / \
       G   H
```

| 术语 | 定义 | 上图示例 |
|------|------|---------|
| **节点的度 (Degree)** | 一个节点拥有的**子树数目** | deg(A)=2, deg(B)=2, deg(C)=1, deg(D)=0 |
| **树的度** | 树中所有节点的度的**最大值** | max(2,2,1,0,2,0,0,0) = 2 |
| **叶子节点 (Leaf)** | 度为 0 的节点 | D, G, H, F |
| **分支节点** | 度 ≥ 1 的节点 | A, B, C, E |
| **节点的层次 (Level)** | 根为第 1 层，其孩子为第 2 层，以此类推 | A→1, B/C→2, D/E/F→3, G/H→4 |
| **树的高度/深度 (Height)** | 树中节点的**最大层次** | 高度 = 4 |
| **路径** | 从一个节点到另一个节点经过的节点序列 | A→B→E→G |
| **路径长度** | 路径上经过的**边数** | A→B→E→G 路径长度=3 |
| **双亲 (Parent)** | 有子树的节点是其子树根的双亲 | B 是 D 和 E 的双亲 |
| **兄弟 (Sibling)** | 同一双亲的孩子互为兄弟 | D 和 E 是兄弟 |
| **祖先 (Ancestor)** | 从根到该节点路径上的所有节点 | G 的祖先: A, B, E |
| **子孙 (Descendant)** | 以某节点为根的子树中的所有节点 | B 的子孙: D, E, G, H |
| **森林 (Forest)** | m (m≥0) 棵互不相交的树的集合 | 删除根 A 后，得到以 B、C 为根的森林 |

## 1.2 满二叉树与完全二叉树

### 满二叉树 (Perfect Binary Tree)

**定义**：一棵高度为 h 的二叉树，如果它包含 **2^h - 1** 个节点，则称为满二叉树。即每一层都被完全填满。

> 注意：英文资料中 `full binary tree` 有时指“每个节点的孩子数只能是 0 或 2”的严格二叉树；这里的“满二叉树”对应 `perfect binary tree`。

```
高度 h = 3 的满二叉树，共 2³-1 = 7 个节点

         1
       /   \
      2     3
     / \   / \
    4   5 6   7
```

**特点**：
- 第 i 层有 **2^(i-1)** 个节点
- 叶子节点全部在最后一层
- 不存在度为 1 的节点（每个分支节点都有左、右两个孩子）

### 完全二叉树 (Complete Binary Tree)

**定义**：一棵高度为 h 的二叉树，当且仅当其每个节点都与高度为 h 的满二叉树中编号 1~n 的节点一一对应时，称为完全二叉树。

```
完全二叉树 ✓              非完全二叉树 ✗
      1                        1
    /   \                    /   \
   2     3                  2     3
  / \   /                  / \     \
 4   5 6                  4   5     7
                               ↑ 编号6的位置空了
```

**核心性质**（用顺序存储时极其重要）：

对于编号为 i 的节点（从 1 开始编号）：
- **左孩子**编号 = `2*i`
- **右孩子**编号 = `2*i + 1`
- **双亲**编号 = `i / 2`（整数除法）
- 若 `2*i > n`，则该节点为叶子节点
- 最后一个分支节点的编号 = `n / 2`

**完全二叉树的高度**：
$$h = \lfloor \log_2 n \rfloor + 1$$

## 1.3 二叉树的四种遍历（核心中的核心）

### 1.3.1 遍历定义

**遍历**：按照某种规则，对树中每个节点**恰好访问一次**。

我们用以下这棵树作为贯穿始终的例子：

```
            A
          /   \
         B     C
        / \     \
       D   E     F
          / \
         G   H
```

### 1.3.2 先序遍历 (Preorder) — 根左右

**递归定义**：
1. 访问**根**节点
2. 先序遍历**左**子树
3. 先序遍历**右**子树

**手动推演过程**：

```
访问 A → 进入 A 的左子树
  访问 B → 进入 B 的左子树
    访问 D → D 无左子树，D 无右子树 → 返回
  → 进入 B 的右子树
    访问 E → 进入 E 的左子树
      访问 G → G 无左子树，G 无右子树 → 返回
    → 进入 E 的右子树
      访问 H → H 无左子树，H 无右子树 → 返回
    → 返回
  → 返回
→ 进入 A 的右子树
  访问 C → C 无左子树 → 进入 C 的右子树
    访问 F → F 无左子树，F 无右子树 → 返回
  → 返回
→ 结束
```

**先序序列**：**A B D E G H C F**

### 1.3.3 中序遍历 (Inorder) — 左根右

**递归定义**：
1. 中序遍历**左**子树
2. 访问**根**节点
3. 中序遍历**右**子树

**手动推演**：

```
进入 A 的左子树
  进入 B 的左子树
    进入 D 的左子树（空）→ 访问 D → 进入 D 的右子树（空）→ 返回
  → 访问 B
  → 进入 B 的右子树
    进入 E 的左子树
      进入 G 的左子树（空）→ 访问 G → 进入 G 的右子树（空）→ 返回
    → 访问 E
    → 进入 E 的右子树
      进入 H 的左子树（空）→ 访问 H → 进入 H 的右子树（空）→ 返回
    → 返回
  → 返回
→ 访问 A
→ 进入 A 的右子树
  进入 C 的左子树（空）→ 访问 C → 进入 C 的右子树
    进入 F 的左子树（空）→ 访问 F → 进入 F 的右子树（空）→ 返回
  → 返回
→ 结束
```

**中序序列**：**D B G E H A C F**

### 1.3.4 后序遍历 (Postorder) — 左右根

**递归定义**：
1. 后序遍历**左**子树
2. 后序遍历**右**子树
3. 访问**根**节点

**手动推演**（简化）：

```
左子树先全部处理完 → 右子树全部处理完 → 最后才访问根

D → G → H → E → B → F → C → A
```

**后序序列**：**D G H E B F C A**

> **记忆口诀**：先序=根左右，中序=左根右，后序=左右根。"先中后"指的是**根的位置**。

### 1.3.5 层序遍历 (Level-order) — 逐层从左到右

**方法**：使用**队列**。先将根入队，然后循环：出队一个节点、访问它、把它的左右孩子入队。

```
队列变化过程：
初始: [A]
出队 A，访问 A，入队 B, C → [B, C]
出队 B，访问 B，入队 D, E → [C, D, E]
出队 C，访问 C，入队 F    → [D, E, F]
出队 D，访问 D，无孩子    → [E, F]
出队 E，访问 E，入队 G, H → [F, G, H]
出队 F，访问 F，无孩子    → [G, H]
出队 G，访问 G，无孩子    → [H]
出队 H，访问 H，无孩子    → []
```

**层序序列**：**A B C D E F G H**

### 1.3.6 四种遍历的 C++ 实现

```cpp
#include <iostream>
#include <queue>
using namespace std;

// 二叉树节点定义
struct TreeNode {
    char data;
    TreeNode* left;
    TreeNode* right;
    TreeNode(char d) : data(d), left(nullptr), right(nullptr) {}
};

// ==================== 先序遍历（递归） ====================
void preorder(TreeNode* root) {
    if (root == nullptr) return;
    cout << root->data << " ";   // 1. 访问根
    preorder(root->left);         // 2. 遍历左子树
    preorder(root->right);        // 3. 遍历右子树
}

// ==================== 中序遍历（递归） ====================
void inorder(TreeNode* root) {
    if (root == nullptr) return;
    inorder(root->left);          // 1. 遍历左子树
    cout << root->data << " ";   // 2. 访问根
    inorder(root->right);         // 3. 遍历右子树
}

// ==================== 后序遍历（递归） ====================
void postorder(TreeNode* root) {
    if (root == nullptr) return;
    postorder(root->left);        // 1. 遍历左子树
    postorder(root->right);       // 2. 遍历右子树
    cout << root->data << " ";   // 3. 访问根
}

// ==================== 层序遍历（队列） ====================
void levelorder(TreeNode* root) {
    if (root == nullptr) return;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        TreeNode* cur = q.front();
        q.pop();
        cout << cur->data << " ";
        if (cur->left)  q.push(cur->left);
        if (cur->right) q.push(cur->right);
    }
}

// ==================== 非递归中序遍历（用栈） ====================
// 这是面试/考试高频考点
#include <stack>
void inorder_iterative(TreeNode* root) {
    stack<TreeNode*> s;
    TreeNode* cur = root;
    while (cur != nullptr || !s.empty()) {
        // 一路向左，把沿途节点全部压栈
        while (cur != nullptr) {
            s.push(cur);
            cur = cur->left;
        }
        // 弹出栈顶（最左的未访问节点）
        cur = s.top();
        s.pop();
        cout << cur->data << " ";   // 访问
        // 转向右子树
        cur = cur->right;
    }
}

// ==================== 构建测试树并运行 ====================
int main() {
    //            A
    //          /   \
    //         B     C
    //        / \     \
    //       D   E     F
    //          / \
    //         G   H

    TreeNode* A = new TreeNode('A');
    TreeNode* B = new TreeNode('B');
    TreeNode* C = new TreeNode('C');
    TreeNode* D = new TreeNode('D');
    TreeNode* E = new TreeNode('E');
    TreeNode* F = new TreeNode('F');
    TreeNode* G = new TreeNode('G');
    TreeNode* H = new TreeNode('H');

    A->left = B;  A->right = C;
    B->left = D;  B->right = E;
    C->right = F;
    E->left = G;  E->right = H;

    cout << "先序: "; preorder(A);       cout << endl;
    cout << "中序: "; inorder(A);        cout << endl;
    cout << "后序: "; postorder(A);      cout << endl;
    cout << "层序: "; levelorder(A);     cout << endl;
    cout << "中序(非递归): "; inorder_iterative(A); cout << endl;

    return 0;
}
```

**输出**：
```
先序: A B D E G H C F
中序: D B G E H A C F
后序: D G H E B F C A
层序: A B C D E F G H
中序(非递归): D B G E H A C F
```

## 1.4 由遍历序列恢复二叉树

### 1.4.1 核心原理

> **定理**：在节点值互不相同的前提下，给定一棵二叉树的**中序序列**和**先序序列**（或后序序列），可以唯一确定这棵二叉树。仅有先序+后序**不能**唯一确定。

**为什么中序是必须的？**
- 先序/后序可以确定**根节点**（先序的第一个元素 / 后序的最后一个元素）。
- 中序可以确定**左右子树的划分**——根节点在中序序列中的位置，将序列分为左子树部分和右子树部分。

### 1.4.2 先序 + 中序 → 恢复二叉树

**例题**：已知先序序列 `A B D E G H C F`，中序序列 `D B G E H A C F`，恢复二叉树。

**第一步**：先序第一个元素 `A` 是根节点。

```
中序: D B G E H | A | C F
      ←左子树→   根   ←右子树→
```

左子树中序：`D B G E H`（5 个节点）
右子树中序：`C F`（2 个节点）

**第二步**：从先序中取出对应的子序列。

```
先序: A | B D E G H | C F
      根  ←左子树→    ←右子树→
```

**第三步**：递归处理左子树。

左子树先序 `B D E G H`，中序 `D B G E H`：
- 根 = B
- 中序：`D | B | G E H` → 左子树`D`，右子树`G E H`
- 先序：`B | D | E G H`

继续递归...左子树只有 `D`，它就是叶子。
右子树先序 `E G H`，中序 `G E H`：
- 根 = E
- 中序：`G | E | H` → 左子树`G`，右子树`H`

**第四步**：递归处理右子树。

右子树先序 `C F`，中序 `C F`：
- 根 = C
- 中序：空 | `C` | `F` → 左子树空，右子树`F`

**最终恢复**：

```
            A
          /   \
         B     C
        / \     \
       D   E     F
          / \
         G   H
```

### 1.4.3 后序 + 中序 → 恢复二叉树

**方法完全类似**，只是根节点变成后序序列的**最后一个**元素。

**例题**：后序 `D G H E B F C A`，中序 `D B G E H A C F`。

- 后序最后一个 `A` → 根
- 中序中找 A → 左子树 `D B G E H`，右子树 `C F`
- 后序分割：`D G H E B`（左子树后序）、`F C`（右子树后序）
- 递归处理...

### 1.4.4 C++ 实现：先序+中序恢复二叉树

```cpp
#include <iostream>
#include <string>
using namespace std;

struct TreeNode {
    char data;
    TreeNode* left;
    TreeNode* right;
    TreeNode(char d) : data(d), left(nullptr), right(nullptr) {}
};

// pre: 先序序列, in: 中序序列
TreeNode* buildTree(string pre, string in) {
    if (pre.empty()) return nullptr;
    
    // 1. 先序第一个字符就是根
    char rootVal = pre[0];
    TreeNode* root = new TreeNode(rootVal);
    
    // 2. 在中序中找到根的位置
    int rootPos = in.find(rootVal);
    
    // 3. 分割出左子树和右子树的先序、中序
    string leftIn  = in.substr(0, rootPos);          // 左子树的中序
    string rightIn = in.substr(rootPos + 1);          // 右子树的中序
    string leftPre = pre.substr(1, leftIn.size());    // 左子树的先序
    string rightPre = pre.substr(1 + leftIn.size());  // 右子树的先序
    
    // 4. 递归构建
    root->left  = buildTree(leftPre, leftIn);
    root->right = buildTree(rightPre, rightIn);
    
    return root;
}

// 验证：后序遍历输出
void postorder(TreeNode* root) {
    if (!root) return;
    postorder(root->left);
    postorder(root->right);
    cout << root->data;
}

int main() {
    string preorder = "ABDEGHCF";
    string inorder  = "DBGEHACF";
    
    TreeNode* root = buildTree(preorder, inorder);
    
    cout << "恢复后的后序遍历: ";
    postorder(root);
    cout << endl;
    // 应输出: DGHEBFCA
    
    return 0;
}
```

## 1.5 二叉排序树 (BST) — 完整实现

### 1.5.1 BST 的定义

**二叉排序树 (Binary Search Tree, BST)** 是一棵二叉树，它要么为空，要么满足：

1. 左子树上所有节点的值 **<** 根节点的值
2. 右子树上所有节点的值 **>** 根节点的值
3. 左右子树也分别是 BST

```
         50
       /    \
      30     70
     / \    / \
    20  40  60  80
```

**中序遍历 BST 得到的是一个递增的有序序列**：20, 30, 40, 50, 60, 70, 80

这是 BST 最本质的性质！

### 1.5.2 BST 的查找

```
查找 40：
50 → 40 < 50，走左
30 → 40 > 30，走右
40 → 找到！

查找 45：
50 → 45 < 50，走左
30 → 45 > 30，走右
40 → 45 > 40，走右
nullptr → 查找失败
```

**时间复杂度**：
- 最好情况（平衡）：O(log n)
- 最坏情况（退化为链表）：O(n)

### 1.5.3 BST 的插入

**原则**：新插入的节点一定是**叶子节点**。查找失败的位置就是插入位置。

```
在上面的树中插入 45：
50 → 走左 → 30 → 走右 → 40 → 走右 → nullptr → 插入！

         50
       /    \
      30     70
     / \    / \
    20  40  60  80
          \
          45  ← 新插入
```

### 1.5.4 BST 的删除（重点难点！）

删除节点分**三种情况**：

**情况一：删除叶子节点**
直接删除，将其父节点对应的指针置空。

```
删除 20：
         50              50
       /    \   →      /    \
      30     70       30     70
     / \    / \        \    / \
    20  40  60  80     40  60  80
```

**情况二：删除只有一个子树的节点**
用它的孩子替代它的位置。

```
删除 30（假设 30 只有右孩子 40）：
         50              50
       /    \   →      /    \
      30     70       40     70
        \   / \      / \
        40 60  80   ...
```

**情况三：删除有两个子树的节点（最复杂）**

有两种等价策略：
- **方法 A**：用该节点的**中序前驱**（左子树中最大的节点）替换它，然后删除前驱
- **方法 B**：用该节点的**中序后继**（右子树中最小的节点）替换它，然后删除后继

```
删除 50（方法 A：用中序前驱 40 替代）：

中序前驱 = 左子树中一路向右走到底的节点 = 40

         50              40
       /    \   →      /    \
      30     70       30     70
     / \    / \      /      / \
    20  40  60  80  20     60  80

（把 50 的值替换为 40，然后删除原来的 40 节点）
```

```
删除 50（方法 B：用中序后继 60 替代）：

中序后继 = 右子树中一路向左走到底的节点 = 60

         50              60
       /    \   →      /    \
      30     70       30     70
     / \    / \      / \      \
    20  40  60  80  20  40     80
```

### 1.5.5 BST 的完整 C++ 实现

```cpp
#include <iostream>
using namespace std;

struct BSTNode {
    int key;
    BSTNode* left;
    BSTNode* right;
    BSTNode(int k) : key(k), left(nullptr), right(nullptr) {}
};

// ==================== 查找 ====================
BSTNode* search(BSTNode* root, int key) {
    if (root == nullptr || root->key == key)
        return root;
    if (key < root->key)
        return search(root->left, key);
    else
        return search(root->right, key);
}

// ==================== 插入 ====================
BSTNode* insert(BSTNode* root, int key) {
    if (root == nullptr) {
        return new BSTNode(key);  // 找到空位，创建新节点
    }
    if (key < root->key) {
        root->left = insert(root->left, key);
    } else if (key > root->key) {
        root->right = insert(root->right, key);
    }
    // key == root->key 时不插入（不允许重复）
    return root;
}

// ==================== 找最小值节点 ====================
BSTNode* findMin(BSTNode* root) {
    while (root->left != nullptr)
        root = root->left;
    return root;
}

// ==================== 删除 ====================
BSTNode* remove(BSTNode* root, int key) {
    if (root == nullptr) return nullptr;
    
    if (key < root->key) {
        root->left = remove(root->left, key);
    } else if (key > root->key) {
        root->right = remove(root->right, key);
    } else {
        // 找到了要删除的节点
        
        // 情况一 & 二：没有孩子 或 只有一个孩子
        if (root->left == nullptr) {
            BSTNode* temp = root->right;
            delete root;
            return temp;    // 用右孩子（可能是 nullptr）替代
        }
        if (root->right == nullptr) {
            BSTNode* temp = root->left;
            delete root;
            return temp;    // 用左孩子替代
        }
        
        // 情况三：有两个孩子
        // 找中序后继（右子树的最小节点）
        BSTNode* successor = findMin(root->right);
        root->key = successor->key;    // 用后继的值替换当前节点
        root->right = remove(root->right, successor->key);  // 删除后继
    }
    return root;
}

// ==================== 中序遍历（验证有序性） ====================
void inorder(BSTNode* root) {
    if (root == nullptr) return;
    inorder(root->left);
    cout << root->key << " ";
    inorder(root->right);
}

int main() {
    BSTNode* root = nullptr;
    
    // 依次插入
    int keys[] = {50, 30, 70, 20, 40, 60, 80};
    for (int k : keys) {
        root = insert(root, k);
    }
    
    cout << "初始 BST 中序遍历: ";
    inorder(root);
    cout << endl;
    // 输出: 20 30 40 50 60 70 80
    
    // 查找
    BSTNode* found = search(root, 40);
    cout << "查找 40: " << (found ? "找到" : "未找到") << endl;
    
    found = search(root, 45);
    cout << "查找 45: " << (found ? "找到" : "未找到") << endl;
    
    // 删除叶子节点
    root = remove(root, 20);
    cout << "删除 20 后: ";
    inorder(root);
    cout << endl;
    // 输出: 30 40 50 60 70 80
    
    // 删除有两个孩子的节点
    root = remove(root, 50);
    cout << "删除 50 后: ";
    inorder(root);
    cout << endl;
    // 输出: 30 40 60 70 80
    
    return 0;
}
```

# 第二章 线索二叉树 (Threaded Binary Tree)

---

## 2.1 为什么需要线索二叉树？

### 问题的提出

一棵 n 个节点的二叉树有 **2n 个指针域**（每个节点有 left 和 right）。其中只有 **n-1 个指针域**被实际使用（指向孩子），剩余 **n+1 个指针域**为 `nullptr`。

> **核心思想**：能否利用这些空闲的指针域来存储有用的信息？——比如存储**遍历的前驱和后继**？

### 空指针统计证明

| | 指针域总数 | 非空指针数 | 空指针数 |
|---|---|---|---|
| 公式 | 2n | n-1（每个非根节点被一个指针指向） | 2n-(n-1) = **n+1** |

例如一棵有 7 个节点的完全二叉树，空指针数 = 7+1 = 8。

## 2.2 线索化的原理

**线索 (Thread)**：将空的指针域利用起来：
- 若某节点的 `left` 指针为空 → 让它指向该节点的**中序前驱**
- 若某节点的 `right` 指针为空 → 让它指向该节点的**中序后继**

为了区分指针到底指向的是**孩子**还是**线索**，需要增加两个标志位：

```cpp
struct ThreadNode {
    char data;
    ThreadNode* left;
    ThreadNode* right;
    int ltag;  // 0: left 指向左孩子;  1: left 指向前驱（线索）
    int rtag;  // 0: right 指向右孩子; 1: right 指向后继（线索）
};
```

## 2.3 图解中序线索二叉树

原始二叉树：
```
         A
       /   \
      B     C
     / \     \
    D   E     F
```

中序遍历序列：**D B E A C F**

| 节点 | 中序前驱 | 中序后继 | left 指向 | ltag | right 指向 | rtag |
|------|---------|---------|----------|------|-----------|------|
| D | 无（D是序列首） | B | nullptr(前驱) | 1 | B(后继) | 1 |
| B | D | E | D(左孩子) | 0 | E(右孩子) | 0 |
| E | B | A | B(前驱) | 1 | A(后继) | 1 |
| A | E | C | B(左孩子) | 0 | C(右孩子) | 0 |
| C | A | F | A(前驱) | 1 | F(右孩子) | 0 |
| F | C | 无（F是序列尾） | C(前驱) | 1 | nullptr(后继) | 1 |

```
线索化后的示意图（虚线表示线索）：

     nullptr ←-- D ----→ B ←---- E ----→ A ---- C ----→ F ---→ nullptr
                  ↑       |  ↑      |      |  ↑      |
                 ltag=1   实际孩子  ltag=1  实际孩子  ltag=1 实际孩子
                 rtag=1            rtag=1            rtag=1
```

## 2.4 中序线索化的 C++ 实现

```cpp
#include <iostream>
using namespace std;

struct ThreadNode {
    char data;
    ThreadNode* left;
    ThreadNode* right;
    int ltag;  // 0=孩子, 1=线索（前驱）
    int rtag;  // 0=孩子, 1=线索（后继）
    ThreadNode(char d) : data(d), left(nullptr), right(nullptr), ltag(0), rtag(0) {}
};

// 全局变量：指向中序遍历中刚刚访问过的节点
ThreadNode* pre = nullptr;

// ==================== 中序线索化 ====================
void inThread(ThreadNode* p) {
    if (p == nullptr) return;
    
    inThread(p->left);   // 递归线索化左子树
    
    // --- 处理当前节点 p ---
    if (p->left == nullptr) {
        p->left = pre;    // 左指针指向前驱
        p->ltag = 1;
    }
    if (pre != nullptr && pre->right == nullptr) {
        pre->right = p;   // 前驱的右指针指向当前节点（后继）
        pre->rtag = 1;
    }
    pre = p;              // 更新 pre 为当前节点
    
    inThread(p->right);  // 递归线索化右子树
}

// 对外接口：创建中序线索二叉树
void createInThread(ThreadNode* root) {
    pre = nullptr;
    if (root != nullptr) {
        inThread(root);
        // 处理最后一个节点的右指针
        pre->right = nullptr;
        pre->rtag = 1;
    }
}

// ==================== 利用线索进行中序遍历（无需栈！无需递归！） ====================

// 找到中序遍历的第一个节点（最左下的节点）
ThreadNode* firstNode(ThreadNode* p) {
    while (p->ltag == 0)  // 一路向左走到底
        p = p->left;
    return p;
}

// 找到节点 p 在中序遍历中的后继
ThreadNode* nextNode(ThreadNode* p) {
    if (p->rtag == 1)       // 右指针是线索，直接就是后继
        return p->right;
    else                     // 右指针是右孩子，后继是右子树中最左下的节点
        return firstNode(p->right);
}

// 中序遍历（利用线索，非递归，无栈）
void inOrderByThread(ThreadNode* root) {
    if (root == nullptr) return;
    for (ThreadNode* p = firstNode(root); p != nullptr; p = nextNode(p)) {
        cout << p->data << " ";
    }
    cout << endl;
}

int main() {
    // 构建二叉树
    ThreadNode* A = new ThreadNode('A');
    ThreadNode* B = new ThreadNode('B');
    ThreadNode* C = new ThreadNode('C');
    ThreadNode* D = new ThreadNode('D');
    ThreadNode* E = new ThreadNode('E');
    ThreadNode* F = new ThreadNode('F');

    A->left = B;  A->right = C;
    B->left = D;  B->right = E;
    C->right = F;

    // 线索化
    createInThread(A);

    // 利用线索进行中序遍历
    cout << "中序线索遍历: ";
    inOrderByThread(A);
    // 输出: D B E A C F

    return 0;
}
```

## 2.5 线索二叉树的意义

| 对比项 | 普通二叉树 | 线索二叉树 |
|--------|-----------|-----------|
| 找中序后继 | 需要栈或递归，O(h) | 有线索时 O(1)，无线索时 O(h) |
| 找中序前驱 | 需要栈或递归 | 有线索时 O(1) |
| 空间利用 | n+1 个指针浪费 | 全部利用 |
| 遍历 | 需要栈空间 O(h) | **O(1) 空间**遍历 |
| 缺点 | — | 插入/删除需要维护线索，较复杂 |

> **考试要点**：线索二叉树的核心价值 = **空间复杂度 O(1) 的遍历** + **快速找前驱后继**。

# 第三章 并查集 (Union-Find / Disjoint Set)

---

## 3.1 什么是并查集？

**并查集**是一种用于管理**元素分组**的数据结构。它支持两种操作：

1. **Find(x)**：查找元素 x 属于哪个集合（返回集合的**代表元素**，即根节点）
2. **Union(x, y)**：将元素 x 所在的集合和元素 y 所在的集合**合并**

### 生活类比

想象一个班级要分组做项目：
- 初始时，每个人是一个独立的"组"
- `Union(小明, 小红)` → 小明和小红合成一组
- `Union(小红, 小李)` → 小李也加入了小明和小红的组
- `Find(小明) == Find(小李)` → 检查小明和小李是否在同一组 → **是的**

## 3.2 并查集的存储结构

并查集用**树（森林）**来表示，但通常用**数组**实现（不需要指针）。

```
parent[] 数组：parent[i] 表示节点 i 的父节点
- 如果 parent[i] == i，说明 i 是根节点（代表元素）
- 有的实现中 parent[i] == -1 表示 i 是根节点
```

**初始状态**（8 个元素，每个元素自成一组）：

```
下标:   0  1  2  3  4  5  6  7
parent: 0  1  2  3  4  5  6  7    ← 每个元素的父亲是自己

图示:  {0} {1} {2} {3} {4} {5} {6} {7}
       8 棵只有根的树
```

**执行 Union(1, 2)**：把 1 的根挂到 2 的根下面（或反过来）

```
下标:   0  1  2  3  4  5  6  7
parent: 0  2  2  3  4  5  6  7
              ↑
        1 的父亲变成了 2

图示:  {0} {2-1} {3} {4} {5} {6} {7}
             2
            /
           1
```

**执行 Union(3, 4)，Union(1, 3)**：

```
Union(3,4) 后:    4
                 /
                3

Union(1,3) 后:    4
                 /
                3
               /
              2
             /
            1
```

## 3.3 基础实现

```cpp
#include <iostream>
using namespace std;

const int MAXN = 1005;
int parent[MAXN];

// ==================== 初始化 ====================
void init(int n) {
    for (int i = 0; i < n; i++) {
        parent[i] = i;   // 每个元素的父亲是自己
    }
}

// ==================== 查找（Find） ====================
// 找到 x 所属集合的根节点
int find(int x) {
    while (parent[x] != x) {
        x = parent[x];   // 一路向上走
    }
    return x;
}

// ==================== 合并（Union） ====================
void unite(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX != rootY) {
        parent[rootX] = rootY;  // 把 x 的根挂到 y 的根下面
    }
}

// ==================== 判断是否在同一集合 ====================
bool isConnected(int x, int y) {
    return find(x) == find(y);
}
```

## 3.4 路径压缩优化（极其重要！）

**问题**：如果树退化为链状，每次 `find` 需要 O(n) 时间。

```
退化的例子：
0 ← 1 ← 2 ← 3 ← 4 ← 5
find(5) 需要走 5 步
```

**路径压缩**：在 `find` 的过程中，把沿途经过的所有节点**直接挂到根节点**下面。

```
find(5) 执行路径压缩后：

    0
  / | \ \ \
 1  2  3  4  5     ← 所有节点直接连根
```

```cpp
// ==================== 路径压缩版 Find ====================
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);  // 递归压缩：直接挂到根上
    }
    return parent[x];
}
```

**一行代码，效果巨大**：find 操作的均摊时间复杂度从 O(n) 降到接近 **O(1)**（准确说是 O(α(n))，α 是反阿克曼函数，增长极慢）。

## 3.5 按秩合并优化

**思想**：合并时，把**矮树**挂到**高树**下面，避免树变得更高。

```cpp
int rank_arr[MAXN];  // 树的高度（秩）

void init(int n) {
    for (int i = 0; i < n; i++) {
        parent[i] = i;
        rank_arr[i] = 0;   // 初始高度为 0
    }
}

void unite(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX == rootY) return;
    
    // 矮树挂到高树下面
    if (rank_arr[rootX] < rank_arr[rootY]) {
        parent[rootX] = rootY;
    } else if (rank_arr[rootX] > rank_arr[rootY]) {
        parent[rootY] = rootX;
    } else {
        parent[rootY] = rootX;
        rank_arr[rootX]++;   // 等高时，新树高度 +1
    }
}
```

## 3.6 完整示例：判断连通性

```cpp
#include <iostream>
using namespace std;

const int MAXN = 1005;
int parent[MAXN];

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);  // 路径压缩
    return parent[x];
}

void unite(int x, int y) {
    int rx = find(x), ry = find(y);
    if (rx != ry) parent[rx] = ry;
}

bool isConnected(int x, int y) {
    return find(x) == find(y);
}

int main() {
    int n = 8;
    for (int i = 0; i < n; i++) parent[i] = i;
    
    // 合并操作
    unite(0, 1);
    unite(1, 2);
    unite(3, 4);
    unite(5, 6);
    unite(6, 7);
    
    // 查询
    cout << "0 和 2 是否连通: " << (isConnected(0, 2) ? "是" : "否") << endl;  // 是
    cout << "0 和 3 是否连通: " << (isConnected(0, 3) ? "是" : "否") << endl;  // 否
    cout << "5 和 7 是否连通: " << (isConnected(5, 7) ? "是" : "否") << endl;  // 是
    
    // 合并两个组
    unite(2, 3);
    cout << "合并后，0 和 4 是否连通: " << (isConnected(0, 4) ? "是" : "否") << endl;  // 是
    
    // 统计有多少个不同的集合
    int groups = 0;
    for (int i = 0; i < n; i++) {
        if (find(i) == i) groups++;
    }
    cout << "共有 " << groups << " 个独立集合" << endl;
    
    return 0;
}
```

> **并查集的典型应用**：判断图的连通性、Kruskal 算法中判断是否形成环、社交网络中判断两人是否在同一个圈子。完整示例为了简洁只写了路径压缩；实际高性能写法通常会同时使用路径压缩和按秩/按大小合并。

# 第四章 平衡二叉树 (AVL Tree)

---

## 4.1 为什么 BST 不够好？

BST 的查找效率取决于树的高度。如果我们按照 **1, 2, 3, 4, 5** 的顺序插入 BST：

```
1
 \
  2
   \
    3
     \
      4
       \
        5
```

BST 退化成了一条链！查找的时间复杂度退化为 **O(n)**。

**解决方案**：在每次插入/删除后，通过**旋转**操作保持树的**平衡**，使树高始终保持在 **O(log n)**。

## 4.2 AVL 树的定义

**AVL 树**（Adelson-Velsky and Landis Tree）是一种**自平衡二叉搜索树**，满足：

> 每个节点的**左右子树高度差**（称为**平衡因子，Balance Factor**）的绝对值不超过 1。

$$\text{平衡因子} = \text{左子树高度} - \text{右子树高度} \in \{-1, 0, 1\}$$

```
    合法的 AVL 树            不合法（BF=2）
        30 [BF=1]              30 [BF=2]
       /  \                   /
      20   40 [BF=0]        20
     /                      /
    10                     10
 [BF=0]                 [BF=0]
```

## 4.3 AVL 的四种旋转

当插入一个节点导致某个祖先节点的平衡因子变为 ±2 时，需要通过**旋转**来恢复平衡。

### 4.3.1 LL 旋转（右单旋）

**触发条件**：在**失衡节点的左孩子的左子树**上插入导致失衡。

```
失衡前:            失衡后(A 的 BF=2):    LL 右旋后:
    A [BF=1]         A [BF=2]              B [BF=0]
   /                /                     / \
  B [BF=0]         B [BF=1]             C    A
                  /
                 C   ← 新插入
```

**详细旋转过程**：

```
         A                    B
        / \                  / \
       B   AR    →         C    A
      / \                 /    / \
     C   BR              新   BR  AR
    /
   新节点

步骤:
1. B 成为新的根
2. A 成为 B 的右孩子
3. B 的原右子树 BR 成为 A 的左子树
```

### 4.3.2 RR 旋转（左单旋）

**触发条件**：在**失衡节点的右孩子的右子树**上插入导致失衡。

```
  A [BF=-2]              B [BF=0]
   \                    / \
    B [BF=-1]    →    A    C
     \               / \
      C             AR  BL
     新插入

步骤:
1. B 成为新的根
2. A 成为 B 的左孩子
3. B 的原左子树 BL 成为 A 的右子树
```

### 4.3.3 LR 旋转（左右双旋）

**触发条件**：在**失衡节点的左孩子的右子树**上插入导致失衡。

```
     A [BF=2]
    /
   B [BF=-1]
    \
     C   ← 新插入

解法：先对 B 做左旋(RR)，再对 A 做右旋(LL)

第一步：对 B 左旋           第二步：对 A 右旋
     A                         A                    C
    /                         /                    / \
   B          →              C          →        B    A
    \                       /
     C                     B
```

**更一般的情况**：

```
         A                    A                    C
        / \                  / \                  / \
       B   AR    →         C    AR     →        B    A
      / \                 / \                  / \  / \
     BL  C              B    CR              BL CL CR AR
        / \            / \
       CL  CR         BL  CL

第一步：对 B 左旋(把 C 提上来)     第二步：对 A 右旋(把 C 提上来)
```

### 4.3.4 RL 旋转（右左双旋）

**触发条件**：在**失衡节点的右孩子的左子树**上插入导致失衡。

```
  A [BF=-2]
   \
    B [BF=1]
   /
  C   ← 新插入

解法：先对 B 做右旋(LL)，再对 A 做左旋(RR)

第一步：对 B 右旋           第二步：对 A 左旋
  A                         A                    C
   \                         \                  / \
    B          →              C        →       A    B
   /                           \
  C                             B
```

### 4.3.5 口诀总结

| 失衡类型 | 插入位置 | 旋转方式 | 记忆口诀 |
|---------|---------|---------|---------|
| **LL** | 左孩子的左子树 | **右单旋** | 左左→右旋 |
| **RR** | 右孩子的右子树 | **左单旋** | 右右→左旋 |
| **LR** | 左孩子的右子树 | **先左旋再右旋** | 左右→先左后右 |
| **RL** | 右孩子的左子树 | **先右旋再左旋** | 右左→先右后左 |

## 4.4 AVL 树的 C++ 实现

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

struct AVLNode {
    int key;
    AVLNode* left;
    AVLNode* right;
    int height;   // 以该节点为根的子树高度
    AVLNode(int k) : key(k), left(nullptr), right(nullptr), height(1) {}
};

// ==================== 辅助函数 ====================
int getHeight(AVLNode* node) {
    return node ? node->height : 0;
}

int getBalance(AVLNode* node) {
    return node ? getHeight(node->left) - getHeight(node->right) : 0;
}

void updateHeight(AVLNode* node) {
    node->height = 1 + max(getHeight(node->left), getHeight(node->right));
}

// ==================== 右旋（LL 情况） ====================
//       A              B
//      / \            / \
//     B   AR   →    BL   A
//    / \                 / \
//   BL  BR             BR  AR
AVLNode* rightRotate(AVLNode* A) {
    AVLNode* B  = A->left;
    AVLNode* BR = B->right;
    
    B->right = A;      // A 变成 B 的右孩子
    A->left  = BR;     // B 的原右子树变成 A 的左子树
    
    updateHeight(A);   // 先更新 A（A 现在是 B 的孩子）
    updateHeight(B);   // 再更新 B
    
    return B;          // B 成为新的根
}

// ==================== 左旋（RR 情况） ====================
//     A                B
//    / \              / \
//   AL  B     →     A   BR
//      / \         / \
//     BL  BR      AL  BL
AVLNode* leftRotate(AVLNode* A) {
    AVLNode* B  = A->right;
    AVLNode* BL = B->left;
    
    B->left  = A;
    A->right = BL;
    
    updateHeight(A);
    updateHeight(B);
    
    return B;
}

// ==================== 插入 ====================
AVLNode* insert(AVLNode* node, int key) {
    // 1. 标准 BST 插入
    if (node == nullptr)
        return new AVLNode(key);
    
    if (key < node->key)
        node->left = insert(node->left, key);
    else if (key > node->key)
        node->right = insert(node->right, key);
    else
        return node;  // 不允许重复
    
    // 2. 更新高度
    updateHeight(node);
    
    // 3. 检查平衡因子，决定是否旋转
    int balance = getBalance(node);
    
    // LL: 左子树的左子树偏重
    if (balance > 1 && key < node->left->key)
        return rightRotate(node);
    
    // RR: 右子树的右子树偏重
    if (balance < -1 && key > node->right->key)
        return leftRotate(node);
    
    // LR: 左子树的右子树偏重
    if (balance > 1 && key > node->left->key) {
        node->left = leftRotate(node->left);   // 先对左孩子左旋
        return rightRotate(node);               // 再对自己右旋
    }
    
    // RL: 右子树的左子树偏重
    if (balance < -1 && key < node->right->key) {
        node->right = rightRotate(node->right); // 先对右孩子右旋
        return leftRotate(node);                // 再对自己左旋
    }
    
    return node;  // 无需旋转
}

// ==================== 中序遍历 ====================
void inorder(AVLNode* root) {
    if (!root) return;
    inorder(root->left);
    cout << root->key << "(BF=" << getBalance(root) << ") ";
    inorder(root->right);
}

// ==================== 层序遍历（显示树结构） ====================
#include <queue>
void levelOrder(AVLNode* root) {
    if (!root) return;
    queue<AVLNode*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            AVLNode* cur = q.front(); q.pop();
            cout << cur->key << " ";
            if (cur->left)  q.push(cur->left);
            if (cur->right) q.push(cur->right);
        }
        cout << endl;
    }
}

int main() {
    AVLNode* root = nullptr;
    
    // 插入序列：1,2,3,4,5,6,7
    // 如果是普通 BST，会退化成链表
    // AVL 树会自动调整
    for (int i = 1; i <= 7; i++) {
        root = insert(root, i);
    }
    
    cout << "中序遍历（应为有序）: ";
    inorder(root);
    cout << endl;
    
    cout << "层序遍历（显示树形）:" << endl;
    levelOrder(root);
    
    cout << "树高: " << getHeight(root) << endl;
    // 7 个节点的 AVL 树，高度应为 3（而非退化的 7）
    
    return 0;
}
```

## 4.5 AVL 树的性能分析

| 操作 | 时间复杂度 |
|------|----------|
| 查找 | O(log n) — 保证 |
| 插入 | O(log n) — 最多旋转 1~2 次 |
| 删除 | O(log n) — 可能需要旋转 O(log n) 次 |

> **AVL 树 vs 普通 BST**：AVL 牺牲了少量的插入/删除性能（需要旋转），换取了**稳定的 O(log n) 查找性能**。

# 第五章 红黑树 (Red-Black Tree)

---

## 5.1 红黑树的动机

AVL 树虽好，但它对**平衡**的要求过于严格——每个节点的左右子树高度差不超过 1。这意味着：
- 每次插入/删除都可能触发旋转
- 删除操作可能需要**从删除位置一路向上旋转到根**

**红黑树**放宽了平衡条件，不追求严格的高度平衡，而是追求一种**"近似平衡"**：

> 红黑树保证最长路径不超过最短路径的 **2 倍**。

这样，插入最多旋转 2 次，删除最多旋转 3 次。

## 5.2 红黑树的五条性质

一棵红黑树是满足以下**全部五条性质**的二叉搜索树：

| 编号 | 性质 | 含义 |
|------|------|------|
| ① | 每个节点要么是**红色**，要么是**黑色** | 颜色二选一 |
| ② | **根节点**是黑色 | 根必黑 |
| ③ | 每个**叶子节点 (NIL)** 是黑色 | 这里的叶子指空节点，不是真正的数据节点 |
| ④ | 红色节点的两个孩子必须都是**黑色** | **不红红**（不能有连续的红节点） |
| ⑤ | 从任一节点到其所有后代 NIL 节点的路径上，**黑色节点数目相同** | **黑路同** |

### 5.2.1 性质解读

**性质④**（不红红）的推论：红色节点的父亲必须是黑色。

**性质⑤**（黑路同）中的"黑色节点数目"称为**黑高 (Black Height)**。

```
一棵合法的红黑树示例（B=黑, R=红）：

           13B
          /    \
        8R      17R
       / \      / \
      1B  11B  15B  25B
       \           / \
        6R       22R  27R
```

验证：
- 性质②: 根 13 是黑色 ✓
- 性质④: 红色节点 8 的孩子 1,11 都是黑色 ✓；红色节点 17 的孩子 15,25 都是黑色 ✓
- 性质⑤: 从根到每个 NIL 的黑色节点数都是 3 ✓

## 5.3 红黑树 vs AVL 树

| 对比 | AVL 树 | 红黑树 |
|------|--------|--------|
| 平衡标准 | 左右高度差 ≤ 1 | 黑路同 + 不红红 |
| 树高 | ≤ 1.44 log₂(n+2) | ≤ 2 log₂(n+1) |
| 查找 | 更快（更矮） | 略慢 |
| 插入 | 可能旋转多次 | 最多旋转 2 次 |
| 删除 | 可能旋转 O(log n) 次 | 最多旋转 3 次 |
| 适用场景 | 查找密集 | 插入/删除频繁 |
| 实际应用 | 数据库索引 | C++ `std::map/set`、Java `TreeMap`、Linux 内核 |

## 5.4 红黑树的插入（概述）

**步骤**：
1. 按照 BST 规则插入新节点
2. 新节点着为**红色**（因为插入红色节点不会违反性质⑤，只可能违反性质④）
3. 如果违反了红黑性质（主要是"不红红"），通过**变色**和**旋转**来修复

### 插入修复的三种情况

设新插入节点为 N，父节点 P，祖父节点 G，叔叔节点 U。

**前提**：P 是红色（否则不需要修复）。

| 情况 | 条件 | 操作 |
|------|------|------|
| **Case 1** | 叔叔 U 是红色 | P 和 U 变黑，G 变红，然后把 G 当作新插入节点继续向上检查 |
| **Case 2** | U 是黑色，N 是 P 的"内侧"孩子 | 对 P 做旋转（把 N 转上来），转化为 Case 3 |
| **Case 3** | U 是黑色，N 是 P 的"外侧"孩子 | 对 G 做旋转，P 和 G 变色 |

### Case 1 图解（叔叔红色 → 变色）

```
       G(B)               G(R)  ← G 变红，继续向上检查
      / \                / \
    P(R)  U(R)   →    P(B)  U(B)
    /                  /
   N(R)              N(R)
```

### Case 3 图解（叔叔黑色，外侧 → 旋转+变色）

```
       G(B)               P(B)
      / \                / \
    P(R)  U(B)   →    N(R)  G(R)
    /                        \
   N(R)                      U(B)

步骤: 对 G 右旋, P 变黑, G 变红
```

## 5.5 红黑树的核心要点（考试重点）

> ⚠️ **红黑树的代码实现非常复杂**（完整代码约 300-400 行），考试通常**不要求手写完整代码**，但要求：

1. **能说出红黑树的 5 条性质**
2. **理解插入修复的基本思路**（新节点着红色，可能违反"不红红"，通过变色和旋转修复）
3. **知道红黑树与 AVL 的区别和各自的优劣势**
4. **能判断一棵给定的树是否是合法的红黑树**

### 判断是否为合法红黑树的方法

给一棵带颜色的 BST，依次检查：
1. 根是黑色？
2. 有没有连续的红色节点（红色节点的孩子是否都是黑色）？
3. 从根到每个 NIL 的路径上，黑色节点数是否都相同？

如果全部满足，则是合法红黑树。

### 练习题

```
判断以下是否是合法红黑树：

    树 1:              树 2:              树 3:
       10B                10B                10B
      / \                / \                / \
     5R  15B            5R  15R            5B  15B
    / \                / \                /
   3B  7B            3B  7B            3R

答案：
树 1: ✓ 合法 (根黑、不红红、黑路同 = 2)
树 2: 检查性质⑤：
      左路径 10B→5R→3B→NIL 黑节点=2
      右路径 10B→15R→NIL 黑节点=1  ✗ 不合法（黑路不同）
树 3: ✓ 合法 (根黑、不红红、
      所有路径黑节点数=2)
```

# 第六章 B 树 (B-Tree)

---

## 6.1 为什么需要 B 树？

之前学的 BST、AVL、红黑树都是**二叉树**——每个节点最多两个孩子。当数据量非常大（如数据库中的百万级记录），数据存储在**磁盘**上时：

**问题**：二叉树即使平衡，高度也是 O(log₂n)。对于 n = 1,000,000，高度约 20。每访问一个节点就是一次**磁盘 I/O**，20 次 I/O 太慢了！

**解决方案**：让每个节点存储**多个关键字**、拥有**多个孩子**，大幅降低树高。

> **B 树**就是这样一种**多路平衡搜索树**。

## 6.2 B 树的定义

一棵 **m 阶 B 树**满足以下性质：

| 性质 | 说明 |
|------|------|
| ① | 每个节点最多有 **m 个孩子**（最多 m-1 个关键字） |
| ② | 除根节点外，每个节点至少有 **⌈m/2⌉-1 个关键字**；若它不是叶子节点，则至少有 **⌈m/2⌉ 个孩子** |
| ③ | 根节点如果不是叶子节点，则至少有 **2 个孩子**（至少 1 个关键字）；如果整棵树只有根节点，根也可以是叶子节点 |
| ④ | 所有**叶子节点在同一层** |
| ⑤ | 节点中的关键字**从小到大排列**，关键字两侧的子树中的所有值分别小于和大于该关键字 |

### 5 阶 B 树的约束

| | 关键字数 | 孩子数 |
|---|---|---|
| 根节点 | 1 ~ 4 | 0 或 2 ~ 5 |
| 非根节点 | 2 ~ 4 | 0 或 3 ~ 5（非叶节点至少 ⌈5/2⌉=3 个孩子） |

### B 树的节点结构

```
一个 m 阶 B 树节点：

  ┌────┬────┬────┬────┬────┬────┬────┬────┬────┐
  │ P0 │ K1 │ P1 │ K2 │ P2 │ K3 │ P3 │ .. │ Pn │
  └────┴────┴────┴────┴────┴────┴────┴────┴────┘

  Ki: 关键字 (K1 < K2 < K3 < ...)
  Pi: 指向子树的指针
  P0 指向的子树中所有值 < K1
  P1 指向的子树中所有值在 K1 和 K2 之间
  Pn 指向的子树中所有值 > Kn
```

### 示例：一棵 3 阶 B 树（2-3 树）

3 阶 B 树：每个节点最多 3 个孩子（2 个关键字），非根节点至少 2 个孩子（1 个关键字）。

```
              [30]
            /      \
        [10, 20]   [40, 50]
       /   |   \   /   |   \
     [5] [15] [25] [35] [45] [55,60]
```

## 6.3 B 树的查找

**过程**：类似 BST，但每个节点内部需要**在多个关键字中查找**。

```
在上面的 3 阶 B 树中查找 45：

1. 根节点 [30]：45 > 30 → 走右子树
2. 节点 [40, 50]：40 < 45 < 50 → 走中间的子树
3. 节点 [45]：找到！
```

**每一层**的查找 = 一次磁盘 I/O + 节点内部的顺序/二分查找（内存操作，很快）。

## 6.4 B 树的插入

**核心规则**：新关键字一定插入到**叶子节点**中。如果插入后叶子节点的关键字数超过 m-1，则需要**分裂 (Split)**。

### 插入过程详解（5 阶 B 树，每个节点最多 4 个关键字）

**初始空树，依次插入 3, 7, 1, 5, 9, 2, 4, 6, 8**：

**Step 1-4：插入 3, 7, 1, 5**（直接放入根节点）

```
[1, 3, 5, 7]  ← 4 个关键字，刚好满
```

**Step 5：插入 9**（根节点已满，触发分裂！）

```
插入 9 后（假设先临时放入）：
[1, 3, 5, 7, 9]  ← 5 个关键字，超过 4，必须分裂！

分裂方法：取中间关键字 5 上提为新的根
         [5]
        /   \
   [1, 3]  [7, 9]
```

**Step 6：插入 2**

```
2 < 5 → 走左子树 [1, 3] → 插入得 [1, 2, 3]

         [5]
        /   \
  [1, 2, 3]  [7, 9]
```

**Step 7：插入 4**

```
4 < 5 → 走左子树 [1, 2, 3] → 插入得 [1, 2, 3, 4]

         [5]
        /   \
[1, 2, 3, 4] [7, 9]  ← 左子节点满了
```

**Step 8：插入 6**

```
6 > 5 → 走右子树 [7, 9] → 插入得 [6, 7, 9]

         [5]
        /   \
[1, 2, 3, 4] [6, 7, 9]
```

**Step 9：插入 8**

```
8 > 5 → 走右子树 [6, 7, 9] → 插入得 [6, 7, 8, 9]

         [5]
        /   \
[1, 2, 3, 4] [6, 7, 8, 9]  ← 右子节点满了
```

如果接下来再插入 10：

```
10 > 5 → [6,7,8,9] → 插入得 [6,7,8,9,10] → 超出！分裂！

取中间值 8 上提到父节点：
         [5, 8]
        /  |   \
[1,2,3,4] [6,7] [9,10]
```

## 6.5 B 树的删除（概述）

删除关键字的情况比较复杂，分为：

**情况一：删除的关键字在叶子节点**
- 直接删除，如果删除后关键字数 ≥ ⌈m/2⌉-1，无需调整
- 如果不够，需要向兄弟**借**关键字或与兄弟**合并**

**情况二：删除的关键字在内部节点**
- 用该关键字的**前驱**或**后继**（相邻子树中的最大/最小值）替换它
- 然后问题转化为在叶子节点中删除

### 借和合并的图解

**借关键字**（兄弟够借）：

```
删除 [1,2] 中的 1，左节点只剩 [2]，不够
但右兄弟 [6,7,9] 有富余：

     [5, 8]                    [6, 8]
    /  |   \         →       /   |   \
 [2] [6,7,9] [10]        [2,5] [7,9] [10]

父节点的 5 下移到左孩子，右兄弟的 6 上移到父节点
```

**合并**（兄弟也不够借）：

```
     [5, 8]                    [8]
    /  |   \         →       /    \
  [2] [6,7] [9,10]       [2,5,6,7] [9,10]

2 不够，右兄弟 [6,7] 也刚好满足最低要求不能借
→ 父节点的 5 下移，与 [2] 和 [6,7] 合并
```

# 第七章 B+ 树

---

## 7.1 B+ 树 vs B 树

B+ 树是 B 树的变体，**在数据库和文件系统中使用更广泛**。

| 特性 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储位置 | 所有节点都可以存数据 | **只有叶子节点**存储数据 |
| 内部节点 | 存储关键字 + 数据 + 指针 | **只存储关键字 + 指针**（索引） |
| 叶子节点 | 不一定连通 | **所有叶子节点用链表串联** |
| 关键字重复 | 不重复 | 内部节点的关键字**在叶子中重复出现** |
| 范围查询 | 需要中序遍历（复杂） | 叶子链表直接顺序扫描（高效） |
| 查找路径 | 可能在内部节点就找到 | **必须查到叶子节点** |

## 7.2 B+ 树的结构图解

一棵 3 阶 B+ 树：

```
                      [30, 60]                    ← 内部节点（只是索引）
                    /    |     \
            [10, 20]  [30, 50]  [60, 80]          ← 内部节点
           /  |  \    /  |  \   /  |  \
          ↓   ↓   ↓  ↓   ↓  ↓  ↓   ↓   ↓
叶子层:  [5,10] → [15,20] → [25,30] → [40,50] → [55,60] → [70,80] → [90,95]
                                                                        
         ↑————————————— 叶子节点用链表串联 ————————————————↑
```

**关键观察**：
- 内部节点 30 只起**路由**作用，用来选择继续向下搜索的孩子分支
- 真正的数据全在叶子节点
- 叶子节点之间有**顺序链表**，方便范围查询

## 7.3 B+ 树的查找

### 单点查找

```
查找 50：
1. 根 [30, 60]：30 ≤ 50 < 60 → 走中间
2. 内部 [30, 50]：50 ≥ 50 → 走右
3. 叶子 [40, 50]：找到 50 ✓

注意：即使在内部节点看到 50，也必须继续往下走到叶子节点！
```

### 范围查找（B+ 树的杀手锏！）

```
查找 20 ≤ key ≤ 55：

1. 先找到 20 所在的叶子节点 [15, 20]
2. 然后沿叶子链表顺序扫描：
   [15, 20] → [25, 30] → [40, 50] → [55, 60]
   取出 20, 25, 30, 40, 50, 55
3. 扫描到 60 > 55 时停止

整个过程只需要读取几个连续的叶子节点，非常高效！
```

对比 B 树做范围查找，需要反复执行中序遍历、回溯，效率低得多。

## 7.4 B+ 树的插入

与 B 树类似，但有细微差别：

**核心差别**：分裂时，中间关键字的**副本**上提到父节点（而不是移动），因为叶子节点要保留所有数据。

```
5 阶 B+ 树叶子节点插入导致分裂：

叶子 [10, 20, 30, 40] 插入 25：
→ [10, 20, 25, 30, 40] 超出！

分裂（取中间值 25 的副本上提）：
  父节点增加关键字 25
     ↓
  [10, 20] [25, 30, 40]    ← 注意 25 在叶子中仍然保留！
     ↑—————→↑（链表连接）
```

## 7.5 为什么数据库用 B+ 树？

| 优势 | 原因 |
|------|------|
| **更低的树高** | 内部节点不存数据，能容纳更多关键字和指针 → 扇出更大 → 树更矮 |
| **稳定的查找性能** | 每次查找都走到叶子，路径长度相同 |
| **高效的范围查询** | 叶子链表支持顺序扫描 |
| **更好的缓存利用** | 内部节点小，更容易缓存在内存中 |

> **实际应用**：MySQL InnoDB 引擎的索引就是 B+ 树！

## 8.1 各种树结构的全面对比

| 树结构 | 查找 | 插入 | 删除 | 适用场景 |
|--------|------|------|------|---------|
| **BST** | O(h)，最坏 O(n) | O(h) | O(h) | 简单场景，数据随机 |
| **AVL** | O(log n) | O(log n) | O(log n) | 查找密集型 |
| **红黑树** | O(log n) | O(log n)，旋转≤2 | O(log n)，旋转≤3 | 频繁增删（STL map/set） |
| **B 树** | O(log_m n) | O(log_m n) | O(log_m n) | 磁盘存储，数据库 |
| **B+ 树** | O(log_m n) | O(log_m n) | O(log_m n) | 数据库索引，范围查询 |
| **并查集** | α(n) ≈ O(1) | — | — | 连通性判断，Kruskal |

## 8.2 考试必备速查表

| n 个节点的完全二叉树中叶子节点数 | ⌈n/2⌉ |

### 遍历序列恢复规则

| 已知 | 能否唯一确定？ |
|------|-------------|
| 先序 + 中序 | ✓ 能 |
| 后序 + 中序 | ✓ 能 |
| 先序 + 后序 | ✗ 不能（中序缺失无法划分左右） |
| 层序 + 中序 | ✓ 能 |

### BST 删除三种情况速记

```
叶子 → 直接删
单孩子 → 孩子顶上
双孩子 → 前驱/后继替换，再删前驱/后继
```

### AVL 旋转速记

```
LL → 右旋    （左左→右）
RR → 左旋    （右右→左）
LR → 先左旋再右旋
RL → 先右旋再左旋
```

### 红黑树五条性质速记口诀

```
左根右（BST性质）
根叶黑（性质②③）
不红红（性质④）
黑路同（性质⑤）
```

### B 树关键参数（m 阶）

```
根节点: 关键字 [1, m-1]，孩子 0 或 [2, m]
其他节点: 关键字 [⌈m/2⌉-1, m-1]，孩子 0 或 [⌈m/2⌉, m]
所有叶子在同一层
```

## 8.3 经典练习题

### 练习 1：手写遍历结果

给定二叉树：
```
         1
       /   \
      2     3
     / \   /
    4   5 6
       /
      7
```

写出四种遍历序列。

**答案**：
- 先序（根左右）：1 2 4 5 7 3 6
- 中序（左根右）：4 2 7 5 1 6 3
- 后序（左右根）：4 7 5 2 6 3 1
- 层序：1 2 3 4 5 6 7

### 练习 2：由遍历恢复二叉树

已知中序 `C B E D F A H G`，后序 `C E F D B H G A`，恢复二叉树。

**解**：
1. 后序最后一个 A → 根
2. 中序中 A 的位置：左子树 `C B E D F`，右子树 `H G`
3. 后序对应：左子树后序 `C E F D B`，右子树后序 `H G`
4. 左子树：后序最后一个 B → 左子树的根。中序 `C | B | E D F`
5. B 的左子树 `C`（叶子），B 的右子树中序 `E D F`，后序 `E F D`，根 D
6. D 的左子树 `E`，右子树 `F`
7. 右子树：后序最后一个 G → 右子树的根。中序 `H | G`
8. G 的左子树 `H`

```
结果:
         A
       /   \
      B     G
     / \   /
    C   D H
       / \
      E   F
```

### 练习 3：BST 操作

从空树开始，依次插入 50, 30, 70, 20, 40, 60, 80, 35, 45，画出最终的 BST。然后删除 30。

**插入结果**：
```
           50
         /    \
       30      70
      / \     / \
    20   40  60  80
        / \
       35  45
```

**删除 30**（用中序前驱 20 替换）：
```
           50
         /    \
       20      70
        \     / \
        40   60  80
       / \
      35  45
```

或用中序后继 35 替换：
```
           50
         /    \
       35      70
      / \     / \
    20   40  60  80
          \
          45
```

### 练习 4：判断红黑树合法性

```
       20(B)
      /    \
    10(R)  30(R)
    /       \
   5(B)    35(B)
```

- 性质②: 根 20 黑色 ✓
- 性质④: 红 10 的孩子 5 是黑 ✓；红 30 的孩子 35 是黑 ✓
- 性质⑤:
  - 20→10→5→NIL: 黑节点 = 20, 5 → 2个
  - 20→10→NIL(10的右): 黑节点 = 20 → 1个 ✗

**不合法！** 路径 20→10→NIL(右) 只有 1 个黑节点，但 20→10→5→NIL 有 2 个。
