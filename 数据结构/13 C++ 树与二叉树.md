# 阶段八：树（Tree）完全教程

> 树是数据结构中从"线性"迈向"非线性"的关键转折点。前面学过的数组、链表、栈、队列都是线性结构——数据元素之间是"一对一"关系；而树是**层次结构**——数据元素之间是**"一对多"**关系。掌握树形结构，是理解数据库索引、文件系统、编译器语法树、网络路由等核心计算机技术的基础。

---

## Part 1 ｜ 树的基础概念

## 第一章 树的定义

### 1.1 什么是树？

**树（Tree）** 是 $n$（$n \geq 0$）个节点的**有限集合**。当 $n = 0$ 时，称为**空树**。

任何一棵非空树必须满足以下条件：

1. **有且仅有一个**特殊的节点，称为**根（Root）**
2. 其余节点可以分为 $m$（$m \geq 0$）个**互不相交**的有限集合 $T_1, T_2, \dots, T_m$，每个集合本身又是一棵树，称为根的**子树（Subtree）**

这是一个典型的**递归定义**——树由更小的树组成。

```
               A              ← 根节点
             / | \
            B  C  D           ← A 的三棵子树的根
           / \    |
          E   F   G           ← B 的子树、D 的子树
             / \
            H   I             ← F 的子树
```

用生活中的例子来理解：

| 场景 | 根 | 子树 | 叶子 |
|------|----|------|------|
| 公司组织架构 | CEO | 各部门 | 普通员工 |
| 文件系统 | 根目录 `/` | 子目录 | 文件 |
| 家族族谱 | 祖先 | 各支脉 | 当代后人 |
| HTML 页面 | `<html>` | `<head>`、`<body>` | 具体标签 |

**关键理解**：树的递归性质是所有树形算法的基础。"处理一棵树"往往可以分解为"处理根节点 + 递归处理每棵子树"。

### 1.2 树的基本术语

这些术语在后续学习中会反复出现，必须烂熟于心。

我们以下面这棵树为例来逐一讲解：

```
                A
              / | \
             B  C  D
            / \    |
           E   F   G
              / \
             H   I
```

#### （1）节点相关术语

| 术语 | 含义 | 示例 |
|------|------|------|
| **节点（Node）** | 树中的每一个数据元素 | A, B, C, D, E, F, G, H, I |
| **根节点（Root）** | 树中唯一没有前驱的节点 | A |
| **叶子节点（Leaf）**<br>又称终端节点 | **度为 0** 的节点（没有孩子） | E, H, I, C, G |
| **分支节点**<br>又称非终端节点 | **度不为 0** 的节点（有孩子） | A, B, D, F |
| **内部节点** | 除根节点外的分支节点 | B, D, F |

#### （2）关系相关术语

| 术语 | 含义 | 示例 |
|------|------|------|
| **孩子（Child）** | 节点的子树的根称为该节点的孩子 | B、C、D 是 A 的孩子 |
| **双亲/父节点（Parent）** | 某节点的直接上层节点 | A 是 B 的双亲 |
| **兄弟（Sibling）** | 同一双亲的孩子 | B、C、D 互为兄弟 |
| **祖先（Ancestor）** | 从根到该节点路径上的所有节点 | H 的祖先：F、B、A |
| **子孙（Descendant）** | 以某节点为根的子树中的所有节点 | B 的子孙：E、F、H、I |
| **堂兄弟** | 双亲在同一层的节点 | E 和 G 互为堂兄弟 |

#### （3）度、层次、高度相关术语

| 术语 | 含义 | 示例 |
|------|------|------|
| **节点的度（Degree）** | 该节点拥有的**子树个数**（即孩子数） | A 的度 = 3，B 的度 = 2，C 的度 = 0 |
| **树的度** | 树中所有节点度的**最大值** | 此树的度 = 3（因为 A 有 3 个孩子） |
| **层次（Level）** | 根为第 1 层，孩子为第 2 层，依此类推 | A 在第 1 层，H 在第 4 层 |
| **深度（Depth）** | 从**根**到该节点的路径上的**边数**（或层数-1） | H 的深度 = 3 |
| **高度（Height）** | 从该节点到**最远叶子**的路径上的**边数** | A 的高度 = 3，B 的高度 = 2 |
| **树的高度/深度** | 树中节点的最大层数（或根的高度） | 此树的高度 = 4（按层数计）或 3（按边数计） |
| **路径** | 从节点 X 到节点 Y 所经过的节点序列 | A→B→F→H 是一条路径 |
| **路径长度** | 路径上经过的**边数** | A→B→F→H 的路径长度 = 3 |

> ⚠️ **高度/深度的计数方式**
> 不同教材对"高度"和"深度"的起始值定义不一致：
> - **王道/严蔚敏**：根节点在第 1 层，树的高度 = 最大层数（上面例子中树高 = 4）
> - **部分国外教材**：根节点深度 = 0，树的高度 = 根到最远叶子的边数（上面例子中树高 = 3）
>
> 本教程以**王道规范**为主（根在第 1 层），但会在必要时说明两种计数法。**做题时一定要看清题目的定义！**

#### （4）其他术语

| 术语 | 含义 |
|------|------|
| **有序树** | 各子树从左到右有固定次序，不能互换 |
| **无序树** | 各子树没有次序关系 |
| **森林（Forest）** | $m$（$m \geq 0$）棵互不相交的树的集合 |

> **树与森林的关系**：对树的根节点进行删除操作，树就变成了森林；给森林中所有树加一个共同的根，森林就变成了树。

### 1.3 树与线性结构的对比

| 维度 | 线性结构 | 树形结构 |
|------|----------|----------|
| 元素关系 | 一对一 | 一对多 |
| 首/根 | 有且仅有一个首元素（无前驱） | 有且仅有一个根节点（无双亲） |
| 尾/叶 | 有且仅有一个尾元素（无后继） | 可以有多个叶子节点（无孩子） |
| 中间元素 | 有且仅有一个前驱、一个后继 | 有且仅有一个双亲、可以有多个孩子 |
| 典型实例 | 数组、链表、栈、队列 | 文件系统、组织架构、二叉树 |

## 第二章 树的性质

树有一系列重要的数学性质，**是考研选择题的常客**，也是理解后续算法的基础。

### 2.1 性质一：节点数 = 边数 + 1

> **树中节点总数 $n$ = 总边数 $e$ + 1**，即 $n = e + 1$。

**证明**：
- 除了根节点之外，每个节点都有且仅有一条边连接到它的双亲
- 因此边数 = 非根节点数 = $n - 1$
- 所以 $n = (n-1) + 1 = e + 1$

```
       A          节点数 = 9
      /|\          边数 = 8（每条线段就是一条边）
     B C D        9 = 8 + 1 ✓
    / \  |
   E   F  G
      / \
     H   I
```

### 2.2 性质二：度与节点数的关系

> **总边数 = 所有节点度之和**

因为每条边恰好对应一个"从双亲指向孩子"的关系，而节点的度就是它指向孩子的边数。

结合性质一：

$$n = 1 + \sum_{i=1}^{n} degree(node_i) = 1 + n_1 + 2n_2 + 3n_3 + \cdots$$

其中 $n_k$ 表示度为 $k$ 的节点个数。

**推论**：$n_0 + n_1 + n_2 + \cdots + n_m = 1 + 0 \cdot n_0 + 1 \cdot n_1 + 2 \cdot n_2 + \cdots + m \cdot n_m$

化简得：

$$n_0 = 1 + n_2 + 2n_3 + 3n_4 + \cdots + (m-1)n_m$$

**特别地，对于二叉树**（$m = 2$）：$n_0 = n_2 + 1$，即**叶子节点数 = 度为 2 的节点数 + 1**。这是一个非常重要的结论！

### 2.3 性质三：$m$ 叉树的节点数范围

> **高度为 $h$ 的 $m$ 叉树**（$m \geq 2$）：
> - **最少**节点数：$h$（退化成一条链，每层一个节点）
> - **最多**节点数：$\dfrac{m^h - 1}{m - 1}$（每层都满）

这是等比数列求和：$1 + m + m^2 + \cdots + m^{h-1} = \dfrac{m^h - 1}{m - 1}$

```
高度 h = 3 的二叉树（m = 2）：

最少 3 个节点：         最多 7 个节点：
     A                       A
      \                    /   \
       B                  B     C
        \               / \   / \
         C             D   E F   G

    节点数 = 3          节点数 = 2³ - 1 = 7
```

### 2.4 性质四：节点数为 $n$ 的 $m$ 叉树最小高度

> $$h_{min} = \lceil \log_m(n(m-1) + 1) \rceil$$

这是由"最多节点数"公式反推而来的——当每层都放满时，高度最小。

### 2.5 常见考点练习

**【例题 1】** 一棵度为 4 的树有 20 个度为 4 的节点，10 个度为 3 的节点，1 个度为 2 的节点，10 个度为 1 的节点。求叶子节点个数。

**解**：

利用公式 $n = 1 + 总边数 = 1 + (4 \times 20 + 3 \times 10 + 2 \times 1 + 1 \times 10 + 0 \times n_0)$

即 $n_0 + 10 + 1 + 10 + 20 = 1 + 80 + 30 + 2 + 10$

$n_0 + 41 = 123$

$n_0 = 82$

**答：叶子节点有 82 个。**

**【例题 2】** 已知一棵完全二叉树有 500 个节点，求叶子节点个数。

**解**：完全二叉树中度为 1 的节点最多只有 1 个（当 $n$ 为偶数时有 1 个，$n$ 为奇数时有 0 个）。

$n = 500$（偶数），所以 $n_1 = 1$。

由 $n = n_0 + n_1 + n_2$ 和 $n_0 = n_2 + 1$：

$500 = n_0 + 1 + (n_0 - 1) = 2n_0$

$n_0 = 250$

**答：叶子节点有 250 个。**

## Part 2 ｜ 二叉树（Binary Tree）

## 第三章 二叉树的定义和基本术语

### 3.1 什么是二叉树？

**二叉树（Binary Tree）** 是 $n$（$n \geq 0$）个节点的有限集合，该集合或者为空集（空二叉树），或者由一个**根节点**和两棵互不相交的、分别称为根的**左子树**和**右子树**的二叉树组成。

```
        A             ← 根
       / \
      B   C           ← A 的左子树、右子树
     / \   \
    D   E   F         ← B 的左右子树、C 的右子树
```

**二叉树的特点**：

1. 每个节点**最多**有 2 棵子树（度 ≤ 2）
2. 子树有**左右之分**，次序不能颠倒（有序树）
3. 即使只有一棵子树，也要区分是**左子树**还是**右子树**

### 3.2 二叉树 vs 度为 2 的树

这是一个常见的混淆点，必须区分清楚：

| 区别 | 二叉树 | 度为 2 的有序树 |
|------|--------|-----------------|
| 子树数量 | 每个节点**最多** 2 个孩子 | 树中**至少存在**一个度为 2 的节点 |
| 空树 | 可以是空树 | 至少有 3 个节点（根 + 2 个孩子） |
| 左右区分 | 严格区分左右子树 | 当只有一个孩子时不区分左右 |
| 只有一个孩子 | 必须明确是左孩子还是右孩子 | 无所谓 |

下面两棵二叉树是**不同**的，但作为度为 2 的有序树则是**相同**的：

```
二叉树 1：           二叉树 2：
    A                    A
   /                      \
  B                        B

（B 是 A 的左孩子）  （B 是 A 的右孩子）
```

### 3.3 五种基本形态

二叉树有且仅有以下五种基本形态：

```
①空树    ②只有根   ③只有左子树  ④只有右子树  ⑤左右都有

 (空)        A           A            A           A
                        /              \         / \
                       B                B       B   C
```

### 3.4 特殊的二叉树

#### （1）满二叉树（Perfect Binary Tree）

**定义**：高度为 $h$（从第 1 层算起），且有 $2^h - 1$ 个节点的二叉树。**每一层都达到了最大节点数。**

```
         1
        / \
       2   3
      / \ / \
     4  5 6  7

高度 h = 3，节点数 = 2³ - 1 = 7
```

**特点**：
- 只有度为 0（叶子）和度为 2 的节点，**没有度为 1 的节点**
- 第 $i$ 层有 $2^{i-1}$ 个节点
- 叶子全部在最后一层
- 按层序编号后，节点 $i$ 的左孩子编号为 $2i$，右孩子为 $2i+1$，双亲为 $\lfloor i/2 \rfloor$

#### （2）完全二叉树（Complete Binary Tree）

**定义**：高度为 $h$ 的二叉树，当且仅当其每个节点都与高度为 $h$ 的满二叉树中编号 $1 \sim n$ 的节点一一对应时，称为完全二叉树。

通俗理解：**在满二叉树的基础上，从最后一层的最右边开始连续删去若干节点**。

```
完全二叉树：             不是完全二叉树：
      1                        1
     / \                      / \
    2   3                    2   3
   / \ /                   / \   \
  4  5 6                  4   5   7
                                  ↑ 编号 6 缺失，7 却在
```

**完全二叉树的重要性质**（考试必考！）：

| 性质 | 内容 |
|------|------|
| 性质 1 | 具有 $n$ 个节点的完全二叉树的高度 $h = \lfloor \log_2 n \rfloor + 1$ |
| 性质 2 | 如果 $n$ 为奇数，则所有分支节点都有左右孩子（$n_1 = 0$）|
| 性质 3 | 如果 $n$ 为偶数，则编号最大的分支节点（编号 $n/2$）只有左孩子（$n_1 = 1$）|
| 性质 4 | 叶子节点只出现在最后两层 |
| 性质 5 | 如果某节点只有一个孩子，那一定是**左孩子** |

#### （3）二叉排序树（BST, Binary Search Tree）

**定义**：或者是空树，或者具有如下性质：

- 若左子树非空，则左子树上**所有节点**的值 **<** 根节点的值
- 若右子树非空，则右子树上**所有节点**的值 **>** 根节点的值
- 左右子树也分别是二叉排序树

```
         30
        /  \
      15    50
     / \   / \
    10 20 40  60

中序遍历结果：10 15 20 30 40 50 60（有序！）
```

> BST 的详细操作将在本教程 Part 6 中专门讲解。

#### （4）平衡二叉树（AVL Tree）

**定义**：树上任一节点的**左子树高度与右子树高度之差的绝对值不超过 1**，并且左右子树本身也是平衡二叉树。

> 平衡二叉树的详细内容将在进阶教程中讲解。

## 第四章 二叉树的性质

这些性质是做题和写代码的基础，务必理解并记住。

### 4.1 性质一：叶子与度为 2 的节点

> **在任意二叉树中：$n_0 = n_2 + 1$**
>
> 叶子节点数 = 度为 2 的节点数 + 1

**证明**：

二叉树只有三种节点——度为 0、1、2，因此：

$$n = n_0 + n_1 + n_2 \quad \cdots (1)$$

总边数 = 各节点度之和（每条边对应一个孩子）：

$$e = 0 \cdot n_0 + 1 \cdot n_1 + 2 \cdot n_2 = n_1 + 2n_2 \quad \cdots (2)$$

又因为 $n = e + 1$（树的基本性质）：

$$n_0 + n_1 + n_2 = n_1 + 2n_2 + 1$$

$$n_0 = n_2 + 1 \quad \blacksquare$$

### 4.2 性质二：第 $i$ 层的最大节点数

> **二叉树的第 $i$ 层最多有 $2^{i-1}$ 个节点**（$i \geq 1$）

```
第 1 层：2⁰ = 1  个节点（根）
第 2 层：2¹ = 2  个节点
第 3 层：2² = 4  个节点
第 4 层：2³ = 8  个节点
...
第 i 层：2^(i-1) 个节点
```

### 4.3 性质三：高度为 $h$ 的二叉树的最大节点数

> **高度为 $h$ 的二叉树最多有 $2^h - 1$ 个节点**

等比数列求和：$1 + 2 + 4 + \cdots + 2^{h-1} = 2^h - 1$

### 4.4 性质四：完全二叉树节点编号规律

> 对一棵有 $n$ 个节点的**完全二叉树**，按层序从 1 开始编号，则对于编号为 $i$ 的节点：

| 关系 | 编号 | 条件 |
|------|------|------|
| **双亲** | $\lfloor i/2 \rfloor$ | $i > 1$ 时存在 |
| **左孩子** | $2i$ | $2i \leq n$ 时存在 |
| **右孩子** | $2i + 1$ | $2i + 1 \leq n$ 时存在 |
| **所在层** | $\lfloor \log_2 i \rfloor + 1$ | — |

```
编号规律图示：
           1
         /   \
        2     3
       / \   / \
      4   5 6   7
     /
    8

节点 3 的双亲 = ⌊3/2⌋ = 1 → A    ✓
节点 3 的左孩子 = 2×3 = 6          ✓
节点 3 的右孩子 = 2×3+1 = 7        ✓
节点 5 的双亲 = ⌊5/2⌋ = 2 → B    ✓
```

> **为什么这个性质如此重要？**
> 它是**顺序存储**二叉树的基础！用数组存储完全二叉树时，完全靠下标的算术运算就能找到父子关系，不需要指针。堆（Heap）就是基于这个性质实现的。

### 4.5 性质五：完全二叉树的高度

> **具有 $n$ 个节点的完全二叉树的高度为 $\lfloor \log_2 n \rfloor + 1$**

（或从 0 开始计高度时为 $\lfloor \log_2 n \rfloor$）

### 4.6 常见考点练习

**【例题 1】** 一棵二叉树有 67 个叶子节点，有 34 个度为 1 的节点，求总节点数。

**解**：由 $n_0 = n_2 + 1$ 得 $n_2 = n_0 - 1 = 66$

$n = n_0 + n_1 + n_2 = 67 + 34 + 66 = 167$

**答：总节点数为 167。**

**【例题 2】** 一棵完全二叉树的第 6 层有 8 个叶子节点，求最多有多少个节点？

**解**：

- 第 6 层最多有 $2^5 = 32$ 个节点
- 其中 8 个是叶子（无孩子），24 个是分支节点（有孩子在第 7 层）
- 要使节点数最多，第 7 层应全满：$24 \times 2 = 48$ 个节点
- 第 1~6 层全满：$2^6 - 1 = 63$ 个节点
- 总数 = $63 + 48 = 111$

**答：最多 111 个节点。**

## 第五章 二叉树的存储结构

### 5.1 顺序存储

#### 5.1.1 基本思想

用一个**一维数组**按照**完全二叉树的编号顺序**来存储二叉树的节点。

```
完全二叉树：                数组存储：
         A                 下标: [0]  [1]  [2]  [3]  [4]  [5]
        / \                内容:  A    B    C    D    E    F
       B   C
      / \ /
     D  E F

编号从 0 开始时：
  左孩子 = 2i + 1
  右孩子 = 2i + 2
  双亲   = (i - 1) / 2
```

> **注意**：教材中编号有时从 1 开始（王道），有时从 0 开始（代码实现常用）。同一段代码或同一道题中必须统一编号规则。

#### 5.1.2 非完全二叉树的顺序存储

如果是非完全二叉树，需要把"缺失"的节点位置空出来（填 0 或特殊标记），以保证父子关系仍可通过下标计算。

```
非完全二叉树：              编号（按完全二叉树位置）：
         A                          1
        / \                        / \
       B   C                      2   3
        \   \                      \   \
         D   E                      5   7

数组存储（编号从 1 开始）：
下标: [1]  [2]  [3]  [4]  [5]  [6]  [7]
内容:  A    B    C    ∅    D    ∅    E

[4] 和 [6] 空着，浪费了空间。
```

#### 5.1.3 顺序存储的优缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 空间 | 不需要额外指针 | 非完全二叉树可能浪费大量空间 |
| 访问 | 通过下标 O(1) 找父子 | — |
| 适用场景 | **完全二叉树**（堆）| 稀疏二叉树不适合 |

**最坏情况**：一个高度为 $h$ 的右斜树（每个节点只有右孩子），只有 $h$ 个节点，却需要 $2^h - 1$ 个数组空间！

```
        A               数组需要 2⁴ - 1 = 15 个位置
         \              但只存了 4 个有效节点
          B             空间利用率极低！
           \
            C
             \
              D
```

#### 5.1.4 C++ 实现

```cpp
#include <iostream>
#include <vector>
using namespace std;

class ArrayBinaryTree {
private:
    vector<int> tree;  // 用 0 表示空节点
    int capacity;

public:
    ArrayBinaryTree(int cap) : capacity(cap) {
        tree.resize(cap + 1, 0);  // 编号从 1 开始，[0] 不用
    }

    void setNode(int index, int value) {
        if (index >= 1 && index <= capacity) {
            tree[index] = value;
        }
    }

    int parent(int i) {
        if (i <= 1 || i > capacity) return -1;
        return tree[i / 2] != 0 ? i / 2 : -1;
    }

    int leftChild(int i) {
        int left = 2 * i;
        if (left > capacity) return -1;
        return tree[left] != 0 ? left : -1;
    }

    int rightChild(int i) {
        int right = 2 * i + 1;
        if (right > capacity) return -1;
        return tree[right] != 0 ? right : -1;
    }

    void levelOrder() {
        for (int i = 1; i <= capacity; i++) {
            if (tree[i] != 0) {
                cout << tree[i] << " ";
            }
        }
        cout << endl;
    }
};
```

### 5.2 链式存储

#### 5.2.1 二叉链表（最常用）

每个节点包含三个域：**数据域** + **左指针** + **右指针**。

```
节点结构：
┌──────┬──────┬──────┐
│ left  │ data │ right│
└──┬───┘──────└───┬──┘
   │              │
   ↓              ↓
 左子树          右子树
```

**C++ 定义**：

```cpp
struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};
```

**图示**：

```
二叉树：            链式存储：
      A
     / \           ┌───┬───┬───┐
    B   C          │ ● │ A │ ● │
   / \              └─┬─┘   └─┬─┘
  D   E               │       │
                       ↓       ↓
                 ┌───┬───┬───┐  ┌───┬───┬───┐
                 │ ● │ B │ ● │  │ ∅ │ C │ ∅ │
                 └─┬─┘   └─┬─┘  └───┘   └───┘
                   │       │
                   ↓       ↓
            ┌───┬───┬───┐  ┌───┬───┬───┐
            │ ∅ │ D │ ∅ │  │ ∅ │ E │ ∅ │
            └───┘   └───┘  └───┘   └───┘
```

#### 5.2.2 三叉链表

在二叉链表的基础上，每个节点增加一个**指向双亲**的指针。

```
节点结构：
┌──────┬──────┬──────┬────────┐
│ left  │ data │ right│ parent │
└──────┘──────┘──────┘────────┘
```

```cpp
struct TriTreeNode {
    int data;
    TriTreeNode* left;
    TriTreeNode* right;
    TriTreeNode* parent;

    TriTreeNode(int val) 
        : data(val), left(nullptr), right(nullptr), parent(nullptr) {}
};
```

**何时使用三叉链表？**
- 当需要频繁访问某节点的双亲时（如 AVL 树的旋转、红黑树的调整）
- 代价是每个节点多占一个指针的空间

#### 5.2.3 空指针个数

> **含有 $n$ 个节点的二叉链表中，有 $n + 1$ 个空指针。**

**证明**：

- $n$ 个节点共有 $2n$ 个指针域（每个节点有 left 和 right）
- 其中被使用的指针 = 边数 = $n - 1$
- 空指针数 = $2n - (n - 1) = n + 1$

> 这 $n + 1$ 个空指针在后续"线索二叉树"中会被利用起来！

#### 5.2.4 两种存储方式的对比

| 维度 | 顺序存储 | 链式存储 |
|------|----------|----------|
| 空间 | 完全二叉树最优，稀疏树浪费大 | 额外指针开销，但不浪费 |
| 找父节点 | O(1)，下标计算 | O(n)，需遍历（除非用三叉链表）|
| 找左右孩子 | O(1)，下标计算 | O(1)，直接指针 |
| 插入/删除 | 可能需要大量移动 | O(1)，修改指针 |
| 适用场景 | 完全二叉树、堆 | 一般二叉树 |

## Part 3 ｜ 二叉树的遍历

## 第六章 先序、中序、后序遍历

### 6.1 什么是遍历？

**遍历（Traversal）** 是指按照某种规则，**不重复、不遗漏**地访问二叉树中的**每一个**节点。

为什么遍历是核心操作？因为二叉树是非线性结构，不像数组那样可以简单地从头扫到尾。我们需要一种系统化的方法来"线性化"地访问所有节点。

二叉树的递归结构：**根节点 D + 左子树 L + 右子树 R**。

不同的排列顺序产生了不同的遍历方式：

| 遍历方式 | 访问顺序 | 名称由来 |
|----------|----------|----------|
| **先序（Pre-order）** | **D** → L → R | 根在**前面** |
| **中序（In-order）** | L → **D** → R | 根在**中间** |
| **后序（Post-order）** | L → R → **D** | 根在**后面** |

> **"先、中、后"描述的是根节点被访问的时机**——先于子树、在子树中间、后于子树。而左子树永远先于右子树。

### 6.2 一个完整的例子

我们用下面这棵树来演示所有遍历方式：

```
           A
          / \
         B   C
        / \   \
       D   E   F
          / \
         G   H
```

#### 先序遍历（DLR）：A B D E G H C F

**思路**：先访问根 → 先序遍历左子树 → 先序遍历右子树

```
访问 A → 进入左子树
  访问 B → 进入左子树
    访问 D → 无子树，回退
  回到 B → 进入右子树
    访问 E → 进入左子树
      访问 G → 无子树，回退
    回到 E → 进入右子树
      访问 H → 无子树，回退
回到 A → 进入右子树
  访问 C → 无左子树 → 进入右子树
    访问 F → 无子树，回退

结果：A B D E G H C F
```

#### 中序遍历（LDR）：D B G E H A C F

**思路**：中序遍历左子树 → 访问根 → 中序遍历右子树

```
进入 A 的左子树
  进入 B 的左子树
    D 无左子树 → 访问 D → D 无右子树，回退
  访问 B
  进入 B 的右子树
    进入 E 的左子树
      G 无左子树 → 访问 G → G 无右子树，回退
    访问 E
    进入 E 的右子树
      H 无左子树 → 访问 H → H 无右子树，回退
访问 A
进入 A 的右子树
  C 无左子树 → 访问 C
  进入 C 的右子树
    F 无左子树 → 访问 F → F 无右子树，回退

结果：D B G E H A C F
```

#### 后序遍历（LRD）：D G H E B F C A

**思路**：后序遍历左子树 → 后序遍历右子树 → 访问根

```
进入 A 的左子树
  进入 B 的左子树
    D 无子树 → 访问 D
  进入 B 的右子树
    进入 E 的左子树
      G 无子树 → 访问 G
    进入 E 的右子树
      H 无子树 → 访问 H
    访问 E
  访问 B
进入 A 的右子树
  C 无左子树
  进入 C 的右子树
    F 无子树 → 访问 F
  访问 C
访问 A

结果：D G H E B F C A
```

### 6.3 手绘秘诀：包络线法

画遍历结果最快的方法是**包络线法**（又叫"路径法"）：

**步骤**：
1. 从根节点左侧开始，沿着树的外轮廓画一条线，逆时针绕一圈回到根节点右侧
2. 对于**先序**：经过节点**左侧**时记录
3. 对于**中序**：经过节点**下方**时记录
4. 对于**后序**：经过节点**右侧**时记录

```
           A·
          / \
        ·B·  ·C·
        / \    \
      ·D·  ·E·  ·F·
           / \
         ·G· ·H·

包络线从 A 的左侧出发，逆时针环绕整棵树：

       ↓ A ↑
      ↓/ \↑
     ↓B↑  ↓C↑
    ↓/\↑   ↓\↑
   ↓D↑↓E↑   ↓F↑
      ↓/\↑
     ↓G↑↓H↑

先序（过左侧）：A B D E G H C F  ✓
中序（过下方）：D B G E H A C F  ✓
后序（过右侧）：D G H E B F C A  ✓
```

### 6.4 递归实现

三种遍历的递归代码极其简洁，**只是 "访问" 那一行的位置不同**：

```cpp
#include <iostream>
using namespace std;

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

// ============ 先序遍历 ============
void preOrder(TreeNode* root) {
    if (root == nullptr) return;
    cout << root->data << " ";   // 【访问根】
    preOrder(root->left);         // 遍历左子树
    preOrder(root->right);        // 遍历右子树
}

// ============ 中序遍历 ============
void inOrder(TreeNode* root) {
    if (root == nullptr) return;
    inOrder(root->left);          // 遍历左子树
    cout << root->data << " ";   // 【访问根】
    inOrder(root->right);         // 遍历右子树
}

// ============ 后序遍历 ============
void postOrder(TreeNode* root) {
    if (root == nullptr) return;
    postOrder(root->left);        // 遍历左子树
    postOrder(root->right);       // 遍历右子树
    cout << root->data << " ";   // 【访问根】
}
```

**递归执行过程可视化**（以先序遍历一棵简单树为例）：

```
       1
      / \
     2   3

调用栈展开：
preOrder(1)
├── 输出 1
├── preOrder(2)
│   ├── 输出 2
│   ├── preOrder(null) → return
│   └── preOrder(null) → return
└── preOrder(3)
    ├── 输出 3
    ├── preOrder(null) → return
    └── preOrder(null) → return

输出：1 2 3
```

### 6.5 非递归实现（用栈模拟）

下面代码会用到：

```cpp
#include <algorithm>
#include <stack>
#include <vector>
using namespace std;
```

递归本质上是利用系统的调用栈来实现的。我们可以用自己维护的**显式栈**来模拟这一过程。

#### 先序遍历（非递归）

**思路**：
1. 根节点入栈
2. 循环：弹栈并访问，然后先压入右孩子，再压入左孩子（因为栈是后进先出，所以左孩子先出来）

```cpp
vector<int> preOrderIterative(TreeNode* root) {
    vector<int> result;
    if (root == nullptr) return result;

    stack<TreeNode*> stk;
    stk.push(root);

    while (!stk.empty()) {
        TreeNode* node = stk.top();
        stk.pop();
        result.push_back(node->data);

        if (node->right) stk.push(node->right);  // 先压右
        if (node->left)  stk.push(node->left);   // 再压左
    }
    return result;
}
```

**执行过程**：

```
       A
      / \
     B   C
    / \
   D   E

栈状态（左边是栈顶）：
初始：[A]
弹 A，输出 A，压入 C、B  → 栈 [B, C]
弹 B，输出 B，压入 E、D  → 栈 [D, E, C]
弹 D，输出 D，无孩子      → 栈 [E, C]
弹 E，输出 E，无孩子      → 栈 [C]
弹 C，输出 C，无孩子      → 栈 []

结果：A B D E C  ✓
```

#### 中序遍历（非递归）

**思路**：
1. 一路向左走到底，沿途节点全部入栈
2. 走到底后弹栈、访问，然后转向右子树
3. 重复以上过程

```cpp
vector<int> inOrderIterative(TreeNode* root) {
    vector<int> result;
    stack<TreeNode*> stk;
    TreeNode* curr = root;

    while (curr != nullptr || !stk.empty()) {
        // 一路向左，全部入栈
        while (curr != nullptr) {
            stk.push(curr);
            curr = curr->left;
        }
        // 弹栈、访问、转右
        curr = stk.top();
        stk.pop();
        result.push_back(curr->data);
        curr = curr->right;
    }
    return result;
}
```

**执行过程**：

```
       A
      / \
     B   C
    / \
   D   E

步骤 1：一路向左 → A入栈、B入栈、D入栈。栈 [D, B, A]
步骤 2：弹D，输出D，D无右子树。curr = null
步骤 3：弹B，输出B，B有右子树E。curr = E
步骤 4：E入栈，E无左子树。栈 [E, A]
步骤 5：弹E，输出E，E无右子树。curr = null
步骤 6：弹A，输出A，A有右子树C。curr = C
步骤 7：C入栈，C无左子树。栈 [C]
步骤 8：弹C，输出C，C无右子树。curr = null。栈空，结束。

结果：D B E A C  ✓
```

#### 后序遍历（非递归）

后序遍历的非递归实现是三种中**最复杂的**，因为根节点是最后才访问的，我们需要知道"右子树是否已经遍历完毕"。

**方法：反向先序 + 翻转**（最好理解）

先序是 DLR，如果我们改为 DRL（先右后左），再把结果翻转，就得到了 LRD（后序）。

```cpp
vector<int> postOrderIterative(TreeNode* root) {
    vector<int> result;
    if (root == nullptr) return result;

    stack<TreeNode*> stk;
    stk.push(root);

    while (!stk.empty()) {
        TreeNode* node = stk.top();
        stk.pop();
        result.push_back(node->data);

        if (node->left)  stk.push(node->left);   // 先压左
        if (node->right) stk.push(node->right);  // 再压右
    }
    // result 现在是 DRL 序列，翻转得到 LRD
    reverse(result.begin(), result.end());
    return result;
}
```

**方法：单栈 + prev 指针**（更正统的写法）

```cpp
vector<int> postOrderIterative2(TreeNode* root) {
    vector<int> result;
    if (root == nullptr) return result;

    stack<TreeNode*> stk;
    TreeNode* curr = root;
    TreeNode* prev = nullptr;  // 记录上一个访问过的节点

    while (curr != nullptr || !stk.empty()) {
        // 一路向左
        while (curr != nullptr) {
            stk.push(curr);
            curr = curr->left;
        }

        curr = stk.top();

        // 如果没有右子树，或者右子树已经访问过
        if (curr->right == nullptr || curr->right == prev) {
            result.push_back(curr->data);
            prev = curr;
            stk.pop();
            curr = nullptr;  // 不要再向左走了
        } else {
            curr = curr->right;  // 转向右子树
        }
    }
    return result;
}
```

### 6.6 三种遍历的时空复杂度

| 维度 | 递归 | 非递归 |
|------|------|--------|
| 时间复杂度 | O(n) | O(n) |
| 空间复杂度 | O(h)（系统调用栈） | O(h)（显式栈） |

其中 $h$ 是树的高度。最坏情况（退化成链）$h = n$，最好情况（完全二叉树）$h = \log_2 n$。

## 第七章 层序遍历

### 7.1 什么是层序遍历？

**层序遍历（Level-order Traversal）** 又叫**广度优先遍历（BFS）**，按照从上到下、从左到右的顺序，**逐层访问**二叉树的节点。

```
           A          层序遍历结果：A B C D E F G H
          / \
         B   C        第 1 层：A
        / \   \       第 2 层：B C
       D   E   F     第 3 层：D E F
          / \         第 4 层：G H
         G   H
```

### 7.2 实现思路：队列

层序遍历需要用到**队列**——先访问本层的节点，同时将下一层的孩子入队。

**算法步骤**：
1. 根节点入队
2. 循环直到队列为空：
   - 出队一个节点，访问它
   - 将它的左孩子入队（如果存在）
   - 将它的右孩子入队（如果存在）

### 7.3 代码实现

```cpp
#include <queue>
#include <vector>
using namespace std;

vector<int> levelOrder(TreeNode* root) {
    vector<int> result;
    if (root == nullptr) return result;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        TreeNode* node = q.front();
        q.pop();
        result.push_back(node->data);

        if (node->left)  q.push(node->left);
        if (node->right) q.push(node->right);
    }
    return result;
}
```

**执行过程**：

```
           A
          / \
         B   C
        / \   \
       D   E   F

队列状态（左边是队头）：
初始：  [A]
出A：   [B, C]        输出 A
出B：   [C, D, E]     输出 B
出C：   [D, E, F]     输出 C
出D：   [E, F]        输出 D
出E：   [F]           输出 E
出F：   []            输出 F

结果：A B C D E F  ✓
```

### 7.4 变体：分层输出

很多题目要求按层分组输出，而不是混在一起。只需要在每层开始时记录当前队列长度：

```cpp
vector<vector<int>> levelOrderByLevel(TreeNode* root) {
    vector<vector<int>> result;
    if (root == nullptr) return result;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int levelSize = q.size();  // 当前层的节点数
        vector<int> currentLevel;

        for (int i = 0; i < levelSize; i++) {
            TreeNode* node = q.front();
            q.pop();
            currentLevel.push_back(node->data);

            if (node->left)  q.push(node->left);
            if (node->right) q.push(node->right);
        }

        result.push_back(currentLevel);
    }
    return result;
}
```

输出示例：`[[A], [B, C], [D, E, F], [G, H]]`

### 7.5 四种遍历的总结对比

| 遍历方式 | 数据结构 | 顺序 | 特点 | 典型应用 |
|----------|----------|------|------|----------|
| 先序 DLR | 栈（递归/显式） | 根→左→右 | 第一个是根 | 复制树、序列化 |
| 中序 LDR | 栈（递归/显式） | 左→根→右 | BST 中序有序 | BST 验证、排序 |
| 后序 LRD | 栈（递归/显式） | 左→右→根 | 最后一个是根 | 删除树、表达式求值 |
| 层序 | 队列 | 逐层 | 按层访问 | 最短路径、分层操作 |

## 第八章 由遍历序列构造二叉树

### 8.1 核心问题

**给定遍历序列，能否唯一确定一棵二叉树？**

| 已知序列组合 | 能否唯一确定？ | 原因 |
|-------------|----------------|------|
| 先序 + 中序 | ✅ 能 | — |
| 后序 + 中序 | ✅ 能 | — |
| 层序 + 中序 | ✅ 能 | — |
| 先序 + 后序 | ❌ 不能 | 无法区分左右子树 |
| 只有一种序列 | ❌ 不能 | 信息不足 |

**关键理解**：中序序列是"分割器"——它能告诉我们哪些节点在左子树、哪些在右子树。而先序/后序/层序能告诉我们根是谁。两者结合才能唯一确定。

### 8.2 先序 + 中序 → 构造二叉树

**核心思路**：
1. 先序的**第一个元素**一定是根
2. 在中序中找到根，根**左边的元素**属于左子树，根**右边的元素**属于右子树
3. 递归处理左子树和右子树

**【例题】**

```
先序：A B D E G H C F
中序：D B G E H A C F
```

**第 1 步**：先序首元素 A 是根

```
中序中找到 A：
    D B G E H | A | C F
    ←左子树→       ←右子树→

左子树节点：{D, B, G, E, H}（5 个）
右子树节点：{C, F}（2 个）
```

**第 2 步**：递归处理左子树

```
左子树的先序（从先序中取前 5 个，跳过 A）：B D E G H
左子树的中序：D B G E H

先序首元素 B 是左子树的根

中序中找到 B：
    D | B | G E H
   左子树   右子树

B 的左子树：{D}
B 的右子树：{G, E, H}
```

**第 3 步**：继续递归

```
B 的左子树：先序 [D]，中序 [D] → 叶子节点 D

B 的右子树：先序 [E G H]，中序 [G E H]
  E 是根，G 在左，H 在右
  → E 的左孩子 G，右孩子 H
```

**第 4 步**：递归处理右子树

```
右子树的先序（最后 2 个）：C F
右子树的中序：C F

先序首元素 C 是根

中序中找到 C：
    (空) | C | F
         左    右

C 无左子树，右子树只有 F
```

**最终结果**：

```
           A
          / \
         B   C
        / \   \
       D   E   F
          / \
         G   H
```

### 8.3 C++ 实现：先序 + 中序构造二叉树

```cpp
#include <vector>
#include <unordered_map>
using namespace std;

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

class BuildTree {
private:
    unordered_map<int, int> inorderMap;  // 值 → 中序下标
    int preIndex;

    TreeNode* build(vector<int>& preorder, int inLeft, int inRight) {
        if (inLeft > inRight) return nullptr;

        int rootVal = preorder[preIndex++];
        TreeNode* root = new TreeNode(rootVal);

        int inMid = inorderMap[rootVal];  // 根在中序中的位置

        root->left  = build(preorder, inLeft, inMid - 1);
        root->right = build(preorder, inMid + 1, inRight);

        return root;
    }

public:
    TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
        preIndex = 0;
        for (int i = 0; i < inorder.size(); i++) {
            inorderMap[inorder[i]] = i;
        }
        return build(preorder, 0, inorder.size() - 1);
    }
};
```

**关键点**：
- 用哈希表存储中序序列中每个值的下标，这样查找根的位置只需 O(1)
- `preIndex` 全局递增，因为先序序列的顺序天然反映了"根→左→右"的访问顺序

### 8.4 后序 + 中序 → 构造二叉树

与先序类似，但有两个区别：
1. 后序的**最后一个元素**是根
2. 递归时**先构建右子树**，再构建左子树（因为后序从后往前看是"根→右→左"）

```
后序：D G H E B F C A
中序：D B G E H A C F

后序最后一个元素 A 是根
在中序中找到 A → 左子树 {D,B,G,E,H}，右子树 {C,F}
```

```cpp
class BuildTreePost {
private:
    unordered_map<int, int> inorderMap;
    int postIndex;

    TreeNode* build(vector<int>& postorder, int inLeft, int inRight) {
        if (inLeft > inRight) return nullptr;

        int rootVal = postorder[postIndex--];  // 从后往前取
        TreeNode* root = new TreeNode(rootVal);

        int inMid = inorderMap[rootVal];

        root->right = build(postorder, inMid + 1, inRight);  // 先右
        root->left  = build(postorder, inLeft, inMid - 1);   // 后左

        return root;
    }

public:
    TreeNode* buildTree(vector<int>& inorder, vector<int>& postorder) {
        postIndex = postorder.size() - 1;
        for (int i = 0; i < inorder.size(); i++) {
            inorderMap[inorder[i]] = i;
        }
        return build(postorder, 0, inorder.size() - 1);
    }
};
```

### 8.5 为什么先序 + 后序不能唯一确定？

**反例**：

```
先序：A B
后序：B A

可能是：         也可能是：
    A                A
   /                  \
  B                    B

两棵不同的树，先序和后序完全相同！
```

原因：当一个节点只有一个孩子时，先序和后序无法区分这个孩子是左孩子还是右孩子。只有中序遍历能通过根的位置来划分左右子树。

> **但是**，如果**二叉树的每个节点要么有两个孩子要么是叶子**（没有度为 1 的节点），则先序 + 后序可以唯一确定。

## Part 4 ｜ 树与森林

## 第九章 树的存储结构

前面几章都在讨论**二叉树**的存储和遍历。现在回到**一般的树**——每个节点可以有任意多个孩子。我们需要专门的存储方式。

### 9.1 双亲表示法

**思路**：每个节点记录自己的数据和**双亲**的下标。

```cpp
#define MAX_SIZE 100

struct PTNode {
    int data;
    int parent;  // 双亲在数组中的下标，根节点的 parent = -1
};

struct PTree {
    PTNode nodes[MAX_SIZE];
    int n;  // 节点总数
};
```

**存储示例**：

```
树：       A
         / | \
        B  C  D
       / \
      E   F

双亲表示法（数组）：
┌──────┬──────┬────────┐
│ 下标 │ data │ parent │
├──────┼──────┼────────┤
│  0   │  A   │  -1    │  ← 根节点
│  1   │  B   │   0    │  ← B 的双亲是 A (下标 0)
│  2   │  C   │   0    │
│  3   │  D   │   0    │
│  4   │  E   │   1    │  ← E 的双亲是 B (下标 1)
│  5   │  F   │   1    │
└──────┴──────┴────────┘
```

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 找双亲 | O(1) | 直接读 parent 字段 |
| 找孩子 | O(n) | 需要遍历整个数组 |
| 找根 | O(1) 或 O(n) | 约定根在 [0] 则 O(1) |

**优点**：找双亲极快
**缺点**：找孩子需要遍历

### 9.2 孩子表示法

**思路**：每个节点存储一个**孩子链表**，链表中依次记录所有孩子的下标。

```cpp
struct ChildNode {
    int childIndex;   // 孩子在数组中的下标
    ChildNode* next;  // 下一个孩子
};

struct CTNode {
    int data;
    ChildNode* firstChild;  // 第一个孩子的指针
};

struct CTree {
    CTNode nodes[MAX_SIZE];
    int n;
};
```

**存储示例**：

```
树：       A
         / | \
        B  C  D
       / \
      E   F

孩子表示法：
节点 │ data │ 孩子链表
─────┼──────┼─────────────
 [0] │  A   │ → 1 → 2 → 3 → null
 [1] │  B   │ → 4 → 5 → null
 [2] │  C   │ → null
 [3] │  D   │ → null
 [4] │  E   │ → null
 [5] │  F   │ → null
```

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 找孩子 | O(k) | 遍历该节点的孩子链表，$k$ 为孩子个数 |
| 找双亲 | O(n) | 需要遍历所有节点的孩子链表 |

### 9.3 孩子兄弟表示法（最重要！）

**思路**：每个节点只保存两个指针——**第一个孩子**和**下一个兄弟**。

```cpp
struct CSNode {
    int data;
    CSNode* firstChild;    // 第一个孩子
    CSNode* nextSibling;   // 下一个兄弟（右兄弟）
};
```

**存储示例**：

```
原始树：           孩子兄弟表示法（实质是二叉树！）：
       A                    A
     / | \                 /
    B  C  D               B
   / \                   / \
  E   F                 E   C
                         \   \
                          F   D
```

**转化过程详解**：

```
对于 A：第一个孩子 = B，没有兄弟 → A.firstChild = B, A.nextSibling = null
对于 B：第一个孩子 = E，下一个兄弟 = C → B.firstChild = E, B.nextSibling = C
对于 C：没有孩子，下一个兄弟 = D → C.firstChild = null, C.nextSibling = D
对于 D：没有孩子，没有兄弟 → D.firstChild = null, D.nextSibling = null
对于 E：没有孩子，下一个兄弟 = F → E.firstChild = null, E.nextSibling = F
对于 F：没有孩子，没有兄弟 → F.firstChild = null, F.nextSibling = null
```

> **关键认识**：孩子兄弟表示法本质上就是把一棵一般的树转换成了一棵**二叉树**！这就是"树与二叉树转换"的存储基础。

### 9.4 三种存储方式对比

| 存储方式 | 找双亲 | 找孩子 | 找兄弟 | 空间开销 | 适用场景 |
|----------|--------|--------|--------|----------|----------|
| 双亲表示法 | O(1) | O(n) | O(n) | 低 | 并查集 |
| 孩子表示法 | O(n) | O(k) | O(n) | 中 | 需频繁找孩子 |
| 孩子兄弟表示法 | O(n) | O(1) 找第一个孩子，遍历全部孩子为 O(k) | O(1) 找下一个兄弟 | 低 | 树↔二叉树转换 |

## 第十章 树、森林与二叉树的转换

### 10.1 树 → 二叉树

**口诀**：**"左孩子右兄弟"**

**规则**：
1. 在每个节点的所有兄弟之间加一条连线
2. 对每个节点，只保留它到**第一个孩子**的连线，删去到其他孩子的连线
3. 以根为轴心，将整棵树顺时针旋转 45°

**详细步骤图示**：

```
原始树：                第 1 步：兄弟连线        第 2 步：只留长子连线
       A                     A                      A
     / | \                 / | \                   /
    B  C  D               B--C--D                 B
   / \                   / \                     / \
  E   F                 E---F                   E   C
                                                 \   \
                                                  F   D

第 3 步：顺时针旋转 45° → 得到二叉树：
        A
       /
      B
     / \
    E   C
     \   \
      F   D
```

### 10.2 森林 → 二叉树

**规则**：
1. 先把森林中每棵树分别转换为二叉树
2. 把第一棵二叉树的根作为最终结果的根
3. 依次将后续二叉树的根作为前一棵的**右子树**

```
森林有三棵树：
 A        E        H
/ \       |       / \
B  C      F      I   J
   |      |
   D      G

第 1 步：分别转换为二叉树：
  A       E       H
 /       /       /
B       F       I
 \      /        \
  C    G          J
 /
D

第 2 步：依次连为右子树：
    A
   / \
  B   E
   \ / \
   C F   H
  / /   /
 D G   I
        \
         J
```

### 10.3 二叉树 → 树

这是"树 → 二叉树"的逆操作。

**规则**：
1. 如果节点 X 的**左孩子**为 Y，则 Y 是 X 的**第一个孩子**
2. 沿着 Y 的**右链**（Y 的右孩子、右孩子的右孩子...）一路走到底，这些节点全都是 X 的孩子
3. 删掉所有"右链"连线，改为"双亲→孩子"连线

```
二叉树：               还原的树：
    A                      A
   /                     / | \
  B                     B  C  D
 / \
E   C
 \   \
  F   D

过程：
A 的左孩子 B → B 是 A 的第一个孩子
B 的右链：B→C→D → C、D 也是 A 的孩子
B 的左孩子 E → E 是 B 的第一个孩子
E 的右链：E→F → F 也是 B 的孩子
```

### 10.4 二叉树 → 森林

**规则**：
1. 从根节点开始，沿着根的**右链**断开
2. 分别将得到的每棵二叉树转换为树

```
二叉树：               断开右链：            还原为森林：
    A                 A    E    H            A        E       H
   / \               /    /    /            / \       |      / \
  B   E             B    F    I            B   C     F      I   J
   \ / \             \  /      \              |      |
   C F   H           C G       J              D     G
  / /   /           /
 D G   I           D
        \
         J
```

### 10.5 转换规律总结

| 转换方向 | 关键 |
|----------|------|
| 树 → 二叉树 | 左孩子右兄弟 |
| 二叉树 → 树 | 左孩子 = 第一个孩子，沿右链 = 其余孩子 |
| 森林 → 二叉树 | 各树先独立转换，再将根连成右链 |
| 二叉树 → 森林 | 先断根的右链，再各自转为树 |

## 第十一章 树和森林的遍历

### 11.1 树的遍历

| 遍历方式 | 描述 | 对应二叉树遍历 |
|----------|------|----------------|
| **先根遍历** | 先访问根，再依次先根遍历各子树 | **先序遍历** |
| **后根遍历** | 先依次后根遍历各子树，再访问根 | **中序遍历** |

```
树：         A
           / | \
          B  C  D
         / \
        E   F

先根遍历：A B E F C D（对应二叉树的先序遍历）
后根遍历：E F B C D A（对应二叉树的中序遍历）
```

> **为什么树的后根遍历对应二叉树的中序遍历？**
>
> 把树用"左孩子右兄弟"转为二叉树后：
> - 树的"先访问根"= 二叉树的"先访问当前节点"
> - 树的"先遍历子树们"= 二叉树的"先遍历左子树"
> - 树的"后根 = 先遍历所有子树再访问根"，在二叉树中变成了"左→根→右"= 中序

### 11.2 森林的遍历

| 遍历方式 | 描述 | 对应二叉树遍历 |
|----------|------|----------------|
| **先序遍历** | 访问第一棵树的根 → 先序遍历第一棵树的子树构成的森林 → 先序遍历剩余树构成的森林 | **先序遍历** |
| **中序遍历** | 中序遍历第一棵树的子树构成的森林 → 访问第一棵树的根 → 中序遍历剩余树构成的森林 | **中序遍历** |

### 11.3 遍历对应关系总结

这张表**必须记住**：

| 树 | 森林 | 二叉树 |
|----|------|--------|
| 先根遍历 | 先序遍历 | **先序遍历** |
| 后根遍历 | 中序遍历 | **中序遍历** |

## Part 5 ｜ 哈夫曼树（Huffman Tree）

## 第十二章 哈夫曼树与哈夫曼编码

### 12.1 基本概念

#### （1）权（Weight）

**权** 是赋予树中**叶子节点**的一个数值，通常代表某种"重要程度"或"出现频率"。

#### （2）带权路径长度（WPL, Weighted Path Length）

**节点的带权路径长度**：从根到该节点的**路径长度**（边数）× 该节点的**权值**。

**树的带权路径长度（WPL）**：所有**叶子节点**的带权路径长度之和。

$$WPL = \sum_{i=1}^{n} w_i \times l_i$$

其中 $w_i$ 是第 $i$ 个叶子的权值，$l_i$ 是从根到它的路径长度。

**例子**：

```
树 1：                    树 2：                   树 3：
       ○                     ○                       ○
      / \                  / \                     / \
     ○   ○                1   ○                   7   ○
    / \  / \                  / \                     / \
   7  5  2  1                2   ○                   5   ○
                                 / \                     / \
                                5   7                   1   2

WPL1 = 7×2 + 5×2         WPL2 = 1×1 + 2×2    WPL3 = 7×1 + 5×2
     + 2×2 + 1×2               + 5×3 + 7×3         + 1×3 + 2×3
     = 30                      = 41                 = 26
```

**相同的叶子节点，不同的树形结构，WPL 可能差别巨大！**

#### （3）哈夫曼树

> **哈夫曼树（Huffman Tree）**，又叫**最优二叉树**，是指在含有 $n$ 个带权叶子节点的所有二叉树中，**WPL 最小**的那棵。

### 12.2 哈夫曼树的构造算法

**哈夫曼算法**（贪心策略）：

1. 将 $n$ 个权值看作 $n$ 棵只有根节点的树，构成**森林** $F$
2. 从 $F$ 中选出**两棵根节点权值最小**的树，合并为一棵新树（新树的根的权值 = 两棵子树根的权值之和）
3. 从 $F$ 中删掉这两棵树，把新树加入 $F$
4. 重复第 2-3 步，直到 $F$ 中只剩一棵树

**构造示例**：给定权值 {1, 2, 3, 5, 7}

```
第 1 步：选出最小的 1 和 2，合并
   3              3    5    7
  / \           (新树)
 1   2

第 2 步：选出最小的 3 和 3(新树)，合并
      6           5    7
     / \
    3   3
   / \
  1   2

第 3 步：选出最小的 5 和 6，合并
        11         7
       /  \
      5    6
          / \
         3   3
        / \
       1   2

第 4 步：选出最小的 7 和 11，合并
            18
           /  \
          7    11
              /  \
             5    6
                 / \
                3   3
               / \
              1   2

WPL = 7×1 + 5×2 + 3×3 + 1×4 + 2×4 = 7 + 10 + 9 + 4 + 8 = 38
```

**哈夫曼树的特点**：

| 特点 | 说明 |
|------|------|
| 每次合并产生一个新节点 | $n$ 个叶子的哈夫曼树共有 $2n - 1$ 个节点 |
| **没有度为 1 的节点** | 每次合并都是两棵树 → 只有度为 0 和度为 2 的节点 |
| 权值越大的叶子越靠近根 | 这就是 WPL 最小的原因 |
| 形状不唯一 | 但 WPL 唯一最小 |

### 12.3 哈夫曼编码

#### 为什么需要哈夫曼编码？

在通信或存储中，用**不等长编码**来表示字符：出现频率高的字符用短编码，频率低的用长编码，可以大幅减少总编码长度。

**前缀码**：任何一个字符的编码都不是另一个字符编码的**前缀**，这样接收方可以无歧义地解码。

> **哈夫曼编码是一种最优的前缀码。**

#### 编码方法

1. 以字符频率为权值构建哈夫曼树
2. 从根到每个叶子的路径：**向左走编 0，向右走编 1**

**例子**：字符 A(7)、B(5)、C(3)、D(1)、E(2)

```
构造的哈夫曼树：
            18
           /  \
          7    11
          A   /  \
             5    6
             B   / \
                3   3
                C  / \
                  1   2
                  D   E

编码：
A：0            （路径长度 1）
B：10           （路径长度 2）
C：110          （路径长度 3）
D：1110         （路径长度 4）
E：1111         （路径长度 4）
```

**WPL = 7×1 + 5×2 + 3×3 + 1×4 + 2×4 = 38 bit**

如果用等长编码（3 位），总长 = (7+5+3+1+2)×3 = 54 bit，哈夫曼编码节省了约 30%！

### 12.4 C++ 实现

```cpp
#include <iostream>
#include <queue>
#include <vector>
#include <string>
#include <unordered_map>
using namespace std;

struct HuffNode {
    char ch;         // 字符（仅叶子节点有意义）
    int freq;        // 频率/权值
    HuffNode* left;
    HuffNode* right;

    HuffNode(char c, int f)
        : ch(c), freq(f), left(nullptr), right(nullptr) {}
};

// 优先队列比较器：频率小的优先
struct Compare {
    bool operator()(HuffNode* a, HuffNode* b) {
        return a->freq > b->freq;
    }
};

// 构建哈夫曼树
HuffNode* buildHuffmanTree(unordered_map<char, int>& freqMap) {
    priority_queue<HuffNode*, vector<HuffNode*>, Compare> pq;

    for (auto& [ch, freq] : freqMap) {
        pq.push(new HuffNode(ch, freq));
    }

    while (pq.size() > 1) {
        HuffNode* left = pq.top();  pq.pop();
        HuffNode* right = pq.top(); pq.pop();

        HuffNode* parent = new HuffNode('\0', left->freq + right->freq);
        parent->left = left;
        parent->right = right;

        pq.push(parent);
    }

    return pq.top();
}

// 生成编码表
void generateCodes(HuffNode* root, string code,
                   unordered_map<char, string>& codeMap) {
    if (root == nullptr) return;

    if (!root->left && !root->right) {
        codeMap[root->ch] = code;
        return;
    }

    generateCodes(root->left, code + "0", codeMap);
    generateCodes(root->right, code + "1", codeMap);
}

// 编码
string encode(const string& text, unordered_map<char, string>& codeMap) {
    string encoded;
    for (char ch : text) {
        encoded += codeMap[ch];
    }
    return encoded;
}

// 解码
string decode(const string& encoded, HuffNode* root) {
    string decoded;
    HuffNode* curr = root;
    for (char bit : encoded) {
        if (bit == '0') curr = curr->left;
        else            curr = curr->right;

        if (!curr->left && !curr->right) {
            decoded += curr->ch;
            curr = root;
        }
    }
    return decoded;
}

// 释放内存
void freeTree(HuffNode* root) {
    if (root == nullptr) return;
    freeTree(root->left);
    freeTree(root->right);
    delete root;
}

int main() {
    unordered_map<char, int> freqMap = {
        {'A', 7}, {'B', 5}, {'C', 3}, {'D', 1}, {'E', 2}
    };

    HuffNode* root = buildHuffmanTree(freqMap);

    unordered_map<char, string> codeMap;
    generateCodes(root, "", codeMap);

    cout << "=== 哈夫曼编码表 ===" << endl;
    for (auto& [ch, code] : codeMap) {
        cout << ch << " : " << code << endl;
    }

    string text = "ABCDE";
    string encoded = encode(text, codeMap);
    cout << "\n原文: " << text << endl;
    cout << "编码: " << encoded << endl;

    string decoded = decode(encoded, root);
    cout << "解码: " << decoded << endl;

    freeTree(root);
    return 0;
}
```

## Part 6 ｜ 二叉排序树（BST）

## 第十三章 二叉排序树

### 13.1 定义

**二叉排序树（Binary Search Tree, BST）**，又叫**二叉查找树**，或者为空树，或者满足以下性质：

1. 若左子树非空，则左子树上所有节点的值 **< 根节点的值**
2. 若右子树非空，则右子树上所有节点的值 **> 根节点的值**
3. 左右子树也分别是二叉排序树

```
        30
       /  \
     15    50
    / \   / \
   10 20 40  60
  /       \
 5        45

中序遍历：5 10 15 20 30 40 45 50 60（升序！）
```

> **核心性质**：BST 的中序遍历结果是一个**递增有序序列**。这也是验证一棵树是否为 BST 的最简单方法。

### 13.2 查找操作

**算法思想**：从根开始，利用 BST 的有序性，逐步缩小搜索范围——像二分查找一样。

```
查找 40：
        30        30 < 40，去右子树
       /  \
     15   [50]    50 > 40，去左子树
    / \   / \
   10 20[40] 60   40 == 40，找到了！
```

**递归实现**：

```cpp
TreeNode* search(TreeNode* root, int key) {
    if (root == nullptr || root->data == key) {
        return root;
    }
    if (key < root->data) {
        return search(root->left, key);
    } else {
        return search(root->right, key);
    }
}
```

**非递归实现**（效率更高，推荐）：

```cpp
TreeNode* searchIterative(TreeNode* root, int key) {
    while (root != nullptr && root->data != key) {
        if (key < root->data) {
            root = root->left;
        } else {
            root = root->right;
        }
    }
    return root;
}
```

**时间复杂度**：
- 最好情况（平衡BST）：O(log n)
- 最坏情况（退化成链）：O(n)

### 13.3 插入操作

**算法思想**：
1. 如果树为空，新节点就是根
2. 如果插入值 < 当前节点，递归插入到左子树
3. 如果插入值 > 当前节点，递归插入到右子树
4. BST 一般不允许重复值（重复值的处理取决于具体要求）

**BST 的插入永远在叶子位置——新节点一定是叶子！**

```
插入 25 到下面的 BST：

        30                  30
       /  \                /  \
     15    50   →        15    50
    / \   / \           / \   / \
   10 20 40  60        10 20 40  60
                           \
                           25

查找路径：30→15→20→右子树为空→插入
```

**代码实现**：

```cpp
TreeNode* insert(TreeNode* root, int val) {
    if (root == nullptr) {
        return new TreeNode(val);
    }
    if (val < root->data) {
        root->left = insert(root->left, val);
    } else if (val > root->data) {
        root->right = insert(root->right, val);
    }
    // val == root->data 时不插入（不允许重复）
    return root;
}
```

**非递归实现**：

```cpp
TreeNode* insertIterative(TreeNode* root, int val) {
    TreeNode* newNode = new TreeNode(val);
    if (root == nullptr) return newNode;

    TreeNode* curr = root;
    TreeNode* parent = nullptr;

    while (curr != nullptr) {
        parent = curr;
        if (val < curr->data) {
            curr = curr->left;
        } else if (val > curr->data) {
            curr = curr->right;
        } else {
            delete newNode;  // 重复值，不插入
            return root;
        }
    }

    if (val < parent->data) {
        parent->left = newNode;
    } else {
        parent->right = newNode;
    }

    return root;
}
```

### 13.4 构建 BST

给定一个序列，依次插入即可构建 BST。**不同的插入顺序会得到不同形状的 BST！**

```
序列 {30, 15, 50, 10, 20, 40, 60}（中间值先插入）：
        30                  ← 平衡的 BST
       /  \
     15    50
    / \   / \
   10 20 40  60

序列 {10, 15, 20, 30, 40, 50, 60}（有序插入）：
   10
    \
     15               ← 退化成右斜链！
      \                  查找效率退化为 O(n)
       20
        \
         30
          \
           40
            \
             50
              \
               60
```

> 这就是为什么需要 AVL、红黑树等**自平衡** BST——防止退化。

### 13.5 删除操作（重点难点！）

BST 的删除是三个操作中**最复杂的**，需要分三种情况讨论。

设要删除的节点为 `p`：

#### 情况一：`p` 是叶子节点（度为 0）

直接删除即可，不影响 BST 性质。

```
删除 20：
        30                  30
       /  \                /  \
     15    50   →        15    50
    / \   / \           /    / \
   10 20 40  60        10   40  60
```

#### 情况二：`p` 只有一个孩子（度为 1）

用 `p` 的孩子替代 `p` 的位置。

```
删除 15（只有左孩子 10）：
        30                  30
       /  \                /  \
     15    50   →        10    50
    /     / \                / \
   10    40  60             40  60
```

#### 情况三：`p` 有两个孩子（度为 2）

这是最复杂的情况。不能简单地删除，需要找到一个替代者：

**方法一**：用 `p` 的**中序前驱**（左子树中的最大节点）替代 `p`
**方法二**：用 `p` 的**中序后继**（右子树中的最小节点）替代 `p`

```
删除 30（有两个孩子）：

方法：用中序前驱 20（左子树最大值）替代

        30                  20
       /  \                /  \
     15    50   →        15    50
    / \   / \           /    / \
   10 20 40  60        10   40  60

或者用中序后继 40（右子树最小值）替代：

        30                  40
       /  \                /  \
     15    50   →        15    50
    / \   / \           / \     \
   10 20 40  60        10 20    60
```

**为什么这样做是正确的？**

中序前驱是左子树中最大的值，它一定比左子树其他所有节点大，也一定比右子树所有节点小——放在被删节点位置后，BST 性质仍然成立。而且中序前驱是左子树最右边的节点，它**最多只有一个左孩子**，所以删除它就回到了情况一或情况二。

**代码实现**：

```cpp
TreeNode* findMin(TreeNode* node) {
    while (node->left != nullptr) {
        node = node->left;
    }
    return node;
}

TreeNode* deleteNode(TreeNode* root, int key) {
    if (root == nullptr) return nullptr;

    if (key < root->data) {
        root->left = deleteNode(root->left, key);
    } else if (key > root->data) {
        root->right = deleteNode(root->right, key);
    } else {
        // 找到了要删除的节点

        // 情况一：叶子节点
        if (root->left == nullptr && root->right == nullptr) {
            delete root;
            return nullptr;
        }

        // 情况二：只有一个孩子
        if (root->left == nullptr) {
            TreeNode* temp = root->right;
            delete root;
            return temp;
        }
        if (root->right == nullptr) {
            TreeNode* temp = root->left;
            delete root;
            return temp;
        }

        // 情况三：有两个孩子
        // 找右子树中最小的节点（中序后继）
        TreeNode* successor = findMin(root->right);
        root->data = successor->data;  // 用后继的值替换
        root->right = deleteNode(root->right, successor->data);  // 删除后继
    }
    return root;
}
```

**删除操作的完整图示**：

```
删除度为 2 的节点 50：

原始 BST：
           30
          /  \
        15    50          ← 要删除
       / \   / \
      10 20 40  60
              \   \
              45  70

第 1 步：找 50 的中序后继 = 右子树最小值 = 60
第 2 步：用 60 的值替换 50
第 3 步：在右子树中删除原来的 60（它最多有一个右孩子）

           30
          /  \
        15    60          ← 用 60 替换了 50
       / \   / \
      10 20 40  70        ← 原来 60 的位置由 70 顶替
              \
              45

BST 性质仍然成立 ✓
```

### 13.6 BST 的性能分析

| 操作 | 平均时间 | 最坏时间 | 说明 |
|------|----------|----------|------|
| 查找 | O(log n) | O(n) | 退化成链时最坏 |
| 插入 | O(log n) | O(n) | 先查找再插入 |
| 删除 | O(log n) | O(n) | 先查找再删除 |

**平均查找长度（ASL）**：

对于随机构建的 BST，平均查找长度约为 $O(\log n)$。但如果输入序列是有序的，BST 退化为链表，ASL = $O(n)$。

**查找成功的 ASL 计算**：

```
        30              查找每个节点的比较次数：
       /  \             30: 1 次（根节点）
     15    50           15: 2 次
    / \   / \           50: 2 次
   10 20 40  60         10: 3 次
                        20: 3 次
                        40: 3 次
                        60: 3 次

ASL(成功) = (1+2+2+3+3+3+3) / 7 = 17/7 ≈ 2.43
```

**查找失败的 ASL**：

查找失败时会走到空指针。需要计算走到每个"失败位置"的比较次数。

```
        30              失败位置（空指针）和到达的比较次数：
       /  \
     15    50           <10:  3次 (30→15→10→空)
    / \   / \           10~15: 3次
   10 20 40  60         15~20: 3次
                        20~30: 3次
                        30~40: 3次
                        40~50: 3次
                        50~60: 3次
                        >60:  3次

ASL(失败) = (3+3+3+3+3+3+3+3) / 8 = 24/8 = 3
```

### 13.7 完整的 BST 实现

```cpp
#include <iostream>
#include <queue>
using namespace std;

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

class BST {
private:
    TreeNode* root;

    TreeNode* insertHelper(TreeNode* node, int val) {
        if (node == nullptr) return new TreeNode(val);
        if (val < node->data)
            node->left = insertHelper(node->left, val);
        else if (val > node->data)
            node->right = insertHelper(node->right, val);
        return node;
    }

    TreeNode* findMin(TreeNode* node) {
        while (node->left) node = node->left;
        return node;
    }

    TreeNode* deleteHelper(TreeNode* node, int key) {
        if (node == nullptr) return nullptr;

        if (key < node->data) {
            node->left = deleteHelper(node->left, key);
        } else if (key > node->data) {
            node->right = deleteHelper(node->right, key);
        } else {
            if (!node->left && !node->right) {
                delete node;
                return nullptr;
            }
            if (!node->left) {
                TreeNode* temp = node->right;
                delete node;
                return temp;
            }
            if (!node->right) {
                TreeNode* temp = node->left;
                delete node;
                return temp;
            }
            TreeNode* successor = findMin(node->right);
            node->data = successor->data;
            node->right = deleteHelper(node->right, successor->data);
        }
        return node;
    }

    void inorderHelper(TreeNode* node) {
        if (node == nullptr) return;
        inorderHelper(node->left);
        cout << node->data << " ";
        inorderHelper(node->right);
    }

    void preorderHelper(TreeNode* node) {
        if (node == nullptr) return;
        cout << node->data << " ";
        preorderHelper(node->left);
        preorderHelper(node->right);
    }

    int heightHelper(TreeNode* node) {
        if (node == nullptr) return 0;
        int leftH = heightHelper(node->left);
        int rightH = heightHelper(node->right);
        return 1 + max(leftH, rightH);
    }

    int countNodesHelper(TreeNode* node) {
        if (node == nullptr) return 0;
        return 1 + countNodesHelper(node->left) + countNodesHelper(node->right);
    }

    int countLeavesHelper(TreeNode* node) {
        if (node == nullptr) return 0;
        if (!node->left && !node->right) return 1;
        return countLeavesHelper(node->left) + countLeavesHelper(node->right);
    }

    void destroyHelper(TreeNode* node) {
        if (node == nullptr) return;
        destroyHelper(node->left);
        destroyHelper(node->right);
        delete node;
    }

public:
    BST() : root(nullptr) {}

    ~BST() { destroyHelper(root); }

    void insert(int val) {
        root = insertHelper(root, val);
    }

    bool search(int key) {
        TreeNode* curr = root;
        while (curr) {
            if (key == curr->data) return true;
            if (key < curr->data) curr = curr->left;
            else curr = curr->right;
        }
        return false;
    }

    void remove(int key) {
        root = deleteHelper(root, key);
    }

    void inorder() {
        inorderHelper(root);
        cout << endl;
    }

    void preorder() {
        preorderHelper(root);
        cout << endl;
    }

    void levelOrder() {
        if (root == nullptr) return;
        queue<TreeNode*> q;
        q.push(root);
        while (!q.empty()) {
            TreeNode* node = q.front();
            q.pop();
            cout << node->data << " ";
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        cout << endl;
    }

    int height() { return heightHelper(root); }
    int countNodes() { return countNodesHelper(root); }
    int countLeaves() { return countLeavesHelper(root); }
};

int main() {
    BST tree;

    // 插入
    int values[] = {30, 15, 50, 10, 20, 40, 60};
    for (int v : values) {
        tree.insert(v);
    }

    cout << "=== BST 基本操作演示 ===" << endl;

    cout << "中序遍历（应该有序）: ";
    tree.inorder();

    cout << "先序遍历: ";
    tree.preorder();

    cout << "层序遍历: ";
    tree.levelOrder();

    cout << "树高: " << tree.height() << endl;
    cout << "节点数: " << tree.countNodes() << endl;
    cout << "叶子数: " << tree.countLeaves() << endl;

    // 查找
    cout << "\n查找 20: " << (tree.search(20) ? "找到" : "未找到") << endl;
    cout << "查找 25: " << (tree.search(25) ? "找到" : "未找到") << endl;

    // 删除叶子
    cout << "\n删除叶子 10:" << endl;
    tree.remove(10);
    cout << "中序: "; tree.inorder();

    // 删除度为 1 的节点
    cout << "删除度为1的节点 15:" << endl;
    tree.remove(15);
    cout << "中序: "; tree.inorder();

    // 删除度为 2 的节点
    cout << "删除度为2的节点 50:" << endl;
    tree.remove(50);
    cout << "中序: "; tree.inorder();

    return 0;
}
```

## 第十四章 二叉树的常见操作合集

这一章汇总了二叉树中最常见的辅助操作，是遍历知识的直接应用。

### 14.1 求树的高度

```cpp
int height(TreeNode* root) {
    if (root == nullptr) return 0;
    return 1 + max(height(root->left), height(root->right));
}
```

**思路**：空树高度为 0；非空树高度 = 1 + 左右子树高度的最大值。这是典型的**后序遍历**思想。

### 14.2 统计节点个数

```cpp
int countNodes(TreeNode* root) {
    if (root == nullptr) return 0;
    return 1 + countNodes(root->left) + countNodes(root->right);
}
```

### 14.3 统计叶子节点个数

```cpp
int countLeaves(TreeNode* root) {
    if (root == nullptr) return 0;
    if (root->left == nullptr && root->right == nullptr) return 1;
    return countLeaves(root->left) + countLeaves(root->right);
}
```

### 14.4 求第 k 层的节点个数

```cpp
int countAtLevel(TreeNode* root, int k) {
    if (root == nullptr) return 0;
    if (k == 1) return 1;
    return countAtLevel(root->left, k - 1) + countAtLevel(root->right, k - 1);
}
```

### 14.5 判断两棵树是否相同

```cpp
bool isSameTree(TreeNode* p, TreeNode* q) {
    if (p == nullptr && q == nullptr) return true;
    if (p == nullptr || q == nullptr) return false;
    return p->data == q->data
        && isSameTree(p->left, q->left)
        && isSameTree(p->right, q->right);
}
```

### 14.6 判断是否为镜像（对称）二叉树

```cpp
bool isMirror(TreeNode* left, TreeNode* right) {
    if (left == nullptr && right == nullptr) return true;
    if (left == nullptr || right == nullptr) return false;
    return left->data == right->data
        && isMirror(left->left, right->right)
        && isMirror(left->right, right->left);
}

bool isSymmetric(TreeNode* root) {
    if (root == nullptr) return true;
    return isMirror(root->left, root->right);
}
```

### 14.7 翻转二叉树（镜像翻转）

```cpp
TreeNode* invertTree(TreeNode* root) {
    if (root == nullptr) return nullptr;
    TreeNode* temp = root->left;
    root->left = invertTree(root->right);
    root->right = invertTree(temp);
    return root;
}
```

### 14.8 判断是否为 BST

利用中序遍历的有序性：

```cpp
bool isValidBST(TreeNode* root, long long minVal = LLONG_MIN,
                long long maxVal = LLONG_MAX) {
    if (root == nullptr) return true;
    if (root->data <= minVal || root->data >= maxVal) return false;
    return isValidBST(root->left, minVal, root->data)
        && isValidBST(root->right, root->data, maxVal);
}
```
