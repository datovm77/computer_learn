## 第一章 栈的基本概念

### 1.1 什么是栈？

**栈（Stack）** 是一种 **操作受限的线性表**——它只允许在 **一端** 进行插入和删除操作。

生活类比：

| 场景 | 类比 |
|------|------|
| 一摞盘子 | 只能从最上面取/放盘子 |
| 手枪弹夹 | 最后压入的子弹最先射出 |
| 浏览器"后退" | 最近访问的页面最先返回 |
| Ctrl+Z 撤销 | 最近的操作最先被撤销 |

共同规律：**最后进去的，最先出来。**

### 1.2 LIFO 原则

栈遵循 **LIFO（Last In, First Out）——后进先出** 原则。

```
        ┌─────────┐
        │    4    │ ← 栈顶（Top）：最后进来，最先出去
        ├─────────┤
        │    3    │
        ├─────────┤
        │    2    │
        ├─────────┤
        │    1    │ ← 栈底（Bottom）：最先进来，最后出去
        └─────────┘
```

**核心术语**：

- **栈顶（Top）**：允许操作的那一端
- **栈底（Bottom）**：不允许操作的那一端
- **入栈 / 压栈（Push）**：将元素放到栈顶
- **出栈 / 弹栈（Pop）**：将栈顶元素取走

> **关键理解**：栈本质上还是线性表（数组或链表），只是人为限制了操作的位置——只能在一端操作。这种"限制"反而带来了简单、高效的特性。

### 1.3 栈的数学性质——卡特兰数

**经典考题**：n 个不同元素按 1, 2, 3, ..., n 的顺序依次入栈，合法的出栈序列有多少种？

答案是 **卡特兰数（Catalan Number）**：`C(n) = (2n)! / ((n+1)! × n!)`

这里默认所有元素按固定顺序入栈，出栈时只允许从栈顶弹出。`n=0` 时也有 1 种合法序列：空序列。

| n | 合法出栈序列数 |
|---|----------------|
| 0 | 1 |
| 1 | 1 |
| 2 | 2 |
| 3 | 5 |
| 4 | 14 |

**举例**：3 个元素（1, 2, 3）入栈，5 种合法出栈序列：

- `1 2 3`：入1→出1，入2→出2，入3→出3
- `1 3 2`：入1→出1，入2→入3→出3→出2
- `2 1 3`：入1→入2→出2→出1，入3→出3
- `2 3 1`：入1→入2→出2，入3→出3→出1
- `3 2 1`：入1→入2→入3→出3→出2→出1

> **为什么 `3 1 2` 不合法？** 要先出3就必须先把1和2压入栈，此时栈中是 `[1, 2]`（2在栈顶），出3后下一个只能出2，不可能跳过2先出1。

#### 如何判断一个出栈序列是否合法？

**模拟法**：用一个栈去模拟入栈/出栈过程。

```
判断 [3, 1, 2] 是否是 1,2,3 入栈的合法出栈序列：

期望出栈: 3
  入栈1 → 栈[1]，栈顶≠3
  入栈2 → 栈[1,2]，栈顶≠3
  入栈3 → 栈[1,2,3]，栈顶==3，出栈 → 栈[1,2] ✓

期望出栈: 1
  栈顶是2 ≠ 1，需要继续入栈
  但所有元素都已入栈！→ 无法得到1 → ❌ 不合法！
```

这个模拟方法在编程题中非常实用，代码如下：

```cpp
#include <iostream>
#include <stack>
#include <vector>
using namespace std;

// 判断 outSeq 是否是 1,2,...,n 依次入栈的合法出栈序列
bool isValidPopSequence(const vector<int> &outSeq, int n) {
    if ((int)outSeq.size() != n) return false;

    stack<int> st;
    int pushNext = 1;  // 下一个要入栈的元素

    for (int i = 0; i < n; i++) {
        // 一直入栈，直到栈顶等于期望的出栈元素
        while (pushNext <= n && (st.empty() || st.top() != outSeq[i])) {
            st.push(pushNext++);
        }
        // 如果栈已空，或栈顶仍然不等于期望值，说明不合法
        if (st.empty() || st.top() != outSeq[i]) return false;
        st.pop();  // 匹配成功，出栈
    }
    return true;
}

int main() {
    vector<int> seq1 = {3, 2, 1};
    vector<int> seq2 = {3, 1, 2};
    cout << isValidPopSequence(seq1, 3) << endl;  // 1 (合法)
    cout << isValidPopSequence(seq2, 3) << endl;  // 0 (不合法)
    return 0;
}
```

### 1.4 栈的抽象数据类型（ADT）

不论用什么方式实现，栈都必须支持以下操作：

| 操作          | 描述                     | 时间复杂度 |
| ------------- | ------------------------ | ---------- |
| `InitStack()` | 初始化空栈               | O(1)       |
| `Push(x)`     | 元素 x 压入栈顶          | O(1)       |
| `Pop()`       | 弹出栈顶元素             | O(1)       |
| `GetTop()`    | 返回栈顶元素（不弹出）   | O(1)       |
| `IsEmpty()`   | 判断栈是否为空           | O(1)       |
| `IsFull()`    | 判断是否已满（仅顺序栈） | O(1)       |

固定容量顺序栈和链栈的核心操作通常都是 **O(1)**，这是栈的一大优势。动态数组实现的 `Push` 在扩容那一次可能是 O(n)，但均摊仍是 O(1)。

## 第二章 栈的顺序存储实现（顺序栈）

### 2.1 基本思路

用一个 **数组** 存放栈中的元素，再用一个变量 `top` 来指示栈顶的位置。

```
  索引:    [0]  [1]  [2]  [3]  [4]  ...  [MaxSize-1]
  内容:     10   20   30    _    _   ...     _
                       ↑
                     top=2（栈顶元素在索引2）
```

### 2.2 `top` 指针的两种约定

`top` 的初始值有 **两种常见约定**，直接影响代码写法：

#### 约定一：`top = -1`（指向栈顶元素本身）

```
空栈:      top = -1
入栈:      S.data[++S.top] = x;      先移指针，再放元素
出栈:      x = S.data[S.top--];      先取元素，再移指针
栈满:      top == MaxSize - 1
栈空:      top == -1
元素个数:  top + 1
```

#### 约定二：`top = 0`（指向栈顶元素的下一个空位）

```
空栈:      top = 0
入栈:      S.data[S.top++] = x;      先放元素，再移指针
出栈:      x = S.data[--S.top];      先移指针，再取元素
栈满:      top == MaxSize
栈空:      top == 0
元素个数:  top
```

> **⚠️ 考试务必注意**：两种写法的 `++/--` 位置恰好相反。做题前一定先看清题目用哪种约定！**两种都必须掌握。**

本教程以 **约定一（`top = -1`）** 为主。

### 2.3 完整实现

```cpp
#include <iostream>
using namespace std;

const int MaxSize = 100;

// ========== 顺序栈的定义 ==========
struct SqStack {
    int data[MaxSize];  // 存储栈元素的数组
    int top;            // 栈顶指针（指向栈顶元素的下标）
};

// ========== 初始化 ==========
void InitStack(SqStack &S) {
    S.top = -1;  // 空栈：top 为 -1，表示没有任何元素
}

// ========== 判空 ==========
bool IsEmpty(SqStack S) {
    return S.top == -1;
}

// ========== 判满 ==========
bool IsFull(SqStack S) {
    return S.top == MaxSize - 1;
}

// ========== 入栈 Push ==========
// 步骤：(1) 检查栈满  (2) top+1  (3) 放入元素
bool Push(SqStack &S, int x) {
    if (IsFull(S)) {
        cout << "栈满，无法入栈！" << endl;
        return false;
    }
    S.data[++S.top] = x;  // 先移指针，再存数据
    return true;
}

// ========== 出栈 Pop ==========
// 步骤：(1) 检查栈空  (2) 取出元素  (3) top-1
bool Pop(SqStack &S, int &x) {
    if (IsEmpty(S)) {
        cout << "栈空，无法出栈！" << endl;
        return false;
    }
    x = S.data[S.top--];  // 先取数据，再移指针
    return true;
}

// ========== 获取栈顶元素（不弹出） ==========
bool GetTop(SqStack S, int &x) {
    if (IsEmpty(S)) return false;
    x = S.data[S.top];  // 只读取，不修改 top
    return true;
}

// ========== 测试 ==========
int main() {
    SqStack S;
    InitStack(S);

    Push(S, 10);
    Push(S, 20);
    Push(S, 30);

    int x;
    GetTop(S, x);
    cout << "栈顶元素: " << x << endl;  // 30

    Pop(S, x); cout << "出栈: " << x << endl;  // 30
    Pop(S, x); cout << "出栈: " << x << endl;  // 20
    Pop(S, x); cout << "出栈: " << x << endl;  // 10
    Pop(S, x);  // 栈空，无法出栈！
    return 0;
}
```

### 2.4 入栈出栈过程图解

追踪 `Push(10), Push(20), Push(30), Pop, Pop, Pop` 的完整过程：

```
初始:         top=-1    栈: []              空栈
Push(10):     top=0     栈: [10]            data[0]=10
Push(20):     top=1     栈: [10,20]         data[1]=20
Push(30):     top=2     栈: [10,20,30]      data[2]=30
Pop()→30:     top=1     栈: [10,20]         取出data[2]
Pop()→20:     top=0     栈: [10]            取出data[1]
Pop()→10:     top=-1    栈: []              取出data[0]
```

> **注意**：出栈只是把 `top` 减 1，`data[2]` 里的 30 其实还在数组里，只是逻辑上已不在栈中。下次 Push 时会自然覆盖它。

### 2.5 共享栈（双端栈）

**场景**：需要两个栈，且总元素不超过 MaxSize，可以让它们共享同一个数组，从两端向中间生长。

```
      栈0 从左往右增长 →              ← 栈1 从右往左增长
    ┌────┬────┬────┬────┬────┬────┬────┬────┐
    │ a  │ b  │ c  │    │    │    │ y  │ z  │
    └────┴────┴────┴────┴────┴────┴────┴────┘
      0    1    2    3    4    5    6    7
               ↑                   ↑
             top0=2              top1=6
```

- **栈0**：从左开始向右增长，`top0` 初始为 `-1`
- **栈1**：从右开始向左增长，`top1` 初始为 `MaxSize`
- **栈满条件**：`top0 + 1 == top1`（两个栈顶紧邻）

```cpp
struct SharedStack {
    int data[MaxSize];
    int top0;  // 栈0 的栈顶
    int top1;  // 栈1 的栈顶
};

void InitSharedStack(SharedStack &S) {
    S.top0 = -1;
    S.top1 = MaxSize;
}

bool Push(SharedStack &S, int stackId, int x) {
    if (S.top0 + 1 == S.top1) return false;  // 栈满
    if (stackId == 0)
        S.data[++S.top0] = x;  // 栈0：top0 右移
    else
        S.data[--S.top1] = x;  // 栈1：top1 左移
    return true;
}

bool Pop(SharedStack &S, int stackId, int &x) {
    if (stackId == 0) {
        if (S.top0 == -1) return false;
        x = S.data[S.top0--];
    } else {
        if (S.top1 == MaxSize) return false;
        x = S.data[S.top1++];
    }
    return true;
}
```

> **优势**：空间利用率高——一个栈用得多时可以"借"另一个栈的空间。

### 2.6 动态顺序栈（补充）

静态数组版顺序栈容量固定，可能溢出或浪费空间。实际工程中常用动态数组保存栈元素：容量不足时申请更大的连续空间，把旧元素复制过去，再释放旧空间。

- 单次扩容时需要复制已有元素，最坏时间复杂度为 O(n)
- 如果按固定倍数增长容量，连续多次 Push 的均摊时间复杂度仍为 O(1)
- C++ 中 `std::stack` 是容器适配器，默认底层容器通常是 `std::deque`；如果直接用 `std::vector` 实现栈，也会采用动态扩容思想

## 第三章 栈的链式存储实现（链栈）

### 3.1 为什么需要链栈？

顺序栈的缺点：静态数组大小固定，可能溢出或浪费。
**链栈**用链表实现，每次入栈动态分配节点，通常不需要预设容量；但它仍然可能因为内存耗尽而无法继续入栈。

### 3.2 设计：链表的哪一端做栈顶？

| 方案                | Push             | Pop            |
| ------------------- | ---------------- | -------------- |
| **头部做栈顶（✅）** | O(1) 头插        | O(1) 头删      |
| 尾部做栈顶（❌）     | O(1)（需尾指针） | O(n)（找前驱） |

**结论**：一定用链表 **头部做栈顶**——入栈就是头插，出栈就是头删，都是 O(1)。

### 3.3 不带头节点的链栈——完整实现

```cpp
#include <iostream>
using namespace std;

struct LinkNode {
    int data;
    LinkNode *next;
};

// ========== 初始化 ==========
void InitStack(LinkNode *&top) {
    top = nullptr;  // 空栈
}

// ========== 判空 ==========
bool IsEmpty(LinkNode *top) {
    return top == nullptr;
}

// ========== 入栈 Push ==========
// 相当于链表的"头插法"
void Push(LinkNode *&top, int x) {
    LinkNode *newNode = new LinkNode;
    newNode->data = x;
    newNode->next = top;  // 新节点指向原栈顶
    top = newNode;        // top 指向新节点
}

// ========== 出栈 Pop ==========
// 相当于链表的"头删法"
bool Pop(LinkNode *&top, int &x) {
    if (IsEmpty(top)) {
        cout << "栈空，无法出栈！" << endl;
        return false;
    }
    x = top->data;
    LinkNode *temp = top;
    top = top->next;  // top 移到下一个节点
    delete temp;      // 释放旧栈顶节点
    return true;
}

// ========== 获取栈顶 ==========
bool GetTop(LinkNode *top, int &x) {
    if (IsEmpty(top)) return false;
    x = top->data;
    return true;
}

// ========== 销毁 ==========
void DestroyStack(LinkNode *&top) {
    int temp;
    while (!IsEmpty(top)) Pop(top, temp);
}

// ========== 测试 ==========
int main() {
    LinkNode *top;
    InitStack(top);

    Push(top, 10);
    Push(top, 20);
    Push(top, 30);

    int x;
    GetTop(top, x);
    cout << "栈顶: " << x << endl;  // 30

    Pop(top, x); cout << "出栈: " << x << endl;  // 30
    Pop(top, x); cout << "出栈: " << x << endl;  // 20
    Pop(top, x); cout << "出栈: " << x << endl;  // 10

    DestroyStack(top);
    return 0;
}
```

### 3.4 入栈出栈过程图解

```
Push(10):  top → [10|nullptr]
Push(20):  top → [20|·] → [10|nullptr]
Push(30):  top → [30|·] → [20|·] → [10|nullptr]

Pop()→30:  top → [20|·] → [10|nullptr]       释放30节点
Pop()→20:  top → [10|nullptr]                  释放20节点
Pop()→10:  top → nullptr                       释放10节点，空栈
```

### 3.5 带头节点的链栈

```cpp
// 初始化：创建头节点
void InitStack(LinkNode *&head) {
    head = new LinkNode;
    head->next = nullptr;
}

bool IsEmpty(LinkNode *head) {
    return head->next == nullptr;
}

// 入栈：在头节点后面插入（头插法）
void Push(LinkNode *head, int x) {
    LinkNode *newNode = new LinkNode;
    newNode->data = x;
    newNode->next = head->next;
    head->next = newNode;
}

// 出栈：删除头节点后面的第一个节点
bool Pop(LinkNode *head, int &x) {
    if (IsEmpty(head)) return false;
    LinkNode *temp = head->next;
    x = temp->data;
    head->next = temp->next;
    delete temp;
    return true;
}
```

> 带头节点的好处：Push 和 Pop 不需要处理空栈特殊情况，代码更统一。

> **注意**：如果链栈使用了头节点，销毁栈时除了释放所有数据节点，也要 `delete head` 释放头节点本身。

### 3.6 顺序栈 vs 链栈 对比

| 维度     | 顺序栈               | 链栈                       |
| -------- | -------------------- | -------------------------- |
| 存储方式 | 连续数组             | 离散节点+指针              |
| 栈满     | 有固定容量限制       | 通常无固定容量限制，但可能内存耗尽 |
| 空间     | 可能浪费或溢出       | 不预留连续空位，但每节点多一个指针 |
| 时间     | O(1)，无额外开销     | O(1)，但有 new/delete 开销 |
| 缓存     | ✅ 连续内存，缓存友好 | ❌ 节点分散，缓存不友好     |
| 适用     | 元素数量可预估       | 元素数量变化大             |

> **工程实践**：大多数场景用顺序栈 / `std::stack`，缓存友好，性能更好。

### 3.7 附：C++ STL 中的栈

实际编程中直接使用 `<stack>`：

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> st;
    st.push(10);
    st.push(20);
    st.push(30);

    cout << "栈顶: " << st.top() << endl;   // 30
    cout << "大小: " << st.size() << endl;   // 3

    while (!st.empty()) {
        cout << st.top() << " ";
        st.pop();   // 注意：STL 的 pop() 没有返回值！
    }
    // 输出: 30 20 10
    return 0;
}
```

| 方法      | 功能         | 注意           |
| --------- | ------------ | -------------- |
| `push(x)` | 入栈         |                |
| `pop()`   | 出栈         | **不返回值；空栈调用是未定义行为** |
| `top()`   | 返回栈顶引用 | 空栈调用是未定义行为 |
| `empty()` | 是否为空     |                |
| `size()`  | 元素个数     |                |

## 第四章 栈的应用——括号匹配

### 4.1 问题描述

给定包含 `()[]{}` 的字符串，判断括号是否正确匹配。

```
✅ 合法:   ()    ()[]{}    {[()]}    a*(b+c)
❌ 不合法:  (]    ([)]      {(}       (((
```

### 4.2 为什么用栈？

括号匹配天然符合"后进先出"：**最内层（最晚出现）的左括号必须最先匹配。**

### 4.3 算法

```
从左到右逐字符扫描：
  遇到左括号 ( [ {  → 压入栈
  遇到右括号 ) ] }  →
    栈空？          → ❌ 右括号多余
    弹出栈顶，配对？ → 不配对则 ❌ 类型不匹配
扫描结束后：栈空 → ✅；栈不空 → ❌ 左括号多余
```

**三种失败情况**：

| 类型 | 示例 | 何时发现 |
|------|------|---------|
| 右括号多余 | `())` | 遇到右括号时栈已空 |
| 类型不匹配 | `(]` | 栈顶左括号与右括号不配对 |
| 左括号多余 | `(()` | 扫描结束栈不为空 |

### 4.4 手动模拟

以 `{[()]}` 为例：

```
字符   操作              栈状态      说明
──────────────────────────────────────────────
 {    左括号，入栈        {
 [    左括号，入栈        { [
 (    左括号，入栈        { [ (
 )    右括号，栈顶(匹配   { [        弹出(
 ]    右括号，栈顶[匹配   {          弹出[
 }    右括号，栈顶{匹配   空         弹出{
──────────────────────────────────────────────
扫描结束，栈空 → ✅ 匹配成功
```

以 `([)]` 为例：

```
 (    入栈                (
 [    入栈                ( [
 )    栈顶是[，与)不匹配 → ❌ 失败
```

### 4.5 完整实现

```cpp
#include <iostream>
#include <stack>
#include <string>
using namespace std;

bool isMatch(char left, char right) {
    return (left == '(' && right == ')')
        || (left == '[' && right == ']')
        || (left == '{' && right == '}');
}

bool bracketMatch(const string &expr) {
    stack<char> st;

    for (char ch : expr) {
        if (ch == '(' || ch == '[' || ch == '{') {
            st.push(ch);
        }
        else if (ch == ')' || ch == ']' || ch == '}') {
            if (st.empty()) return false;          // 右括号多余
            char topCh = st.top(); st.pop();
            if (!isMatch(topCh, ch)) return false; // 类型不匹配
        }
        // 非括号字符直接跳过
    }
    return st.empty();  // 左括号多余则栈不空
}

int main() {
    cout << bracketMatch("{[()]}")  << endl;  // 1
    cout << bracketMatch("([)]")   << endl;  // 0
    cout << bracketMatch("((")     << endl;  // 0
    cout << bracketMatch("})")     << endl;  // 0
    cout << bracketMatch("")       << endl;  // 1（空串合法）
    cout << bracketMatch("a*(b+c)-[d]") << endl;  // 1
    return 0;
}
```

### 4.6 考试向：不用 STL 的手写版

为突出括号匹配逻辑，下面代码默认输入长度不超过 `MaxSize`；真实工程代码应在 `Push` 中检查栈满。

```cpp
#include <iostream>
#include <cstring>
using namespace std;

const int MaxSize = 100;
struct SqStack {
    char data[MaxSize];
    int top;
};
void InitStack(SqStack &S) { S.top = -1; }
bool IsEmpty(SqStack S) { return S.top == -1; }
void Push(SqStack &S, char x) { S.data[++S.top] = x; }
char Pop(SqStack &S) { return S.data[S.top--]; }

bool bracketMatch(const char *expr) {
    SqStack S; InitStack(S);
    for (int i = 0; expr[i]; i++) {
        char ch = expr[i];
        if (ch == '(' || ch == '[' || ch == '{') {
            Push(S, ch);
        } else if (ch == ')' || ch == ']' || ch == '}') {
            if (IsEmpty(S)) return false;
            char top = Pop(S);
            if ((top=='(' && ch!=')') || (top=='[' && ch!=']') || (top=='{' && ch!='}'))
                return false;
        }
    }
    return IsEmpty(S);
}
```

## 第五章 栈的应用——表达式求值

这是栈最经典、最重要的应用。本章讲清楚三种表达式及其转换与求值。

本章代码默认输入表达式合法；基础版本只覆盖 `+ - * /`、整数或单字符操作数，不处理一元负号、非法括号和除零等异常情况。

> **补充**：如果表达式以空格分隔 token，或需要支持多位变量名、浮点数、幂运算 `^`、一元负号，就不能再逐字符简单扫描，需要先做词法切分（tokenize），再按 token 处理。

### 5.1 三种表达式

#### 中缀表达式（Infix）—— 人类日常写法

运算符在两个操作数**中间**：`a + b`，`3 + 4 * 2`

- 优点：符合人类阅读习惯
- 缺点：需要括号和优先级规则，计算机处理麻烦

#### 后缀表达式（Postfix / 逆波兰 RPN）—— 计算机偏爱

运算符在两个操作数**后面**：`a b +`，`3 4 2 * +`

- 优点：不需要括号、不需要优先级，从左到右扫一遍就能求值

#### 前缀表达式（Prefix / 波兰 PN）

运算符在两个操作数**前面**：`+ a b`，`+ 3 * 4 2`

#### 对照表

| 中缀            | 后缀            | 前缀            |
| --------------- | --------------- | --------------- |
| `a + b`         | `a b +`         | `+ a b`         |
| `a + b * c`     | `a b c * +`     | `+ a * b c`     |
| `(a + b) * c`   | `a b + c *`     | `* + a b c`     |
| `a * (b + c)`   | `a b c + *`     | `* a + b c`     |
| `a + b * c - d` | `a b c * + d -` | `- + a * b c d` |

> **记忆要点**：不管哪种表达式，**操作数的相对顺序始终不变**，只是运算符位置不同。

### 5.2 运算符优先级

| 优先级 | 运算符     |
| ------ | ---------- |
| 最高   | `( )` 括号 |
| 高     | `*` `/`    |
| 低     | `+` `-`    |

同优先级**左结合**（从左到右），如 `3-2-1 = (3-2)-1 = 0`。

### 5.3 中缀转后缀——手算方法（加括号法）

**步骤**：
1. 按运算优先级给每步运算加括号
2. 把每个运算符移到对应**右括号的后面**
3. 去掉所有括号

**示例1**：`a + b * c`

```
原式:       a + b * c
加括号:     (a + (b * c))
移运算符:   (a   (b   c) *) +     ← 每个运算符移到它的右括号后面
去括号:     a b c * +
```

**示例2**：`(a + b) * c - d`

```
原式:       (a + b) * c - d
加括号:     (((a + b) * c) - d)
移运算符:   (((a   b)+   c)*   d)-
去括号:     a b + c * d -
```

**示例3**：`a * b + c * d`

```
原式:       a * b + c * d
加括号:     ((a * b) + (c * d))
移运算符:   ((a   b)*   (c   d)*)+
去括号:     a b * c d * +
```

### 5.4 中缀转前缀——手算方法

与转后缀对称，把运算符移到对应**左括号的前面**：

```
a + b * c
加括号:    (a + (b * c))
移运算符:  +(a   *(b   c))      ← 每个运算符移到它的左括号前面
去括号:    + a * b c
```

### 5.5 中缀转后缀——算法实现

#### 算法流程

```
准备：一个运算符栈 opStack，一个输出序列 postfix

从左到右扫描中缀表达式的每个 token：

  ① 操作数（数字/变量）→ 直接加入 postfix

  ② 左括号 (  → 压入 opStack

  ③ 右括号 )  → 不断弹出 opStack 栈顶加入 postfix，
                  直到遇到 (，弹出 ( 但不输出

  ④ 运算符 op → 当栈不空 且 栈顶不是 ( 且 栈顶优先级 ≥ op：
                    弹出栈顶加入 postfix
                  然后将 op 压入 opStack

扫描结束 → 将 opStack 中所有运算符弹出加入 postfix
```

**为什么弹出"优先级 ≥ 当前"的？** 因为后缀表达式从左到右求值，优先级更高或相同的运算应该先执行，所以应先输出。

#### 代码实现

```cpp
#include <cctype>
#include <iostream>
#include <stack>
#include <string>
using namespace std;

int precedence(char op) {
    if (op == '+' || op == '-') return 1;
    if (op == '*' || op == '/') return 2;
    return 0;
}

bool isOperator(char ch) {
    return ch == '+' || ch == '-' || ch == '*' || ch == '/';
}

string infixToPostfix(const string &infix) {
    string postfix;
    stack<char> opStack;

    for (char ch : infix) {
        if (isalnum(ch)) {
            // ① 操作数：直接输出
            postfix += ch;
        }
        else if (ch == '(') {
            // ② 左括号：入栈
            opStack.push(ch);
        }
        else if (ch == ')') {
            // ③ 右括号：弹出到左括号
            while (!opStack.empty() && opStack.top() != '(') {
                postfix += opStack.top();
                opStack.pop();
            }
            opStack.pop();  // 弹出 '(' 不输出
        }
        else if (isOperator(ch)) {
            // ④ 运算符：弹出优先级 ≥ 自己的
            while (!opStack.empty() && opStack.top() != '('
                   && precedence(opStack.top()) >= precedence(ch)) {
                postfix += opStack.top();
                opStack.pop();
            }
            opStack.push(ch);
        }
    }

    // 弹出所有剩余运算符
    while (!opStack.empty()) {
        postfix += opStack.top();
        opStack.pop();
    }
    return postfix;
}

int main() {
    cout << infixToPostfix("a+b*c")       << endl;  // abc*+
    cout << infixToPostfix("(a+b)*c")     << endl;  // ab+c*
    cout << infixToPostfix("a*(b+c)")     << endl;  // abc+*
    cout << infixToPostfix("a+b*c-d/e")   << endl;  // abc*+de/-
    cout << infixToPostfix("(a+b)*(c-d)") << endl;  // ab+cd-*
    return 0;
}
```

#### 手动模拟：`a+b*c-d`

```
字符  操作                         输出       运算符栈
──────────────────────────────────────────────────────
 a   操作数，输出                   a
 +   栈空，入栈                    a          +
 b   操作数，输出                   ab         +
 *   栈顶+优先级<*，入栈           ab         + *
 c   操作数，输出                   abc        + *
 -   栈顶*优先级≥-，弹*输出        abc*       +
     栈顶+优先级≥-，弹+输出        abc*+
     栈空，-入栈                    abc*+      -
 d   操作数，输出                   abc*+d     -
──────────────────────────────────────────────────────
结束  弹出-                         abc*+d-    空
```

结果：`abc*+d-` ✅

#### 手动模拟：`(a+b)*c`

```
字符  操作                         输出       运算符栈
──────────────────────────────────────────────────────
 (   左括号入栈                                (
 a   操作数，输出                   a          (
 +   栈顶是(，停止比较，入栈       a          ( +
 b   操作数，输出                   ab         ( +
 )   弹出到(为止，+输出            ab+        空
 *   栈空，入栈                    ab+        *
 c   操作数，输出                   ab+c       *
──────────────────────────────────────────────────────
结束  弹出*                         ab+c*      空
```

结果：`ab+c*` ✅

### 5.6 后缀表达式求值

后缀表达式求值非常简单，**只需一个操作数栈**，不需要优先级和括号。

#### 算法

```
从左到右扫描后缀表达式：
  操作数 → 压入栈
  运算符 → 弹出两个数：先弹的是 right，后弹的是 left
           计算 left op right，结果压入栈
扫描结束，栈中唯一元素即为结果
```

> **⚠️ 关键易错点**：先弹出的是**右操作数**，后弹出的是**左操作数**。减法和除法中顺序搞反会出错！

#### 代码实现

```cpp
#include <cctype>
#include <iostream>
#include <stack>
#include <string>
using namespace std;

int calc(int left, int right, char op) {
    switch (op) {
        case '+': return left + right;
        case '-': return left - right;
        case '*': return left * right;
        case '/': return left / right;
        default:  return 0;
    }
}

int evalPostfix(const string &postfix) {
    stack<int> st;
    for (char ch : postfix) {
        if (isdigit(ch)) {
            st.push(ch - '0');  // 字符转数字
        } else {
            int right = st.top(); st.pop();  // 先弹 = 右操作数
            int left  = st.top(); st.pop();  // 后弹 = 左操作数
            st.push(calc(left, right, ch));
        }
    }
    return st.top();
}

int main() {
    // 3 + 4*2 = 11
    cout << evalPostfix("342*+") << endl;     // 11
    // (3+4)*2 = 14
    cout << evalPostfix("34+2*") << endl;     // 14
    // 5 + 3*2 - 8/4 = 9
    cout << evalPostfix("532*+84/-") << endl; // 9
    return 0;
}
```

#### 手动模拟：`342*+`（中缀 `3+4*2`）

```
字符  操作                  栈
──────────────────────────────
 3   入栈                   3
 4   入栈                   3, 4
 2   入栈                   3, 4, 2
 *   弹2,4 → 4*2=8         3, 8
 +   弹8,3 → 3+8=11        11
──────────────────────────────
结果：11 ✅
```

#### 手动模拟：`532*+84/-`（中缀 `5+3*2-8/4`）

```
字符  操作                  栈
──────────────────────────────
 5   入栈                   5
 3   入栈                   5, 3
 2   入栈                   5, 3, 2
 *   弹2,3 → 3*2=6         5, 6
 +   弹6,5 → 5+6=11        11
 8   入栈                   11, 8
 4   入栈                   11, 8, 4
 /   弹4,8 → 8/4=2         11, 2
 -   弹2,11 → 11-2=9       9
──────────────────────────────
结果：9 ✅ （5+6-2=9）
```

### 5.7 前缀表达式求值

与后缀对称，但**从右到左**扫描，弹出顺序也相反。

```
从右到左扫描前缀表达式：
  操作数 → 压入栈
  运算符 → 弹出两个数：先弹的是 left，后弹的是 right
           计算 left op right，结果压入栈
```

```cpp
int evalPrefix(const string &prefix) {
    stack<int> st;
    for (int i = prefix.size() - 1; i >= 0; i--) {
        char ch = prefix[i];
        if (isdigit(ch)) {
            st.push(ch - '0');
        } else {
            int left  = st.top(); st.pop();  // 先弹 = 左操作数（与后缀相反！）
            int right = st.top(); st.pop();
            st.push(calc(left, right, ch));
        }
    }
    return st.top();
}
```

这段代码复用前面后缀求值中的 `calc` 函数，并同样默认前缀表达式只包含单位数字和二元运算符。

### 5.8 用两个栈直接计算中缀表达式

可以把"中缀转后缀"和"后缀求值"合并为一个过程，用 **操作数栈 + 运算符栈** 直接求值：

```cpp
#include <cctype>
#include <iostream>
#include <stack>
#include <string>
using namespace std;

int precedence(char op) {
    if (op == '+' || op == '-') return 1;
    if (op == '*' || op == '/') return 2;
    return 0;
}

void applyOp(stack<int> &nums, char op) {
    int right = nums.top(); nums.pop();
    int left  = nums.top(); nums.pop();
    switch (op) {
        case '+': nums.push(left + right); break;
        case '-': nums.push(left - right); break;
        case '*': nums.push(left * right); break;
        case '/': nums.push(left / right); break;
    }
}

int evalInfix(const string &expr) {
    stack<int> nums;    // 操作数栈
    stack<char> ops;    // 运算符栈

    for (int i = 0; i < (int)expr.size(); i++) {
        char ch = expr[i];

        if (isdigit(ch)) {
            // 处理多位数
            int num = 0;
            while (i < (int)expr.size() && isdigit(expr[i])) {
                num = num * 10 + (expr[i] - '0');
                i++;
            }
            i--;  // 回退一步（for循环会再+1）
            nums.push(num);
        }
        else if (ch == '(') {
            ops.push(ch);
        }
        else if (ch == ')') {
            while (ops.top() != '(') {
                applyOp(nums, ops.top());
                ops.pop();
            }
            ops.pop();  // 弹出 '('
        }
        else if (ch == '+' || ch == '-' || ch == '*' || ch == '/') {
            while (!ops.empty() && ops.top() != '('
                   && precedence(ops.top()) >= precedence(ch)) {
                applyOp(nums, ops.top());
                ops.pop();
            }
            ops.push(ch);
        }
    }

    while (!ops.empty()) {
        applyOp(nums, ops.top());
        ops.pop();
    }

    return nums.top();
}

int main() {
    cout << evalInfix("3+4*2")     << endl;  // 11
    cout << evalInfix("(3+4)*2")   << endl;  // 14
    cout << evalInfix("10+20*3")   << endl;  // 70
    cout << evalInfix("100-(50+30)") << endl; // 20
    return 0;
}
```

| 知识点             | 要点                                     |
| ------------------ | ---------------------------------------- |
| 中缀转后缀（手算） | 加括号 → 运算符移到**右**括号后 → 去括号 |
| 中缀转前缀（手算） | 加括号 → 运算符移到**左**括号前 → 去括号 |
| 中缀转后缀（算法） | 运算符栈，遇到优先级 ≥ 的弹出输出        |

| 前缀求值           | 右→左扫描，操作数栈，先弹=**左**操作数   |
| 中缀直接求值       | 操作数栈 + 运算符栈，合并转换与求值      |

## 第六章 栈在递归中的应用

### 6.1 什么是递归？

**递归（Recursion）**：函数直接或间接地调用自身。

```cpp
// 最经典的递归：计算阶乘 n!
int factorial(int n) {
    if (n <= 1) return 1;        // 递归终止条件（基准情形）
    return n * factorial(n - 1); // 递归调用
}
```

递归的两个必要条件：

1. **递归终止条件（Base Case）**：什么时候停下来，不再递归
2. **递归递推关系（Recursive Case）**：如何把大问题分解为更小的子问题

> 没有终止条件的递归 = 无限递归 → 栈溢出（Stack Overflow）！

### 6.2 递归的本质——调用栈

递归之所以能工作，是因为**程序运行时会使用线程调用栈（Call Stack）**。每次函数调用时，运行时会在调用栈上分配一块空间（称为**栈帧 / 活动记录**），用来保存：

- 函数的参数
- 局部变量
- 返回地址（调用结束后回到哪里继续执行）

当函数返回时，对应的栈帧被弹出（销毁），控制权回到调用者。

#### 以 `factorial(4)` 为例

调用过程（入栈）：

```
调用 factorial(4)
  → 调用 factorial(3)
    → 调用 factorial(2)
      → 调用 factorial(1)
        → 达到终止条件，返回 1

此时调用栈的状态（从下到上）：
┌──────────────────────────────┐
│ factorial(1): n=1, return 1  │ ← 栈顶（最后调用的）
├──────────────────────────────┤
│ factorial(2): n=2, 等待结果  │
├──────────────────────────────┤
│ factorial(3): n=3, 等待结果  │
├──────────────────────────────┤
│ factorial(4): n=4, 等待结果  │ ← 栈底（最先调用的）
└──────────────────────────────┘
```

返回过程（出栈）：

```
factorial(1) 返回 1       → 弹出栈帧，回到 factorial(2)
factorial(2) 返回 2*1=2   → 弹出栈帧，回到 factorial(3)
factorial(3) 返回 3*2=6   → 弹出栈帧，回到 factorial(4)
factorial(4) 返回 4*6=24  → 弹出栈帧，回到主调函数

最终结果：24
```

> **关键理解**：递归的"先调用后返回"就是"后进先出"——这正是栈的 LIFO 特性！所以递归本质上就是利用了调用栈。

### 6.3 递归的经典示例

#### 示例1：斐波那契数列

```
F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)
```

```cpp
// 朴素递归（效率低，有大量重复计算）
int fib(int n) {
    if (n <= 0) return 0;
    if (n == 1) return 1;
    return fib(n - 1) + fib(n - 2);
}
```

以 `fib(5)` 为例，递归展开形成一棵树：

```
                    fib(5)
                   /      \
              fib(4)      fib(3)
             /     \      /    \
         fib(3)  fib(2) fib(2) fib(1)
         /   \    / \    / \
     fib(2) fib(1) ...  ...
      / \
  fib(1) fib(0)
```

可以看到 `fib(3)` 被计算了 2 次，`fib(2)` 被计算了 3 次——大量重复！

- 时间复杂度：O(2ⁿ)——指数级，非常慢
- 空间复杂度：O(n)——递归深度为 n

> **优化方向**：用循环（迭代）替代递归，或者用"记忆化搜索"消除重复计算。这就是动态规划的思想，后续会详细学习。

#### 示例2：汉诺塔（Tower of Hanoi）

经典递归问题：将 n 个盘子从 A 柱移到 C 柱，借助 B 柱，规则是大盘子不能放在小盘子上面。

```cpp
#include <iostream>
using namespace std;

// 将 n 个盘子从 from 移到 to，借助 aux
void hanoi(int n, char from, char to, char aux) {
    if (n <= 0) return;  // 没有盘子时不需要移动
    if (n == 1) {
        // 只剩 1 个盘子，直接移
        cout << from << " -> " << to << endl;
        return;
    }
    // 第一步：把上面 n-1 个从 from 移到 aux（借助 to）
    hanoi(n - 1, from, aux, to);

    // 第二步：把最底下那个从 from 移到 to
    cout << from << " -> " << to << endl;

    // 第三步：把 n-1 个从 aux 移到 to（借助 from）
    hanoi(n - 1, aux, to, from);
}

int main() {
    hanoi(3, 'A', 'C', 'B');
    return 0;
}
```

输出（3 个盘子需要 7 步）：

```
A -> C
A -> B
C -> B
A -> C
B -> A
B -> C
A -> C
```

n 个盘子（n ≥ 1）需要 **2ⁿ - 1** 步移动，递归调用栈最深 n 层。

### 6.4 递归转迭代——用显式栈模拟递归

**核心思想**：既然递归本质上是利用调用栈，那我们可以**自己维护一个栈**来模拟递归过程，把递归转为迭代。

#### 为什么要这么做？

1. **避免栈溢出**：线程调用栈大小有限（通常 1~8 MB），深度递归会溢出。自己维护的显式栈通常可承载更深规模，但仍受可用内存限制
2. **性能优化**：函数调用有额外开销（保存寄存器、参数传递等），迭代通常更快
3. **考试要求**：很多题目要求写出非递归版本

#### 示例：用栈模拟阶乘

```cpp
#include <iostream>
#include <stack>
using namespace std;

int factorialIterative(int n) {
    stack<int> st;

    // 模拟"递归调用"过程：把所有需要计算的值压栈
    // 相当于递归展开 factorial(n) → factorial(n-1) → ... → factorial(1)
    for (int i = n; i >= 1; i--) {
        st.push(i);
    }

    // 模拟"递归返回"过程：从最小的开始，逐步计算
    int result = 1;
    while (!st.empty()) {
        result *= st.top();
        st.pop();
    }

    return result;
}

int main() {
    cout << factorialIterative(5) << endl;  // 120
    return 0;
}
```

当然，阶乘太简单了，直接循环就行，不需要显式栈。下面看更有意义的例子。

#### 示例：用栈模拟二叉树的递归遍历（预告）

这是数据结构后续章节"树"的重要内容。递归版中序遍历：

```cpp
void inorder(TreeNode *root) {
    if (!root) return;
    inorder(root->left);
    visit(root);
    inorder(root->right);
}
```

非递归版（用显式栈）：

```cpp
void inorderIterative(TreeNode *root) {
    stack<TreeNode*> st;
    TreeNode *curr = root;

    while (curr || !st.empty()) {
        // 一路向左走到底，沿途节点入栈
        while (curr) {
            st.push(curr);
            curr = curr->left;
        }
        // 弹出栈顶节点并访问
        curr = st.top(); st.pop();
        visit(curr);
        // 转向右子树
        curr = curr->right;
    }
}
```

> 这个例子完美展示了"用显式栈代替调用栈"的思想。学完树的章节后回来看会更加清晰。

### 6.5 递归 vs 迭代 对比

| 维度       | 递归                               | 迭代（循环/显式栈）        |
| ---------- | ---------------------------------- | -------------------------- |
| 代码可读性 | ✅ 通常更简洁直观                   | ❌ 可能较冗长               |
| 性能       | ❌ 函数调用开销                     | ✅ 开销更小                 |
| 栈空间     | ❌ 受调用栈大小限制                 | ✅ 显式栈通常更可控         |
| 栈溢出风险 | ❌ 深度大时容易溢出                 | ✅ 可控                     |
| 适用场景   | 问题天然具有递归结构时（如树遍历） | 追求性能或需要避免栈溢出时 |

> **实用原则**：能用循环简单实现的就用循环；问题结构天然递归（如树、图的DFS）时，先写递归版，需要优化时再转迭代。

### 6.6 尾递归优化（补充）

如果递归调用是函数的**最后一条语句**（即"尾递归"），编译器有可能将其优化为循环，消除栈帧的累积。

```cpp
// 普通递归（不是尾递归）：return n * factorial(n-1)
// 乘法是最后执行的操作，不是递归调用
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 先递归，再乘 → 不是尾递归
}

// 改写为尾递归：用累加器参数传递中间结果
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 递归调用就是最后操作 → 尾递归
}
```

> **注意**：C++ 标准不保证尾递归优化，但多数编译器（如 GCC、Clang）在开启优化（`-O2`）时会执行此优化。考试中了解概念即可。

### 7.2 必须牢记的核心结论

#### 顺序栈两种约定对比

|          | top 初始 = -1    | top 初始 = 0       |
| -------- | ---------------- | ------------------ |
| top 含义 | 指向栈顶元素     | 指向栈顶的下一空位 |
| 入栈     | `data[++top]=x`  | `data[top++]=x`    |
| 出栈     | `x=data[top--]`  | `x=data[--top]`    |
| 栈空     | `top==-1`        | `top==0`           |
| 栈满     | `top==MaxSize-1` | `top==MaxSize`     |
| 元素数   | `top+1`          | `top`              |

#### 表达式核心速记

| 操作           | 关键规则                          |
| -------------- | --------------------------------- |
| 手算中缀→后缀  | 运算符移到对应**右括号后面**      |
| 手算中缀→前缀  | 运算符移到对应**左括号前面**      |
| 后缀求值       | 从**左→右**，先弹=**右**操作数    |
| 前缀求值       | 从**右→左**，先弹=**左**操作数    |
| 中缀转后缀算法 | 运算符栈，弹出优先级 **≥** 当前的 |

#### 链栈要点

| 要点         | 结论                             |
| ------------ | -------------------------------- |
| 哪端做栈顶？ | 链表**头部**（头插头删 O(1)）    |
| 需要判满吗？ | 通常不需要固定容量判满，但要考虑内存耗尽 |
| 何时用链栈？ | 元素数量无法预估、变化大时       |

### 7.3 经典易错点

**❶ top 的两种约定别搞混**

```cpp
// 约定一 (top=-1): 先++再赋值
S.data[++S.top] = x;  // ✅
S.data[S.top++] = x;  // ❌ 这是约定二的写法！
```

**❷ 后缀求值弹出顺序**

```cpp
// 后缀求值中遇到运算符：
int right = st.top(); st.pop();  // 先弹的是右操作数！
int left  = st.top(); st.pop();
result = left - right;  // 不是 right - left！
```

如果搞反了，`8 3 -` 会算成 `3-8=-5` 而不是正确的 `8-3=5`。

**❸ 中缀转后缀时 `(` 的处理**

左括号 `(` 入栈后，**不参与优先级比较**。遇到运算符时，如果栈顶是 `(`，停止弹出，直接入栈。`(` 只有遇到 `)` 时才被弹出。

**❹ 共享栈栈满条件**

```cpp
// 是 top0+1 == top1，不是 top0 == top1
if (S.top0 + 1 == S.top1)  // 栈满 ✅
```

**❺ STL stack 的 pop() 无返回值**

```cpp
// 错误写法：
int x = st.pop();  // ❌ 编译错误！pop() 返回 void

// 正确写法：
int x = st.top();  // 先取值
st.pop();           // 再弹出
```

### 7.4 经典练习题

#### 题目1：出栈序列判断

> 元素 1,2,3,4,5 依次入栈，以下哪个不是合法出栈序列？
> A. 3 2 1 5 4
> B. 3 1 2 5 4
> C. 1 5 4 3 2
> D. 4 5 3 2 1

**解题方法**：用模拟法，逐个验证。

以 B 选项 `3 1 2 5 4` 为例：

```
期望出3：入1入2入3，栈[1,2,3]，出3 → 栈[1,2] ✓
期望出1：栈顶是2≠1，但所有≤3的已入栈，无法再入栈得到1
         必须先出2才能出1 → ❌ 不合法
```

**答案：B**

#### 题目2：手算中缀转后缀

> 将 `A + B * (C - D) - E / F` 转换为后缀表达式。

```
加括号:     ((A + (B * (C - D))) - (E / F))
移运算符:   ((A   (B   (C   D)-  )*  )+   (E   F)/  )-
去括号:     A B C D - * + E F / -
```

**答案：`A B C D - * + E F / -`**

验证：用算法模拟

```
A → 输出A                         输出:A        栈:
+ → 入栈                          输出:A        栈:+
B → 输出B                         输出:AB       栈:+
* → 栈顶+优先级<*，入栈           输出:AB       栈:+ *
( → 入栈                          输出:AB       栈:+ * (
C → 输出C                         输出:ABC      栈:+ * (
- → 栈顶是(，入栈                 输出:ABC      栈:+ * ( -
D → 输出D                         输出:ABCD     栈:+ * ( -
) → 弹出到(：弹-                  输出:ABCD-    栈:+ *
- → 栈顶*优先级≥-，弹*            输出:ABCD-*   栈:+
     栈顶+优先级≥-，弹+           输出:ABCD-*+  栈:
     入栈-                         输出:ABCD-*+  栈:-
E → 输出E                         输出:ABCD-*+E 栈:-
/ → 栈顶-优先级</，入栈          输出:ABCD-*+E 栈:- /
F → 输出F                         输出:ABCD-*+EF 栈:- /
结束→弹出/和-                      输出:ABCD-*+EF/-
```

结果一致 ✅

#### 题目3：后缀表达式求值

> 计算后缀表达式 `6 2 3 + - 3 8 2 / + * 2 ^ 3 +` 的值。

```
6           栈: 6
```

### 7.5 学习建议

1. **先理解概念，再写代码**：栈很简单，但它的应用（表达式求值）需要反复练习
2. **两种 top 约定都要会**：考试可能用任意一种出题
3. **手算中缀转后缀一定要熟练**：考试高频考点，加括号法最稳
4. **后缀求值的弹出顺序务必记牢**：先弹=右操作数
5. **自己手动模拟几遍**：拿纸笔画栈的变化过程，比看十遍代码都有效
6. **STL `stack` 要会用**：实际做题和工程中不会手写栈
