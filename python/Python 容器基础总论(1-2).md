## 1.1 为什么要学习"容器"？

在编程世界中，你几乎无法回避一个基本问题：**如何组织和管理数据？**

假设你要记录一个班级 50 个学生的成绩。你当然可以定义 50 个变量：

```python
score_1 = 90
score_2 = 85
score_3 = 78
# ... 还要继续为每个学生单独定义变量
```

这种做法不仅笨拙，而且完全无法扩展——当学生数量变为 500 甚至 5000 时，你不可能手动管理这么多变量。

**容器（Container）** 就是解决这一问题的核心工具。它是一种**能够存储多个数据元素的对象**，让你可以用一个名字来管理一组数据：

```python
scores = [90, 85, 78, 92, 88, ...]  # 一个列表，存储所有成绩
```

在 Python 中，容器不仅仅是"装数据的盒子"。Python 为不同的使用场景设计了**多种容器类型**，它们在内部结构、存储方式、操作特点上各有不同。理解这些容器的分类和特性，是 Python 编程的核心基础之一。

---

## 1.2 什么是容器？——从日常生活到编程概念

### 1.2.1 日常生活中的容器隐喻

在日常生活中，"容器"无处不在：

|    日常容器    |            特点            |    编程类比     |
| :------------: | :------------------------: | :-------------: |
|  **购物清单**  |   有顺序、可重复、可修改   | `list`（列表）  |
| **身份证号码** |      有顺序、固定不变      | `tuple`（元组） |
|   **电话簿**   | 按名字查电话号码（键值对） | `dict`（字典）  |
|   **抽奖箱**   |       不重复、无顺序       |  `set`（集合）  |

这些隐喻帮助我们建立直觉：不同的容器适合不同的使用场景。

### 1.2.2 编程中的容器定义

在编程中，**容器（Container）** 的定义更加精确：

> **容器是一种数据结构，用于存储、组织和管理一组元素，并提供对这些元素的访问和操作接口。**

Python 中的容器具有以下共同特征：

1. **可以包含多个元素**：容器内可以存放零个、一个或多个数据项。
2. **支持成员测试**：可以使用 `in` 关键字判断某个元素是否在容器中。
3. **可以遍历**：可以使用 `for` 循环逐个访问容器中的元素。
4. **有大小**：可以使用 `len()` 函数获取容器中元素的数量。

```python
# 一个简单的容器示例
fruits = ["苹果", "香蕉", "橘子"]

# 1. 包含多个元素
print(len(fruits))          # 输出: 3

# 2. 成员测试
print("苹果" in fruits)     # 输出: True
print("西瓜" in fruits)     # 输出: False

# 3. 遍历
for fruit in fruits:
    print(fruit)
# 输出:
# 苹果
# 香蕉
# 橘子
```

### 1.2.3 容器与"值"的区别

初学者需要明确一个概念：**单个值不是容器**。

```python
x = 42          # 这是一个整数，不是容器
name = "Alice"  # 字符串在 Python 中被视为序列容器（稍后解释）
scores = [90, 85, 78]  # 这是一个列表，是容器
```

虽然字符串 `str` 在技术上也属于序列类型（后面会详述），但在本教程中，我们主要聚焦于 Python 的**四大核心容器**：`list`、`tuple`、`dict`、`set`。

## 1.3 Python 四大内置容器一览

Python 提供了四种最基础、最常用的内置容器类型。它们是你日常编程中 90% 以上场景的解决方案：

### 1.3.1 列表（list）——有序、可变、可重复

**列表**是 Python 中最灵活、最常用的容器。可以把它想象成一个**可以随时修改的有序清单**。

```python
# 创建一个列表
shopping_list = ["牛奶", "面包", "鸡蛋", "牛奶"]  # 可以有重复元素

# 💡 现代 Python (3.9+) 严谨开发中建议使用类型注解声明容器（针对学术和工程实践）：
scores: list[int] = [90, 85, 78]  # 明确该列表专门用于存储整数

# 列表是有序的：元素有固定的位置（索引）
print(shopping_list[0])   # 输出: 牛奶（第一个元素）
print(shopping_list[2])   # 输出: 鸡蛋（第三个元素）

# 列表是可变的：可以修改、添加、删除元素
shopping_list[1] = "全麦面包"         # 修改元素
shopping_list.append("黄油")          # 添加元素
shopping_list.remove("鸡蛋")          # 删除元素
print(shopping_list)
# 输出: ['牛奶', '全麦面包', '牛奶', '黄油']
```

**关键特征**：
- ✅ 有序（Ordered）：元素有确定的位置，可以通过索引访问
- ✅ 可变（Mutable）：创建后可以修改内容
- ✅ 可重复（Duplicates allowed）：同一个值可以出现多次

### 1.3.2 元组（tuple）——有序、不可变、可重复

**元组**可以看作是"冻结的列表"。一旦创建，就不能修改其内容。

```python
# 创建一个元组
coordinates = (3.14, 2.71, 1.41)

# 元组是有序的：可以通过索引访问
print(coordinates[0])   # 输出: 3.14
print(coordinates[2])   # 输出: 1.41

# 元组是不可变的：不能修改！
# coordinates[0] = 9.99   # ❌ TypeError: 'tuple' object does not support item assignment

# 元组可以有重复元素
repeats = (1, 2, 2, 3, 3, 3)
print(repeats)   # 输出: (1, 2, 2, 3, 3, 3)
```

**关键特征**：
- ✅ 有序（Ordered）：元素有确定的位置
- ❌ 不可变（Immutable）：创建后不能修改
- ✅ 可重复（Duplicates allowed）：同一个值可以出现多次

**你可能会问**：既然列表什么都能做，为什么还要元组？我们在第 2 章和第 9 章会详细讨论。先记住一个更严谨的结论：元组更适合表达"这组数据不应被修改"的语义；而当元组中的元素也都可哈希时，它还可以作为字典键或集合元素。

### 1.3.3 字典（dict）——键值对映射、可变、键唯一

**字典**是一种以**键-值对（key-value pair）** 组织数据的容器。你可以把它想象成一本真正的词典：通过"单词"（键）来查找"释义"（值）。

```python
# 创建一个字典
student = {
    "name": "Alice",
    "age": 20,
    "major": "计算机科学"
}

# 通过键访问值
print(student["name"])    # 输出: Alice
print(student["age"])     # 输出: 20

# 字典是可变的：可以添加、修改、删除键值对
student["gpa"] = 3.8           # 添加新的键值对
student["age"] = 21            # 修改已有键的值
del student["major"]           # 删除键值对
print(student)
# 输出: {'name': 'Alice', 'age': 21, 'gpa': 3.8}

# 键是唯一的：重复的键会覆盖旧值
data = {"a": 1, "b": 2, "a": 99}
print(data)   # 输出: {'a': 99, 'b': 2}  ← 后面的 "a": 99 覆盖了 "a": 1
```

**关键特征**：
- ✅ 键值映射（Key-Value Mapping）：通过键来存取值
- ✅ 可变（Mutable）：可以增删改键值对
- ✅ 键唯一（Unique keys）：每个键只能出现一次
- ✅ 键需可哈希（Hashable keys）：常见如 `str`、`int`、只含可哈希元素的 `tuple`
- ✅ 插入有序（自 Python 3.7 起）：按插入顺序保持键值对的排列

### 1.3.4 集合（set）——无序、可变、元素唯一

**集合**是一种专门用于**去重和集合运算**的容器。它借鉴了数学中"集合"的概念。

```python
# 创建一个集合
unique_numbers = {1, 2, 3, 2, 1, 4}
print(unique_numbers)   # 例如: {1, 2, 3, 4}  ← 自动去重，但显示顺序不保证

# 集合是无序的：不能通过索引访问
# print(unique_numbers[0])   # ❌ TypeError: 'set' object is not subscriptable

# 集合是可变的
unique_numbers.add(5)         # 添加元素
unique_numbers.discard(1)     # 删除元素
print(unique_numbers)   # 例如: {2, 3, 4, 5}  ← 显示顺序不保证

# 集合支持数学集合运算
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
print(a & b)    # 例如: {3, 4}
print(a | b)    # 例如: {1, 2, 3, 4, 5, 6}
print(a - b)    # 例如: {1, 2}
```

> 补充说明：空集合必须写成 `set()`，因为字面量 `{}` 在 Python 中表示空字典，而不是空集合。

**关键特征**：
- ❌ 无序（Unordered）：元素没有固定位置，不能索引
- ✅ 可变（Mutable）：可以增删元素
- ✅ 唯一（Unique）：自动去除重复元素
- ✅ 元素需可哈希（Hashable elements）：因此列表、字典、普通集合都不能直接作为集合元素

## 1.4 容器的三大分类维度

现在我们已经认识了四大容器，接下来用三个**分类维度**来系统地理解它们之间的关系。

### 1.4.1 第一维度：序列（Sequence）vs 映射（Mapping）vs 集合（Set）

这是**最重要的分类维度**，它决定了容器的基本组织方式：

#### （一）序列（Sequence）

**序列**是一种**按位置排列元素**的容器。每个元素都有一个确定的位置编号（索引），你可以通过这个编号来访问它。

序列的核心特征：
- 元素有固定的先后顺序
- 支持通过整数索引访问元素（如 `seq[0]`、`seq[1]`）
- 支持切片操作（如 `seq[1:3]`）
- 许多常见序列支持 `+`（拼接）和 `*`（重复）操作，但这并不是所有序列类型都具备的能力（例如 `range` 就不支持直接拼接）

Python 中的序列类型包括：
- `list`（列表）——可变序列
- `tuple`（元组）——不可变序列
- `str`（字符串）——不可变字符序列
- `range`（范围）——不可变的等差整数序列
- `bytes`（字节序列）——不可变
- `bytearray`（字节数组）——可变

```python
# 序列的共同操作
my_list = [10, 20, 30, 40, 50]
my_tuple = (10, 20, 30, 40, 50)
my_str = "Hello"

# 索引访问
print(my_list[0])     # 10
print(my_tuple[0])    # 10
print(my_str[0])      # H

# 切片
print(my_list[1:4])   # [20, 30, 40]
print(my_tuple[1:4])  # (20, 30, 40)
print(my_str[1:4])    # ell

# 拼接
print([1, 2] + [3, 4])        # [1, 2, 3, 4]
print((1, 2) + (3, 4))        # (1, 2, 3, 4)
print("Hello" + " World")     # Hello World

# 重复
print([0] * 3)        # [0, 0, 0]
print((0,) * 3)       # (0, 0, 0)
print("Ha" * 3)       # HaHaHa
```

#### （二）映射（Mapping）

**映射**是一种通过**键（key）来访问值（value）** 的容器。它不关心元素的"位置"，而是关心元素的"名字"。

映射的核心特征：
- 数据以键值对的形式存储
- 通过键来存取值（而非通过位置索引）
- 键必须唯一
- 键必须是可哈希的（hashable）

Python 中的映射类型包括：
- `dict`（字典）——最常用的映射类型
- `collections.OrderedDict`——保持插入顺序的字典（Python 3.7+ 后 `dict` 本身已保持顺序）
- `collections.defaultdict`——带默认值的字典
- `collections.ChainMap`——链式映射

```python
# 映射的核心操作：通过键存取
phone_book = {
    "张三": "13800001111",
    "李四": "13900002222",
    "王五": "13700003333"
}

# 通过"键"访问"值"，而不是通过位置
print(phone_book["李四"])     # 输出: 13900002222

# 不能用位置索引！
# print(phone_book[0])        # ❌ KeyError
```

#### （三）集合（Set）

**集合**是一种**只关心元素是否存在**的容器。它不关心元素的位置，也不存储键值对。

集合的核心特征：
- 元素唯一，自动去重
- 无序，不能通过索引访问
- 支持数学集合运算（交、并、差、对称差）
- 高效的成员测试（`in` 操作非常快）

Python 中的集合类型包括：
- `set`（集合）——可变集合
- `frozenset`（冻结集合）——不可变集合

```python
# 集合的核心操作：去重与集合运算
tags_blog1 = {"Python", "编程", "入门", "教程"}
tags_blog2 = {"Python", "进阶", "编程", "项目"}

# 两篇博客共同的标签
print(tags_blog1 & tags_blog2)    # 例如: {'Python', '编程'}（顺序不保证）

# 两篇博客所有的标签（去重）
print(tags_blog1 | tags_blog2)    # 例如: {'Python', '编程', '入门', '教程', '进阶', '项目'}（顺序不保证）
```

### 分类总结表

|   分类   |        代表类型        |    访问方式    |        特点        |
| :------: | :--------------------: | :------------: | :----------------: |
| **序列** | `list`、`tuple`、`str` | 按位置（索引） |   有序、支持切片   |
| **映射** |         `dict`         |  按键（key）   |   键值对、键唯一   |
| **集合** |   `set`、`frozenset`   |   无直接访问   | 唯一、支持集合运算 |

### 1.4.2 第二维度：可变（Mutable）vs 不可变（Immutable）

这一维度回答的问题是：**容器创建后，能否修改其内容？**

|        特性        |       可变容器        |         不可变容器          |
| :----------------: | :-------------------: | :-------------------------: |
|        定义        | 创建后可以增删改元素  |  创建后内容固定，不能修改   |
|        代表        | `list`、`dict`、`set` | `tuple`、`frozenset`、`str` |
|  能否作为字典的键  |      通常 ❌ 不能      |      视是否可哈希而定       |
| 能否作为集合的元素 |      通常 ❌ 不能      |      视是否可哈希而定       |

```python
# 可变——可以"原地"修改
my_list = [1, 2, 3]
my_list[0] = 99
print(my_list)    # [99, 2, 3]  ← 原列表被修改了

# 不可变——尝试修改会报错
my_tuple = (1, 2, 3)
# my_tuple[0] = 99   # ❌ TypeError
```

> 💡 **深层含义**：可变性不仅影响你能对容器做什么操作，还会强烈影响对象是否可能是"可哈希"的，从而影响它能否被用作字典的键或集合的元素。这个概念非常重要，我们将在第 2 章深入讨论。

### 1.4.3 第三维度：有序（Ordered）vs 无序（Unordered）

这一维度回答的问题是：**元素是否有确定的排列顺序？**

|     特性     |            有序容器             |        无序容器        |
| :----------: | :-----------------------------: | :--------------------: |
|     定义     |      元素有固定的排列顺序       | 元素没有固定的排列顺序 |
|     代表     | `list`、`tuple`、`dict`（3.7+） |   `set`、`frozenset`   |
| 是否支持索引 |          序列类型支持           |         不支持         |

> ⚠️ **特别说明**：`dict` 自 Python 3.7 起保证按**插入顺序**维护键值对。这意味着你遍历字典时，键值对的顺序与你插入它们的顺序一致。但字典仍然不支持整数位置索引（不能写 `my_dict[0]` 来访问第一个键值对）。

## 1.5 容器分类的"全景图"

让我们把上述三个维度综合起来，形成一张完整的分类表：

|  容器类型   | 大分类 |   有序性   |  可变性  | 元素唯一性 |    字面量语法    |
| :---------: | :----: | :--------: | :------: | :--------: | :--------------: |
|   `list`    |  序列  |   ✅ 有序   |  ✅ 可变  |  ❌ 可重复  |   `[1, 2, 3]`    |
|   `tuple`   |  序列  |   ✅ 有序   | ❌ 不可变 |  ❌ 可重复  |   `(1, 2, 3)`    |
|    `str`    |  序列  |   ✅ 有序   | ❌ 不可变 |  ❌ 可重复  |    `"hello"`     |
|   `dict`    |  映射  | ✅ 插入有序 |  ✅ 可变  |  ✅ 键唯一  |    `{"a": 1}`    |
|    `set`    |  集合  |   ❌ 无序   |  ✅ 可变  |   ✅ 唯一   |   `{1, 2, 3}`    |
| `frozenset` |  集合  |   ❌ 无序   | ❌ 不可变 |   ✅ 唯一   | `frozenset(...)` |

> 💡 **记忆技巧**：
> - **列表**（list）= 可变 + 有序 + 可重复 → 最通用
> - **元组**（tuple）= 不可变 + 有序 + 可重复 → 冻结的列表
> - **字典**（dict）= 可变 + 键值对 + 键唯一 → 以名查值
> - **集合**（set）= 可变 + 无序 + 唯一 → 去重与集合运算

## 1.6 容器之间的关系与层次

### 1.6.1 抽象层次关系

Python 的容器类型并不是孤立存在的，它们之间有清晰的抽象层次关系。Python 在 `collections.abc` 模块中定义了一套**抽象基类（Abstract Base Classes, ABC）**，用于描述"一个类型支持哪些协议"，而不是简单地给出一棵单继承树。对初学者而言，可以先把它理解为下面这张更准确的能力关系图：

```
Container（支持 in）
Iterable（支持 for）
Sized（支持 len()）
Collection = Container + Iterable + Sized
├── Sequence（有序、支持索引与切片）
│   ├── MutableSequence → list
│   └── 不可变序列 → tuple、str、range
├── Mapping（键值映射）
│   └── MutableMapping → dict
└── Set（元素唯一、支持集合运算）
    ├── MutableSet → set
    └── 不可变集合 → frozenset
```

你现在不需要记住这些抽象基类的细节（它们属于进阶内容），但了解这个关系有助于你理解：**不同容器共享了哪些通用操作，又各自有哪些独特能力**。

### 1.6.2 容器之间的转换

Python 的容器之间可以方便地相互转换：

```python
# 列表 → 元组
my_list = [1, 2, 3]
my_tuple = tuple(my_list)      # (1, 2, 3)

# 元组 → 列表
my_list2 = list(my_tuple)      # [1, 2, 3]

# 列表 → 集合（自动去重）
my_list3 = [1, 2, 2, 3, 3, 3]
my_set = set(my_list3)         # {1, 2, 3}

# 集合 → 列表
my_list4 = list(my_set)        # [1, 2, 3]（顺序不保证）

# 列表的列表 → 字典
pairs = [("name", "Alice"), ("age", 20)]
my_dict = dict(pairs)          # {'name': 'Alice', 'age': 20}

# 字典 → 键的列表
keys_list = list(my_dict)      # ['name', 'age']

# 字典 → 值的列表
values_list = list(my_dict.values())  # ['Alice', 20]

# 字典 → (键, 值) 元组的列表
items_list = list(my_dict.items())    # [('name', 'Alice'), ('age', 20)]
```

> 补充说明：`dict()` 从键值对序列构造字典时，如果同一个键重复出现，后面的值会覆盖前面的值。

> 💡 **核心理解**：容器转换的本质是**迭代**。`list(x)`、`tuple(x)`、`set(x)` 的工作方式都是：从 `x` 中逐个取出元素，放到新的容器中。对于字典，默认迭代的是键。

## 1.7 "字符串是容器吗？"——澄清一个常见困惑

严格来说，**字符串（`str`）确实是序列容器**。它满足序列的所有条件：

```python
s = "Hello"

# 有序，支持索引
print(s[0])       # H
print(s[4])       # o

# 支持切片
print(s[1:4])     # ell

# 支持遍历
for ch in s:
    print(ch, end=" ")
# 输出: H e l l o
print()    # 补一个换行，避免后续输出接在同一行

# 支持 len()
print(len(s))     # 5

# 支持 in
print("ell" in s)  # True
```

但在本教程中，我们将字符串视为**"文本处理的专用类型"**而非通用容器。原因如下：

1. 字符串的元素只能是单个字符（长度为 1 的字符串），而列表/元组可以存放任意类型。
2. 字符串有大量文本处理专用方法（如 `split()`、`strip()`、`replace()`），这些与通用容器操作关系不大。
3. 字符串是不可变的，行为上更接近元组。

因此，本教程聚焦于四大核心容器：**list、tuple、dict、set**。字符串将作为理解序列概念的辅助参考，但不会单独设章讲解。

## 2.2 可变（Mutable）与不可变（Immutable）

### 2.2.1 基本定义

**可变（Mutable）**：对象创建后，其**内容**可以被修改，而**对象本身的身份（内存地址）不变**。

**不可变（Immutable）**：对象创建后，其**内容**不能被修改。任何看似"修改"的操作，实际上都是创建了一个新的对象。

```python
# ========== 可变类型：list ==========
a = [1, 2, 3]
print(id(a))       # 例如: 140234567890
a[0] = 99          # 修改第一个元素
print(a)           # [99, 2, 3]
print(id(a))       # 仍然是: 140234567890  ← id 不变！同一个对象被"原地修改"了

# ========== 不可变类型：tuple ==========
b = (1, 2, 3)
print(id(b))       # 例如: 140234567456
# b[0] = 99        # ❌ TypeError: 'tuple' object does not support item assignment
```

> 🔑 **关键理解**：`id()` 返回对象在当前运行期的身份标识；在 CPython 中通常可近似理解为内存地址。对可变对象做原地修改时，`id()` 通常保持不变；而对不可变对象的所谓"修改"，本质上往往是让变量重新绑定到一个新对象。需要注意：`id()` 的变化与否受实现细节和优化影响，不应把它当作业务逻辑的判断依据。

### 2.2.2 用 id() 深入理解可变性

让我们通过更多的例子来体会这一点：

```python
# ========== 不可变类型的"修改"实际上是创建新对象 ==========

# 整数是不可变的
x = 10
print(f"x = {x}, id = {id(x)}")     # x = 10, id = 140234500
x = x + 1
print(f"x = {x}, id = {id(x)}")     # x = 11, id = 140234532  ← id 变了！
# 注意：x + 1 创建了一个新的整数对象 11，然后让 x 指向这个新对象
# 原来的整数 10 并没有变成 11

# 字符串是不可变的
s = "hello"
print(f"s = {s}, id = {id(s)}")       # s = hello, id = 140234600
s = s + " world"
print(f"s = {s}, id = {id(s)}")       # s = hello world, id = 140234700  ← id 变了！
# s + " world" 创建了一个全新的字符串 "hello world"

# 元组是不可变的
t = (1, 2, 3)
print(f"t = {t}, id = {id(t)}")       # t = (1, 2, 3), id = 140234800
t = t + (4,)
print(f"t = {t}, id = {id(t)}")       # t = (1, 2, 3, 4), id = 140234900  ← id 变了！
# t + (4,) 创建了一个全新的元组
```

```python
# ========== 可变类型的修改不会创建新对象 ==========

# 列表是可变的
a = [1, 2, 3]
print(f"a = {a}, id = {id(a)}")       # a = [1, 2, 3], id = 140234000
a.append(4)
print(f"a = {a}, id = {id(a)}")       # a = [1, 2, 3, 4], id = 140234000  ← id 不变！
a[0] = 99
print(f"a = {a}, id = {id(a)}")       # a = [99, 2, 3, 4], id = 140234000  ← id 还是不变！
a.extend([5, 6])
print(f"a = {a}, id = {id(a)}")       # a = [99, 2, 3, 4, 5, 6], id = 140234000  ← 始终不变

# 字典是可变的
d = {"a": 1}
print(f"d = {d}, id = {id(d)}")       # d = {'a': 1}, id = 140235000
d["b"] = 2
print(f"d = {d}, id = {id(d)}")       # d = {'a': 1, 'b': 2}, id = 140235000  ← id 不变

# 集合是可变的
s = {1, 2, 3}
print(f"s = {s}, id = {id(s)}")       # s = {1, 2, 3}, id = 140236000
s.add(4)
print(f"s = {s}, id = {id(s)}")       # s = {1, 2, 3, 4}, id = 140236000  ← id 不变
```

### 2.2.3 Python 中常见类型的可变性分类

|       不可变类型        |               可变类型                |
| :---------------------: | :-----------------------------------: |
|      `int`（整数）      |            `list`（列表）             |
|    `float`（浮点数）    |            `dict`（字典）             |
|    `bool`（布尔值）     |             `set`（集合）             |
|     `str`（字符串）     |              `bytearray`              |
|     `tuple`（元组）     | 大多数用户自定义实例（取决于类的设计） |
| `frozenset`（冻结集合） |                                       |
|   `bytes`（字节序列）   |                                       |
|    `complex`（复数）    |                                       |
|   `NoneType`（None）    |                                       |
|         `range`         |                                       |

> 💡 **更严谨地说**：所谓"某个对象可变/不可变"，描述的是这个对象的状态能否在原地改变。对用户自定义类而言，这通常取决于它是否暴露了会修改内部状态的方法，而不是仅凭"这是类实例"就能机械判断。

### 2.2.4 可变性的"陷阱"：别名与共享引用

这是初学者最容易踩的坑之一。理解这个概念可以帮你避免大量难以排查的 Bug。

**Python 中，变量不是"盒子"，而是"标签"。**

在 C/C++ 中，你可以把变量想象成一个盒子，值存在盒子里。但在 Python 中，变量更像是贴在对象上的标签。多个标签可以贴在同一个对象上。

```python
# ========== 不可变类型：别名是安全的 ==========
a = 10
b = a          # b 和 a 指向同一个整数对象 10
print(id(a) == id(b))   # True —— 指向同一个对象

b = b + 1      # b 现在指向新的整数对象 11
print(a)       # 10 —— a 没有被影响
print(b)       # 11
# 因为整数是不可变的，b + 1 创建了新对象，b 的标签转贴到了新对象上
# a 的标签仍然贴在旧对象 10 上

# ========== 可变类型：别名可能导致"意外修改" ==========
x = [1, 2, 3]
y = x          # y 和 x 指向同一个列表对象
print(id(x) == id(y))   # True —— 指向同一个对象

y.append(4)    # 通过 y 修改列表
print(x)       # [1, 2, 3, 4] ← x 也被修改了！
print(y)       # [1, 2, 3, 4]
# 因为 x 和 y 指向同一个列表对象，通过 y 的修改当然会影响 x
```

这个行为可以用图示理解：

```
# 赋值 y = x 后：
x ──┐
    ├──→ [1, 2, 3]     ← 两个变量指向同一个列表对象
y ──┘

# y.append(4) 后：
x ──┐
    ├──→ [1, 2, 3, 4]  ← 修改作用在同一个对象上
y ──┘
```

> ⚠️ **重要教训**：对于可变对象，**赋值 `=` 不是复制，而是创建别名（alias）**。如果你需要一个独立的副本，必须显式复制；本章后文会补充讨论浅拷贝和深拷贝。

### 2.2.5 可变性与函数参数传递

这个陷阱在函数中尤为常见：

```python
def add_element(lst, item):
    """向列表末尾添加一个元素"""
    lst.append(item)

my_list = [1, 2, 3]
add_element(my_list, 4)
print(my_list)   # [1, 2, 3, 4] ← 原列表被修改了！

# 对比：不可变类型不会有这个问题
def add_one(x):
    """尝试给 x 加 1"""
    x = x + 1       # 创建了新的整数对象，只是 x 这个局部变量指向了新对象
    return x

n = 10
add_one(n)
print(n)    # 10 ← 原变量不受影响
```

**原理解释**：Python 的参数传递机制不是"传值"也不是"传引用"（这是 C++ 的概念），而是**"传对象引用"（pass by object reference）**。具体来说：

- 函数参数 `lst` 和外部变量 `my_list` 指向同一个列表对象。
- 在函数内部调用 `lst.append(item)` 修改了这个共享对象。
- 所以外部的 `my_list` 也受到了影响。

对于不可变类型，函数内部的 `x = x + 1` 实际上让 `x` 指向了一个新对象，不影响外部变量。

### 2.2.6 经典陷阱：可变默认参数

这是 Python 教学中必须单独强调的经典问题，因为它同时牵涉到**函数定义时机**和**可变对象共享**。

先看一个容易写错的版本：

```python
def append_item(item, target=[]):
    """错误示例：target 会在多次调用之间被共享"""
    target.append(item)
    return target

print(append_item(1))   # [1]
print(append_item(2))   # [1, 2]  ← 很多人本来以为会得到 [2]
print(append_item(3))   # [1, 2, 3]
```

为什么会这样？因为**默认参数只在函数定义时求值一次**。上面的 `target=[]` 不是"每次调用都创建一个新列表"，而是在定义函数时创建了一个列表，后续所有没有显式传入 `target` 的调用，都会复用这个同一个列表对象。

正确写法是把默认值设为 `None`，再在函数内部创建新列表：

```python
def append_item_safe(item, target=None):
    """推荐写法：每次需要时再创建新列表"""
    if target is None:
        target = []
    target.append(item)
    return target

print(append_item_safe(1))   # [1]
print(append_item_safe(2))   # [2]
print(append_item_safe(3))   # [3]
```

> 🔑 **经验法则**：函数默认参数中，尽量不要直接写 `[]`、`{}`、`set()` 这类可变对象。若参数语义是"可选，未提供时新建一个容器"，首选 `None` 作为哨兵值。

### 2.2.7 可变性与哈希：为什么可变对象不能做字典的键？

这是一个非常重要的设计约束：

```python
# 不可变对象可以作为字典的键
d = {}
d[(1, 2, 3)] = "一个元组作为键"       # ✅ 元组可以
d["hello"] = "一个字符串作为键"         # ✅ 字符串可以
d[42] = "一个整数作为键"               # ✅ 整数可以
print(d)

# 可变对象不能作为字典的键
# d[[1, 2, 3]] = "列表作为键"         # ❌ TypeError: unhashable type: 'list'
# d[{1, 2}] = "集合作为键"            # ❌ TypeError: unhashable type: 'set'
```

**为什么有这个限制？**

字典和集合的底层都使用了**哈希表（Hash Table）** 数据结构。哈希表会先根据对象的**哈希值（hash value）** 选择候选位置，再结合相等性比较来确认是不是同一个键/元素。

- 如果一个对象参与判等与哈希计算的状态是**可变的**，那它在放入字典或集合后就可能发生变化。
- 一旦这种变化破坏了"哈希值稳定"或"相等关系稳定"的前提，哈希表就可能无法在原有位置正确找到它。
- 因此，作为字典键或集合元素的对象，必须满足：**在其生命周期内，参与 `__hash__` / `__eq__` 的状态应保持稳定**。

因此，Python 规定：**只有不可变的（准确地说是"可哈希的"）对象才能作为字典的键和集合的元素**。

```python
# hash() 函数可以查看对象的哈希值
print(hash(42))           # 42
print(hash("hello"))      # 某个整数值
print(hash((1, 2, 3)))    # 某个整数值

# 可变对象没有哈希值
# print(hash([1, 2, 3]))  # ❌ TypeError: unhashable type: 'list'
# print(hash({1, 2}))     # ❌ TypeError: unhashable type: 'set'
```

> 💡 **精确表述**：能否作为字典键的真正判据是**是否可哈希（hashable）**，而非简单的"是否可变"。在入门阶段，你可以先记成："大多数常见不可变内置对象通常可哈希，而常见可变内置对象通常不可哈希"。但两者并不完全等价，例如：`tuple` 虽然不可变，但只要内部含有不可哈希元素，它自己也不可哈希。

### 2.2.8 特殊情况：元组中包含可变元素

元组本身是不可变的，但如果元组中包含了可变对象（如列表），情况会变得微妙：

```python
# 元组中包含列表
t = (1, 2, [3, 4])
print(t)   # (1, 2, [3, 4])

# 不能修改元组的"指向"
# t[0] = 99           # ❌ TypeError
# t[2] = [5, 6]       # ❌ TypeError（不能让 t[2] 指向另一个对象）

# 但可以修改元组中列表的内容！
t[2].append(5)
print(t)   # (1, 2, [3, 4, 5])  ← 元组"变了"？

# 更精确地说：元组本身没变（它仍然包含三个元素，第三个元素仍然是同一个列表对象）
# 变的是列表对象的内容
```

图示理解：

```
t ──→ tuple 对象:
         [0] ──→ 整数 1
         [1] ──→ 整数 2
         [2] ──→ list 对象: [3, 4]      ← 元组记住的是"指向这个列表"
                              ↓               而列表的内容是列表自己管的
                          list 对象: [3, 4, 5]  ← 列表内容变了
```

元组保证的不可变性是：**它的每个"槽位"始终指向同一个对象**。但如果某个槽位指向的是一个可变对象，那个可变对象自身的内容变化，不受元组的控制。

**这也导致了一个重要后果**：

```python
# 包含可变元素的元组不可哈希
t1 = (1, 2, "hello")     # 所有元素都不可变
print(hash(t1))           # ✅ 可以哈希

t2 = (1, 2, [3, 4])       # 包含可变元素（列表）
# print(hash(t2))         # ❌ TypeError: unhashable type: 'list'
```

> 🔑 **核心结论**：元组是否可哈希，取决于它的所有元素是否都可哈希。如果元组中包含列表或集合等可变对象，则该元组也不可哈希，不能用作字典的键或集合的元素。

### 2.2.9 补充：浅拷贝、深拷贝与嵌套容器

前面我们已经看到：`y = x` 并不会复制对象，只是让 `y` 和 `x` 指向同一个对象。实际编程里，更常见的问题是：**我明明复制了，为什么修改副本还是影响了原对象？**

这就涉及 **浅拷贝（shallow copy）** 与 **深拷贝（deep copy）**。

```python
import copy

original = [[1, 2], [3, 4]]

# 浅拷贝：只复制最外层容器，内部元素仍然共享
shallow = copy.copy(original)
shallow[0].append(99)

print(original)   # [[1, 2, 99], [3, 4]]  ← 原对象也受影响
print(shallow)    # [[1, 2, 99], [3, 4]]

# 深拷贝：递归复制嵌套对象
deep = copy.deepcopy(original)
deep[1].append(88)

print(original)   # [[1, 2, 99], [3, 4]]      ← 不受 deep 的修改影响
print(deep)       # [[1, 2, 99], [3, 4, 88]]
```

你也可以把浅拷贝理解为：

- 最外层容器是新的
- 里面装的元素引用，很多仍然指向原来的子对象

而深拷贝则是：

- 最外层是新的
- 内层嵌套对象也尽量递归复制成新的

另一个非常经典、而且与容器教学高度相关的陷阱是二维列表初始化：

```python
# 错误写法：三行其实引用的是同一个内部列表
matrix_bad = [[0] * 3] * 3
matrix_bad[0][1] = 7
print(matrix_bad)   # [[0, 7, 0], [0, 7, 0], [0, 7, 0]]

# 正确写法：每一行都是独立的新列表
matrix_good = [[0] * 3 for _ in range(3)]
matrix_good[0][1] = 7
print(matrix_good)  # [[0, 7, 0], [0, 0, 0], [0, 0, 0]]
```

> 🔑 **经验法则**：只要容器里还嵌套着其他可变对象，就要格外警惕"共享引用"。`=` 只是绑定；切片、`list()`、`copy.copy()` 通常只是浅拷贝；需要完全隔离时，优先考虑 `copy.deepcopy()`。

### 2.2.10 可变性对比总结

|      特性      | 可变类型（list, dict, set） | 不可变类型（tuple, str, int, frozenset） |
| :------------: | :-------------------------: | :--------------------------------------: |
|    修改内容    |       ✅ 可以原地修改        |             ❌ 必须创建新对象             |
|    赋值行为    |    创建别名（共享引用）     |      创建别名，但修改时不影响原对象      |
|  能否做字典键  |           ❌ 不能            |     ✅ 能（需要所有嵌套元素也可哈希）     |
| 能否做集合元素 |           ❌ 不能            |                   ✅ 能                   |
|   函数内修改   |       会影响外部变量        |              不影响外部变量              |
|  `id()` 行为   |      修改前后 id 不变       |            "修改"前后 id 改变            |

## 2.3 有序（Ordered）与无序（Unordered）

### 2.3.1 基本定义

**有序（Ordered）**：容器中的元素有确定的排列顺序。元素的位置在容器的生命周期内保持稳定（除非你主动修改）。

**无序（Unordered）**：容器中的元素没有固定的排列顺序。你不能依赖元素的排列位置。

```python
# ========== 有序：list ==========
fruits = ["苹果", "香蕉", "橘子"]
# 第一次遍历
for f in fruits:
    print(f, end=" ")
# 输出: 苹果 香蕉 橘子
print()

# 第一百次遍历，顺序一样
for f in fruits:
    print(f, end=" ")
# 输出: 苹果 香蕉 橘子
print()

# 可以通过位置索引访问
print(fruits[0])    # 苹果
print(fruits[2])    # 橘子

# ========== 无序：set ==========
colors = {"红", "绿", "蓝"}
# 遍历时顺序不保证
for c in colors:
    print(c, end=" ")
# 可能输出: 蓝 红 绿（每次运行可能不同）
print()

# 不能通过位置索引访问
# print(colors[0])    # ❌ TypeError: 'set' object is not subscriptable
```

### 2.3.2 "有序"的准确含义

在 Python 的容器术语中，"有序"一词经常引起困惑。让我们精确区分它的两种含义：

#### 含义一："插入有序"（Insertion-ordered）

指元素**按照插入的先后顺序**排列。这是 Python 中"有序"的最常见含义。

```python
# list 保持插入顺序
lst = []
lst.append("第一个")
lst.append("第二个")
lst.append("第三个")
print(lst)   # ['第一个', '第二个', '第三个'] ← 按插入顺序排列

# dict（Python 3.7+）也保持插入顺序
d = {}
d["c"] = 3
d["a"] = 1
d["b"] = 2
print(list(d.keys()))   # ['c', 'a', 'b'] ← 按插入顺序，不是按字母排序
```

#### 含义二："排序有序"（Sorted）

指元素**按某种比较规则排列**（如从小到大、字母序等）。Python 的内置容器**没有自动排序**的功能。如果你需要排序，需要显式调用 `sort()` 或 `sorted()`。

```python
numbers = [3, 1, 4, 1, 5, 9]
print(numbers)             # [3, 1, 4, 1, 5, 9] ← 没有自动排序

numbers.sort()              # 显式排序
print(numbers)             # [1, 1, 3, 4, 5, 9] ← 现在是排序有序的
```

> 💡 **澄清**：当我们说 `list` 和 `tuple` 是"有序的"，我们指的是**"插入有序"**——元素按你放入的顺序保存。它们不会自动帮你排序。

### 2.3.3 字典的"有序性"详解

字典的有序性是一个经常被讨论的话题。让我们梳理它的历史：

|    Python 版本    |                         字典行为                         |
| :---------------: | :------------------------------------------------------: |
| Python 3.5 及更早 |               字典**无序**，遍历顺序不确定               |
|    Python 3.6     | CPython 实现中字典**事实上有序**（实现细节，非语言保证） |
|    Python 3.7+    |            **语言规范**保证字典按插入顺序排列            |

```python
# Python 3.7+ 中，字典按插入顺序排列
student = {}
student["name"] = "Alice"
student["age"] = 20
student["major"] = "CS"

for key, value in student.items():
    print(f"{key}: {value}")
# 输出（保证顺序）:
# name: Alice
# age: 20
# major: CS
```

> ⚠️ **注意**：虽然字典保持插入顺序，但字典**仍然不支持位置索引**。你不能写 `student[0]` 来访问第一个键值对。字典的访问方式始终是通过键：`student["name"]`。

还有一个很容易忽略的边界细节：

- **更新已有键的值，不会改变这个键的位置**
- **删除一个键再重新插入，它会出现在末尾**

这说明字典的"有序"是**插入顺序语义**，而不是"按键自动排序"或"随便修改后位置都不变"。

如果你确实需要"通过位置访问字典的第 n 个键值对"，可以这样做（但这不是常规操作）：

```python
student = {"name": "Alice", "age": 20, "major": "CS"}

# 获取第一个键值对
first_key = list(student.keys())[0]       # "name"
first_value = list(student.values())[0]   # "Alice"

# 更好的方式：使用 next() 和 iter()
first_key = next(iter(student))           # "name"  ← 不需要创建完整列表
```

### 2.3.4 有序性的实际影响

有序性决定了你能对容器做哪些操作：

|        操作        | 有序容器（list, tuple） | 无序容器（set） |  字典（dict）   |
| :----------------: | :---------------------: | :-------------: | :-------------: |
|   索引访问 `[i]`   |            ✅            |        ❌        | ❌（通过键访问） |
|    切片 `[i:j]`    |            ✅            |        ❌        |        ❌        |
|      `+` 拼接      |            ✅            |        ❌        |        ❌        |
|      `*` 重复      |            ✅            |        ❌        |        ❌        |
| `index()` 查找位置 |            ✅            |        ❌        |        ❌        |
|   `sort()` 排序    |      ✅（仅 list）       |        ❌        |        ❌        |
|    遍历顺序确定    |            ✅            |        ❌        |    ✅（3.7+）    |

```python
# 有序容器的专属操作
lst = [10, 20, 30, 40, 50]

# 索引
print(lst[2])          # 30

# 切片
print(lst[1:4])        # [20, 30, 40]

# 拼接
print(lst + [60, 70])  # [10, 20, 30, 40, 50, 60, 70]

# 重复
print([0] * 5)         # [0, 0, 0, 0, 0]

# 查找位置
print(lst.index(30))   # 2

# 反转
lst.reverse()
print(lst)             # [50, 40, 30, 20, 10]
```

```python
# 无序容器（set）不支持这些操作
s = {10, 20, 30, 40, 50}

# s[0]              # ❌ TypeError
# s[1:3]            # ❌ TypeError
# s + {60}          # ❌ TypeError
# s * 3             # ❌ TypeError
# s.index(30)       # ❌ AttributeError
# s.sort()          # ❌ AttributeError
```

### 2.3.5 "无序"真的意味着"随机"吗？

一个常见的误解是：无序 = 随机。实际上并非如此。

```python
# 集合的遍历顺序在单次程序运行中通常是一致的
s = {3, 1, 4, 1, 5, 9}
print(s)   # 例如: {1, 3, 4, 5, 9}

# 多次遍历同一个集合（在同一次程序运行中），顺序通常一样
for _ in range(3):
    print(list(s))
# 输出（在同一次运行中通常一致）:
# [1, 3, 4, 5, 9]
# [1, 3, 4, 5, 9]
# [1, 3, 4, 5, 9]
```

"无序"的准确含义是：

1. **语言规范不保证任何特定的顺序**：即使你观察到某种顺序，也不能依赖它。
2. **不同版本的 Python、不同的运行环境，顺序可能不同**。
3. **现代 Python 出于安全原因对 `str`、`bytes` 等类型的哈希使用随机化种子**（hash randomization），所以同一程序多次运行，字符串集合的遍历顺序可能不同。

> 🔑 **实用建议**：如果你需要有序的集合，应该先将集合转换为列表再排序，或者使用 `sorted()` 函数。不要依赖集合的遍历顺序。

```python
s = {3, 1, 4, 1, 5, 9}
ordered_list = sorted(s)    # [1, 3, 4, 5, 9] ← 得到有序的列表
```

## 2.4 可重复（Duplicates）与唯一（Unique）

### 2.4.1 基本定义

**可重复（Duplicates allowed）**：容器中**同一个值可以出现多次**，每次出现都被视为独立的元素。

**唯一（Unique）**：容器中**每个值最多出现一次**，重复添加无效。

```python
# ========== 可重复：list、tuple ==========
scores = [90, 85, 90, 78, 85]
print(scores)        # [90, 85, 90, 78, 85] ← 90 和 85 各出现两次，完全保留
print(len(scores))   # 5（每个都算一个独立元素）

t = (1, 2, 2, 3, 3, 3)
print(t)             # (1, 2, 2, 3, 3, 3)
print(t.count(3))    # 3（元组支持 count 方法来统计重复元素个数）

# ========== 唯一：set ==========
unique_scores = {90, 85, 90, 78, 85}
print(unique_scores)       # 例如: {90, 85, 78} ← 重复的被自动去除，但显示顺序不保证
print(len(unique_scores))  # 3

# 重复添加无效
unique_scores.add(90)
print(unique_scores)       # 例如: {90, 85, 78} ← 没有任何变化，显示顺序不保证
```

### 2.4.2 字典的"唯一性"

字典的唯一性体现在**键**上，而不是值上：

```python
# 键必须唯一
d = {"apple": 3, "banana": 5, "apple": 7}
print(d)   # {'apple': 7, 'banana': 5} ← 后面的 "apple": 7 覆盖了 "apple": 3

# 值可以重复
grades = {"Alice": "A", "Bob": "B", "Charlie": "A", "David": "B"}
print(grades)
# {'Alice': 'A', 'Bob': 'B', 'Charlie': 'A', 'David': 'B'} ← 值 "A" 和 "B" 都重复了，没问题
```

### 2.4.3 "去重"——唯一性的典型应用

唯一性最常见的应用就是**去重**。通过将列表转换为集合，可以快速获得不重复的元素：

```python
# 经典去重模式
raw_data = [1, 3, 5, 3, 1, 7, 5, 9, 1, 3]

# 方法1：转换为 set 再转回 list（不保留原顺序）
unique_data = list(set(raw_data))
print(unique_data)   # [1, 3, 5, 7, 9]（顺序不保证）

# 方法2：保留原顺序的去重（Python 3.7+，利用 dict 的插入有序特性）
unique_ordered = list(dict.fromkeys(raw_data))
print(unique_ordered)   # [1, 3, 5, 7, 9]（保留首次出现的顺序）
```

### 2.4.4 唯一性与判等机制

集合和字典判断"两个元素是否相同"时，会综合使用 **`hash()` 哈希值** 与 **`==` 等值比较**：

- 哈希值用于快速定位候选位置
- 如果哈希值相同，还需要再用 `==` 判断它们是否真的相等
- 仅仅"哈希值碰巧相同"并不意味着它们一定是同一个元素，因为哈希冲突在理论上是可能存在的

对初学者而言，可以先抓住这条实用规则：**若两个可哈希对象相等（`a == b` 为 True），它们必须具有相同的哈希值；而集合/字典会把这样的对象视为同一个键或元素。**

```python
# 整数 1 和浮点数 1.0 在 Python 中相等
print(1 == 1.0)         # True
print(hash(1) == hash(1.0))  # True

# 因此在集合中它们被视为同一个元素
s = {1, 1.0, 2}
print(s)   # {1, 2} ← 1 和 1.0 被认为是同一个元素，保留先出现的那个

# 在字典中同理
d = {1: "integer", 1.0: "float"}
print(d)   # {1: 'float'} ← 键 1.0 覆盖了键 1 的值，但键本身保留先出现的
```

> ⚠️ **注意**：这个行为是很多初学者意想不到的。`True == 1 == 1.0`，`False == 0 == 0.0`，它们在集合中都被视为同一个元素。

```python
# 更极端的例子
s = {True, 1, 1.0}
print(s)   # {True} ← True、1、1.0 三者相等，只保留第一个

s2 = {0, False, 0.0}
print(s2)  # {0} ← 0、False、0.0 三者相等，只保留第一个
```

## 2.5 三组概念的交叉影响

三个维度并不是独立的，它们之间有重要的交叉影响。

### 2.5.1 可变性 × 唯一性 → 哈希约束

集合和字典的键要求**元素/键可哈希**。在常见内置类型中，**可变容器通常不可哈希**，因此：

```python
# 列表不能放入集合
# s = {[1, 2], [3, 4]}   # ❌ TypeError: unhashable type: 'list'

# 元组可以（前提是元组内元素也都可哈希）
s = {(1, 2), (3, 4)}      # ✅
print(s)

# 字典不能做集合的元素
# s = {{"a": 1}}           # ❌ TypeError: unhashable type: 'dict'

# frozenset 可以做集合的元素（不可变集合）
s = {frozenset({1, 2}), frozenset({3, 4})}   # ✅
print(s)
```

### 2.5.2 有序性 × 可变性 → 支持的操作

|            | 有序 + 可变 | 有序 + 不可变 | 无序 + 可变 | 无序 + 不可变 |
| :--------: | :---------: | :-----------: | :---------: | :-----------: |
|    代表    |   `list`    |    `tuple`    |    `set`    |  `frozenset`  |
| 索引/切片  |      ✅      |       ✅       |      ❌      |       ❌       |
|  原地修改  |      ✅      |       ❌       |      ✅      |       ❌       |
|    排序    | ✅ `sort()`  |       ❌       |      ❌      |       ❌       |
|  集合运算  |      ❌      |       ❌       |      ✅      |       ✅       |
| 可做字典键 |      ❌      |       ✅       |      ❌      |       ✅       |

### 2.5.3 实际场景中的概念组合

```python
# 场景1：记录学生考试成绩（可重复、有序 → list）
exam_scores = [78, 85, 92, 85, 78, 90]  # 需要记录每个人的分数，可能重复

# 场景2：存储地理坐标（不可变、有序 → tuple）
location = (39.9042, 116.4074)  # 北京的经纬度，不应被修改

# 场景3：管理用户标签（唯一、无序 → set）
user_tags = {"Python", "编程", "AI"}  # 标签不应重复

# 场景4：学号→姓名映射（键唯一、键值对 → dict）
students = {"2024001": "张三", "2024002": "李四"}

# 场景5：不可修改的配置项集合（唯一、不可变 → frozenset）
ALLOWED_METHODS = frozenset({"GET", "POST", "PUT", "DELETE"})
```

### 2.5.4 容器选型速判

如果你面对一个新问题，不确定该选哪种容器，可以按下面的判断顺序快速决策：

1. **是否需要通过键来查值？**
   - 需要：优先考虑 `dict`
2. **是否最重要的是去重或集合运算？**
   - 需要：优先考虑 `set`；如果还要求不可修改，考虑 `frozenset`
3. **是否需要保持顺序，并且后续还会增删改？**
   - 需要：优先考虑 `list`
4. **是否需要保持顺序，但数据创建后不应再改动，或者要把它作为字典键/集合元素？**
   - 需要：优先考虑 `tuple`

常见教学场景可以这样理解：

|           场景           | 推荐容器 |                 原因                 |
| :----------------------: | :------: | :----------------------------------: |
|      记录实验测量序列      |  `list`  |        有顺序，后续常需要追加或修正        |
|     表示固定坐标或多返回值     | `tuple`  |      有顺序，但语义上不希望被随意修改      |
| 学号到学生信息的对应关系 |  `dict`  |          本质是"键 -> 值"映射          |
|      论文关键词去重与交并集      |  `set`   | 天然去重，且支持交集、并集、差集等集合运算 |

## 2.6 "Python 中一切皆对象"——容器背后的对象模型

要真正理解可变性，你需要了解 Python 的对象模型。

### 2.6.1 对象的三要素

Python 中每个对象都有三个基本属性：

1. **身份（Identity）**：对象在内存中的唯一标识，用 `id()` 获取，创建后不可改变。
2. **类型（Type）**：对象的类型，用 `type()` 获取，决定了对象支持的操作。
3. **值（Value）**：对象包含的数据。可变对象的值可以改变，不可变对象的值不可改变。

```python
x = [1, 2, 3]
print(id(x))      # 身份：例如 2305847234512
print(type(x))    # 类型：<class 'list'>
print(x)          # 值：[1, 2, 3]

# 可变对象：值变了，身份和类型不变
x.append(4)
print(id(x))      # 身份不变
print(type(x))    # 类型不变
print(x)          # 值变了：[1, 2, 3, 4]
```

### 2.6.2 `is` vs `==`：身份比较 vs 值比较

这是一对经常被混淆的运算符：

```python
a = [1, 2, 3]
b = [1, 2, 3]

# == 比较的是"值"是否相等
print(a == b)     # True ← 两个列表的内容一样

# is 比较的是"身份"是否相同（是否是同一个对象）
print(a is b)     # False ← 虽然内容一样，但它们是两个不同的对象

# 赋值创建别名：同一个对象
c = a
print(a is c)     # True ← c 和 a 指向同一个对象
print(a == c)     # True ← 当然值也相等
```

> 🔑 **核心区别**：`is` 关注"是不是同一个对象"，`==` 关注"两个对象按相等规则是否相等"。在多数常见内置类型里，同一对象通常也会 `==` 成立；但像 `float("nan")` 这样的特殊值，或自定义了特殊 `__eq__` 的类型，`a is b` 为 True 也不必然推出 `a == b` 为 True。

> 💡 **实用建议**：日常编程中，绝大多数时候你应该用 `==` 比较值。`is` 主要用于与 `None` 比较：`if x is None:`。

## 2.7 本章核心概念速查表

|  概念  |               定义               |             影响              |
| :----: | :------------------------------: | :---------------------------: |
|  可变  |    创建后可修改内容，id 不变     |  别名共享引用、不能做字典键   |
| 不可变 | 创建后不能修改，"修改"创建新对象 |   安全、可哈希、能做字典键    |
|  有序  |       元素有确定的排列顺序       | 支持索引、切片、保持插入顺序  |
|  无序  |          元素无固定顺序          | 不支持索引、不能依赖遍历顺序  |
| 可重复 |         同一值可出现多次         |    适合需要记录频次的场景     |
|  唯一  |        每个值最多出现一次        | 适合去重、成员测试、集合运算  |
| 可哈希 |        对象有稳定的哈希值        | 能做字典键/集合元素的前提条件 |

## 2.8 本章小结

1. **可变性**是最重要的概念——它决定了对象能否被原地修改，影响别名行为、函数参数传递、以及能否用作字典键。
2. **有序性**决定了你能否通过索引/切片访问元素，以及遍历顺序是否稳定。
3. **唯一性**决定了容器是否自动去重，与哈希机制紧密相关。
4. 三组概念相互关联：可变性影响可哈希性，可哈希性影响能否成为集合/字典的元素/键。
5. 理解 `id()`、`is`、`==` 的区别，是掌握 Python 对象模型的关键。
