## 01 初识对象

### 1.1 什么是"面向对象"？

在正式学习语法之前，我们先用生活中的例子来理解两种编程思维。

#### 面向过程编程（Procedure-Oriented Programming）

你可以把它想象成**一份菜谱**：先做第一步、再做第二步、再做第三步……一步一步按顺序执行。

```python
# 面向过程：制作一杯奶茶
water = "开水"
tea = "茶包泡入" + water          # 第一步：泡茶
sugar = tea + " + 加糖"           # 第二步：加糖
milk = sugar + " + 加牛奶"        # 第三步：加奶
print("奶茶完成:", milk)           # 第四步：完成
```

这种方式的特点是：**关注的是"步骤"和"过程"**，代码像流水线一样一行一行往下走。

#### 面向对象编程（Object-Oriented Programming, OOP）

你可以把它想象成**现实世界的模拟**：世界上有各种各样的"事物"（对象），每个事物都有自己的**属性**（特征）和**行为**（能做什么）。

```
比如一只猫：
├── 属性（特征）：名字是"小橘"、颜色是"橘色"、体重5kg
└── 行为（动作）：吃饭、睡觉、抓老鼠
```

面向对象编程，就是用代码去**描述现实世界中的事物**，把数据（属性）和操作数据的方法（行为）**打包在一起**。

### 1.2 生活中处处是"对象"

| 现实事物 | 属性（特征）             | 行为（能做什么）       |
| -------- | ------------------------ | ---------------------- |
| 一个学生 | 姓名、年龄、学号、班级   | 学习、考试、交作业     |
| 一辆汽车 | 品牌、颜色、油量、速度   | 启动、加速、刹车、熄火 |
| 一部手机 | 品牌、价格、电量、内存   | 打电话、发短信、拍照   |
| 一个闹钟 | 品牌、当前时间、闹铃时间 | 响铃、关闭、设定时间   |

你会发现：**万物皆可为对象**。

### 1.3 "类"与"对象"的关系——模具与月饼

这是面向对象中 **最核心** 的两个概念：

- **类（Class）**：是一个"**模板**"或"**图纸**"，它描述了一类事物共有的属性和行为。
- **对象（Object）**：是根据类这个模板**创建出来的具体实例**。

> **经典比喻**：
> - **类** = 月饼模具（定义了月饼的形状、花纹、大小）
> - **对象** = 用模具压出来的一个个具体月饼（每一个都是独立的）

```
类：学生
├── 属性模板：姓名、年龄、学号
└── 行为模板：学习、考试

对象1：张三，20岁，学号001  ← 一个具体的学生
对象2：李四，21岁，学号002  ← 另一个具体的学生
对象3：王五，19岁，学号003  ← 又一个具体的学生
```

三个对象都是"学生类"创建出来的，但各自的数据不同。

### 1.4 用 Python 初步体验类和对象

先看一段最简单的代码，感受一下：

```python
# 第一步：定义一个类（设计图纸）
class Cat:
    # 类中定义了一个行为（方法）
    def say_hello(self):
        print("喵喵喵~ 你好呀！")

# 第二步：根据类创建对象（用图纸造东西）
cat1 = Cat()    # cat1 是一个Cat类的对象
cat2 = Cat()    # cat2 是另一个Cat类的对象

# 第三步：使用对象
cat1.say_hello()   # 输出：喵喵喵~ 你好呀！
cat2.say_hello()   # 输出：喵喵喵~ 你好呀！
```

**逐行解读**：

1. `class Cat:` —— 使用 `class` 关键字定义了一个名为 `Cat` 的类。注意：类名建议用**大驼峰命名法**（首字母大写，如 `Cat`、`Student`、`MobilePhone`）。
2. `def say_hello(self):` —— 在类内部定义了一个方法（函数）。注意参数 `self`，这个后面会详细讲。
3. `cat1 = Cat()` —— 类名后加 `()` 就是创建对象。这个过程叫做**实例化**。
4. `cat1.say_hello()` —— 用 `.` 来调用对象的方法。

### 1.5 类的基本语法结构

```python
class 类名:
    
    # 类的属性（变量）
    属性名 = 值

    # 类的方法（函数）
    def 方法名(self):
        方法体
```

**关键规则**：
- `class` 是定义类的关键字，后面跟类名和冒号
- 类名推荐使用**大驼峰命名**：`MyClass`、`StudentInfo`
- 类的内容（属性和方法）需要**缩进**
- 方法的第一个参数必须是 `self`（代表对象自身，后面详解）

### 1.6 为什么要学面向对象？

| 对比维度 | 面向过程         | 面向对象               |
| -------- | ---------------- | ---------------------- |
| 思维方式 | 关注步骤和流程   | 关注事物和交互         |
| 代码组织 | 函数为基本单位   | 类和对象为基本单位     |
| 复用性   | 通过函数复用     | 通过类的继承、组合复用 |
| 可维护性 | 项目大了难以维护 | 结构清晰，易于维护     |
| 适用场景 | 简单脚本、小工具 | 大型项目、复杂系统     |

> **小结**：面向对象不是取代面向过程，而是在更复杂的场景中提供更好的代码组织方式。Python 同时支持这两种编程方式。

## 02 类的成员方法

### 2.1 什么是成员方法？

在类中定义的函数，我们称之为**成员方法**（也叫实例方法）。它描述了这个类的对象**能做什么事情**——也就是对象的**行为**。

```python
class Student:
    # 这就是一个成员方法
    def study(self):
        print("学生正在学习...")
    
    # 这也是一个成员方法
    def do_homework(self):
        print("学生正在写作业...")
```

### 2.2 self 关键字——彻底搞懂它

在学习成员方法时，最让新手困惑的就是 `self`。我们来一步步揭开它的面纱。

#### self 是什么？

**`self` 代表的是调用该方法的对象本身。**

当你写 `cat1.say_hello()` 时，Python 其实在背后做了这件事：

```python
Cat.say_hello(cat1)   # Python 自动把 cat1 传给了 self
```

也就是说，`self` 就是 `cat1` 这个对象本身的引用。

#### 为什么需要 self？

因为一个类可以创建很多对象，方法需要知道**是谁在调用我**：

```python
class Cat:
    name = ""
    
    def introduce(self):
        # 通过 self 访问当前对象的属性
        print(f"我是一只猫，我叫{self.name}")

# 创建两个对象
cat1 = Cat()
cat1.name = "小橘"

cat2 = Cat()
cat2.name = "小黑"

cat1.introduce()   # 输出：我是一只猫，我叫小橘
cat2.introduce()   # 输出：我是一只猫，我叫小黑
```

当 `cat1.introduce()` 执行时，`self` 就是 `cat1`，所以 `self.name` 就是 `cat1.name`，即 `"小橘"`。
当 `cat2.introduce()` 执行时，`self` 就是 `cat2`，所以 `self.name` 就是 `cat2.name`，即 `"小黑"`。

#### self 的使用规则

```python
class MyClass:
    def my_method(self):       # ✅ 必须写 self 作为第一个参数
        pass
    
    def another(self, a, b):   # ✅ self 之后可以有其他参数
        pass
    
    def wrong():               # ❌ 不写 self 会报错（调用时）
        pass
```

> **重要规则**：
> 1. `self` 必须是成员方法的**第一个参数**
> 2. `self` 这个名字不是强制的（你可以叫 `this`、`me`），但 **Python 社区统一约定用 `self`**，请务必遵守
> 3. 调用方法时**不需要手动传 self**，Python 会自动传入

### 2.3 带参数的成员方法

成员方法和普通函数一样，可以接收参数：

```python
class Calculator:
    def add(self, a, b):
        result = a + b
        print(f"{a} + {b} = {result}")
    
    def multiply(self, a, b):
        result = a * b
        print(f"{a} × {b} = {result}")

# 创建对象
calc = Calculator()

# 调用方法，传入参数
calc.add(10, 20)        # 输出：10 + 20 = 30
calc.multiply(5, 6)     # 输出：5 × 6 = 30
```

注意：定义时写了 `self, a, b` 三个参数，但调用时只需要传 `a` 和 `b`（`self` 自动传入）。

### 2.4 带返回值的成员方法

```python
class Calculator:
    def add(self, a, b):
        return a + b
    
    def is_even(self, number):
        """判断一个数是否为偶数"""
        return number % 2 == 0

calc = Calculator()

result = calc.add(100, 200)
print(f"计算结果: {result}")      # 输出：计算结果: 300

print(calc.is_even(10))           # 输出：True
print(calc.is_even(7))            # 输出：False
```

### 2.5 在方法中调用其他方法

在一个方法内部，可以通过 `self` 来调用同一个类中的其他方法：

```python
class Student:
    name = ""
    
    def study(self):
        print(f"{self.name}正在学习...")
    
    def do_homework(self):
        print(f"{self.name}正在写作业...")
    
    def daily_routine(self):
        """每日日程：先学习，再写作业"""
        self.study()            # 通过 self 调用本类其他方法
        self.do_homework()      # 通过 self 调用本类其他方法
        print("今天的任务完成了！")

stu = Student()
stu.name = "张三"
stu.daily_routine()
```

输出：
```
张三正在学习...
张三正在写作业...
今天的任务完成了！
```

## 03 类和对象

### 3.1 回顾：类与对象的关系

前面我们已经初步认识了类和对象，现在来系统地理清它们的关系。

```
类（Class）          对象（Object / Instance）
─────────           ───────────────────────
抽象的模板            具体的实例
图纸                 根据图纸建造的房子
cookie 模具          用模具做出来的一块块 cookie
数据类型             该类型的一个具体值
```

**一个类可以创建无数个对象，每个对象都是独立的。**

### 3.2 创建类和对象的完整流程

```python
# ============ 第一步：设计类 ============
class Dog:
    """这是一个狗类"""
    # 属性（描述特征）
    name = ""
    age = 0
    breed = ""     # 品种
    
    # 方法（描述行为）
    def bark(self):
        print(f"{self.name}在叫: 汪汪汪！")
    
    def eat(self, food):
        print(f"{self.name}正在吃{food}")
    
    def show_info(self):
        print(f"名字: {self.name}, 年龄: {self.age}岁, 品种: {self.breed}")


# ============ 第二步：创建对象（实例化） ============
dog1 = Dog()     # 创建第一个对象
dog2 = Dog()     # 创建第二个对象
dog3 = Dog()     # 创建第三个对象


# ============ 第三步：给对象赋予属性值 ============
dog1.name = "旺财"
dog1.age = 3
dog1.breed = "中华田园犬"

dog2.name = "Lucky"
dog2.age = 2
dog2.breed = "金毛"

dog3.name = "豆豆"
dog3.age = 1
dog3.breed = "泰迪"


# ============ 第四步：使用对象 ============
dog1.show_info()      # 名字: 旺财, 年龄: 3岁, 品种: 中华田园犬
dog2.show_info()      # 名字: Lucky, 年龄: 2岁, 品种: 金毛
dog3.bark()           # 豆豆在叫: 汪汪汪！
dog2.eat("骨头")      # Lucky正在吃骨头
```

### 3.3 每个对象都是独立的

这是一个非常重要的概念：**不同对象之间的属性值互不影响**。

```python
class Student:
    name = ""
    score = 0

stu1 = Student()
stu2 = Student()

stu1.name = "张三"
stu1.score = 90

stu2.name = "李四"
stu2.score = 85

# 修改 stu1 的成绩，不会影响 stu2
stu1.score = 95

print(stu1.score)   # 95
print(stu2.score)   # 85  ← 没有受到影响
```

你可以这样理解：每个对象在内存中都有自己**独立的一块空间**，存放自己的属性数据。

```
内存示意图：

stu1 ──→ [ name="张三", score=95 ]    # 独立的内存空间
stu2 ──→ [ name="李四", score=85 ]    # 独立的内存空间
```

### 3.4 用 type() 和 isinstance() 查看对象类型

```python
class Cat:
    pass

class Dog:
    pass

cat = Cat()
dog = Dog()

# type() 查看类型
print(type(cat))    # <class '__main__.Cat'>
print(type(dog))    # <class '__main__.Dog'>

# isinstance() 判断是否是某个类的实例
print(isinstance(cat, Cat))   # True
print(isinstance(cat, Dog))   # False
print(isinstance(dog, Dog))   # True
```

### 3.5 理解：Python 中"万物皆对象"

事实上，你之前学过的所有数据类型，在 Python 中都是**类**，你创建的每一个变量，都是**对象**：

```python
a = 10
b = "hello"
c = [1, 2, 3]
d = {"name": "张三"}

print(type(a))   # <class 'int'>      ← int 是一个类
print(type(b))   # <class 'str'>      ← str 是一个类
print(type(c))   # <class 'list'>     ← list 是一个类
print(type(d))   # <class 'dict'>     ← dict 是一个类
```

你曾经用过的 `"hello".upper()`、`[1,2,3].append(4)` 等操作，其实就是在**调用对象的成员方法**！

```python
my_list = [1, 2, 3]
my_list.append(4)       # append 就是 list 类的成员方法
print(my_list)          # [1, 2, 3, 4]

my_str = "hello"
result = my_str.upper() # upper 就是 str 类的成员方法
print(result)           # HELLO
```

> 所以你其实一直在使用面向对象，只是之前不知道而已！

### 3.6 类属性 vs 对象属性（补充知识）

这是一个稍微进阶的知识点，但理解它能帮你避免很多坑。

```python
class Student:
    # 这是"类属性"：属于类本身，所有对象共享
    school = "Python大学"
    name = ""

stu1 = Student()
stu2 = Student()

# 类属性被所有对象共享
print(stu1.school)    # Python大学
print(stu2.school)    # Python大学

# 通过类名修改类属性，所有对象都受影响
Student.school = "AI大学"
print(stu1.school)    # AI大学
print(stu2.school)    # AI大学

# 但是！如果通过对象修改，会创建一个"对象自己的属性"
stu1.school = "数据大学"    # 这不是修改类属性，是给stu1创建了独立属性
print(stu1.school)    # 数据大学（stu1自己的）
print(stu2.school)    # AI大学（仍然是类属性）
```

> **新手建议**：目前阶段，不用太纠结类属性和对象属性的区别。学完构造方法后，我们会用更规范的方式来定义对象属性。

## 04 构造方法

### 4.1 问题引入：目前的写法有什么不方便？

回顾之前创建对象的方式：

```python
class Student:
    name = ""
    age = 0
    score = 0

# 每次创建对象后，都要手动一个一个赋值
stu1 = Student()
stu1.name = "张三"      # 手动赋值
stu1.age = 20           # 手动赋值
stu1.score = 90         # 手动赋值

stu2 = Student()
stu2.name = "李四"      # 手动赋值
stu2.age = 21           # 手动赋值
stu2.score = 85         # 手动赋值
```

**痛点**：每创建一个对象就要写好几行赋值代码，既麻烦又容易遗漏。

**有没有办法在创建对象的时候就直接把属性传进去呢？** 有！就是**构造方法**。

### 4.2 什么是构造方法？

构造方法是一个特殊的成员方法，名字固定为 `__init__`（前后各两个下划线）。

**它的特殊之处在于**：
1. 在创建对象时会被 **自动调用**（不需要你手动调用）
2. 用于给对象的属性赋初始值

```python
class Student:
    def __init__(self):
        print("构造方法被自动调用了！")

# 创建对象时，__init__ 自动执行
stu = Student()    # 输出：构造方法被自动调用了！
```

你没有写 `stu.__init__()`，但它自动执行了——这就是构造方法的特性。

### 4.3 用构造方法初始化属性

```python
class Student:
    def __init__(self, name, age, score):
        # 通过 self.属性名 = 值，给对象创建属性并赋值
        self.name = name
        self.age = age
        self.score = score
    
    def show_info(self):
        print(f"姓名: {self.name}, 年龄: {self.age}, 成绩: {self.score}")

# 创建对象时直接传入属性值 —— 简洁多了！
stu1 = Student("张三", 20, 90)
stu2 = Student("李四", 21, 85)
stu3 = Student("王五", 19, 92)

stu1.show_info()   # 姓名: 张三, 年龄: 20, 成绩: 90
stu2.show_info()   # 姓名: 李四, 年龄: 21, 成绩: 85
stu3.show_info()   # 姓名: 王五, 年龄: 19, 成绩: 92
```

**执行流程详解**：

当你写 `stu1 = Student("张三", 20, 90)` 时，Python 做了以下事情：

```
第1步：在内存中创建一个新的 Student 对象
第2步：自动调用 __init__ 方法，并把参数传入：
       self = 新创建的对象（也就是即将赋值给 stu1 的那个对象）
       name = "张三"
       age = 20
       score = 90
第3步：执行 __init__ 方法体：
       self.name = "张三"   → 给对象添加 name 属性
       self.age = 20        → 给对象添加 age 属性
       self.score = 90      → 给对象添加 score 属性
第4步：把这个创建好的对象赋值给变量 stu1
```

### 4.4 self.属性 的理解

`self.name = name` 这行代码中：
- 等号左边的 `self.name` 是**对象的属性**（给对象身上"贴"一个叫 name 的标签）
- 等号右边的 `name` 是**传入的参数值**

它们名字可以不一样（但通常保持一致，这是编程习惯）：

```python
class Student:
    def __init__(self, n, a, s):
        self.name = n        # 参数叫 n，属性叫 name，也可以
        self.age = a
        self.score = s
```

但推荐参数名和属性名保持一致，代码更清晰：

```python
class Student:
    def __init__(self, name, age, score):
        self.name = name     # ✅ 推荐：参数名和属性名一致
        self.age = age
        self.score = score
```

### 4.5 构造方法中可以设置默认值

和普通函数一样，构造方法的参数也可以有默认值：

```python
class Student:
    def __init__(self, name, age=18, score=0):
        self.name = name
        self.age = age
        self.score = score
    
    def show_info(self):
        print(f"姓名: {self.name}, 年龄: {self.age}, 成绩: {self.score}")

# 只传必填参数
stu1 = Student("张三")
stu1.show_info()      # 姓名: 张三, 年龄: 18, 成绩: 0

# 传入所有参数

# 使用关键字参数
stu3 = Student("王五", score=88)
stu3.show_info()      # 姓名: 王五, 年龄: 18, 成绩: 88
```

## 05 魔术方法

### 5.1 什么是魔术方法？

在 Python 中，有一类特殊的方法，它们的名字前后各有**两个下划线**，如 `__init__`、`__str__`、`__lt__` 等。这些方法被称为**魔术方法**（Magic Methods），也叫 **双下方法**（Dunder Methods，dunder = double underscore）。

**它们的特点**：
- 名字格式固定：`__名字__`
- 不需要你手动调用，在**特定场景下会自动触发**
- 可以让你的类具有像内置类型一样的行为

你已经学过了第一个魔术方法 `__init__`（创建对象时自动调用），下面我们来学习更多。

### 5.2 `__str__` 方法——定义对象的字符串表示

#### 问题引入

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

stu = Student("张三", 20)
print(stu)    # 输出：<__main__.Student object at 0x000001A2B3C4D5E6>
```

直接 `print` 一个对象，输出的是一串看不懂的内存地址信息。我们希望输出有意义的内容。

#### 解决方案：定义 `__str__`

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __str__(self):
        return f"Student(name={self.name}, age={self.age})"

stu = Student("张三", 20)
print(stu)          # 输出：Student(name=张三, age=20)   ← 清晰多了！
print(str(stu))     # 输出：Student(name=张三, age=20)
```

**触发时机**：当你对对象使用 `print()`、`str()` 或 f-string 时，Python 会自动调用 `__str__` 方法。

```python
# 以下三种方式都会触发 __str__
print(stu)                        # 触发 __str__
s = str(stu)                      # 触发 __str__
msg = f"学生信息: {stu}"           # 触发 __str__
print(msg)   # 输出：学生信息: Student(name=张三, age=20)
```

> **注意**：`__str__` 方法必须 **返回一个字符串（return）**，不是 print。

### 5.3 `__lt__` 方法——定义小于比较

`lt` 是 less than（小于）的缩写。

#### 问题引入

```python
stu1 = Student("张三", 20)
stu2 = Student("李四", 21)
print(stu1 < stu2)    # ❌ TypeError: '<' not supported
```

两个自定义对象不能直接用 `<` 比较。但定义了 `__lt__` 之后就可以了。

#### 解决方案：定义 `__lt__`

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __str__(self):
        return f"Student(name={self.name}, age={self.age})"
    
    def __lt__(self, other):
        """定义 < 和 > 的行为，按年龄比较"""
        return self.age < other.age

stu1 = Student("张三", 20)
stu2 = Student("李四", 21)

print(stu1 < stu2)    # True  （20 < 21）
print(stu1 > stu2)    # False （自动反转：相当于 stu2 < stu1，即 21 < 20 = False）
```

**参数说明**：
- `self`：`<` 左边的对象
- `other`：`<` 右边的对象

> **小知识**：定义了 `__lt__` 之后，`>` 也能自动使用（Python 会自动反转比较）。

### 5.4 `__le__` 方法——定义小于等于比较

`le` 是 less than or equal（小于等于）的缩写。

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __le__(self, other):
        """定义 <= 和 >= 的行为"""
        return self.age <= other.age

stu1 = Student("张三", 20)
stu2 = Student("李四", 20)
stu3 = Student("王五", 21)

print(stu1 <= stu2)   # True   (20 <= 20)
print(stu1 >= stu3)   # False  (20 >= 21 → False)
print(stu3 >= stu1)   # True   (21 >= 20 → True)
```

### 5.5 `__eq__` 方法——定义相等比较

`eq` 是 equal（等于）的缩写。

#### 问题引入

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

stu1 = Student("张三", 20)
stu2 = Student("张三", 20)

print(stu1 == stu2)    # False ❗ 
```

虽然两个学生的属性完全相同，但 `==` 默认比较的是**内存地址**（是不是同一个对象），所以返回 `False`。

#### 解决方案：定义 `__eq__`

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __eq__(self, other):
        """当 name 和 age 都相同时，认为两个学生相等"""
        return self.name == other.name and self.age == other.age

stu1 = Student("张三", 20)
stu2 = Student("张三", 20)
stu3 = Student("李四", 21)

print(stu1 == stu2)    # True  ✓（名字和年龄都相同）
print(stu1 == stu3)    # False ✓（不同）
```

### 5.6 魔术方法汇总表

| 魔术方法               | 触发时机                        | 作用                       |
| ---------------------- | ------------------------------- | -------------------------- |
| `__init__(self, ...)`  | 创建对象时自动调用              | 初始化对象属性             |
| `__str__(self)`        | `print(对象)` 或 `str(对象)` 时 | 返回对象的字符串描述       |
| `__lt__(self, other)`  | 使用 `<` 或 `>` 比较时          | 定义大小比较规则           |
| `__le__(self, other)`  | 使用 `<=` 或 `>=` 比较时        | 定义大小等于比较规则       |
| `__eq__(self, other)`  | 使用 `==` 比较时                | 定义相等判断规则           |
| `__repr__(self)`       | 交互式环境输出 / `repr(对象)`   | 返回对象的"官方"字符串表示 |
| `__add__(self, other)` | 使用 `+` 运算时                 | 定义加法行为               |
| `__len__(self)`        | 使用 `len(对象)` 时             | 定义长度                   |

> 目前只需掌握前5个，其余作为了解。

### 5.7 综合示例——完整的学生类

```python
class Student:
    """学生类 —— 演示多个魔术方法"""
    
    def __init__(self, name, age, score):
        self.name = name
        self.age = age
        self.score = score
    
    def __str__(self):
        return f"Student({self.name}, {self.age}岁, {self.score}分)"
    
    def __lt__(self, other):
        """按成绩比较大小"""
        return self.score < other.score
    
    def __le__(self, other):
        """按成绩比较（小于等于）"""
        return self.score <= other.score
    
    def __eq__(self, other):
        """姓名和成绩都相同则认为相等"""
        return self.name == other.name and self.score == other.score


# 创建学生对象
stu1 = Student("张三", 20, 90)
stu2 = Student("李四", 21, 85)
stu3 = Student("王五", 19, 95)
stu4 = Student("张三", 22, 90)   # 和 stu1 同名同分但不同龄

# __str__
print(stu1)              # Student(张三, 20岁, 90分)

# __lt__ 和 __gt__
print(stu1 > stu2)       # True  (90 > 85)
print(stu2 < stu3)       # True  (85 < 95)

# __le__
print(stu1 <= stu3)      # True  (90 <= 95)

# __eq__
print(stu1 == stu4)      # True  (同名同分)
print(stu1 == stu2)      # False

# 排序（因为定义了 __lt__，所以 sorted 可以直接用！）
students = [stu1, stu2, stu3]
sorted_students = sorted(students)   # 按成绩从低到高排序
for s in sorted_students:
    print(s)
```

输出：
```
Student(张三, 20岁, 90分)
True
True
True
True
False
Student(李四, 21岁, 85分)
Student(张三, 20岁, 90分)
Student(王五, 19岁, 95分)
```

> **注意最后一个亮点**：因为我们定义了 `__lt__`，所以 Python 内置的 `sorted()` 函数可以直接对我们的对象列表进行排序！

## 06 封装

### 6.1 面向对象三大特性

面向对象有三大核心特性：

| 特性     | 关键词        | 简单理解                                   |
| -------- | ------------- | ------------------------------------------ |
| **封装** | Encapsulation | 把数据和方法包在一起，对外隐藏细节         |
| **继承** | Inheritance   | 子类继承父类的属性和方法（后续章节讲）     |
| **多态** | Polymorphism  | 同一操作对不同对象有不同表现（后续章节讲） |

本节我们重点学习 **封装**。

### 6.2 什么是封装？

#### 生活中的封装

**ATM 取款机**就是一个很好的封装案例：

```
ATM取款机：
├── 对外暴露的操作（公开的）：
│   ├── 插卡
│   ├── 输入密码
│   ├── 选择取款金额
│   └── 取出现金
│
└── 内部隐藏的细节（私有的）：
    ├── 如何验证密码（加密算法）
    ├── 如何与银行服务器通信
    ├── 如何计算钞票张数
    └── 保险柜里的现金总量
```

你只需要知道"怎么操作"，而不需要知道"内部怎么运作"。这就是封装的思想：
- **对外提供简单的接口**
- **对内隐藏复杂的实现细节**

#### 编程中的封装

在编程中，封装指的是：
1. 将**属性**和**方法**包装到类中
2. 对外提供**公开的方法**（接口）
3. 将不想让外部直接访问的属性和方法**设为私有**

### 6.3 私有成员——用下划线实现

在 Python 中，通过在属性名或方法名前加**两个下划线** `__` 来标记为**私有**。

#### 私有属性

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner          # 公开属性：户主姓名
        self.__balance = balance    # 私有属性：余额（前面加 __）
    
    def show_balance(self):
        """公开方法：查询余额"""
        print(f"{self.owner}的余额为: {self.__balance}元")
    
    def deposit(self, amount):
        """公开方法：存款"""
        if amount > 0:
            self.__balance += amount
            print(f"存入{amount}元，当前余额: {self.__balance}元")
        else:
            print("存款金额必须大于0！")
    
    def withdraw(self, amount):
        """公开方法：取款"""
        if amount > self.__balance:
            print(f"余额不足！当前余额: {self.__balance}元")
        elif amount <= 0:
            print("取款金额必须大于0！")
        else:
            self.__balance -= amount
            print(f"取出{amount}元，当前余额: {self.__balance}元")


# 使用
acc = BankAccount("张三", 10000)

# ✅ 通过公开方法操作
acc.show_balance()         # 张三的余额为: 10000元
acc.deposit(5000)          # 存入5000元，当前余额: 15000元
acc.withdraw(3000)         # 取出3000元，当前余额: 12000元

# ❌ 直接访问私有属性会报错
# print(acc.__balance)     # AttributeError: 'BankAccount' object has no attribute '__balance'
```

**为什么要把余额设为私有？**

如果余额是公开的，任何人都可以直接修改：

```python
# 如果 balance 是公开的：
acc.balance = 99999999    # 随意修改余额？这太危险了！
acc.balance = -100        # 余额变成负数？不合理！
```

但如果余额是私有的，只能通过 `deposit()` 和 `withdraw()` 方法来操作，我们就可以**在方法中加入验证逻辑**，确保数据安全。

#### 私有方法

```python
class Phone:
    def __init__(self, brand):
        self.brand = brand
        self.__voltage = 0   # 私有属性：内部电压
    
    def call(self, number):
        """公开方法：打电话"""
        # 内部调用私有方法
        self.__check_network()
        print(f"正在拨打 {number}...")
    
    def __check_network(self):
        """私有方法：检查网络（内部使用，外部不可调用）"""
        print("正在检查网络信号...")
        print("网络正常 ✓")

phone = Phone("华为")

# ✅ 调用公开方法
phone.call("10086")
# 输出：
# 正在检查网络信号...
# 网络正常 ✓
# 正在拨打 10086...

# ❌ 调用私有方法会报错
# phone.__check_network()   # AttributeError
```

### 6.4 封装的完整规则总结

```python
class MyClass:
    def __init__(self):
        self.public_var = "我是公开属性"         # 公开属性：外部可以访问
        self.__private_var = "我是私有属性"       # 私有属性：外部不能直接访问
    
    def public_method(self):                     # 公开方法：外部可以调用
        print("我是公开方法")
        self.__private_method()                  # 公开方法内部可以调用私有方法
    
    def __private_method(self):                  # 私有方法：外部不能直接调用
        print("我是私有方法")
        print(self.__private_var)                # 内部可以访问私有属性

obj = MyClass()

# ✅ 公开成员
print(obj.public_var)        # 我是公开属性
obj.public_method()          # 我是公开方法 → 我是私有方法 → 我是私有属性

# ❌ 私有成员
# print(obj.__private_var)   # 报错
# obj.__private_method()     # 报错
```

**记忆口诀**：

```
公开成员：类内类外都能用
私有成员：只能在类内部使用（方法里通过 self 访问）
标记方式：名字前加两个下划线 __
```

### 6.5 为什么需要封装？——封装的好处

#### 好处一：保护数据安全

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.__age = age      # 私有：年龄不能随便改
    
    def get_age(self):
        """获取年龄"""
        return self.__age
    
    def set_age(self, new_age):
        """设置年龄（带验证）"""
        if 0 < new_age < 150:
            self.__age = new_age
            print(f"年龄已更新为: {self.__age}")
        else:
            print(f"年龄 {new_age} 不合法！请输入 1-149 之间的整数")

stu = Student("张三", 20)

# 通过方法安全地修改
stu.set_age(21)       # 年龄已更新为: 21
stu.set_age(-5)       # 年龄 -5 不合法！请输入 1-149 之间的整数
stu.set_age(200)      # 年龄 200 不合法！请输入 1-149 之间的整数

print(stu.get_age())  # 21
```

#### 好处二：简化使用复杂度

```python
class Car:
    def __init__(self, brand):
        self.brand = brand
        self.__engine_started = False
        self.__fuel_injected = False
        self.__ignition_done = False
    
    def start(self):
        """一键启动（对外简单的接口）"""
        print(f"【{self.brand}】一键启动中...")
        self.__check_fuel()       # 内部步骤1
        self.__inject_fuel()      # 内部步骤2
        self.__ignite()           # 内部步骤3
        self.__engine_started = True
        print("启动成功！可以出发了 🚗")
    
    def __check_fuel(self):
        """私有方法：检查油量"""
        print("  ├── 检查油量... OK")
    
    def __inject_fuel(self):
        """私有方法：喷油"""
        print("  ├── 喷油... OK")
        self.__fuel_injected = True
    
    def __ignite(self):
        """私有方法：点火"""
        print("  ├── 点火... OK")
        self.__ignition_done = True


car = Car("宝马")
car.start()
# 用户只需要调用 start()，不需要关心内部的 check_fuel、inject_fuel、ignite
```

输出：
```
【宝马】一键启动中...
  ├── 检查油量... OK
  ├── 喷油... OK
  ├── 点火... OK
启动成功！可以出发了 🚗
```

### 6.6 Python 的"名义上"的私有（补充知识）

严格来说，Python 的私有机制并不是"真正的"私有，而是一种**命名改写**（Name Mangling）。Python 会把 `__属性名` 自动改写成 `_类名__属性名`：

```python
class MyClass:
    def __init__(self):
        self.__secret = "秘密"

obj = MyClass()

# 直接访问会报错
# print(obj.__secret)         # AttributeError

# 但通过改写后的名字可以强行访问（不推荐！）
print(obj._MyClass__secret)   # 秘密 ← 能访问，但千万别这么做！
```

> **重要提醒**：虽然技术上可以这样访问，但这违背了封装的设计初衷。在实际开发中，**绝对不要这样做**。这就像虽然你能撬开别人家的锁，但你不应该这样做。

### 6.7 单下划线 `_` 的约定（补充知识）

在 Python 社区中，还有一种用**单下划线**开头的命名约定：

```python
class MyClass:
    def __init__(self):
        self.name = "公开属性"          # 完全公开
        self._hint = "受保护属性"        # 单下划线：约定"请别在外部使用"（但技术上可以）
        self.__secret = "私有属性"       # 双下划线：真正的私有
```

| 命名方式 | 含义                   | 外部能否访问               |
| -------- | ---------------------- | -------------------------- |
| `name`   | 公开                   | ✅ 可以                     |
| `_name`  | 约定受保护（"请别用"） | ✅ 技术上可以，但不推荐     |
| `__name` | 私有                   | ❌ 不可以（会触发名称改写） |

class StudentScore:
    """学生成绩管理类"""
    
    def __init__(self, name):
        self.name = name
        self.__scores = []       # 私有属性：成绩列表
    
    def add_score(self, score):
        """添加成绩（带验证）"""
        if not isinstance(score, (int, float)):
            print(f"错误：成绩必须是数字，收到了 {type(score).__name__} 类型")
            return
        if score < 0 or score > 100:
            print(f"错误：成绩 {score} 不合法，必须在 0~100 之间")
            return
        self.__scores.append(score)
        print(f"✓ 已添加成绩: {score}")
    
    def get_average(self):
        """计算平均成绩"""
        if len(self.__scores) == 0:
            return 0
        return sum(self.__scores) / len(self.__scores)
    
    def get_highest(self):
        """获取最高成绩"""
        if len(self.__scores) == 0:
            return 0
        return max(self.__scores)
    
    def show_all(self):
        """显示完整的成绩报告"""
        print(f"====== {self.name} 的成绩报告 ======")
        if len(self.__scores) == 0:
            print("  暂无成绩")

            for i, score in enumerate(self.__scores, 1):
                print(f"  第{i}次成绩: {score}分")
            print(f"  ─────────────────────")
            print(f"  最高分: {self.get_highest()}分")
            print(f"  平均分: {self.get_average():.1f}分")
            print(f"  共{len(self.__scores)}次考试")
        print(f"{'=' * 32}")
    
    def __str__(self):
        avg = self.get_average()
        count = len(self.__scores)
        return f"StudentScore({self.name}, {count}次考试, 平均分:{avg:.1f})"
    
    def __lt__(self, other):
        """按平均分比较"""
        return self.get_average() < other.get_average()

### 练习题中的封装体现

让我们回顾一下这个练习中封装的体现：

| 封装体现 | 代码位置             | 说明                                                   |
| -------- | -------------------- | ------------------------------------------------------ |
| 私有属性 | `self.__scores = []` | 成绩列表不允许外部直接访问和篡改                       |
| 数据验证 | `add_score()` 方法   | 通过公开方法间接操作私有数据，添加了类型检查和范围检查 |
| 接口简洁 | `get_average()`      | 外部只需要调用方法就能获取结果，不需要知道内部如何计算 |
| 信息隐藏 | `show_all()`         | 对外提供完整的格式化输出，隐藏了内部数据的存储方式     |
