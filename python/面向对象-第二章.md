## 08 继承的基础语法

### 一、为什么需要继承？

在实际开发中，我们经常会遇到这样的场景：多个类之间存在**共同的属性和方法**。例如：

```python
# 没有继承的写法：大量重复代码
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def eat(self):
        print(f"{self.name}正在吃东西")

    def sleep(self):
        print(f"{self.name}正在睡觉")

    def bark(self):
        print(f"{self.name}正在汪汪叫")


class Cat:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def eat(self):
        print(f"{self.name}正在吃东西")

    def sleep(self):
        print(f"{self.name}正在睡觉")

    def catch_mouse(self):
        print(f"{self.name}正在抓老鼠")
```

你会发现，`Dog` 和 `Cat` 中 `__init__`、`eat`、`sleep` 几乎一模一样，只有 `bark` 和 `catch_mouse` 不同。如果再加上 `Bird`、`Fish`……代码会越来越臃肿。

> **继承**就是为了解决这类问题：把**公共的部分**抽取到一个**父类（基类）**中，让其他**子类（派生类）**去继承它，从而实现**代码复用**。

---

### 二、继承的基本概念

| 术语                     | 含义                                                    |
| ------------------------ | ------------------------------------------------------- |
| **父类（Parent Class）** | 也叫基类（Base Class）、超类（Super Class），被继承的类 |
| **子类（Child Class）**  | 也叫派生类（Derived Class），继承别人的类               |
| **继承（Inheritance）**  | 子类自动拥有父类的属性和方法的机制                      |

用一张关系图来理解：

```
        Animal（父类）
       /      \
    Dog        Cat（子类）
```

`Dog` 和 `Cat` 都是 `Animal`，它们共享 `Animal` 的属性和方法，同时还可以有自己独特的行为。

---

### 三、单继承的语法

#### 3.1 基本语法格式

```python
class 父类名:
    # 父类的属性和方法
    pass

class 子类名(父类名):
    # 子类的属性和方法
    pass
```

**关键点：子类在定义时，在类名后面的小括号 `()` 中写上父类的名字。**

#### 3.2 第一个继承案例

```python
# 第一步：定义父类
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def eat(self):
        print(f"{self.name}正在吃东西")

    def sleep(self):
        print(f"{self.name}正在睡觉")


# 第二步：定义子类，继承父类
class Dog(Animal):
    def bark(self):
        print(f"{self.name}正在汪汪叫")


class Cat(Animal):
    def catch_mouse(self):
        print(f"{self.name}正在抓老鼠")


# 第三步：使用子类创建对象
dog = Dog("旺财", 3)
print(f"狗的名字：{dog.name}，年龄：{dog.age}")
dog.eat()       # 继承自父类的方法
dog.sleep()     # 继承自父类的方法
dog.bark()      # 子类自己的方法

print("=" * 40)

cat = Cat("咪咪", 2)
print(f"猫的名字：{cat.name}，年龄：{cat.age}")
cat.eat()         # 继承自父类的方法
cat.sleep()       # 继承自父类的方法
cat.catch_mouse() # 子类自己的方法
```

**运行结果：**

```
狗的名字：旺财，年龄：3
旺财正在吃东西
旺财正在睡觉
旺财正在汪汪叫
========================================
猫的名字：咪咪，年龄：2
咪咪正在吃东西
咪咪正在睡觉
咪咪正在抓老鼠
```

> **核心理解：** 子类 `Dog` 没有定义 `__init__`、`eat`、`sleep`，但是通过继承 `Animal`，它自动拥有了这些属性和方法。这就是继承的威力——**子类天生拥有父类的一切（公开的属性和方法）**。

#### 3.3 子类无法访问父类的私有成员

回顾一下：以双下划线 `__` 开头的成员会触发**名称改写（name mangling）**，通常把它当作"私有成员"来使用。

```python
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age
        self.__health = 100  # 私有属性

    def eat(self):
        print(f"{self.name}正在吃东西")

    def __secret(self):      # 私有方法
        print("这是Animal的秘密方法")


class Dog(Animal):
    def try_access(self):
        # print(self.__health)   # ❌ 报错！无法访问父类的私有属性
        # self.__secret()        # ❌ 报错！无法访问父类的私有方法
        print(self.name)         # ✅ 可以访问父类的公开属性


dog = Dog("旺财", 3)
dog.try_access()
# dog.__health     # ❌ 报错
# dog.__secret()   # ❌ 报错
```

> **结论：** 父类中以 `__` 开头的成员不会被子类以原名字直接访问。通常把它当作类内部使用的成员，而不是给子类直接调用的接口。

### 四、多继承的语法

Python 支持**多继承**——一个子类可以同时继承**多个父类**。

#### 4.1 基本语法格式

```python
class 子类名(父类1, 父类2, 父类3, ...):
    # 子类的属性和方法
    pass
```

#### 4.2 多继承案例

```python
class Animal:
    def eat(self):
        print("动物在吃东西")


class Flyable:
    def fly(self):
        print("可以在天空飞翔")


class Swimmable:
    def swim(self):
        print("可以在水中游泳")


# Duck 同时继承三个父类
class Duck(Animal, Flyable, Swimmable):
    def quack(self):
        print("嘎嘎嘎~")


duck = Duck()
duck.eat()    # 来自 Animal
duck.fly()    # 来自 Flyable
duck.swim()   # 来自 Swimmable
duck.quack()  # Duck 自己的方法
```

**运行结果：**

```
动物在吃东西
可以在天空飞翔
可以在水中游泳
嘎嘎嘎~
```

#### 4.3 多继承中的同名方法——谁优先？（MRO）

当多个父类中有**同名方法**时，这类简单例子里会表现为按继承列表从左到右优先匹配；更准确地说，Python 实际按 MRO 来决定调用哪个：

```python
class A:
    def show(self):
        print("我是 A 的 show 方法")

class B:
    def show(self):
        print("我是 B 的 show 方法")

# 注意继承顺序：先 A 后 B
class C(A, B):
    pass

c = C()
c.show()  # 输出：我是 A 的 show 方法
```

```python
# 调换继承顺序：先 B 后 A
class D(B, A):
    pass

d = D()
d.show()  # 输出：我是 B 的 show 方法
```

> **规则：** 多个父类中有同名成员时，Python 会按 **MRO（方法解析顺序）** 查找。对这种简单示例来说，确实表现为**继承列表中靠前（左边）的父类优先**。
>
> 你可以通过 `类名.__mro__` 或 `类名.mro()` 查看完整的方法解析顺序（Method Resolution Order）。

```python
print(C.__mro__)
# (<class 'C'>, <class 'A'>, <class 'B'>, <class 'object'>)

print(D.__mro__)
# (<class 'D'>, <class 'B'>, <class 'A'>, <class 'object'>)
```

> 查找规则：Python 按 MRO 依次查找；在当前这个例子里，顺序正好体现为从左到右。

### 五、关于 `object` 类——所有类的祖宗

在 Python 3 中，**所有类都默认继承自 `object` 类**，即使你没有显式写出来。

```python
# 以下两种写法完全等价
class MyClass:
    pass

class MyClass(object):
    pass
```

`object` 是 Python 中最顶级的父类，所有类都是它的子类。你可以验证：

```python
print(isinstance(42, object))        # True
print(isinstance("hello", object))   # True
print(isinstance([1, 2], object))    # True

class Dog:
    pass

print(issubclass(Dog, object))       # True
```

### 六、继承的完整案例：员工管理

```python
# 父类：Employee（员工）
class Employee:
    def __init__(self, name, employee_id, salary):
        self.name = name
        self.employee_id = employee_id
        self.salary = salary

    def show_info(self):
        print(f"员工姓名：{self.name}")
        print(f"员工工号：{self.employee_id}")
        print(f"员工薪资：{self.salary}")

    def work(self):
        print(f"{self.name}正在工作...")


# 子类1：Programmer（程序员）
class Programmer(Employee):
    def write_code(self):
        print(f"程序员{self.name}正在写代码")


# 子类2：Designer（设计师）
class Designer(Employee):
    def design(self):
        print(f"设计师{self.name}正在做设计")


# 子类3：Manager（经理）
class Manager(Employee):

# 创建对象并使用
p = Programmer("张三", "P001", 15000)

p.work()          # 继承自 Employee
p.write_code()    # Programmer 独有
print()

d = Designer("李四", "D001", 12000)
d.show_info()
d.design()
print()

m = Manager("王五", "M001", 20000)
m.show_info()
m.manage()
```

## 09 复写父类成员和调用父类成员

### 一、什么是复写（Override）？

继承后，子类自动拥有父类的属性和方法。但有时候，父类的方法**不完全适用于子类**，子类需要**改写**它。

> **复写（重写/覆盖/Override）：** 子类中定义了一个**与父类同名**的属性或方法，子类的版本会**覆盖**父类的版本。

生活中的例子：
- 父亲开了一家餐馆，菜单上有"红烧肉"。
- 儿子继承了餐馆，但他觉得父亲的红烧肉做法不够好，于是用自己的新配方重新做——**这就是复写**。
- 顾客点"红烧肉"，吃到的是儿子的版本，而不是父亲的版本。

---

### 二、复写父类方法

#### 2.1 基本语法

只需要在子类中**定义与父类同名的方法**即可：

```python
class 父类:
    def 方法名(self):
        # 父类的实现
        pass

class 子类(父类):
    def 方法名(self):    # 同名方法，复写！
        # 子类的新实现
        pass
```

#### 2.2 案例演示

```python
class Animal:
    def speak(self):
        print("动物发出声音")

    def show_type(self):
        print("我是一个动物")


class Dog(Animal):
    # 复写父类的 speak 方法
    def speak(self):
        print("汪汪汪！🐕")

    # 复写父类的 show_type 方法
    def show_type(self):
        print("我是一只狗")


class Cat(Animal):
    # 复写父类的 speak 方法
    def speak(self):
        print("喵喵喵！🐈")

    # 复写父类的 show_type 方法
    def show_type(self):
        print("我是一只猫")


# 创建对象
dog = Dog()
dog.speak()       # 输出：汪汪汪！🐕      （调用的是Dog自己的版本）
dog.show_type()   # 输出：我是一只狗

cat = Cat()
cat.speak()       # 输出：喵喵喵！🐈       （调用的是Cat自己的版本）
cat.show_type()   # 输出：我是一只猫

# 父类本身不受影响
animal = Animal()
animal.speak()    # 输出：动物发出声音     （父类的版本依然存在）
```

> **重点理解：** 复写不会修改父类本身，只是子类的对象在调用该方法时，会用子类的版本"遮盖"父类的版本。

---

### 三、复写父类属性

属性也可以复写，道理相同：

```python
class Phone:
    producer = "APPLE"           # 类属性（厂商）

    def __init__(self):
        self.price = 9999        # 实例属性（价格）


class MyPhone(Phone):
    producer = "HUAWEI"          # 复写类属性

    def __init__(self):
        super().__init__()       # 先调用父类的__init__（下面会详细讲）
        self.price = 6999        # 复写实例属性


phone = MyPhone()
print(phone.producer)  # 输出：HUAWEI（子类复写后的值）
print(phone.price)     # 输出：6999（子类复写后的值）
```

---

### 四、调用父类成员（核心重点）

复写是"完全替换"了父类的方法。但很多时候，我们并不想完全抛弃父类的实现，而是想**在父类实现的基础上，增加新功能**。

> 这就需要在子类中**调用父类的成员**。

Python 提供了 **两种方式** 来调用父类的成员。

#### 方式一：使用 `父类名.方法名(self, ...)` 直接调用

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name}发出声音")


class Dog(Animal):
    def __init__(self, name, breed):
        # 方式一：通过父类名调用父类的 __init__
        Animal.__init__(self, name)     # 注意：必须手动传入 self
        self.breed = breed              # 子类新增的属性

    def speak(self):
        # 方式一：通过父类名调用父类的 speak
        Animal.speak(self)              # 注意：必须手动传入 self
        print(f"{self.name}：汪汪汪！（品种：{self.breed}）")


dog = Dog("旺财", "金毛")
dog.speak()
```

**运行结果：**

```
旺财发出声音
旺财：汪汪汪！（品种：金毛）
```

#### 方式二：使用 `super().方法名(...)` 调用（推荐 ✅）

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name}发出声音")


class Dog(Animal):
    def __init__(self, name, breed):
        # 方式二：通过 super() 调用父类的 __init__
        super().__init__(name)          # 不需要传 self，更简洁
        self.breed = breed

    def speak(self):
        # 方式二：通过 super() 调用父类的 speak
        super().speak()                 # 不需要传 self
        print(f"{self.name}：汪汪汪！（品种：{self.breed}）")


dog = Dog("旺财", "金毛")
dog.speak()
```

**运行结果与方式一完全相同。**

#### 两种方式的对比

| 对比项            | `父类名.方法(self, ...)` | `super().方法(...)` |
| ----------------- | ------------------------ | ------------------- |
| 是否需要传 `self` | ✅ 需要手动传             | ❌ 不需要            |
| 父类名变化时      | 需要逐个修改             | 自动适应            |
| 多继承兼容性      | 可能有问题               | 自动按 MRO 查找     |
| **推荐程度**      | 一般                     | **强烈推荐** ✅      |

### 五、综合案例：手机升级

```python
class Phone:
    """基础手机"""
    def __init__(self, brand, price):
        self.brand = brand
        self.price = price

    def call(self):
        print(f"使用{self.brand}手机打电话")

    def show_info(self):
        print(f"品牌：{self.brand}")
        print(f"价格：{self.price}元")


class SmartPhone(Phone):
    """智能手机，在基础手机上升级"""
    def __init__(self, brand, price, os):
        super().__init__(brand, price)   # 调用父类初始化
        self.os = os                     # 新增属性：操作系统

    def call(self):
        """复写：增加视频通话功能"""
        super().call()                   # 先执行父类的打电话
        print("同时支持视频通话！📹")     # 新增功能

    def show_info(self):
        """复写：展示更多信息"""
        super().show_info()              # 先展示父类的基本信息
        print(f"操作系统：{self.os}")      # 新增展示

    def install_app(self):
        """子类独有的方法"""
        print(f"在{self.brand}上安装应用程序")


# 使用
sp = SmartPhone("华为", 5999, "HarmonyOS")
sp.show_info()
print("-" * 30)
sp.call()
print("-" * 30)
sp.install_app()
```

**运行结果：**

```
品牌：华为
价格：5999元
操作系统：HarmonyOS
------------------------------
使用华为手机打电话
同时支持视频通话！📹
------------------------------
在华为上安装应用程序
```

### 六、本节小结

| 知识点           | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| 复写（Override） | 子类中定义与父类同名的属性/方法，覆盖父类的版本              |
| 调用父类方式一   | `父类名.方法名(self, 参数)`                                  |
| 调用父类方式二   | `super().方法名(参数)` **（推荐）**                          |
| 典型使用场景     | 在 `__init__` 中先调用 `super().__init__(...)` 完成父类初始化，再添加子类新属性 |
| 复写 + 调用父类  | 先 `super().方法()` 保留父类功能，再添加子类新功能           |

## 10 变量的类型注解

### 一、为什么需要类型注解？

Python 是一门**动态类型语言**——变量不需要声明类型，随时可以赋值为任何类型：

```python
x = 10          # x 是 int
x = "hello"     # x 变成了 str
x = [1, 2, 3]   # x 变成了 list
```

这种灵活性很方便，但也带来了问题：

**问题 1：阅读代码时不清楚变量是什么类型**

```python
def process(data):
    # data 是什么？字符串？列表？字典？完全不知道！
    result = data.split(",")  # 猜它是字符串？
    return result
```

**问题 2：IDE（如 PyCharm）无法提供有效的代码提示**

```python
def handle(info):
    info.  # IDE 不知道 info 是什么类型，无法弹出可用方法的提示
```

> **类型注解（Type Hint）** 就是为了解决这些问题：在代码中**标注**变量的预期类型，**帮助开发者阅读代码，帮助 IDE 提供智能提示**。

**重要声明：**
> Python 的类型注解**只是提示性的**，不是强制性的。即使你注解了 `x: int = 10`，然后写 `x = "hello"`，Python **不会报错**。类型注解不影响程序运行，它更像是一种"说明书"。

---

### 二、变量类型注解的基本语法

#### 2.1 语法格式

```python
变量名: 类型 = 值
```

#### 2.2 基础类型注解

```python
# 整数
age: int = 18

# 浮点数
height: float = 1.75

# 字符串
name: str = "张三"

# 布尔值
is_student: bool = True

# 空值
nothing: None = None
```

#### 2.3 先注解后赋值

类型注解也可以**先声明，后赋值**（不常用，但语法合法）：

```python
age: int       # 先声明类型
age = 18       # 后赋值
```

---

### 三、容器类型的注解

#### 3.1 基础容器注解（不指定元素类型）

```python
my_list: list = [1, 2, 3]
my_tuple: tuple = (1, 2, 3)
my_set: set = {1, 2, 3}
my_dict: dict = {"name": "张三", "age": 18}
```

这种写法只告诉你"这是一个列表/元组/集合/字典"，但**不知道里面装的是什么类型的元素**。

#### 3.2 详细容器注解（指定元素类型）

使用 `list[元素类型]`、`dict[键类型, 值类型]` 等方式，可以精确描述容器内部的元素类型：

```python
# 列表：元素都是 int
scores: list[int] = [90, 85, 78, 92]

# 元组：需要标注每一个元素的类型（因为元组是固定长度的）
student: tuple[str, int, float] = ("张三", 18, 92.5)

# 集合：元素都是 str
tags: set[str] = {"Python", "Java", "Go"}

# 字典：键是 str，值是 int
age_map: dict[str, int] = {"张三": 18, "李四": 20, "王五": 22}
```

> **注意：** `list[int]`、`dict[str, int]` 这种写法在 **Python 3.9+** 版本直接支持。如果你使用的是 Python 3.8 或更早版本，需要从 `typing` 模块导入：
>
> ```python
> from typing import List, Dict, Tuple, Set
> 
> scores: List[int] = [90, 85, 78]
> age_map: Dict[str, int] = {"张三": 18}
> ```
>
> 本教程以 Python 3.9+ 语法为主。

#### 3.3 嵌套容器注解

容器类型可以嵌套：

```python
# 列表中的每个元素是一个列表，内部列表的元素是 int
matrix: list[list[int]] = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# 字典：键是 str，值是一个 int 列表
scores: dict[str, list[int]] = {
    "张三": [90, 85, 78],
    "李四": [88, 92, 76]
}
```

### 四、类对象类型注解

变量也可以注解为**自定义类**的类型：

```python
class Student:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

    def study(self):
        print(f"{self.name}正在学习")


# 注解变量为 Student 类型
stu: Student = Student("张三", 18)
stu.study()  # IDE 可以智能提示 Student 的方法
```

多个对象的注解：

```python
stu_1: Student = Student("张三", 18)
stu_2: Student = Student("李四", 20)

# 列表中的元素是 Student 类型
students: list[Student] = [stu_1, stu_2]

# IDE 智能提示：students[0]. 后会弹出 name、age、study 等
```

---

### 五、类型注解的第二种方式：注释法

除了 `变量: 类型 = 值` 这种语法格式，还可以通过**注释**的方式添加类型注解：

```python
age = 18                         # type: int
name = "张三"                     # type: str
scores = [90, 85, 78]            # type: list[int]
info = {"name": "张三", "age": 18}  # type: dict[str, object]
```

> 两种方式的对比：

| 方式                 | 示例                    | 适用场景         |
| -------------------- | ----------------------- | ---------------- |
| **语法格式**（推荐） | `age: int = 18`         | 新代码，主流选择 |
| **注释格式**         | `age = 18  # type: int` | 兼容旧代码       |

一般推荐使用第一种 `变量: 类型 = 值` 的方式，更直观、更现代。

---

### 六、类型注解只是"建议"，不是"强制"

**类型注解不会影响程序的运行，Python 不会在运行时检查类型注解是否正确。**

```python
age: int = "这其实是一个字符串"  # 不会报错！Python 照样运行
print(age)   # 输出：这其实是一个字符串
print(type(age))  # 输出：<class 'str'>
```

虽然 Python 解释器不会报错，但好的 IDE（如 PyCharm、VS Code）会给你**黄色警告提示**，告诉你类型不匹配。

> **类型注解的价值在于：**
> 1. **帮助 IDE 提供智能代码补全**
> 2. **帮助开发者（包括你自己和同事）理解代码意图**
> 3. **可以配合 `mypy` 等工具做静态类型检查**

---

| 知识点               | 示例                                   |
| -------------------- | -------------------------------------- |
| 基础类型注解         | `name: str = "张三"`                   |
| 容器类型注解（简单） | `my_list: list = [1, 2]`               |
| 容器类型注解（详细） | `my_list: list[int] = [1, 2]`          |
| 元组注解             | `info: tuple[str, int] = ("张三", 18)` |

## 11 函数和方法的类型注解

### 一、为函数的参数添加类型注解

在定义函数时，我们可以为**每一个参数**标注预期的类型。

#### 1.1 语法格式

```python
def 函数名(参数1: 类型, 参数2: 类型, ...):
    函数体
```

#### 1.2 基础案例

```python
def greet(name: str, age: int):
    print(f"你好，我叫{name}，今年{age}岁")


greet("张三", 18)  # 正常调用
```

有了类型注解后，当你在调用 `greet()` 时，IDE 会提示你：
- 第一个参数 `name` 应该传 `str`
- 第二个参数 `age` 应该传 `int`

#### 1.3 容器类型参数注解

```python
def calculate_average(scores: list[int]) -> float:
    return sum(scores) / len(scores)


def show_student(info: dict[str, str]):
    for key, value in info.items():
        print(f"{key}: {value}")


calculate_average([90, 85, 78, 92])
show_student({"name": "张三", "class": "三年二班"})
```

---

### 二、为函数的返回值添加类型注解

#### 2.1 语法格式

```python
def 函数名(参数: 类型, ...) -> 返回值类型:
    函数体
    return 返回值
```

使用箭头 `->` 在参数列表之后标注返回值类型。

#### 2.2 案例演示

```python
# 返回 int
def add(a: int, b: int) -> int:
    return a + b


# 返回 str
def get_greeting(name: str) -> str:
    return f"你好，{name}！"


# 返回 bool
def is_adult(age: int) -> bool:
    return age >= 18


# 返回 list
def get_even_numbers(nums: list[int]) -> list[int]:
    return [n for n in nums if n % 2 == 0]


# 没有返回值的函数，标注为 None
def say_hello(name: str) -> None:
    print(f"Hello, {name}!")


# 调用
result1 = add(10, 20)           # IDE 知道 result1 是 int
result2 = get_greeting("张三")   # IDE 知道 result2 是 str
result3 = is_adult(20)          # IDE 知道 result3 是 bool
result4 = get_even_numbers([1, 2, 3, 4, 5, 6])  # IDE 知道 result4 是 list[int]
```

---

### 三、方法的类型注解

类中的方法与函数的类型注解规则完全相同，只是 `self` 参数不需要注解：

```python
class Calculator:
    def __init__(self, name: str) -> None:
        self.name: str = name

    def add(self, a: int, b: int) -> int:
        return a + b

    def get_info(self) -> str:
        return f"计算器名称：{self.name}"

    def divide(self, a: float, b: float) -> float:
        if b == 0:
            print("除数不能为0")
            return 0.0
        return a / b


calc = Calculator("我的计算器")
result = calc.add(10, 20)       # IDE 提示 result 是 int
info = calc.get_info()          # IDE 提示 info 是 str
```

| 知识点         | 语法                      | 示例                            |
| -------------- | ------------------------- | ------------------------------- |
| 参数类型注解   | `参数: 类型`              | `def func(name: str):`          |
| 返回值类型注解 | `-> 类型`                 | `def func() -> int:`            |
| 无返回值       | `-> None`                 | `def func() -> None:`           |
| 方法类型注解   | 与函数相同，`self` 不注解 | `def add(self, a: int) -> int:` |

## 12 Union 联合类型注解

### 一、什么场景需要 Union？

有些时候，一个变量可能是**多种类型之一**。例如：

```python
# 这个列表里既有 int，又有 str
my_list = [1, "hello", 2, "world"]

# 这个字典的值既有 str，又有 int
person = {"name": "张三", "age": 18}
```

如果用 `list[int]` 注解，不准确（里面还有 str）；用 `list[str]` 注解，也不准确。

> **Union 类型注解**允许我们表示：一个变量可以是**类型 A 或者类型 B**。

---

### 二、Union 的使用方式

#### 方式一：使用 `Union` 从 `typing` 模块导入

```python
from typing import Union

# 变量可以是 int 或 str
my_var: Union[int, str] = 10
my_var = "hello"    # 也合法

# 列表中的元素可以是 int 或 str
my_list: list[Union[int, str]] = [1, "hello", 2, "world"]

# 字典的值可以是 str 或 int
person: dict[str, Union[str, int]] = {
    "name": "张三",     # str
    "age": 18           # int
}
```

#### 方式二：使用 `|` 管道符（Python 3.10+ 推荐）

Python 3.10 及以上版本，可以直接使用 `|` 代替 `Union`，更简洁：

```python
# Python 3.10+ 写法
my_var: int | str = 10
my_var = "hello"

my_list: list[int | str] = [1, "hello", 2, "world"]

person: dict[str, str | int] = {
    "name": "张三",
    "age": 18
}
```

> 两种写法效果完全一样，如果你的 Python 版本 >= 3.10，推荐用 `|` 更简洁。

---

### 三、Union 在函数中的应用

#### 3.1 参数的 Union 注解

```python
from typing import Union


def process_id(user_id: Union[int, str]) -> str:
    """
    用户ID可能是数字（如 1001），也可能是字符串（如 "U1001"）
    统一转为字符串返回
    """
    return str(user_id)


print(process_id(1001))       # 传入 int，合法
print(process_id("U1001"))    # 传入 str，合法
```

#### 3.2 返回值的 Union 注解

```python
from typing import Union


def safe_divide(a: int, b: int) -> Union[float, str]:
    """
    除法运算：正常返回 float 结果，出错返回 str 错误信息
    """
    if b == 0:
        return "错误：除数不能为0"
    return a / b


result1 = safe_divide(10, 3)   # 返回 float: 3.333...
result2 = safe_divide(10, 0)   # 返回 str: "错误：除数不能为0"
print(result1)
print(result2)
```

### 四、综合案例

```python
from typing import Union


class DataProcessor:
    """数据处理器"""

    def __init__(self, name: str) -> None:
        self.name = name
        self.data: list[Union[int, float, str]] = []

    def add_data(self, item: Union[int, float, str]) -> None:
        """添加一条数据，可以是整数、浮点数或字符串"""
        self.data.append(item)
        print(f"已添加：{item}（类型：{type(item).__name__}）")

    def get_numbers(self) -> list[Union[int, float]]:
        """获取所有数字类型的数据"""
        return [item for item in self.data if isinstance(item, (int, float))]

    def get_strings(self) -> list[str]:
        """获取所有字符串类型的数据"""
        return [item for item in self.data if isinstance(item, str)]


# 使用
dp = DataProcessor("我的处理器")
dp.add_data(100)
dp.add_data(3.14)
dp.add_data("hello")
dp.add_data(200)
dp.add_data("world")

print(f"所有数字：{dp.get_numbers()}")    # [100, 3.14, 200]
print(f"所有字符串：{dp.get_strings()}")   # ['hello', 'world']
```

`Optional[X]` 等价于 `Union[X, None]`，表示值可以是 `X` 类型或者 `None`：

```python
from typing import Optional


def find_student(name: str) -> Optional[str]:
    """查找学生，找到返回信息，找不到返回 None"""
    students = {"张三": "三年二班", "李四": "三年三班"}

result = find_student("张三")   # "三年二班"

```

```python
def find_student(name: str) -> str | None:
    ...
```

## 13 多态

### 一、什么是多态？

**多态（Polymorphism）** 是面向对象三大特性之一（封装、继承、**多态**）。

> **多态的含义：** 同一个方法调用，传入不同类型的对象，产生不同的行为效果。

用生活中的例子理解：

> 老板说："工作！"
> - 程序员听到后 → 开始写代码
> - 设计师听到后 → 开始画图
> - 销售听到后 → 开始打电话
>
> **同一个命令（"工作"），不同的人（对象）产生不同的行为** —— 这就是多态。

---

### 二、多态的前提条件

多态需要满足以下条件：

1. **有继承关系**（子类继承父类）
2. **有方法复写**（子类复写父类的方法）
3. **父类类型的变量，引用子类对象**（用父类的角度去使用子类）

---

### 三、多态的基本案例

```python
class Animal:
    """父类：动物"""
    def speak(self):
        print("动物发出声音")


class Dog(Animal):
    """子类：狗"""
    def speak(self):
        print("汪汪汪！🐕")


class Cat(Animal):
    """子类：猫"""
    def speak(self):
        print("喵喵喵！🐈")


class Chicken(Animal):
    """子类：鸡"""
    def speak(self):
        print("咯咯咯！🐔")


# 多态的体现：同一个函数，传入不同类型的对象，产生不同效果
def make_animal_speak(animal: Animal):
    """让动物说话
    参数类型注解为 Animal（父类），但实际可以传入任何 Animal 的子类
    """
    animal.speak()


# 调用同一个函数，传入不同的对象
make_animal_speak(Dog())       # 输出：汪汪汪！🐕
make_animal_speak(Cat())       # 输出：喵喵喵！🐈
make_animal_speak(Chicken())   # 输出：咯咯咯！🐔
```

**运行结果：**

```
汪汪汪！🐕
喵喵喵！🐈
咯咯咯！🐔
```

> **关键理解：**
> `make_animal_speak` 函数的参数类型是 `Animal`（父类），但我们实际传入的是 `Dog`、`Cat`、`Chicken`（子类对象）。调用 `animal.speak()` 时，Python 会根据**实际对象的类型**来调用对应的方法——这就是多态。

---

### 四、抽象类与抽象方法

在上面的例子中，父类 `Animal` 的 `speak` 方法写了 `print("动物发出声音")`。但实际上，"动物发出声音"这个描述太笼统了——我们**并不打算真正使用父类的这个方法**，它只是一个"模板"，提醒子类"你必须实现 `speak` 方法"。

这种情况下，我们可以使用**抽象类**。

#### 4.1 什么是抽象类？

- **抽象类（Abstract Class）：** 包含抽象方法的类，**不能直接实例化**（不能创建对象）。
- **抽象方法（Abstract Method）：** 用 `@abstractmethod` 标记的方法。它可以只写 `pass`，也可以给出默认实现，但抽象类本身仍不能直接实例化，子类需要补全抽象方法后才能实例化。

> 抽象类就像一份**合同/规范**：它规定了子类必须实现哪些方法，但不关心具体怎么实现。

#### 4.2 使用 `abc` 模块定义抽象类

```python
from abc import ABC, abstractmethod


class Animal(ABC):
    """抽象类：动物（不能直接实例化）"""

    @abstractmethod
    def speak(self):
        """抽象方法：子类必须实现"""
        pass

    @abstractmethod
    def move(self):
        """抽象方法：子类必须实现"""
        pass

    def breathe(self):
        """普通方法：子类可以直接继承使用"""
        print("呼吸中...")
```

- `ABC`：Abstract Base Class 的缩写，继承它表明这是一个抽象类。
- `@abstractmethod`：装饰器，标记这个方法是抽象方法，子类必须实现。

#### 4.3 子类必须实现所有抽象方法

```python
from abc import ABC, abstractmethod


class Animal(ABC):
    @abstractmethod
    def speak(self):
        pass

    @abstractmethod
    def move(self):
        pass


class Dog(Animal):
    def speak(self):          # ✅ 实现了抽象方法
        print("汪汪汪！")

    def move(self):           # ✅ 实现了抽象方法
        print("狗在跑")


class Cat(Animal):
    def speak(self):          # ✅ 实现了抽象方法
        print("喵喵喵！")

    def move(self):           # ✅ 实现了抽象方法
        print("猫在走")


# 正常使用子类
dog = Dog()
dog.speak()   # 汪汪汪！
dog.move()    # 狗在跑

cat = Cat()
cat.speak()   # 喵喵喵！
cat.move()    # 猫在走
```

**如果子类没有实现所有抽象方法，会怎样？**

```python
class Fish(Animal):
    def speak(self):
        print("鱼不会说话...")
    # 注意：没有实现 move 方法！


# fish = Fish()  # ❌ 报错！TypeError: Can't instantiate abstract class Fish 
#                #    with abstract method move
```

> **抽象类不能直接实例化：**

```python
# animal = Animal()  # ❌ 报错！TypeError: Can't instantiate abstract class Animal 
#                    #    with abstract methods move, speak
```

### 五、多态的完整应用案例

#### 案例：空调管理系统

```python
from abc import ABC, abstractmethod


# ========== 抽象父类 ==========
class AirConditioner(ABC):
    """空调抽象类（规范/标准）"""

    @abstractmethod
    def cool(self) -> None:
        """制冷"""
        pass

    @abstractmethod
    def heat(self) -> None:
        """制热"""
        pass

    @abstractmethod
    def show_info(self) -> str:
        """展示信息"""
        pass


# ========== 子类实现 ==========
class MideaAC(AirConditioner):
    """美的空调"""

    def cool(self) -> None:
        print("美的空调：强劲制冷，一晚低至一度电！❄️")

    def heat(self) -> None:
        print("美的空调：暖风徐来，温暖整个冬天！☀️")

    def show_info(self) -> str:
        return "品牌：美的 | 型号：KFR-35GW"


class GreeAC(AirConditioner):
    """格力空调"""

    def cool(self) -> None:
        print("格力空调：核心科技，精准控温制冷！❄️")

    def heat(self) -> None:
        print("格力空调：智能变频，均匀暖房！☀️")

    def show_info(self) -> str:
        return "品牌：格力 | 型号：KFR-50LW"


class HaierAC(AirConditioner):
    """海尔空调"""

    def cool(self) -> None:
        print("海尔空调：自清洁制冷，健康好空气！❄️")

    def heat(self) -> None:
        print("海尔空调：3D立体暖风，速热不干燥！☀️")

    def show_info(self) -> str:
        return "品牌：海尔 | 型号：KFR-35GW/03JDM81A"


# ========== 使用多态 ==========
def summer_mode(ac: AirConditioner) -> None:
    """夏天模式：不管什么品牌，都开制冷"""
    print(f"[{ac.show_info()}]")
    ac.cool()
    print()


def winter_mode(ac: AirConditioner) -> None:
    """冬天模式：不管什么品牌，都开制热"""
    print(f"[{ac.show_info()}]")
    ac.heat()
    print()


# 创建不同品牌的空调对象
midea = MideaAC()
gree = GreeAC()
haier = HaierAC()

# 一个列表存储所有空调
all_ac: list[AirConditioner] = [midea, gree, haier]

print("=" * 50)
print("          🌞 夏天来了，全部开制冷！")
print("=" * 50)
for ac in all_ac:
    summer_mode(ac)      # 同一个函数，不同对象，不同行为

print("=" * 50)
print("          ❄️ 冬天来了，全部开制热！")
print("=" * 50)
for ac in all_ac:
    winter_mode(ac)      # 同一个函数，不同对象，不同行为
```

### 六、多态的核心优势

#### 6.1 开闭原则（开放扩展，关闭修改）

看上面的例子，如果要新增一个"奥克斯空调"，我们只需要：

```python
class AuxAC(AirConditioner):
    """奥克斯空调"""
    def cool(self) -> None:
        print("奥克斯空调：高效制冷！❄️")

    def heat(self) -> None:
        print("奥克斯空调：舒适制热！☀️")

    def show_info(self) -> str:
        return "品牌：奥克斯 | 型号：KFR-35GW/BpR3TYA1"
```

然后：

```python
aux = AuxAC()
summer_mode(aux)   # 直接用！不需要修改 summer_mode 函数
winter_mode(aux)   # 直接用！不需要修改 winter_mode 函数
```

> **无需修改已有的函数代码**，只需要新增子类即可。这就是**开闭原则**的体现。

#### 6.2 多态的好处总结

| 好处         | 说明                               |
| ------------ | ---------------------------------- |
| **灵活性**   | 同一段代码可以处理不同类型的对象   |
| **可扩展性** | 新增子类不影响已有代码             |
| **可维护性** | 代码逻辑集中，减少 `if-else` 判断  |
| **规范性**   | 抽象类强制约束子类必须实现特定方法 |

### 七、Python 中多态的特殊说明——"鸭子类型"

Python 是动态语言，它的多态比 Java、C++ 更加灵活。Python 奉行一个理念：

> **"如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子。"** —— 鸭子类型（Duck Typing）

**这意味着：在 Python 中，多态不一定需要继承关系。** 只要对象有对应的方法，就可以使用：

```python
class Dog:
    def speak(self):
        print("汪汪汪！")

class Cat:
    def speak(self):
        print("喵喵喵！")

class Robot:           # Robot 不继承任何动物类
    def speak(self):
        print("你好，我是机器人。")


def make_it_speak(obj):
    """不关心 obj 是什么类型，只要它有 speak 方法就行"""
    obj.speak()


make_it_speak(Dog())     # 汪汪汪！
make_it_speak(Cat())     # 喵喵喵！
make_it_speak(Robot())   # 你好，我是机器人。
```

> `Dog`、`Cat`、`Robot` 之间没有任何继承关系，但它们都有 `speak()` 方法，所以 `make_it_speak` 函数可以正常调用——**这就是鸭子类型**。

但是，虽然 Python 支持鸭子类型，**在实际开发中我们依然推荐使用继承 + 抽象类的方式**，原因是：
1. 代码更规范，意图更清晰
2. IDE 可以给出更好的代码提示
3. 如果忘记实现某个方法，抽象类会在创建对象时直接报错，而鸭子类型可能运行到某个时刻才报错

### 八、本章全部核心概念回顾

```
面向对象三大特性
├── 封装（Encapsulation）   → 将数据和方法包装在类内部，控制访问权限
├── 继承（Inheritance）     → 子类复用父类的代码
└── 多态（Polymorphism）    → 同一方法，不同对象，不同行为
```

| 概念                 | 关键语法/知识                       |
| -------------------- | ----------------------------------- |
| **继承**             | `class Child(Parent):`              |
| **多继承**           | `class C(A, B):`，实际按 MRO 解析     |
| **复写**             | 子类定义同名方法覆盖父类            |
| **调用父类**         | `super().方法()`                    |
| **类型注解（变量）** | `x: int = 10`                       |
| **类型注解（函数）** | `def f(a: int) -> str:`             |
| **Union**            | `Union[int, str]` 或 `int \| str`   |
| **多态**             | 父类引用指向子类对象                |
| **抽象类**           | `class X(ABC):` + `@abstractmethod` |
| **鸭子类型**         | 不看类型看行为                      |

---

> **学习建议：** 以上所有概念是环环相扣的——**先有继承，才有复写；先有继承和复写，才有多态；类型注解帮助你更好地理解和使用多态中的父类类型引用。** 建议将每一节的示例代码都在 IDE 中亲手敲一遍运行，观察结果，加深理解。
